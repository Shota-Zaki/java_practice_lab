Java Silverのオーバーロード解決

オーバーロード解決は「どのメソッドが選ばれるか」を決める問題です。完全一致、拡大変換、ボクシング、可変長引数の優先度を押さえます。

1. 完全一致が最優先

void test(int x) { System.out.println("int"); }
void test(long x) { System.out.println("long"); }
test(10); // int

整数リテラル 10 は int なので、test(int) が選ばれます。

2. 拡大変換はボクシングより優先

void test(long x) { System.out.println("long"); }
void test(Integer x) { System.out.println("Integer"); }
test(10); // long

int から long への拡大変換が、Integer へのボクシングより優先されます。

3. 可変長引数は最後

void test(int x) { System.out.println("one"); }
void test(int... x) { System.out.println("varargs"); }
test(1); // one

可変長引数は、通常の引数で解決できない場合に使われます。

4. nullで曖昧になる

void test(String s) {}
void test(StringBuilder s) {}
test(null); // コンパイルエラー

どちらも参照型で、片方がもう片方のサブタイプではないため曖昧です。

5. より具体的な型が選ばれる

void test(Object o) { System.out.println("Object"); }
void test(String s) { System.out.println("String"); }
test(null); // String

String は Object より具体的なので選ばれます。

優先順位

  1. 完全一致
  2. 拡大プリミティブ変換
  3. ボクシング
  4. 可変長引数

ただし、複数候補が同じ強さで残ると曖昧になりコンパイルエラーです。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java11 / Java17 Silver共通。Silver共通範囲では、バージョン固有構文に逃げず、型、参照、制御構文、クラス、継承、例外の基本ルールだけで判定します。Java17固有の記事はJava17 Silver側に分けています。

本文で必ず押さえること

  • インスタンスに属するものと、クラスに属するstaticなものを分ける。
  • コンストラクタは戻り値型を書かない。メソッドと見た目が似ていても別物。
  • オーバーロードはコンパイル時に引数型で決まる。実行時のオーバーライドとは分ける。

この後の演習で見るポイント

  • 呼び出し元の変数型、引数型、アクセス修飾子を順に確認する。
  • thisとsuperが何を指しているかを明確にする。
  • コンストラクタチェーンでは、最初にどのコンストラクタへ制御が移るかを見る。

ここで落ちる典型ミス

  • staticメソッドからインスタンスフィールドへ直接触れると思い込む。
  • privateメンバが継承先から自由に使えると考える。
  • 同名メソッドを見てすぐオーバーライドと判断する。引数が違えばオーバーロード。

演習前の小課題

同じ名前のメソッドが複数あるコードでは、先に「候補メソッド一覧」を作り、引数型でどれが選ばれるかを判定してください。
