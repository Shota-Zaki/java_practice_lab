Collectors入門

collectはStreamの終点です。どのCollectorを渡したかで、List、Set、Map、文字列、数値集約に変わります。

collectは「集め方」を指定する

collect自体を覚えるより、渡しているCollectorの種類を読むほうが重要です。

Collector結果確認点
toList()List順序や変更可能性を決めつけない。
toSet()Set重複が消える。順序も決めつけない。
joining()String要素型はCharSequence系である必要がある。
toMap()Mapキー重複時の扱いが重要。
groupingBy()Mapキーと値側の型を分けて読む。
String joined = Stream.of("A", "B", "C")
    .collect(Collectors.joining("-"));
System.out.println(joined); // A-B-C

toMapはキー重複に注意

toMapで同じキーが作られると、マージ関数を指定していない場合は実行時例外になります。

Map<Integer, String> map = Stream.of("aa", "bb")
    .collect(Collectors.toMap(String::length, s -> s));
// どちらもキー2になるため、実行時例外
Map<Integer, String> map = Stream.of("aa", "bb")
    .collect(Collectors.toMap(
        String::length,
        s -> s,
        (oldValue, newValue) -> oldValue));
// キー重複時の処理を指定しているためOK
Collectorsは「何に集めるか」よりも「重複、順序、下流Collector、戻り値の型」を見ると安定します。
