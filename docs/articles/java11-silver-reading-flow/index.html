Java11 Silverコード読解フロー

Java11 Silverでは、var、StringBuilder、ArrayList、例外、継承をJava11範囲に限定して精密に読む必要があります。

先に結論

Java11 Silverでは、Java17固有構文やGoldの高度APIを持ち込まず、Java11時点の範囲で判断します。読み方は、コンパイル可否、実行時例外、出力結果の順です。

コンパイル可否で見るところ

  • varはローカル変数で、初期値から型推論できる場所だけで使える。
  • checked例外はthrowsまたはcatchが必要。
  • staticメソッドからインスタンスフィールドへ直接アクセスできない。
  • オーバーロードは引数の型で決まり、戻り値型だけでは区別できない。

実行時例外で見るところ

null参照、配列範囲外、ClassCastException、NumberFormatException、UnsupportedOperationExceptionを疑います。コンパイルできるから安全、という判断は間違いです。

出力問題の読み方

  1. Stringは不変。戻り値を受け取っていなければ元は変わらない。
  2. StringBuilderは変更可能。appendやreverseで自身が変わる。
  3. ArrayListは要素数が変わる。remove(1)とremove(Integer.valueOf(1))を区別する。
  4. オーバーライドは実体のメソッド、フィールド参照は変数の型で決まる。

典型的な誤解

誤解

参照型変数に代入すると、オブジェクトがコピーされる。

正しくは

コピーされるのは参照です。同じオブジェクトを複数の変数が指すことがあります。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java11 Silver。Java11 Silverでは、Java11範囲の文法・APIだけで判定します。record、sealed、switch式、yield、instanceofパターン変数のようなJava17寄りの構文は混ぜません。

本文で必ず押さえること

  • 用語を覚えるだけでなく、コード上でどの判断に使う知識かを確認する。
  • 先にコンパイル可否、次に実行時の流れ、最後に出力を考える。
  • 自信がない箇所は、変数の型と値を紙に分けて書く。

この後の演習で見るポイント

  • 本文の例を少し変えても同じ判断ができるか確認する。
  • 選択肢を消去するときは、なぜ誤りかを言葉で説明する。
  • 関連する演習では、正解番号ではなく判断根拠を残す。

ここで落ちる典型ミス

  • 用語の意味だけ覚えて、コード読解に接続しない。
  • 「なんとなく見たことがある」で選択肢を選ぶ。
  • 解説を読んだあとに、自力で再判定しない。

演習前の小課題

本文のコード例を1行だけ変更し、コンパイル可否・実行時の流れ・出力をもう一度判定してください。
