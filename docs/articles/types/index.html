型・変数・リテラル完全整理

Silverの型問題は、暗記よりも判定順です。リテラルの型、式の型、代入変換、初期化、スコープ、var、配列、参照型を順に切ると、コンパイルエラー・実行時例外・出力を安定して判定できます。

最初の結論: 型問題はこの順番で読む

型・変数・リテラルの問題で一番危険なのは、「値が小さいから入る」「見た目が同じだから同じ」と処理することです。Javaは、コードの見た目ではなく、式が持つ型で判定します。

  1. 宣言できる場所かを見る。フィールド、ローカル変数、引数、戻り値で使える構文が違う。
  2. 初期化済みかを見る。ローカル変数とフィールドで扱いが違う。
  3. 右辺の式の型を決める。整数は基本int、小数は基本double。
  4. 演算で型が昇格するかを見る。byte/short/charは演算で原則int。
  5. 左辺へ代入できるかを見る。拡大変換、縮小変換、定数式の特例、キャストを分ける。
  6. 参照型なら参照先を見る。null、同一性、配列、ラッパークラスを分ける。
試験での落とし穴: byte b = 1 + 2; と byte b = a + 2; は同じではありません。前者は定数式、後者は変数を含む式です。

値・型・変数・リテラル・式・文を分ける

用語を分けないままコードを読むと、ほぼ確実にどこかで誤判定します。特に「リテラル」と「式の型」は別物です。

用語意味例判定ポイント
値実際に扱うデータ。10, true, 'A', オブジェクト値だけで代入可否は決まらない。
型値の扱い方を決める分類。int, char, String代入、演算、比較、メソッド呼び出しを決める。
変数値または参照値を保持する名前。int n, String sプリミティブ型と参照型で保持するものが違う。
リテラルコードに直接書いた値。10, 10L, "abc", nullリテラル自体にも型がある。
式評価すると値と型を持つコード片。a + 1, s.length()byte + byteの式はint。
文処理の単位。int a = 1;, if (...) {}代入式は文の中にあるが、式として値も持つ。

プリミティブ型と参照型

Javaの型は、大きくプリミティブ型と参照型に分かれます。Silverではここを曖昧にすると、null、==、配列、ラッパークラスで落ちます。

分類変数が持つものnull==の意味例
プリミティブ型値そのもの代入不可値の比較int, double, boolean, char
参照型オブジェクトへの参照代入可同じオブジェクトを参照しているかString, 配列, ArrayList, 自作クラス
 
int a = 10;
int b = a;
b = 20;
System.out.println(a); // 10: 値がコピーされただけ

String s = "abc";
String t = s;
System.out.println(s == t); // true: 同じ参照値をコピー
nullの位置: nullは参照型専用です。int n = null;はコンパイルエラーです。String s = null;はコンパイルできますが、s.length()は実行時に落ちます。

8つのプリミティブ型

範囲を丸暗記する必要は限られますが、byte、short、char、int、long、float、double、boolean の役割は正確に分けます。

型分類代表値Silverでの注意点
byte整数-128〜127演算すると原則int。範囲内定数式だけ代入特例あり。
short整数-32768〜32767short + shortはint。
char文字・符号なし整数'A'数値として扱えるが、演算するとint。
int整数10整数リテラルの基本型。
long整数10LLなしの巨大整数は、代入先がlongでも不可。
float浮動小数1.0F小数リテラルは基本doubleなのでFが必要。
double浮動小数1.0小数リテラルの基本型。
boolean真偽値true, false数値型と互換性なし。0/1で代用不可。

数値リテラルの基本型

ソースコードに直接書いた数値は、見た目で型が決まります。代入先の型だけで決まるわけではありません。

リテラル基本型例注意点
整数リテラルint10範囲を超えるならLが必要。
L/l付き整数long10L小文字lは1と紛らわしい。
小数リテラルdouble1.0floatへ直接代入不可。
F/f付き小数float1.0Ffloatに入れるなら基本これ。
D/d付き小数double1.0D明示しなくても小数はdouble。
 
long a = 10;       // OK: int 10 を long へ拡大変換
long b = 10L;      // OK: longリテラル

// float c = 1.0;  // NG: 1.0 は double
float d = 1.0F;    // OK

double e = 1.0F;   // OK: float から double へ拡大変換
// long x = 2147483648;  // NG: 右辺はまず int リテラルとして判定され、int範囲外
long y = 2147483648L;    // OK

整数リテラルの進数表記

先頭の書き方で基数が変わります。特に0始まりの8進数は、実務では避けたい書き方ですが試験ではよく出ます。

表記例10進での値注意
10進数1717通常の表記。
2進数0b10001170bまたは0B。
8進数02117先頭が0。8と9は使えない。
16進数0x11170xまたは0X。
 
int a = 17;
int b = 0b10001;
int c = 021;
int d = 0x11;
System.out.println(a + b + c + d); // 68

// int e = 089; // NG: 8進数に 8/9 は使えない

アンダースコアを置ける場所・置けない場所

_は数値リテラルの読みやすさを上げるだけで、値には影響しません。ただし置ける場所に制限があります。

コード可否理由
int a = 123_456;OK数字の間。
int b = 1__2__3;OK連続しても数字の間なら可。
int c = _123;NGリテラルの先頭。
int d = 123_;NGリテラルの末尾。
float e = 3_.14F;NG小数点の直前。
float f = 3._14F;NG小数点の直後。
long g = 100_L;NG接尾辞Lの直前。
int h = 0x_FF;NG基数接頭辞0xの直後。
int i = 0xF_F;OK数字の間。

識別子のルール

変数名・クラス名・メソッド名などの識別子は、使える文字と使えない文字があります。予約語も使えません。

例可否理由
int price = 100;OK通常の識別子。
int _price = 100;OK先頭_は使える。ただし単独_は不可。
int $price = 100;OK$は使えるが通常は避ける。
int 1price = 100;NG数字で始められない。
int class = 100;NGclassは予約語。
int var = 100;OKvarは予約語ではなく、特定文脈で型推論に使う名前。
int _ = 100;NGJava 9以降、単独_は識別子として使えない。
試験対策: varを変数名にできるかは引っかけです。int var = 1;は可能です。ただしvar a = 1;では型推論として解釈されます。

ローカル変数とフィールドの初期値

フィールドにはデフォルト値があります。ローカル変数にはデフォルト値がありません。ここはSilverで頻出です。

場所デフォルト値未初期化で読むと
インスタンスフィールドある型ごとのデフォルト値。
staticフィールドある型ごとのデフォルト値。
ローカル変数ないコンパイルエラー。
 
class Sample {
    int field;
    String text;

    void test() {

        System.out.println(text);  // null
        // System.out.println(local); // NG: localは初期化されていない
    }
}
型フィールドのデフォルト値
整数型0
浮動小数型0.0
char''
booleanfalse
参照型null

スコープと同名変数

スコープは、その変数名が有効な範囲です。同じ名前を使えるかどうかは、同じスコープ内か、別スコープかで決まります。

int x = 10;
// int x = 20; // NG: 同じスコープで同じローカル変数名は不可

{
    // int x = 30; // NG: 外側のローカル変数と重なる
    int y = 40;    // OK
}
// System.out.println(y); // NG: yはブロック外では見えない

フィールドとローカル変数は同名にできます。この場合、単に名前を書くとローカル変数が優先されます。フィールドを指すにはthisを使います。

class Item {
    int price = 100;

    void setPrice(int price) {
        price = price;       // ローカル変数priceに自分自身を代入しているだけ
        this.price = price;  // フィールドpriceに引数priceを代入
    }
}

代入変換: 拡大変換と縮小変換

代入では、右辺の型を左辺の型として扱えるかを見ます。広い型へは暗黙変換できます。狭い型へは原則キャストが必要です。

int i = 10;
long l = i;        // OK: int から long
float f = l;       // OK: long から float。精度低下の可能性はある
// int x = l;      // NG: long から int
int y = (int) l;   // OK: 明示キャスト
範囲と精度を分ける: longからfloatへの変換はコンパイルできます。ただし精度は失われる可能性があります。試験では「コンパイル可否」と「正確な値」を分けて考えます。

byte・short・charは演算でintになる

小さい整数型は、演算に入ると原則intに昇格します。ここを外すと、型問題の正答率は安定しません。

byte a = 1;
byte b = 2;
// byte c = a + b;       // NG: a + b は int
int d = a + b;           // OK
byte e = (byte)(a + b);  // OK

short s1 = 1;
short s2 = 2;
// short s3 = s1 + s2;   // NG: s1 + s2 は int
char c = 'A';
System.out.println(c);       // A
System.out.println(c + 1);   // 66
// char next = c + 1;        // NG: c + 1 は int
char next = (char)(c + 1);   // OK
System.out.println(next);    // B

二項数値昇格

左右のオペランドの型が違う場合、より広い型へ寄せられます。ざっくり優先順位は double、float、long、int です。

含まれる型演算結果例
doubleが含まれるdouble1 + 2.0
floatが含まれるfloat1L + 2.0F
longが含まれるlong1 + 2L
それ以外の整数型intbyte + short
 
int a = 10;
long b = 20L;
// int c = a + b;  // NG: a + b は long
long d = a + b;    // OK

float f = 1.0F;
double g = 2.0;
// float h = f + g; // NG: f + g は double
double i = f + g;   // OK

定数式なら小さい型へ入る場合がある

右辺がコンパイル時に確定できる定数式で、代入先の範囲内なら、byte、short、charへ代入できます。

byte a = 10;          // OK
byte b = 10 + 20;     // OK: 定数式で範囲内
// byte c = 100 + 100; // NG: 200 は byte の範囲外

short s = 1000 + 2000; // OK
char ch = 65;          // OK: 'A'

変数を含むと、たとえ実際の値が範囲内でも通常は代入できません。

byte a = 10;
// byte b = a + 1; // NG: a + 1 は int

final int x = 10;
byte c = x + 1;    // OK: x はコンパイル時定数

final int y;
y = 10;
// byte d = y;     // NGになり得る: 宣言時初期化された定数式ではない

複合代入は暗黙キャストを含む

+=、-=、*=、/=などの複合代入は、左辺の型へのキャストを含むように動きます。

byte b = 1;
// b = b + 1; // NG: b + 1 は int
b += 1;       // OK: b = (byte)(b + 1) 相当

ただし、安全という意味ではありません。範囲を超えると、オーバーフローした結果になります。

byte b = 127;
b += 1;
System.out.println(b); // -128

オーバーフロー

整数型は範囲を超えると循環します。キャストや複合代入では、コンパイルできるが値が予想外になるケースがあります。

byte b = (byte) 128;
System.out.println(b); // -128

int max = Integer.MAX_VALUE;
System.out.println(max + 1); // -2147483648

「コンパイルエラー」なのか、「実行できるが値が循環する」のかを必ず分けてください。

整数除算と浮動小数点除算

割り算は、左辺にdoubleを置けば小数になる、という話ではありません。先に右辺の式が評価されます。

System.out.println(5 / 2);     // 2
System.out.println(5 / 2.0);   // 2.5
System.out.println(5.0 / 2);   // 2.5

int a = 5;
int b = 2;
double c = a / b;
System.out.println(c);         // 2.0

double c = a / b;は、先にint / intが計算されて2になり、その後doubleに代入されます。

0除算: 整数と浮動小数点で違う

整数の0除算は実行時例外です。浮動小数点の0除算は例外ではなく、InfinityやNaNになります。

// System.out.println(10 / 0);   // 実行時例外: ArithmeticException
System.out.println(10.0 / 0);    // Infinity
System.out.println(-10.0 / 0);   // -Infinity
System.out.println(0.0 / 0);     // NaN

booleanは数値型ではない

Javaのbooleanは、0や1と互換性がありません。C系言語の感覚で読むと間違えます。

boolean flag = true;
// boolean a = 1;  // NG
// int b = true;   // NG

if (flag) {
    System.out.println("OK");
}

// if (1) { }      // NG: 条件式は boolean でなければならない

また、booleanに大小比較は使えません。

boolean a = true;
boolean b = false;
System.out.println(a == b); // OK: false
// System.out.println(a < b); // NG

charは文字だが数値としても扱われる

charは文字を表しますが、内部的には数値コードを持ちます。演算するとintになります。

char a = 'A';
char b = 65;
System.out.println(a);      // A
System.out.println(b);      // A
System.out.println(a == b); // true
System.out.println(a + b);  // 130

文字リテラルはシングルクォート、文字列リテラルはダブルクォートです。

char c1 = 'A';      // OK
String s1 = "A";    // OK
// char c2 = "A";   // NG: Stringはcharではない
// String s2 = 'A';  // NG: charはStringではない

ラッパークラス・オートボクシング・アンボクシング

プリミティブ型には対応するラッパークラスがあります。intならInteger、longならLongです。

Integer a = 10; // オートボクシング: int -> Integer
int b = a;      // アンボクシング: Integer -> int
System.out.println(a + b); // 20

ただし、ボクシングと拡大変換の組み合わせには制限があります。

long a = 10;       // OK: int -> long
Integer b = 10;    // OK: int -> Integer
// Long c = 10;    // NG: int -> long -> Long とはならない
Long d = 10L;      // OK: long -> Long

参照がnullのラッパーをアンボクシングすると、実行時にNullPointerExceptionです。

Integer n = null;
// int x = n; // 実行時例外: NullPointerException

varは型を消す機能ではない

varはローカル変数の型推論です。型がなくなるわけではありません。コンパイル時に右辺から型が決まります。

var a = 10;       // int
var b = 10L;      // long
var c = 1.0;      // double
var d = "abc";    // String

// a = 10L;       // NG: aはintとして推論済み
コード可否理由
var a = 10;OK右辺からintと推論できる。
var a;NG右辺がなく型を推論できない。
var a = null;NGnullだけでは型を決められない。
var a = {1, 2, 3};NG配列初期化子だけでは不可。
var a = new int[] {1, 2, 3};OK右辺の型がint[]と分かる。
private var value;NGvarはフィールドに使えない。
void test(var x) {}NGメソッド引数には使えない。

配列型と初期化の基本

配列は参照型です。変数は配列オブジェクトへの参照を持ちます。

int[] a = new int[3];
System.out.println(a[0]); // 0

a[0] = 10;
int[] b = a;
b[0] = 20;
System.out.println(a[0]); // 20: 同じ配列を参照している

配列初期化子{...}は、宣言と同時に使うか、new 型[]と一緒に使います。

int[] a = {1, 2, 3};       // OK
int[] b;
// b = {1, 2, 3};          // NG
b = new int[] {1, 2, 3};   // OK

多次元配列は、必ずしも長方形である必要はありません。

int[][] data = new int[2][];
data[0] = new int[] {1, 2};
data[1] = new int[] {3, 4, 5};
System.out.println(data[1][2]); // 5

コンパイルエラー・実行時例外・正常実行を分ける

型問題では、結果を3分類で判定してください。

コード結果理由
byte b = 128;コンパイルエラーbyteの範囲外。
byte b = (byte)128;正常実行明示キャスト。値は-128。
int x = 10 / 0;コンパイルは可能だが実行時例外整数0除算。
double x = 10.0 / 0;正常実行Infinity。
String s = null; s.length();実行時例外null参照へのメソッド呼び出し。
int[] a = new int[0]; a[0] = 1;実行時例外配列の範囲外アクセス。

不可能・誤りパターン一覧

誤り理由修正例
float f = 1.0;1.0はdouble。float f = 1.0F;
long n = 2147483648;右辺がint範囲外。long n = 2147483648L;
byte b = a + 1;a + 1はint。byte b = (byte)(a + 1);
boolean b = 1;booleanと数値は互換性なし。boolean b = true;
if (1) {}条件式はboolean必須。if (n == 1) {}
char c = "A";"A"はString。char c = 'A';
var x;型推論できない。var x = 10;
var x = null;nullだけでは型が決まらない。String x = null;
int[] a; a = {1, 2};配列初期化子だけの代入は不可。a = new int[] {1, 2};
Long x = 10;int -> long -> Longとは変換されない。Long x = 10L;

読解ミニ演習

問1: コンパイル可否

byte b = 10 + 20; はコンパイルできますか。

答え: できます。10 + 20は定数式で、結果の30がbyte範囲内だからです。
問2: コンパイル可否

byte a = 10; byte b = a + 20; はコンパイルできますか。

答え: できません。a + 20は変数を含む式で、結果の型はintです。byteへ代入するにはキャストが必要です。
問3: 出力

byte b = 127; b += 1; System.out.println(b); の出力は何ですか。

答え: -128です。複合代入は暗黙キャストを含み、byteの範囲を超えて循環します。
問4: 出力

double d = 5 / 2; System.out.println(d); の出力は何ですか。

答え: 2.0です。先にint / intで2になり、その後doubleへ代入されます。
問5: 実行時例外

Integer n = null; int x = n; はどうなりますか。

答え: コンパイルはできますが、アンボクシング時にNullPointerExceptionが発生します。

試験での最終チェック

  • byte/short/charの演算結果は原則int。
  • 整数リテラルは基本int、小数リテラルは基本double。
  • 定数式の代入特例と、変数を含む式を混同しない。
  • +=は暗黙キャストを含むが、値が安全とは限らない。
  • ローカル変数にはデフォルト値がない。
  • booleanは数値型と互換性がない。
  • varはローカル変数専用。型がなくなるわけではない。
  • 参照型はオブジェクトそのものではなく参照を持つ。
