Java Silverの型変換・var・配列でコンパイルエラーになる例

型の問題は、見た目よりも機械的です。代入できるか、暗黙変換か、キャストが必要か、ローカル変数型推論が使える場所か、配列の形が正しいかを順に見ます。

1. 基本データ型の分類

分類型試験での注意
整数byte, short, int, long整数リテラルは原則 int。long は L を付ける。
浮動小数点float, double小数リテラルは原則 double。float は F が必要。
文字char数値としても扱えるが、文字列ではない。
真偽boolean数値と互換性なし。if (1) は不可。

2. 暗黙の型変換とキャスト

小さい範囲の値を大きい範囲の型へ入れる場合は暗黙変換できます。逆方向はデータ欠損の可能性があるためキャストが必要です。

int i = 10;
long l = i;       // OK: int -> long
float f = l;      // OK: long -> float。ただし精度には注意

long x = 10L;
int y = x;        // NG: long -> int は暗黙変換不可
int z = (int) x;  // OK: 明示的キャスト
落とし穴。 byteやshort同士の演算結果は基本的にintになります。byte b = 1; b = b + 1; はコンパイルエラーです。
byte b = 1;
b = b + 1;     // NG: b + 1 は int
b += 1;        // OK: 複合代入は暗黙のキャストを含む

3. リテラルの読み方

表記意味例
10intリテラルint a = 10;

10.0doubleリテラルdouble d = 10.0;

4. var の使える場所・使えない場所

var は型をなくす機能ではありません。右辺からローカル変数の型を推論するだけです。

場所可否理由
メソッド内のローカル変数OK右辺から型が決まる
拡張for文の変数OK配列・Iterableの要素型から推論
フィールドNGローカル変数ではない
メソッド引数NG引数の型は明示する必要がある
戻り値型NGメソッドシグネチャに使えない
null 初期化NG型を推論できない
var list = new ArrayList<String>(); // OK: ArrayList<String>
var n = 10;                         // OK: int
var x = null;                       // NG: 型を推論できない
private var value = 10;             // NG: フィールドでは使えない

5. 配列の宣言・生成・初期化

コード可否理由
int[] a = new int[3];OK3要素の配列を生成
int[] a = {1,2};NG一次元配列に二次元初期化子
int[] a = (1, 2, 3);OK宣言と同時なら省略初期化可
int[] a; a = (1, 2, 3);NG宣言後は new int[] が必要
a = new int[]{1,2};NG次元が合わない
int[][] array = new int[2][];
array[0] = new int[] {1, 2, 3};
array[1] = new int[] {4, 5};
// 二次元配列は内側の長さが揃っていなくてもよい

この記事を演習で使える知識に変える

対象レベル：Java基礎〜Silver共通。この範囲は、基礎理解とSilverの読解をつなぐ部分です。まず「コンパイルできるか」「実行時にどこまで進むか」「出力は何か」の順で考えます。

本文で必ず押さえること

  • リテラルの型を先に決める。整数は基本int、浮動小数点は基本doubleとして見る。
  • byte・short・charは、計算に入った瞬間にintへ昇格する場面がある。
  • varは「型を省略する記法」ではなく、初期化式から型を確定する仕組みとして読む。

この後の演習で見るポイント

  • 代入の右辺を先に評価し、結果の型が左辺へ入るか確認する。
  • 複合代入だけを例外扱いにせず、内部でキャスト相当の処理が入ると理解する。
  • コンパイル可否を出力結果より先に判定する。

ここで落ちる典型ミス

  • 数値が小さいからbyteへ入る、という雑な判断をしない。式になった時点で型が変わる。
  • varをフィールド、引数、戻り値型で使えると考えるのは誤り。
  • キャストを書けば常に安全、ではない。値の欠落や符号変化まで確認する。

演習前の小課題

次の3行を自分で判定します。byte b = 1; b = b + 1; b += 1; それぞれの違いを「式の型」で説明してください。
