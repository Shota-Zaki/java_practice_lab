型変換・var・配列でコンパイルエラーになる例

型の問題は、見た目よりも機械的です。代入できるか、暗黙変換か、キャストが必要か、ローカル変数型推論が使える場所か、配列の形が正しいかを順に見ます。

値・型・変数・リテラル・式・文を分ける

型の問題で詰まる原因の多くは、用語を全部まとめて「値」として扱ってしまうことです。Javaは、コード上の見た目ではなく、各式が持つ 型 で判定します。

用語意味例注意点
値実際に扱うデータ。10, true, 'A', オブジェクト見た目が似ていても型が違えば別物。
型値の扱い方を決める分類。int, char, String代入、演算、比較、メソッド呼び出しの可否を決める。
変数値または参照値を保持する名前付き領域。int n, String sプリミティブ型は値、参照型はオブジェクトへの参照を持つ。
リテラルコードに直接書いた値。10, 10L, "10", nullリテラル自体にも型がある。
式評価すると値と型を持つコード片。a + 1, str.length()byte + byte の式は int になる。
文処理の単位。int a = 1;, if (...) {}式と文を混同すると、代入式や戻り値の判定を誤る。
判定の芯: byte b = 1; は「1が小さいから入る」ではなく、「int リテラルだが、定数で範囲内なので特例で代入できる」と読みます。

プリミティブ型と参照型の根本差

int a = 10;       // 変数aは数値10そのものを持つ
String s = "abc"; // 変数sはStringオブジェクトへの参照を持つ
String t = s;     // 参照値がコピーされ、同じオブジェクトを見る
分類変数が持つものnull== の意味
プリミティブ型値そのもの代入不可値の比較
参照型オブジェクトへの参照代入可参照先が同じかの比較

null は空文字でも0でもありません。参照型変数が どのオブジェクトも参照していない状態 を表す特別なリテラルです。したがって null.length() のような呼び出しは、コンパイルは通っても実行時に NullPointerException になります。

この単元で最初に固定する結論

型・変数・リテラルの問題は、暗記だけで解くと崩れます。毎回、次の順番で機械的に判定します。

  1. 変数の型を見る。プリミティブ型か参照型か。
  2. 右辺の式の型を見る。リテラル、演算式、メソッド戻り値、null のどれか。
  3. 代入・比較・メソッド呼び出しが可能かを見る。
  4. 最後に、問題が聞いているのが コンパイルエラー、実行時例外、表示結果 のどれかを分ける。
雑に読むと落ちる点: 「値としては入りそう」ではなく「その式の型として代入できるか」で判断します。Javaではここを狙われます。

型とは「値の扱い方」を決めるルール

Javaの変数は、必ず型を持ちます。型は単なる入れ物の名前ではなく、次の3つを決めます。

保持できるもの

int は整数値、boolean は true/false、参照型はオブジェクトへの参照を保持します。

できる操作

int は算術演算できますが、boolean は数値演算できません。参照型はメソッド呼び出しできます。

代入できる範囲

型が違えば、値が似ていても代入できない場合があります。暗黙変換できるかが判定軸です。

問題では「同じ値に見えるが型が違う」コードがよく出ます。例として 1、1L、1.0、'1'、"1" はすべて意味が違います。

プリミティブ型と参照型の根本差

分類代表例変数が直接持つものnull比較の注意
プリミティブ型byte short int long float double char boolean値そのもの不可== は値比較
参照型String、配列、自作クラス、ラッパークラス、Objectオブジェクトへの参照可== は参照先の同一性比較
 
int a = 10;
String s = "java";
Object o = new Object();

// a は 10 という値そのものを持つ
// s と o は、オブジェクトそのものではなく参照を持つ

参照型変数は「オブジェクトの実体」ではなく「オブジェクトへ到達するための参照」を持ちます。この理解がないと、==、ガベージコレクション、null、メソッド引数の問題で誤答します。

8つのプリミティブ型をコード読解用に整理

全部を均等に覚える必要はありません。Javaで特に問われるのは byte、short、char、int、long、float、double、boolean の互換性です。

型分類範囲・特徴問題での狙われ方

short整数-32768〜32767範囲内の定数式は代入可。変数を含む式は int になりやすい。
char文字・整数的0〜65535相当'A'、65、演算後の int 昇格に注意。
int整数整数リテラルの標準何も付かない整数リテラルは基本 int。
long整数int より広いL 接尾辞。l は読みにくいので実務では避ける。
float浮動小数単精度1.0 は double なので float f = 1.0; は不可。
double浮動小数倍精度小数リテラルの標準。

booleanは数値ではない

C言語系の感覚で 0 を false、1 を true のように扱うとJavaでは即アウトです。

boolean a = true;
boolean b = false;

// boolean c = 1;    // NG
// int x = true;     // NG
// if (1) { }        // NG

if (a == true) {     // OK。ただし実務では if (a) が自然
    System.out.println("ok");
}

問題では if、while、三項演算子の条件部に数値を置くひっかけが出ます。条件部は必ず boolean です。

変数の宣言・初期化・代入を分ける

宣言、初期化、代入は似ていますが、問題では分けて読む必要があります。

int a;        // 宣言。ローカル変数ならまだ使えない
int b = 10;   // 宣言 + 初期化
b = 20;       // 代入
操作意味問題での注意
宣言変数名と型を決めるローカル変数は宣言だけでは使えない。
初期化最初の値を入れる使用前に必須。
代入既存変数に値を入れ直すfinal 変数は再代入不可。

フィールドとローカル変数の初期値

フィールドにはデフォルト値があります。ローカル変数にはありません。ここは非常に落としやすいです。

変数の種類宣言場所自動初期化例
フィールドクラス直下されるint は 0、boolean は false、参照型は null
ローカル変数メソッド・コンストラクタ・ブロック内されない使う前に全経路で初期化が必要
 
class Sample {
    int field;          // 0
    boolean flag;       // false
    String text;        // null

    void test() {
        int local;
        // System.out.println(local); // NG: 初期化されていない
        System.out.println(field);    // OK: 0
    }
}

確定代入: 分岐があるローカル変数

ローカル変数は「実行時にたまたま初期化されそう」では足りません。コンパイラが全経路で初期化済みだと判断できる必要があります。

void test(boolean condition) {
    int x;
    if (condition) {
        x = 10;
    }
    // System.out.println(x); // NG: condition が false の経路で未初期化
}
void test(boolean condition) {
    int x;
    if (condition) {
        x = 10;
    } else {
        x = 20;
    }
    System.out.println(x); // OK: どちらの経路でも初期化される
}
判断: ローカル変数の問題は「実行したらどうなるか」より前に、コンパイラが初期化済みと証明できるかを見ます。

参照型変数とnull

null は「参照先がない」ことを表す特別なリテラルです。参照型には代入できますが、プリミティブ型には代入できません。

String s = null;   // OK
Object o = null;   // OK
int[] array = null;// OK

// int n = null;   // NG
// boolean b = null; // NG

null を出力するだけなら "null" と表示されますが、メソッドを呼ぶと NullPointerException です。

String s = null;
System.out.println(s);  // null
// System.out.println(s.length()); // 実行時にNullPointerException

nullと空文字は完全に別物

値意味メソッド呼び出し表示
null参照先がない不可。呼ぶと NullPointerExceptionprintln では null と出る
""長さ0のStringオブジェクト可見た目は何も出ない
 
String a = null;
String b = "";

System.out.println(a);        // null
System.out.println(b);        // 空文字なので見た目は空
System.out.println(b.length());// 0
// System.out.println(a.length()); // NullPointerException

参照のコピー: オブジェクトはコピーされない

参照型変数を別の変数に代入しても、オブジェクト本体が複製されるわけではありません。参照がコピーされます。

StringBuilder a = new StringBuilder("A");
StringBuilder b = a;
b.append("B");
System.out.println(a); // AB

a と b は同じ StringBuilder を参照しています。これは配列、自作クラス、ArrayList でも同じです。

識別子: 変数名・クラス名に使えるもの

識別子は「名前」です。英字、数字、_、$ を使えますが、数字始まりは不可です。予約語も使えません。

例可否理由
int a1 = 10;OK数字始まりではない。
int $price = 100;OK$ は使える。ただし実務では避ける。
int _value = 1;OK_ を含む名前は可。
int 1st = 1;NG数字始まり。
int class = 1;NGclass は予約語。
int _ = 1;NG_ 単独は識別子として使えない。

予約語・文脈キーワードの読み方

予約語は識別子にできません。さらにJava 17では var、record、sealed、permits、non-sealed など、文脈によって特別な意味を持つ語があります。

// int class = 1;   // NG: 予約語
int var = 10;       // OK: var は予約語ではない
var value = 20;     // OK: ここではローカル変数型推論

// record は record 宣言では特別な意味を持つ
record Point(int x, int y) {}

var は「どこでも型推論できる魔法」ではありません。ローカル変数の宣言で、初期化式がある場合にだけ使うのが基本です。

charは文字だが、数値的にも扱われる

char は1文字を表しますが、内部的にはUnicodeコード値を持ちます。そのため、範囲内の整数定数を代入できる場合があります。

char a = 'A';   // OK
char b = 65;    // OK。'A' 相当
// char c = -1; // NG。char の範囲外

char x = 'A';
// char y = x + 1; // NG: x + 1 は int
char z = (char)(x + 1); // OK
注意: 'A' は char、"A" は String です。シングルクォートとダブルクォートの違いは頻出です。

配列も参照型

配列変数も参照型です。配列そのものではなく、配列オブジェクトへの参照を持ちます。

int[] a = {1, 2, 3};
int[] b = a;
b[0] = 99;
System.out.println(a[0]); // 99

a と b は同じ配列オブジェクトを参照します。配列問題で == が出たら、要素比較ではなく参照比較です。

ラッパークラス: プリミティブに見えて参照型

Integer、Long、Double、Boolean などはラッパークラスです。プリミティブ型に対応する参照型です。

Integer a = 10;  // オートボクシング
int b = a;       // アンボクシング

Integer n = null;
// int x = n;    // 実行時にNullPointerException

ラッパークラスは参照型なので null を持てます。しかし、int などに戻すアンボクシング時に null だと実行時例外になります。

よくある誤答パターン

値だけで判断する

byte b = 1 + 2; はOKですが、byte b = x + 2; は x が変数ならNGになりやすいです。式の型を見てください。

nullを文字列だと思う

null は文字列 "null" ではありません。参照先がない状態です。

参照コピーを値コピーと誤解する

参照型の代入ではオブジェクト本体は複製されません。同じオブジェクトを共有する場合があります。

ローカル変数に初期値があると思う

フィールドには初期値がありますが、ローカル変数にはありません。

ミニ演習

Q1. Object obj = null; System.out.println(obj); の結果は？

null と表示されます。println に null 参照を渡すこと自体は可能です。ただし obj.toString() を明示的に呼ぶと NullPointerException です。

Q2. char c = 65; はコンパイルできる？

できます。65 は char の範囲内の整数定数です。ただし int x = 65; char c = x; は暗黙には代入できません。

Q3. Integer n = null; int x = n; は？

コンパイルは通りますが、実行時に NullPointerException です。Integer から int へアンボクシングしようとして、参照先がないためです。

Q4. String a = ""; System.out.println(a.length()); は？

0 と表示されます。空文字は長さ0の String オブジェクトです。null とは違います。

次に読む: 数値・型昇格 / 関連: var・スコープ / 同一性・同値性

この単元の結論: 数値問題は「値」ではなく「式の型」で解く

数値・型昇格の問題は、見た目の値だけで判断すると高確率で間違えます。毎回、次の順番で処理してください。

  1. 各リテラルの型を決める。整数は基本 int、小数は基本 double。
  2. 演算があるかを見る。byte/short/char は演算で原則 int に昇格。
  3. 定数式か変数式かを見る。範囲内の定数式だけ特別に小さい型へ代入できる場合がある。
  4. 代入先の型へ暗黙変換できるかを見る。
  5. 複合代入なら、暗黙キャストを含むことを考慮する。
重要: byte b = 1 + 2; と byte b = a + 2; は同じではありません。前者は定数式、後者は変数を含む式です。

数値リテラルの基本型

まず、ソースコードに直接書いた数値が何型として扱われるかを決めます。

リテラル型例注意
整数リテラルint10範囲を超える整数は L が必要。
L/l付き整数long10L小文字 l は 1 と紛らわしい。
小数リテラルdouble1.0float へ直接代入不可。
F/f付き小数float1.0Ffloat へ入れるなら基本これ。
D/d付き小数double1.0D明示しなくても小数は基本 double。
 
int a = 10;       // 10 は int
long b = 10;      // int から long へ拡大変換
long c = 10L;     // 10L は long

// float d = 1.0; // NG: 1.0 は double
float e = 1.0F;   // OK
double f = 1.0F;  // OK: float から double へ拡大変換

整数リテラルの進数表記

同じ整数でも、先頭の書き方で基数が変わります。特に8進数の 0 始まりは見落としやすいです。

10進数1717通常の表記。
2進数0b10001170b または 0B。
8進数02117先頭が 0。8 と 9 は使えない。
16進数0x11170x または 0X。
 
int a = 17;

int c = 021;
int d = 0x11;
System.out.println(a + b + c + d); // 68

// int e = 089; // NG: 8進数に 8/9 は使えない

アンダースコアのルール

数値リテラルの途中には _ を置けます。読みやすくするための記号で、値には影響しません。ただし置ける場所に制限があります。

コード可否理由
int a = 123_456;OK数字の間。
int b = 1__2__3;OK連続しても数字の間なら可。
int c = _123;NGリテラルの先頭。
int d = 123_;NGリテラルの末尾。
float e = 3_.14F;NG小数点の直前。
float f = 3._14F;NG小数点の直後。
long g = 100_L;NG接尾辞 L の直前。
int h = 0x_FF;NG基数接頭辞 0x の直後。
int i = 0xF_F;OK数字の間。

拡大変換と縮小変換

代入では、右辺の型から左辺の型へ安全に変換できるかを見ます。広い型へは暗黙変換できますが、狭い型へは原則キャストが必要です。

int i = 10;
long l = i;       // OK: int から long
float f = l;      // OK: long から float
// int x = l;     // NG: long から int
int y = (int) l;  // OK: 明示キャスト

ただし、浮動小数では「範囲」と「精度」は別です。long から float への変換は許されますが、精度が失われる可能性はあります。問題では主にコンパイル可否を見ます。

byte・short・charは演算でintに昇格する

ここが最重要です。byte、short、char は、算術演算に入ると基本的に int に昇格します。

byte a = 1;
byte b = 2;
// byte c = a + b; // NG: a + b は int
int d = a + b;     // OK
byte e = (byte)(a + b); // OK: 明示キャスト
char c = 'A';
// char next = c + 1; // NG: c + 1 は int
char next = (char)(c + 1); // OK
覚え方: 小さい整数型の演算結果は、だいたい int になる。代入先が byte/short/char なら警戒。

二項数値昇格の優先順位

左右のオペランドの型が違う場合、広い型に寄せられます。大まかな優先順位は次の通りです。

含まれる型演算結果の型例
double が含まれるdouble1 + 2.0 は double
float が含まれるfloat1L + 2.0F は float
long が含まれるlong1 + 2L は long
それ以外の整数型intbyte + short は int
 
int a = 10;
long b = 20L;
// int c = a + b;  // NG: a + b は long
long d = a + b;    // OK

float f = 1.0F;
double g = 2.0;
// float h = f + g; // NG: f + g は double
double i = f + g;   // OK

定数式なら小さい型へ代入できる場合がある

右辺がコンパイル時に値を確定できる定数式で、代入先の範囲内なら、byte、short、char へ代入できます。

byte a = 10;        // OK
byte b = 10 + 20;   // OK: 定数式で範囲内
// byte c = 100 + 100; // NG: 200 は byte の範囲外

short s = 1000 + 2000; // OK
char ch = 65;          // OK

定数式の例外を知らないと、「整数リテラルはintだから全部NG」と誤判定します。逆に、変数を含む式までOKにしてしまうのも誤りです。

final変数とコンパイル時定数

final が付いていて、宣言時に定数式で初期化されている場合、コンパイル時定数として扱われることがあります。

final int x = 10;
byte a = x;       // OK: x はコンパイル時定数で範囲内
byte b = x + 20;  // OK

int y = 10;
// byte c = y;    // NG: y は変数
// byte d = y + 20; // NG
final int z;
z = 10;
// byte e = z; // NGになり得る: 宣言時初期化のコンパイル時定数ではない
コード読解用判断: final int x = 10; のように宣言と同時に定数で初期化されているかを確認します。

複合代入は暗黙キャストを含む

+=、-=、*=、/= などの複合代入は、左辺の型へのキャストを含むように動きます。

byte b = 1;
// b = b + 1; // NG: b + 1 は int
b += 1;       // OK: b = (byte)(b + 1) 相当

ただし、これは安全という意味ではありません。値が範囲を超えれば、オーバーフローした結果になります。

byte b = 127;
b += 1;
System.out.println(b); // -128

オーバーフロー: コンパイル可否と値を分ける

整数型は範囲を超えると循環します。キャストや複合代入では、コンパイルできるが値が予想外になることがあります。

byte b = (byte)128;
System.out.println(b); // -128

int max = Integer.MAX_VALUE;
System.out.println(max + 1); // -2147483648

Javaでは「コンパイルエラー」なのか「実行できるが値が変わる」のかを分けることが重要です。

整数除算と小数除算

割り算は、オペランドの型で結果が変わります。整数同士なら小数部は切り捨てです。

System.out.println(5 / 2);     // 2
System.out.println(5 / 2.0);   // 2.5
System.out.println(5.0 / 2);   // 2.5

int a = 5;
int b = 2;
double c = a / b;
System.out.println(c);         // 2.0

double c = a / b; は、先に int / int が計算されて 2 になり、その後 double に代入されます。2.5 にはなりません。

余り演算子 %

% は割った余りを返します。出力追跡問題でよく使われます。

System.out.println(10 % 3); // 1
System.out.println(10 % 5); // 0
System.out.println(3 % 10); // 3

符号付きの余りも出題される可能性はありますが、Javaではまず正の整数の基本を確実に処理できれば十分です。

charの演算と出力

char は文字ですが、演算では数値として扱われ、結果は原則 int です。

char c = 'A';
System.out.println(c);     // A
System.out.println(c + 1); // 66
System.out.println((char)(c + 1)); // B

println に char 単体を渡すと文字として出ますが、c + 1 は int なので数値として出ます。

String結合が混ざる数値問題

+ は数値加算にも文字列結合にも使われます。左から評価され、途中で String が出ると以後は文字列結合になります。

System.out.println(1 + 2 + "A"); // 3A
System.out.println("A" + 1 + 2); // A12
System.out.println("A" + (1 + 2)); // A3

これは数値型昇格そのものではありませんが、数値問題の出力選択肢で頻出です。

キャストの優先順位

キャストはどこにかかっているかを正確に見ます。

byte a = 10;
byte b = 20;

byte c = (byte)(a + b); // OK: 合計結果を byte にキャスト
// byte d = (byte)a + b; // NG: (byte)a は byte だが、+ b で int になる

キャストを見つけたら「式全体にかかっているのか、一部だけか」を括弧で確認します。

出力追跡の例: 型を表にして追う

byte b = 10;
int i = 20;
long l = 30L;
System.out.println(b + i + l);
処理型値
b + iint30
30 + llong60
printlnlong値の表示60

出力には L は付きません。L はソースコード上のリテラル表記です。

典型ひっかけコード集

1. byte b = 128;

コンパイルエラー。byte の範囲は -128〜127 です。

2. short s = 10 + 20;

OK。リテラルだけの定数式で、結果30は short の範囲内です。

3. short a = 10, b = 20; short c = a + b;

コンパイルエラー。a + b は int です。

4. float f = 10.0;

コンパイルエラー。10.0 は double です。10.0F にします。

5. int x = 2 * 3L;

コンパイルエラー。3L があるので演算結果は long です。

6. int x = 08;

コンパイルエラー。先頭 0 は8進数で、8進数に 8 は使えません。
