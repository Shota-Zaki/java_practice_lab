Java Goldのジェネリクスとワイルドカード

Java Goldのジェネリクスは、型安全性を守るための仕組みです。ワイルドカードは「読むだけ」「書く制限」を理解すると安定します。

1. List<String> は List<Object> ではない

List<String> strings = new ArrayList<>();
// List<Object> objects = strings; // コンパイルエラー

もし代入できると、objects.add(123) ができてしまい、Stringリストの安全性が壊れます。

2. ? extends T は読む側

List<? extends Number> nums = List.of(1, 2, 3);
Number n = nums.get(0); // OK
// nums.add(4);         // コンパイルエラー

上限境界は、取り出すと少なくとも Number だとわかります。一方、具体的なリスト型が不明なので追加はできません。

3. ? super T は書く側

List<? super Integer> list = new ArrayList<Number>();
list.add(10); // OK
Object x = list.get(0);

下限境界は Integer を追加できます。ただし、取り出し時の型は安全には Object までです。

4. PECS

Producer Extends, Consumer Super。値を提供する側なら extends、値を受け取る側なら super を使う、という目安です。

5. 型消去

ジェネリクスの型情報は主にコンパイル時に使われ、実行時には消去されます。そのため、new T() や instanceof List<String> のようなコードは使えません。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Gold。Goldでは、構文暗記よりもAPIの戻り値型、遅延実行、型推論、例外、リソース管理まで含めて判定します。Silverレベルの文法確認で止まると、ここでは足りません。

本文で必ず押さえること

  • APIの名前だけでなく、戻り値型を追う。Goldではここを外すと連鎖的に崩れる。
  • Streamは終端操作が来るまで処理が実行されない。
  • ラムダ・ジェネリクス・Optionalは、型がどこで確定するかを読む。

この後の演習で見るポイント

  • 中間操作と終端操作を分ける。
  • Optionalは値がある場合と空の場合の分岐を確認する。
  • ジェネリクスでは読み取り専用寄りか、追加できるかをワイルドカードから判断する。

ここで落ちる典型ミス

  • mapとflatMapを同じものとして扱う。
  • Collectorsの戻り値のコレクション型やMapのキーを確認しない。
  • parallelやExecutorを「速くなる機能」とだけ覚え、実行順序や終了処理を見落とす。

演習前の小課題

Streamの問題では、まず「元データ→中間操作→終端操作」の3列で表を作り、各段階の型を書いてください。
