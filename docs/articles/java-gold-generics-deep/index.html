ジェネリクスとワイルドカード

extends、super、PECS、型消去を、読み取りと書き込みの制限から整理します。

不変性を最初に固定する

StringはObjectのサブクラスですが、List<String>はList<Object>のサブタイプではありません。

List<String> strings = new ArrayList<>();
// List<Object> objects = strings; // コンパイルエラー

? extends T は読む側に強い

上限境界は、取り出した値を少なくともTとして扱えます。ただし、実際のリストがList<Integer>なのかList<Double>なのか不明なので、具体的な値を追加できません。

List<? extends Number> nums = List.of(1, 2, 3);
Number n = nums.get(0); // OK
// nums.add(4);         // コンパイルエラー

? super T は書く側に強い

下限境界は、TまたはTのサブクラスの値を追加できます。一方、取り出すときは安全にはObjectまでです。

List<? super Integer> list = new ArrayList<Number>();
list.add(10);       // OK
Object x = list.get(0); // 安全に言えるのはObjectまで
目的選ぶ境界理由
取り出して使いたい? extends T少なくともTとして読める。追加は制限される。
値を追加したい? super TTを受け取れることが保証される。読み取りはObject寄り。
読み書き両方を強くしたいワイルドカードなし具体的な型を固定する。
PECSは暗記語ではなく、Producerならextends、Consumerならsuperという読み替えです。値を「出す側」か「受け取る側」かで判断します。
