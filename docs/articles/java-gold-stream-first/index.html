Java GoldのStream API入門

map・filter・collectの前に、Streamが何をするものか整理します。

Streamはデータ処理の流れ

コレクションの要素を順に処理し、変換・抽出・集約するためのAPIです。

中間操作と終端操作

filterやmapは中間操作、forEachやcollectは終端操作です。終端操作が呼ばれるまで処理されません。

副作用を減らす

Streamは元のコレクションを直接変更するためのものではなく、処理結果を作る考え方に向いています。

例

list.stream()
    .filter(s -> s.length() >= 3)
    .map(String::toUpperCase)
    .forEach(System.out::println);

試験・学習での注意

  • 実行結果を見る前に、まずコンパイルできる形か確認します。
  • 似た構文でも、型・戻り値・アクセス修飾子で結果が変わります。
  • 暗記だけでなく、なぜその結果になるかを言語化してください。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Gold。Goldでは、構文暗記よりもAPIの戻り値型、遅延実行、型推論、例外、リソース管理まで含めて判定します。Silverレベルの文法確認で止まると、ここでは足りません。

本文で必ず押さえること

  • APIの名前だけでなく、戻り値型を追う。Goldではここを外すと連鎖的に崩れる。
  • Streamは終端操作が来るまで処理が実行されない。
  • ラムダ・ジェネリクス・Optionalは、型がどこで確定するかを読む。

この後の演習で見るポイント

  • 中間操作と終端操作を分ける。
  • Optionalは値がある場合と空の場合の分岐を確認する。
  • ジェネリクスでは読み取り専用寄りか、追加できるかをワイルドカードから判断する。

ここで落ちる典型ミス

  • mapとflatMapを同じものとして扱う。
  • Collectorsの戻り値のコレクション型やMapのキーを確認しない。
  • parallelやExecutorを「速くなる機能」とだけ覚え、実行順序や終了処理を見落とす。

演習前の小課題

Streamの問題では、まず「元データ→中間操作→終端操作」の3列で表を作り、各段階の型を書いてください。
