Stream API入門

Streamは「コレクションを別の形に処理する流れ」です。中間操作と終端操作を分ければ、出力や戻り値をかなり正確に追えます。

Streamを読む順番

Streamの問題は、メソッド名を知っているだけでは解けません。必ず次の順番で追います。

  1. 元データの型と要素型を確認する。
  2. 中間操作で要素が残るのか、別の型に変わるのかを確認する。
  3. 終端操作で何が返るのかを確認する。
  4. 終端操作がなければ、原則として処理は流れないと判断する。
分類代表例判定ポイント
元データList<String>, Stream<Integer>要素型を固定する。ここを曖昧にすると後続の型が崩れる。
中間操作filter, map, sorted, peek戻り値は基本的にStream。まだ処理結果そのものではない。
終端操作count, forEach, collect, reduceここで初めて処理が実行され、Streamではない結果に変わる。
List<String> names = List.of("Ann", "Bob", "Chris");
long count = names.stream()
    .filter(s -> s.length() >= 4)
    .count();
System.out.println(count); // 1

filterとmapを混同しない

filterは要素を残すか捨てるかを決めます。mapは要素を別の値へ変換します。どちらも中間操作なので、終端操作が来るまで処理は完了しません。

悪い判断

filterを「値を加工するメソッド」と読む。

良い判断

filterはboolean判定、mapは変換、と役割を分ける。

List<Integer> lengths = List.of("a", "bb", "ccc").stream()
    .filter(s -> s.length() >= 2)  // "bb", "ccc" が残る
    .map(String::length)           // 2, 3 に変わる
    .toList();
Streamの最初の壁は、処理内容よりも「いつ実行されるか」です。中間操作だけを見て出力を決めると、ほぼ確実に誤ります。
