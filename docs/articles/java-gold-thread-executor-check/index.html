ExecutorServiceとFutureの判定

Futureは「いつか終わる処理の結果」です。get、cancel、isDoneの意味を区別します。

Future.getは待つ

get()は結果が出るまで待機します。タスク内で例外が起きた場合、ExecutionExceptionとして受け取ります。

Future<String> f = executor.submit(() -> "done");
String result = f.get(); // 終了まで待つ
メソッド意味誤解しやすい点
isDone()完了しているか結果を取り出すわけではない。
cancel(true)キャンセルを試みる必ず停止できるとは限らない。
isCancelled()キャンセルされたか正常完了とは別。
get(timeout, unit)時間制限つきで待つ時間切れならTimeoutException。

CallableとRunnable

Callableは値を返し、例外を投げられます。Runnableは戻り値がありません。

Callable<Integer> c = () -> 100;
Runnable r = () -> System.out.println("run");
悪い判断

submitした直後に必ずタスクが完了していると思う。

良い判断

完了確認、getによる待機、shutdownの意味を分ける。
