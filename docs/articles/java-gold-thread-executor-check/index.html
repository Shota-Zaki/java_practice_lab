ExecutorService注意点

このページは、ExecutorServiceで特に間違えやすい判定だけを抜き出した確認用です。詳しい解説は並行処理 / Executor / Future完全整理に統合しています。

最低限の判定

項目正しい読み方
submitタスクを投入し、Futureを返す。
get完了まで待ち、結果を取得する。checked例外の処理が必要。
shutdown新規受付停止。投入済みタスクの即停止ではない。
shutdownNow停止を試みる。即停止保証ではない。
複数スレッド同期や待機がなければ出力順は固定しない。
ExecutorService ex = Executors.newSingleThreadExecutor();
Future<String> f = ex.submit(() -> "done");
System.out.println(f.get());
ex.shutdown();
悪い判断

submit直後にタスクが必ず完了していると考える。

良い判断

Future、get、shutdownを別々に読む。
