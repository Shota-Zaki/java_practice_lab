Java Silverのコンパイルエラーと実行時例外の分け方

試験対策と実装理解の両方で使えるよう、例を中心に整理します。

結論

Java Silverでは、まず「コンパイル時に型・文法・到達可能性で止まるのか」、それとも「実行して初めて例外になるのか」を分ける力が必要です。

コンパイルエラーの例

int x;
System.out.println(x); // ローカル変数未初期化

ローカル変数は自動初期化されないため、使用前に代入が必要です。

実行時例外の例

String s = null;
System.out.println(s.length());

型としてはStringなのでコンパイルは通りますが、実行時にNullPointerExceptionになります。
