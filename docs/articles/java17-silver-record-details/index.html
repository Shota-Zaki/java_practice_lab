Java17 Silverのrecordを丁寧に整理

recordのコンストラクタ、アクセサ、equalsをまとめます。

基本

record Item(String name, int price) {}

自動生成

  • アクセサ name()
  • equals/hashCode
  • toString

注意

recordのフィールドは実質finalで、あとから再代入できません。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java17 Silver。Java17 Silverでは、Java17で扱う文法差分を確認します。ただしGold寄りのStream、Optional、Collectors、NIO、並行処理、モジュールの深掘りはここでは扱いません。

本文で必ず押さえること

  • recordは「データを保持するクラスを短く書く構文」として見る。
  • コンポーネントから、private final相当のフィールド、アクセサ、equals/hashCode/toStringが作られる。
  • アクセサ名はgetNameではなくname()の形になる。

この後の演習で見るポイント

  • コンパクトコンストラクタで代入してよい場所と、してはいけない場所を分ける。
  • recordに追加できるメンバと、再代入できないフィールドを確認する。
  • 通常クラスのgetter命名に引っ張られない。

ここで落ちる典型ミス

  • recordのフィールドをあとから書き換えられると考える。
  • アクセサをgetX()だと思い込む。
  • recordをGoldの高度API問題と混ぜて学ぶ。Java17 Silverでは構文の成立を中心に見る。

演習前の小課題

record Item(String name) {} に対して、使える呼び出しが item.name() か item.getName() かを判定し、理由も説明してください。
