プリミティブ型の範囲

Silverで落とされやすい範囲・基本型・数値昇格をまとめます。

型サイズ範囲・特徴注意
byte8bit-128〜127整数リテラルが範囲内なら代入可。演算時はintへ昇格
short16bit-32768〜32767byte + short も基本intになる
char16bit0〜65535負数不可。文字でも数値として演算される
int32bit整数リテラルの基本型整数演算の標準
long64bitL/l接尾辞で明示小文字lは1と紛らわしい
float32bitF/f接尾辞が必要小数リテラルは標準でdouble
double64bit小数リテラルの基本型0.0除算は例外ではなくInfinity/NaN
boolean規定なしtrue / false数値型と相互変換しない

booleanboolean b = 1;不可数値で真偽値を表現しない

String メソッド戻り値・変更有無

Stringは不変です。戻り値を受け取らないと結果を捨てます。

メソッド戻り値元のString変更注意
concatStringしない戻り値を使う
replaceStringしない一致箇所がなければ同内容
substringStringしないendは含まない
trimStringしない前後空白を除去
toLowerCase / toUpperCaseStringしない戻り値を使う
charAtcharしない範囲外でStringIndexOutOfBoundsException
indexOfintしない見つからなければ-1
equalsbooleanしないnull側から呼ぶとNPE

StringBuilder 変更有無

StringBuilderは可変ですが、すべてのメソッドが本体を変更するわけではありません。

メソッド本体変更戻り値試験での注意
appendする自分自身連鎖呼び出し可
insertする自分自身挿入位置の範囲外に注意
deleteする自分自身endは含まない
replaceする自分自身endは含まない
reverseする自分自身元の並びが変わる
substringしないStringStringBuilder本体は変わらない
toStringしないStringStringとして比較・出力したいとき
equalsしないboolean内容比較しない

ArrayList / List.of / Arrays.asList 比較

作り方サイズ変更要素変更null注意
new ArrayList<>()可可可通常の可変List
Arrays.asList(...)不可可可固定サイズ。add/remove不可
List.of(...)不可不可不可不変。null不可
Stream.toList()不可不可実装依存に頼らないJava 16以降。不変List

Optional メソッド

Optionalは値がない可能性を明示する箱です。get()から考えない方が安全です。

メソッド空でない空注意
of(x)Optional[x]nullならNPEnull可能ならofNullable
ofNullable(x)Optional[x]emptynull吸収
get()値NoSuchElementException安易に使わない
orElse(v)値vvは先に評価される
orElseGet(s)値Supplier実行必要時だけ評価
orElseThrow()値NoSuchElementException引数ありは任意例外
map(f)Optional変換後empty戻り値nullならempty
flatMap(f)Optional変換後emptyfはOptionalを返す
filter(p)条件を満たせば維持empty条件不一致でもempty

Path / Files メソッド

Pathはパスの表現、Filesは実ファイル操作です。存在確認するかどうかが頻出です。

メソッド対象意味注意
normalize()Path. や .. を構文上整理ファイル存在確認はしない
toAbsolutePath()Path絶対パス化存在確認はしない
toRealPath()Path実在パス取得存在しないとIOException
resolve()Pathパス結合右辺が絶対パスなら右辺優先
relativize()Path相対パス計算絶対/相対を混ぜると例外
Files.exists()Files存在確認権限等で判断不能もあり得る
Files.list()Files直下をStream化closeが必要
Files.walk()Files再帰的にStream化削除は深い順が必要
Files.readString()Files文字列として読むIOException処理が必要

ExecutorService / Future

submitした例外はFuture.getで見える、という点が頻出です。

API役割戻り値注意
execute(Runnable)実行依頼なし例外取得しにくい
submit(Runnable)実行依頼Futuregetの結果はnull
submit(Callable)値を返す処理Future例外はExecutionExceptionに包まれる
Future.get()結果取得T完了まで待つ
Future.cancel()キャンセル試行boolean実行中停止は保証されない
shutdown()新規受付停止なし投入済みタスクは実行される
shutdownNow()停止試行未開始タスク一覧即時停止は保証されない
awaitTermination()終了待ちboolean先にshutdownが必要
invokeAll()複数実行List>全完了を待つ
invokeAny()複数実行Tどれか1つ成功した結果

キーワード意味注意
