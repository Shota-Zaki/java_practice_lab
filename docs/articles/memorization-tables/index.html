Java試験 覚える表

長い解説を読み直す前に、混同しやすい差分だけを表で確認します。Silverは文法・型・API、GoldはStream・Optional・Collectors・NIO・並行処理・モジュールを中心に整理します。

プリミティブ型の範囲

型サイズ範囲・特徴注意
byte8bit-128〜127整数リテラルが範囲内なら代入可
short16bit-32768〜32767演算時は基本intへ昇格
char16bit0〜65535負数不可。数値としても扱える
int32bit整数リテラルの基本型整数演算の基本
long64bitL/l接尾辞で明示小文字lは1と紛らわしい
float32bitF/f接尾辞が必要小数リテラルは標準でdouble
double64bit小数リテラルの基本型整数0除算と違いInfinityになる場合あり
boolean規定なしtrue / false数値型と互換性なし

StringBuilder 変更有無

メソッド本体変更戻り値試験での注意
appendする自分自身連鎖呼び出し可
insertする自分自身挿入位置の範囲外に注意
deleteする自分自身endは含まない
replaceする自分自身endは含まない
reverseする自分自身元の並びが変わる
substringしないStringここが頻出。StringBuilder本体は変わらない

List作成方法の違い

作り方サイズ変更要素変更null注意
new ArrayList<>()可可可通常の可変List
Arrays.asList(...)不可可可固定サイズ。add/remove不可
List.of(...)不可不可不可不変。null不可
Stream.toList()不可不可実装依存に頼らないJava 16以降。Collectors.toListと違う
collect(Collectors.toList())通常可通常可可仕様上、具体実装に依存しすぎない

Optional 主要メソッド

メソッド空でない空注意
of(x)Optional[x]nullならNPEnull可能ならofNullable
ofNullable(x)Optional[x]emptynull吸収
get()値NoSuchElementException安易に使わない
orElse(v)値vvは先に評価される
orElseGet(s)値Supplier実行必要時だけ評価
map(f)変換後Optionalempty戻り値nullならempty
flatMap(f)fがOptionalを返すemptyOptionalの二重化を避ける

Collectors 頻出比較

Collector用途戻り値の形注意
toList()List化List<T>Stream.toList()とは変更可否が違う
toSet()Set化Set<T>順序・実装型に依存しない
toMap(k,v)Map化Map<K,V>キー重複でIllegalStateException
toMap(k,v,merge)重複キー処理Map<K,V>merge関数の戻り値を確認
groupingBy分類Map<K,List<T>>下流Collectorで形が変わる
partitioningBytrue/false分類Map<Boolean,List<T>>キーはBoolean
mapping分類後に変換下流に依存単独ではなく下流Collectorとして使う

Path / Files 頻出比較

メソッド対象意味注意
normalize()Path. や .. を構文上整理ファイル存在確認はしない
toAbsolutePath()Path絶対パス化存在確認はしない
toRealPath()Path実在パス取得存在しないとIOException
resolve()Pathパス結合右辺が絶対パスなら右辺優先
relativize()Path相対パス計算絶対/相対を混ぜると例外
Files.list()Files直下をStream化closeが必要
Files.walk()Files再帰的にStream化削除は深い順が必要

Executor / Future 重要差分

API役割戻り値注意
execute(Runnable)実行依頼なし例外取得しにくい
submit(Runnable)実行依頼Future<?>getはnull
submit(Callable)値を返す処理Future<T>例外はExecutionExceptionに包まれる
shutdown()新規受付停止なし投入済みタスクは実行される
shutdownNow()停止試行未開始タスク一覧即時停止保証ではない
awaitTermination()終了待ちboolean先にshutdownが必要

module-info.java 主要キーワード

キーワード意味注意
requires他モジュールを読むコンパイル時・実行時に必要
requires transitive依存を利用側へ公開APIとして依存型を見せる場合
requires staticコンパイル時だけ必要実行時必須ではない
exportsパッケージを公開public型でもexportsしないと外から使えない
opensリフレクション用に開く通常のコンパイル参照とは別
usesサービス利用ServiceLoaderと関係
provides ... withサービス実装提供実装クラスを指定
