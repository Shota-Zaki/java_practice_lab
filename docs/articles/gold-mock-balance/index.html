Gold QA

Gold模試 出題バランスQA

模試A〜Dごとに、カテゴリ偏り、難易度偏り、同一問題の混入を確認し、Gold 11 / Gold 17 の模試を再配置した結果。

今回の判断

  • Gold SE 11は、模試A/Bが似すぎ、模試CはCollectors寄り、模試DはNIO・モジュール・並行処理寄りだったため、A〜Dへ再配置。
  • Gold SE 17は、カテゴリ比率は均等だったが、A〜D間で同一問題が多かったため、重複していた問題を分類別の変形問題に差し替え。
  • 各模試内では、同じ分類が固まらないようにカテゴリをラウンドロビンで並べ替え。
  • 解説方針は「コンパイル可否 → API仕様 → 出力・例外」の判定順に統一。

目標バランス

Gold SE 11は80問、Gold SE 17は60問。Stream / Collectors / Optional / ジェネリクス / ラムダ / NIO / 並行処理 / モジュールを中心に、JDBC・日時・総合読解を混ぜる構成。

残る注意点

模試の比率は整えたが、本番の完全な出題比率を再現するものではない。目的は、特定カテゴリだけで点が取れる状態を避け、Gold範囲を横断して読ませること。

次に詰めるなら、各模試の前半・中盤・後半で難易度が偏らないかを確認する。
