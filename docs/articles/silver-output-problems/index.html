出力結果問題の読み方

Java Silverで頻出の出力問題を、どう分解して読むかをまとめます。

出力問題の落とし穴

見た目は出力問題でも、実際にはコンパイルエラーや実行時例外で出力まで到達しない問題があります。まずコード全体が成立しているかを疑う姿勢が必要です。

文字列は不変

replaceやsubstringの戻り値を受け取っているかを見ます。受け取っていなければ元のStringは変わりません。

インクリメントの位置

num++は評価後に増え、++numは増えてから評価されます。式の中に複数出ると混乱しやすいです。

int n = 0;
System.out.println(n++ + "," + ++n); // 0,2

継承とフィールド

メソッドは実体の型で動的に選ばれますが、フィールドは参照型で決まります。この違いは出力問題で頻出です。
