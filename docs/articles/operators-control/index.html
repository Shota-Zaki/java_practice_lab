1. ++num と num++

書き方意味式の値
++num先に増やす増えた後の値
num++後で増やす増える前の値
int num = 0;
System.out.println(num++ + num + "," + ++num + num);
// num++ は 0 を使ってから num が 1
// 0 + 1 = 1
// 文字列連結後、++num で 2
// 結果: 1,22
単独文の num++; と ++num; はどちらも最終値は同じです。違いが出るのは、式の中で値を使うときです。

2. && と &、|| と |

演算子短絡評価右辺評価
&&あり左辺がfalseなら右辺は評価しない
&なし左辺に関係なく右辺も評価する
||あり左辺がtrueなら右辺は評価しない
|なし左辺に関係なく右辺も評価する
static boolean a() { System.out.print("A"); return false; }
static boolean b() { System.out.print("B"); return true; }

if (a() && b()) { } // Aだけ表示。b()は呼ばれない
if (a() & b())  { } // ABが表示。b()も呼ばれる

3. switch文とswitch式

項目switch文switch式
値を返すか返さない返す
フォールスルーcase:形式では起こる->では起こらない
yield通常使わないブロックで値を返すときに使う
網羅性値を返さないので緩いすべての経路で値が必要
int value = 2;
String result = switch (value) {
    case 1 -> "A";
    case 2 -> "B";
    default -> "C";
};

4. ループの抜け方

構文意味注意
break一番内側のswitchまたはループを抜けるswitch内のbreakはループではなくswitchを抜ける
continue現在の反復の残りをスキップして次へ後続コードが実行されない
returnメソッドを終了finallyがあれば先に実行される

5. do-whileの特徴

do-while は条件判定が後ろにあるため、最低1回は実行されます。

int i = 10;
do {
    System.out.println(i);
    i++;
} while (i < 5);
// 10が1回表示される

6. 到達不能コード

continue や return の直後に通常文を書くと、コンパイラが「絶対に実行されない」と判断してコンパイルエラーになることがあります。

for (int i = 0; i < 3; i++) {
    continue;
    System.out.println(i); // 到達不能コードでコンパイルエラー
}
