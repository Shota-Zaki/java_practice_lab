Javaの++iとi++、短絡評価、switch問題の解き方

出力問題は、評価順を雑に見ると外します。前置/後置インクリメント、短絡評価、switch、break/continue、ループ条件を機械的に追います。

1. ++num と num++

書き方意味式の値
++num先に増やす増えた後の値
num++後で増やす増える前の値
int num = 0;
System.out.println(num++ + num + "," + ++num + num);
// num++ は 0 を使ってから num が 1
// 0 + 1 = 1
// 文字列連結後、++num で 2
// 結果: 1,22
単独文の num++; と ++num; はどちらも最終値は同じです。違いが出るのは、式の中で値を使うときです。

2. && と &、|| と |

演算子短絡評価右辺評価
&&あり左辺がfalseなら右辺は評価しない
&なし左辺に関係なく右辺も評価する
||あり左辺がtrueなら右辺は評価しない
|なし左辺に関係なく右辺も評価する
static boolean a() { System.out.print("A"); return false; }
static boolean b() { System.out.print("B"); return true; }

if (a() && b()) { } // Aだけ表示。b()は呼ばれない
if (a() & b())  { } // ABが表示。b()も呼ばれる

3. switch文とswitch式

項目switch文switch式
値を返すか返さない返す
フォールスルーcase:形式では起こる->では起こらない
yield通常使わないブロックで値を返すときに使う
網羅性値を返さないので緩いすべての経路で値が必要
int value = 2;
String result = switch (value) {
    case 1 -> "A";
    case 2 -> "B";
    default -> "C";
};

4. ループの抜け方

構文意味注意
break一番内側のswitchまたはループを抜けるswitch内のbreakはループではなくswitchを抜ける
continue現在の反復の残りをスキップして次へ後続コードが実行されない
returnメソッドを終了finallyがあれば先に実行される

5. do-whileの特徴

do-while は条件判定が後ろにあるため、最低1回は実行されます。

int i = 10;
do {
    System.out.println(i);
    i++;
} while (i < 5);
// 10が1回表示される

6. 到達不能コード

continue や return の直後に通常文を書くと、コンパイラが「絶対に実行されない」と判断してコンパイルエラーになることがあります。

for (int i = 0; i < 3; i++) {
    continue;
    System.out.println(i); // 到達不能コードでコンパイルエラー
}

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java基礎〜Silver共通。この範囲は、基礎理解とSilverの読解をつなぐ部分です。まず「コンパイルできるか」「実行時にどこまで進むか」「出力は何か」の順で考えます。

本文で必ず押さえること

  • 演算子の優先順位だけでなく、左から評価される部分と短絡評価を分ける。
  • if、for、whileは「条件判定→本体→更新」の順番を紙に書くとミスが減る。
  • switch文とswitch式は、値を返すかどうか、break/yieldの役割が違う。

この後の演習で見るポイント

  • 条件式の左辺で結果が確定するか確認する。
  • ループは初期値、条件、更新後の値を1周ずつ表にする。
  • switchはフォールスルーの有無と、defaultの位置を確認する。

ここで落ちる典型ミス

  • &&と&、||と|を同じものとして扱う。
  • 後置インクリメントを、式の中でもすぐ増えた値として使ってしまう。
  • switch式でyieldが必要な場所と不要な場所を混同する。

演習前の小課題

a++ + ++a のような式は暗算で処理しないでください。評価された値と、変数に残る値を別々に書いて判定します。
