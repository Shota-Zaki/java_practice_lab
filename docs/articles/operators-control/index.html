演算子・制御構造完全整理

Silverの演算子・制御構造は、暗記よりも読解手順です。コンパイル可否、式の型、評価順、副作用、分岐、ループ、switch、到達不能コードを順番に切ると、出力問題の精度が上がります。

最初の結論: 出力を見る前に5段階で判定する

演算子・制御構造の問題で一番危険なのは、いきなり出力を計算することです。Javaは、実行結果以前にコンパイルで落ちる問題が多いです。

  1. 構文が成立するか。括弧、セミコロン、case、else、do-while末尾のセミコロンを見る。
  2. 条件式がbooleanか。if、while、for、三項演算子の条件部はbooleanだけ。
  3. 式の型が成立するか。byte + byte、boolean < boolean、switch(long)などを先に落とす。
  4. 評価される式だけを追う。&&、||、三項演算子、switch分岐で評価されない式を消す。
  5. 副作用を表にする。++、--、+=、代入式、メソッド呼び出しによる値の変化を最後に追う。
試験での現実: 「なんとなく上から読む」は弱いです。特に++、短絡評価、switchのフォールスルー、ループの更新式は、手順なしで読むと簡単に外します。

演算子の優先順位は頻出だけ正確に押さえる

優先順位表を全暗記する必要は薄いですが、Silverで頻出の階層は正確に必要です。迷ったら括弧を補って読めるかを確認します。

優先度分類例読解ポイント
高後置a++, a--値を渡してから変数を更新する。
高前置・単項++a, --a, !b, +a, -a先に変数を更新してから値を渡す。
中乗除余*, /, %整数除算、0除算、byte/short/charのint昇格に注意。
中加減・文字列結合+, -+は数値加算かString結合かを左から判定。
中比較<, <=, >, >=booleanや一般の参照型には使えない。
中等価==, !=プリミティブは値、参照型は参照値の比較。
低論理&&, ||, &, |, ^短絡するか、右辺も評価されるかを見る。
低三項条件 ? A : B条件はboolean。実行時に評価されるのは選ばれた側だけ。
低代入=, +=, -=右結合。代入式自体も値を持つ。
補足: 優先順位よりも、コンパイル可否の方が先です。優先順位どおりに計算できても、型が不正なら実行されません。

前置・後置インクリメントは「渡す値」と「残る値」を分ける

++と--は、式の値と変数の更新を同時に扱うため、出力問題の主戦場です。

形式に渡る値変数に残る値例
++a増えた後の値増えた後の値int b = ++a;
a++増える前の値増えた後の値int b = a++;
--a減った後の値減った後の値int b = --a;
a--減る前の値減った後の値int b = a--;
 
int a = 10;
int b = ++a; // aは11になり、bにも11が入る
int c = a++; // cには11が入り、その後aは12になる
System.out.println(a + "," + b + "," + c); // 12,11,11
甘い覚え方: 「前置は先、後置は後」だけでは足りません。「式に渡る値」と「変数の現在値」を分けてください。

複雑な++/--は表に分解する

同じ変数が1つの式に何度も出る場合は、暗算ではなく表で追います。左から評価し、各地点で式に渡る値と評価後の変数値を書きます。

int a = 10;
int b = a++ + a + a-- - a-- + ++a;
System.out.println(b);
部分式式に渡る値評価後のa理由
a++1011後置。10を渡してから増える。
a1111現在値。
a--1110後置。11を渡してから減る。
a--109後置。10を渡してから減る。
++a1010前置。9から10へ増やしてから渡す。

したがって、b = 10 + 11 + 11 - 10 + 10で、出力は32です。

代入式は値を持つ

=や+=は、処理をするだけでなく、式全体として値を持ちます。代入後の値が式の値になります。

int a = 3;
int b = a += 5;
System.out.println(a);     // 8
System.out.println(b);     // 8
System.out.println(a + b); // 16
  1. a += 5が評価され、aは8になる。
  2. a += 5という式全体の値も8。
  3. その8がbへ代入される。

複合代入は単純代入と完全には同じではない

a += bは概念的にはa = a + bですが、代入先型への暗黙キャストを含みます。ここはSilverの頻出です。

byte b = 1;
// b = b + 1; // NG: b + 1 は int
b += 1;       // OK: 内部的には b = (byte)(b + 1) のように扱われる

char c = 'A';
// c = c + 1; // NG: c + 1 は int
c += 1;       // OK
System.out.println(c); // B
要点: byte、short、charは演算で原則intになります。単純代入なら落ちるが、複合代入なら通る、という差を狙われます。

数値演算ではbyte/short/charがintへ昇格する

小さい整数型同士の演算でも、結果は基本的にintです。値が小さいかどうかではなく、式の型で判定します。

byte a = 10;
byte b = 20;
// byte c = a + b; // NG: a + b は int
int d = a + b;     // OK

final byte x = 10;
final byte y = 20;
byte z = x + y;    // OK: コンパイル時定数式で範囲内
式型理由
byte + byteint二項数値昇格。
short + shortint二項数値昇格。
char + charint文字コード値として演算。
int + longlong大きい型へ昇格。
float + doubledouble大きい型へ昇格。

整数除算・剰余・0除算

整数同士の割り算は小数部を切り捨てます。剰余%は余りを返します。0除算は整数と浮動小数で挙動が違います。

System.out.println(5 / 2);    // 2
System.out.println(5 % 2);    // 1
System.out.println(5 / 2.0);  // 2.5

// System.out.println(5 / 0); // 実行時例外 ArithmeticException
System.out.println(5.0 / 0);  // Infinity
System.out.println(0.0 / 0);  // NaN
判定順: 5 / 0は構文・型としてはコンパイル可能です。落ちるのは実行時です。一方、5 / falseはコンパイルエラーです。

オーバーフローはコンパイルエラーにならない

整数型の範囲を超える演算結果は循環します。実行時例外にもなりません。

int max = Integer.MAX_VALUE;
System.out.println(max);     // 2147483647
System.out.println(max + 1); // -2147483648

byte b = 127;
b += 1;
System.out.println(b);       // -128

ただし、リテラル自体がint範囲を超えている場合は別です。

// long a = 2147483648;  // NG: 右辺リテラルがint範囲外
long b = 2147483648L;    // OK

String結合は左から読む

+は数値加算にも文字列結合にも使われます。片方がStringになった地点から、以降の+は文字列結合として進みます。

System.out.println(1 + 2 + "A");     // 3A
System.out.println("A" + 1 + 2);     // A12
System.out.println("A" + (1 + 2));   // A3
System.out.println(1 + 2 + "A" + 3 + 4); // 3A34
出力問題のコツ: +を全部足し算だと思うと落ちます。左から現在の式が数値なのかStringなのかを毎回確定してください。

比較演算子と等価演算子を混同しない

演算子使える主な型boolean参照型注意点
<, <=, >, >=数値型・char不可不可大小比較。
==, !=互換性のあるプリミティブ型・参照型・boolean可可参照型では同一オブジェクト判定。
 
boolean x = true;
boolean y = false;
System.out.println(x == y); // false
// System.out.println(x <= y); // NG: booleanに大小比較はない

String a = new String("x");
String b = new String("x");
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

短絡評価と非短絡評価

&&と||は短絡評価します。&と|は、boolean同士に使った場合でも右辺を評価します。

式左辺右辺結果
false && exprfalse評価しないfalse
true || exprtrue評価しないtrue
false & exprfalse評価する右辺次第で副作用あり
true | exprtrue評価する右辺次第で副作用あり
 
int a = 0;
if (false && ++a > 0) { }
System.out.println(a); // 0

int b = 0;
if (false & ++b > 0) { }
System.out.println(b); // 1
よくある誤読: &と|はビット演算専用ではありません。boolean同士にも使えます。ただし短絡しません。

論理演算子の型と結果

booleanの論理演算と整数のビット演算は、同じ記号でも別の読み方になります。

System.out.println(true & false);  // false
System.out.println(true | false);  // true
System.out.println(true ^ true);   // false
System.out.println(true ^ false);  // true

System.out.println(5 & 3); // 1  : 0101 & 0011 = 0001
System.out.println(5 | 3); // 7  : 0101 | 0011 = 0111
System.out.println(5 ^ 3); // 6  : 0101 ^ 0011 = 0110

&&と||はboolean専用です。整数には使えません。

// int x = 1 && 2; // NG
int y = 1 & 2;     // OK: ビット演算

if / while / do-whileの条件はbooleanだけ

Javaでは、0をfalse、非0をtrueとして扱いません。条件式には必ずboolean式が必要です。

int n = 1;
// if (n) { }       // NG
if (n == 1) { }     // OK

// while (1) { }    // NG
while (n > 0) {
    n--;
}

do-whileは末尾のセミコロンが必要です。

int i = 0;
do {
    i++;
} while (i < 3); // セミコロン必須
System.out.println(i); // 3

if-elseは「近いif」に結びつく

中括弧を省略した場合、elseは直前の対応可能なifに結びつきます。見た目のインデントでは決まりません。

int a = 10;
int b = 20;
if (a > 0)
    if (b < 0)
        System.out.println("A");
    else
        System.out.println("B"); // このelseは内側のifに対応
// B
実務では: 中括弧を省略しない方が安全です。試験では、省略時の結合先を狙われます。

三項演算子は条件・評価側・型を分ける

三項演算子は条件 ? 式1 : 式2です。条件はbooleanです。実行時に評価されるのは選ばれた側だけです。

int a = 10;
int b = 20;
String result = a > b ? "A" : "B";
System.out.println(result); // B

int x = a > 0 ? 1 : 10 / 0;
System.out.println(x); // 1。選ばれない 10 / 0 は実行されない

ただし、選ばれない側でも、構文や型として不正ならコンパイルエラーです。

boolean ok = true;
// int n = ok ? 1 : "x"; // NG: intに代入できる共通の型ではない
Object o = ok ? 1 : "x"; // OK: IntegerまたはStringをObjectとして扱える

for文は初期化・条件・本体・更新の順番で読む

基本のforは、初期化が1回、その後は条件→本体→更新→条件の順に進みます。

for (int i = 0; i < 3; i++) {
    System.out.print(i);
}
// 012
部分実行タイミング省略
初期化式最初に1回だけ可
条件式各ループ開始前可。省略するとtrue扱い。
更新式本体実行後可
 
int i = 0;
for (; i < 3; ) {
    System.out.print(i);
    i++;
}
// 012

// for (;;) { } // 無限ループとして構文上はOK

for文の初期化部で宣言した変数のスコープ

forの初期化部で宣言した変数は、for文の中だけで有効です。

for (int i = 0; i < 3; i++) {
    System.out.print(i);
}
// System.out.println(i); // NG: iはスコープ外

複数変数を宣言する場合、同じ型で宣言します。型の違う変数宣言をカンマで混ぜることはできません。

for (int i = 0, j = 10; i < 3; i++, j--) {
    System.out.println(i + ":" + j);
}

// for (int i = 0, long j = 10; i < 3; i++) { } // NG

拡張for文は配列またはIterableを回す

拡張for文は、配列またはIterableを実装したオブジェクトを順に取り出します。ループ変数は各要素の値または参照値を受け取ります。

int[] nums = {1, 2, 3};
for (int n : nums) {
    n *= 10;
}
System.out.println(nums[0]); // 1。nを変えても配列要素は変わらない

参照型配列の場合、ループ変数が参照するオブジェクトの中身を変えれば、元のオブジェクトにも影響します。

StringBuilder[] arr = { new StringBuilder("A") };
for (StringBuilder sb : arr) {
    sb.append("B");
}
System.out.println(arr[0]); // AB

breakとcontinue

breakはループまたはswitchを抜けます。continueは現在の反復を終え、次の反復へ進みます。

for (int i = 0; i < 5; i++) {
    if (i == 1) continue;
    if (i == 3) break;
    System.out.print(i);
}
// 02

continueの後は、for文なら更新式へ進みます。while文なら条件式へ戻ります。

ラベル付きbreak / continue

ラベルを使うと、外側のループを対象にできます。ただし、ラベルは濫用すると読みにくいので、試験用の読解ポイントとして押さえます。

outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == 1 && j == 1) break outer;
        System.out.print(i + ":" + j + " ");
    }
}
// 0:0 0:1 0:2 1:0
outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) continue outer;
        System.out.print(i + ":" + j + " ");
    }
}
// 0:0 1:0 2:0

到達不能コード

Javaでは、明らかに到達できない文はコンパイルエラーになります。return、throw、break、continueの直後は要注意です。

void test() {
    return;
    // System.out.println("A"); // NG: unreachable statement
}

ただし、条件式の扱いには例外があります。while(false)の本体は到達不能ですが、if(false)の本体はコンパイル上許されます。

if (false) {
    System.out.println("A"); // OK
}

// while (false) {
//     System.out.println("B"); // NG: unreachable statement
// }
注意: 到達不能コードは「実行されないからOK」ではありません。Javaのコンパイラが到達不能と判断する形ならコンパイルエラーです。

switch文の対象型

switchの対象にできる型は限定されています。long、float、double、booleanは使えません。

使える使えない
byte, short, char, intlong, float, double, boolean
対応するラッパークラス、String、enum一般のObject、配列、Listなど
 
String s = "B";
switch (s) {
    case "A": System.out.println("A"); break;
    case "B": System.out.println("B"); break;
    default:  System.out.println("other");
}

// long n = 1L;
// switch (n) { } // NG

caseラベルは互換性のあるコンパイル時定数

caseラベルは、switch対象型と互換性があり、コンパイル時に値が確定している必要があります。

final int A = 1; // コンパイル時定数
final int B;
B = 2;           // finalでもコンパイル時定数ではない
int n = 1;

switch (n) {
    case A: break;    // OK
    // case B: break; // NG
}

値が重複するcaseもコンパイルエラーです。表記が違っても、値が同じなら重複です。

int n = 1;
switch (n) {
    case 1: break;
    // case 01: break; // NG: 01は8進数表記だが値は1
}

switch対象がbyteなどの場合、caseラベルの値が範囲外ならコンパイルエラーです。

byte b = 1;
switch (b) {
    case 1: break;      // OK
    // case 128: break; // NG: byteの範囲外
}

switch文はbreakがなければフォールスルーする

switch文は、一致したcaseから実行が始まり、breakなどで抜けるまで下へ流れます。defaultは最後でなくても構文上は可能です。

int n = 2;
switch (n) {
    case 1: System.out.print("A");
    case 2: System.out.print("B");
    default: System.out.print("C");
}
// BC
int n = 9;
switch (n) {
    default: System.out.print("D");
    case 1: System.out.print("A");
    case 2: System.out.print("B");
}
// DAB
試験対策: defaultは「どこに書いても最後に実行される」わけではありません。実行開始位置の候補になるだけです。開始後は下へ流れます。

switch対象がnullの場合

従来のswitch文で、対象式がnullになると実行時にNullPointerExceptionです。

String s = null;
// switch (s) {
//     case "A": System.out.println("A");
// }
// 実行時にNullPointerException

コンパイル可否と実行時例外を混同しないでください。Stringはswitch対象として使えるため、コンパイルは可能です。落ちるのは実行時です。

switch式とyield

Java 17範囲では、switch式も重要です。switch文は分岐処理ですが、switch式は値を返せます。

int n = 2;
String result = switch (n) {
    case 1 -> "one";
    case 2 -> "two";
    default -> "other";
};
System.out.println(result); // two

->形式はフォールスルーしません。ブロックで複数文を書く場合、値を返すにはyieldを使います。

int n = 3;
String result = switch (n) {
    case 1 -> "one";
    default -> {
        String text = "other";
        yield text;
    }
};
System.out.println(result); // other

switch式では、全ての可能性に対して値を返せる必要があります。

int n = 1;
// String s = switch (n) {
//     case 1 -> "one";
// }; // NG: nが1以外の場合の値がない

String t = switch (n) {
    case 1 -> "one";
    default -> "other";
}; // OK

break・yield・returnを混同しない

キーワード主な場所意味
breakループ、switch文対象のブロックから抜ける。
continueループ次の反復へ進む。
yieldswitch式switch式の値を返す。
returnメソッド、ラムダなどメソッドなどの呼び出し元へ戻る。
 
int n = 2;
String s = switch (n) {
    case 1 -> "A";
    default -> {
        // break "B"; // NG
        yield "B";    // OK
    }
};

コンパイルエラー・実行時例外・正常実行の典型

コード判定理由
if (1) {}コンパイルエラー条件式はbooleanのみ。
true < falseコンパイルエラーbooleanに大小比較はない。
byte b = 1; b = b + 1;コンパイルエラーb + 1はint。
byte b = 1; b += 1;正常複合代入は暗黙キャストを含む。
false && ++a > 0右辺未評価短絡評価。
false & ++a > 0右辺評価非短絡評価。
switch (1L)コンパイルエラーlongはswitch対象不可。
switch ((String)null)実行時例外対象式がnullでNullPointerException。
int n = 10 / 0;実行時例外整数0除算。
double d = 10.0 / 0;正常浮動小数点ではInfinity。

読解ミニ演習

Q1. 代入式

int a = 1; int b = a += 2; System.out.println(a + b);

答え: 6。a += 2でaは3。式全体の値も3なのでbも3。
Q2. booleanの大小比較

boolean x = true; boolean y = false; System.out.println(x > y);

答え: コンパイルエラー。booleanに大小比較は使えない。
Q3. 短絡評価

int a = 0; if (false && ++a > 0) {{}} System.out.println(a);

答え: 0。&&の左辺がfalseなので右辺は評価されない。
Q4. 非短絡評価

int a = 0; if (false & ++a > 0) {{}} System.out.println(a);

答え: 1。&は右辺も評価する。
Q5. switchのdefault位置

int n = 9; switch(n) {{ default: System.out.print("D"); case 1: System.out.print("A"); }}

答え: DA。defaultから開始し、breakがないため下へ流れる。
Q6. 拡張forとプリミティブ配列

int[] a = {{1}}; for (int n : a) {{ n = 9; }} System.out.println(a[0]);

答え: 1。ループ変数nを書き換えても配列要素は変わらない。
Q7. switch式

int n = 2; String s = switch(n) {{ case 1 -> "A"; default -> "B"; }}; System.out.println(s);

答え: B。switch式は値を返す。->形式はフォールスルーしない。
