オブジェクト指向で起こるコンパイルエラー

継承、オーバーライド、コンストラクタ、interfaceで起こるエラーをまとめます。

アクセス修飾子を弱くする

オーバーライドでは、親より弱いアクセス修飾子にできません。publicをprotectedやデフォルトに下げるとエラーです。

class A { public void test() {} }
class B extends A {
    // protected void test() {} // NG
}

戻り値だけ違う同名メソッド

引数が同じで戻り値だけ違うメソッドは定義できません。オーバーロードにもなりません。

class Sample {
    int test() { return 1; }
    // String test() { return "A"; } // NG
}

interface実装メソッドのpublic不足

interfaceの抽象メソッドは暗黙にpublicです。実装側でpublicを付けないとアクセスを弱めた扱いになります。

interface A { void run(); }
class B implements A {
    public void run() {} // publicが必要
}

this()とsuper()の位置

コンストラクタ内のthis()またはsuper()は必ず先頭文です。両方を同時に直接書くこともできません。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java基礎〜Silver共通。この範囲は、基礎理解とSilverの読解をつなぐ部分です。まず「コンパイルできるか」「実行時にどこまで進むか」「出力は何か」の順で考えます。

本文で必ず押さえること

  • 用語を覚えるだけでなく、コード上でどの判断に使う知識かを確認する。
  • 先にコンパイル可否、次に実行時の流れ、最後に出力を考える。
  • 自信がない箇所は、変数の型と値を紙に分けて書く。

この後の演習で見るポイント

  • 本文の例を少し変えても同じ判断ができるか確認する。
  • 選択肢を消去するときは、なぜ誤りかを言葉で説明する。
  • 関連する演習では、正解番号ではなく判断根拠を残す。

ここで落ちる典型ミス

  • 用語の意味だけ覚えて、コード読解に接続しない。
  • 「なんとなく見たことがある」で選択肢を選ぶ。
  • 解説を読んだあとに、自力で再判定しない。

演習前の小課題

本文のコード例を1行だけ変更し、コンパイル可否・実行時の流れ・出力をもう一度判定してください。
