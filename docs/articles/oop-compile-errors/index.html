オブジェクト指向で起こるコンパイルエラー

継承、オーバーライド、コンストラクタ、interfaceで起こるエラーをまとめます。

アクセス修飾子を弱くする

オーバーライドでは、親より弱いアクセス修飾子にできません。publicをprotectedやデフォルトに下げるとエラーです。

class A { public void test() {} }
class B extends A {
    // protected void test() {} // NG
}

戻り値だけ違う同名メソッド

引数が同じで戻り値だけ違うメソッドは定義できません。オーバーロードにもなりません。

class Sample {
    int test() { return 1; }
    // String test() { return "A"; } // NG
}

interface実装メソッドのpublic不足

interfaceの抽象メソッドは暗黙にpublicです。実装側でpublicを付けないとアクセスを弱めた扱いになります。

interface A { void run(); }
class B implements A {
    public void run() {} // publicが必要
}

this()とsuper()の位置

コンストラクタ内のthis()またはsuper()は必ず先頭文です。両方を同時に直接書くこともできません。
