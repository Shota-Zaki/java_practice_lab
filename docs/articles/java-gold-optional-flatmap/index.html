OptionalのmapとflatMap

OptionalのmapとflatMapは、戻り値が通常値なのかOptionalなのかで使い分けます。

mapは中身を変換する

mapに渡す関数は、TからRを返します。結果はOptional<R>になります。

Optional<String> s = Optional.of("java");
Optional<Integer> len = s.map(String::length); // Optional<Integer>

flatMapはOptionalの入れ子を防ぐ

関数の戻り値がすでにOptional<R>なら、mapではなくflatMapを使うと入れ子になりません。

Optional<User> user = findUser();
Optional<String> email = user.flatMap(User::email); // email() が Optional<String> を返す場合
関数の戻り値使うメソッド結果
RmapOptional<R>
Optional<R>flatMapOptional<R>
nullの可能性mapはempty扱いになる場合があるnullを返す設計自体を疑う
Optional<Optional<String>> wrong = user.map(User::email);
Optional<String> right = user.flatMap(User::email);
Optionalの入れ子が見えたら、flatMapの出番です。型を書いてみると判断できます。
