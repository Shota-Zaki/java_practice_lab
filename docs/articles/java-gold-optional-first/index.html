Optional入門

Optionalはnullを隠す魔法ではありません。「値がある場合だけ処理する」ための型です。

先に結論

この単元は、用語を覚えるだけでは安定しません。コードを見た瞬間に、型、戻り値、実行タイミング、副作用の有無を順番に切り分ける必要があります。

Optionalの基本姿勢

Optional<T>は、T型の値があるかもしれない、ないかもしれない状態を表します。中身を無条件に取り出す前提で使うなら、Optionalの意味がありません。

メソッド意味落とし穴
of(value)非null値から作るvalueがnullならNullPointerException。
ofNullable(value)nullならemptyにするnullの可能性がある入力で使う。
empty()空のOptional値は存在しない。
get()値を取り出す空ならNoSuchElementException。無条件使用は危険。
orElse空なら代替値引数は先に評価される。
orElseGet空ならSupplierを実行必要な時だけ代替値を作る。
Optional<String> name = Optional.ofNullable(getName());
String result = name.orElse("unknown");

orElseとorElseGetの違い

どちらも空の場合の代替値を返します。ただし、orElseの引数はOptionalに値があっても評価されます。orElseGetは必要な場合だけSupplierが呼ばれます。

String a = Optional.of("A").orElse(createDefault());
// createDefault() は呼ばれる

String b = Optional.of("A").orElseGet(() -> createDefault());
// createDefault() は呼ばれない
悪い判断

Optionalを使えばNullPointerExceptionが絶対に起きないと考える。

良い判断

Optionalを作る時点、getする時点、map内の戻り値でnullが絡む場所を確認する。

演習前チェック

演習に入る前に、次の4点を自分の言葉で確認してください。

  1. このコードはコンパイルできるか。
  2. 戻り値の型は何か。
  3. 処理はいつ実行されるか。
  4. 元のオブジェクトが変わるのか、新しい結果が返るだけなのか。
