Java Silverのクラス設計・static・recordの要点

クラス問題では、フィールド、ローカル変数、コンストラクタ、this、super、staticの区別が重要です。

1. フィールドとローカル変数

class Sample {
    private String value;

    public void setValue(String value) {
        value = "Hello, " + value; // ローカル変数を書き換えているだけ
    }
}

引数名とフィールド名が同じ場合、メソッド内で単に value と書くとローカル変数が優先されます。フィールドを指すには this.value が必要です。

public void setValue(String value) {
    this.value = "Hello, " + value; // フィールドを書き換える
}

2. コンストラクタ

コンストラクタはインスタンス生成時に実行されます。戻り値型を書きません。クラス名と同じ名前でなければ、ただのメソッドです。

class Sample {
    Sample() { }        // コンストラクタ
    void Sample() { }   // メソッド。コンストラクタではない
}

3. this() と super()

構文意味制約
this(...)同じクラスの別コンストラクタを呼ぶコンストラクタの先頭行のみ
super(...)スーパークラスのコンストラクタを呼ぶコンストラクタの先頭行のみ

this() と super() は同じコンストラクタ内で両方書けません。どちらも先頭行にしか置けないためです。

4. staticの基本

static はクラスに属するメンバです。インスタンスに属するフィールドやメソッドへ、staticメソッドから直接アクセスすることはできません。

class Sample {
    int value;
    static void test() {
        value++; // NG: staticからインスタンスフィールドへ直接アクセス不可
    }
}

アクセスしたいなら、インスタンスを作って参照を通してアクセスします。

static void test() {
    Sample s = new Sample();
    s.value++;
}

6. recordの基本

record はデータ保持用のクラスを簡潔に書く構文です。コンポーネントに対応するフィールド、アクセサ、コンストラクタ、equals、hashCode、toString が自動生成されます。

public record Item(String name, int price) { }

Item item = new Item("Book", 1000);
System.out.println(item.name());  // getName() ではない
System.out.println(item.price());
注意説明
アクセサ名name()。getName()ではない。
フィールド暗黙的に private final。
引数なしコンストラクタ自動では作られない。
値比較コンポーネント値でequals()が自動実装される。
