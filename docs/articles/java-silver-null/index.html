nullとNullPointerException

null代入とメソッド呼び出しの違いを整理します。

値・型・変数・リテラル・式・文を分ける

型の問題で詰まる原因の多くは、用語を全部まとめて「値」として扱ってしまうことです。Javaは、コード上の見た目ではなく、各式が持つ 型 で判定します。

用語意味例注意点
値実際に扱うデータ。10, true, 'A', オブジェクト見た目が似ていても型が違えば別物。
型値の扱い方を決める分類。int, char, String代入、演算、比較、メソッド呼び出しの可否を決める。
変数値または参照値を保持する名前付き領域。int n, String sプリミティブ型は値、参照型はオブジェクトへの参照を持つ。
リテラルコードに直接書いた値。10, 10L, "10", nullリテラル自体にも型がある。
式評価すると値と型を持つコード片。a + 1, str.length()byte + byte の式は int になる。
文処理の単位。int a = 1;, if (...) {}式と文を混同すると、代入式や戻り値の判定を誤る。
判定の芯: byte b = 1; は「1が小さいから入る」ではなく、「int リテラルだが、定数で範囲内なので特例で代入できる」と読みます。

プリミティブ型と参照型の根本差

int a = 10;       // 変数aは数値10そのものを持つ
String s = "abc"; // 変数sはStringオブジェクトへの参照を持つ
String t = s;     // 参照値がコピーされ、同じオブジェクトを見る
分類変数が持つものnull== の意味
プリミティブ型値そのもの代入不可値の比較
参照型オブジェクトへの参照代入可参照先が同じかの比較

null は空文字でも0でもありません。参照型変数が どのオブジェクトも参照していない状態 を表す特別なリテラルです。したがって null.length() のような呼び出しは、コンパイルは通っても実行時に NullPointerException になります。

nullは参照なし

String s = null;
System.out.println(s); // null
// s.length(); // NullPointerException

配列要素の初期値

String[] a = new String[2];
System.out.println(a[0]); // null
// a[0].concat("x"); // NullPointerException

試験注意

「nullを表示する」のか「nullに対してメソッドを呼ぶ」のかを分けてください。

例外処理の追跡手順

  1. try内でどの行まで進むかを見る。
  2. 発生した例外型を確認する。
  3. 上から順にcatchできるかを見る。
  4. finallyがあれば最後に実行する。
  5. catchされなければ、途中出力の後に異常終了する。
try {
    System.out.print("A");
    int x = 10 / 0;
    System.out.print("B");
} catch (ArithmeticException e) {
    System.out.print("C");
} finally {
    System.out.print("D");
}
// ACD

この単元をコードに適用する手順

  1. まず宣言を読む。型、修飾子、メソッド名、引数を固定する。
  2. 次に変更される値を探す。代入、メソッド呼び出し、副作用を表にする。
  3. 最後に出力を組み立てる。途中で止まるなら、そこまでの出力だけを残す。
理解した気になっているだけなら、似たコードに変えた瞬間に間違えます。本文の例を1行ずつ書き換えて、どこで結果が変わるか確認してください。

よくある誤解と修正

誤解正しい見方確認方法
見た目が似ていれば同じ動きになる型、戻り値、参照、修飾子が1つ違うだけで結果は変わる。行ごとに「型」「値」「参照先」をメモする。
実行結果だけ追えばよいコンパイルできなければ実行されない。例外で止まれば後続行は動かない。コンパイル可否 → 実行時例外 → 出力の順に判定する。
暗記した用語で判断できる用語は入口にすぎない。コード上のどの行に効いているかまで見る。本文の例を1行変更し、結果が変わる理由を説明する。

ミニドリル

次の形式で、自分の答えを文章にしてください。正解だけを出しても、弱点は潰れません。

  1. このコードはコンパイルできるか。
  2. 実行時に例外が起きるなら、どの行で起きるか。
  3. 正常に動くなら、出力は何か。
  4. 自分が間違えた場合、原因は「型」「参照」「戻り値」「アクセス」「例外」「評価順」のどれか。
学習時は、選択肢を見る前に自分の判定を作ってください。選択肢を先に見ると、もっともらしい誤答に引っ張られます。
