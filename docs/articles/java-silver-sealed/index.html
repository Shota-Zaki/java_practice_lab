sealedクラス・interfaceの頻出ルール

permits, final, sealed, non-sealedを整理します。

直接サブクラスの制約

sealedクラスの直接サブクラスは、final、sealed、non-sealedのいずれかを指定します。

sealed class A permits B {}
final class B extends A {}

permits外は不可

sealed class A permits B {}
final class C extends A {} // コンパイルエラー

試験注意

sealed問題は、許可されたクラスか、直接サブクラスの修飾子が適切かを確認します。

defaultメソッドの衝突を読む

  1. クラスに具象メソッドがあれば、それが優先。
  2. 複数interfaceのdefaultが衝突したら、実装クラスでoverride必須。
  3. interfaceのstaticメソッドは実装クラスへ継承されない。
  4. interfaceの抽象メソッドを実装するときはpublicを弱められない。
interface A { default void test(){ System.out.println("A"); } }
interface B { default void test(){ System.out.println("B"); } }
class C implements A, B {
    public void test() { A.super.test(); }
}

よくある誤解と修正

誤解正しい見方確認方法
見た目が似ていれば同じ動きになる型、戻り値、参照、修飾子が1つ違うだけで結果は変わる。行ごとに「型」「値」「参照先」をメモする。
実行結果だけ追えばよいコンパイルできなければ実行されない。例外で止まれば後続行は動かない。コンパイル可否 → 実行時例外 → 出力の順に判定する。
暗記した用語で判断できる用語は入口にすぎない。コード上のどの行に効いているかまで見る。本文の例を1行変更し、結果が変わる理由を説明する。

ミニドリル

次の形式で、自分の答えを文章にしてください。正解だけを出しても、弱点は潰れません。

  1. このコードはコンパイルできるか。
  2. 実行時に例外が起きるなら、どの行で起きるか。
  3. 正常に動くなら、出力は何か。
  4. 自分が間違えた場合、原因は「型」「参照」「戻り値」「アクセス」「例外」「評価順」のどれか。
学習時は、選択肢を見る前に自分の判定を作ってください。選択肢を先に見ると、もっともらしい誤答に引っ張られます。

最後に自分で確認すること

この記事を読み終えたら、次の3点を口頭で説明できるか確認します。説明できない場合は、まだ「読んだだけ」で、コードに適用できる状態ではありません。

  • この単元で最初に見るべき場所はどこか。
  • コンパイルエラー、実行時例外、正常出力をどの順番で分けるか。
  • 自分が間違えやすい引っかけは、型・参照・戻り値・アクセス・例外のどれか。
