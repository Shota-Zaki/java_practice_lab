ワイルドカード境界の判定

ワイルドカードは「何が入るか」ではなく「安全に何ができるか」で読みます。

List<?> は何でも入るListではない

List<?>は要素型が不明なListです。何でも入れられるのではなく、何が入っているかわからないため、基本的に追加できません。

List<?> list = List.of("A", "B");
Object value = list.get(0); // OK
// list.add("C");          // コンパイルエラー

上限境界と下限境界の比較

宣言読める型追加できる代表例
List<? extends Number>Number原則追加不可
List<? super Integer>ObjectIntegerやそのサブクラス
List<Integer>IntegerInteger

メソッド引数での使い分け

引数から値を読むだけならextends、引数へ値を追加するならsuperを考えます。

static double sum(List<? extends Number> nums) {
    double total = 0;
    for (Number n : nums) total += n.doubleValue();
    return total;
}

static void addInts(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}
悪い判断

extendsは継承しているから追加できる、と考える。

良い判断

extendsは「読める型の上限が決まる」と考える。具体的な実体型が不明なので追加できない。
