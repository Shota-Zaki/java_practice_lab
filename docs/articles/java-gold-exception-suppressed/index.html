try-with-resourcesと抑制された例外

close時の例外と本体例外の関係を整理します。

本体とcloseの両方で例外が起こる場合

tryブロック内で例外が発生し、さらにcloseでも例外が発生することがあります。

本体の例外が主になる

通常、tryブロック内の例外が主例外となり、close時の例外はsuppressedとして保持されます。

Goldでは詳細理解が必要

Silverでは閉じる順序が重要ですが、Goldでは抑制例外も理解しておくと強いです。

例

catch (Exception e) {
    for (Throwable t : e.getSuppressed()) {
        System.out.println(t);
    }
}

試験・学習での注意

  • 実行結果を見る前に、まずコンパイルできる形か確認します。
  • 似た構文でも、型・戻り値・アクセス修飾子で結果が変わります。
  • 暗記だけでなく、なぜその結果になるかを言語化してください。
