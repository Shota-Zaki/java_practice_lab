try-with-resourcesと抑制例外

try-with-resourcesでは、try本体の例外とclose時の例外が同時に起きることがあります。

リソースは逆順にcloseされる

try-with-resourcesで複数のリソースを宣言した場合、closeは宣言と逆順に呼ばれます。

try (A a = new A("A"); B b = new B("B")) {
    System.out.println("body");
}
// closeは B → A の順

抑制例外とは何か

try本体で例外が発生し、さらにcloseでも例外が発生した場合、主例外はtry本体の例外です。closeで発生した例外は抑制例外として主例外に保持されます。

try (Resource r = new Resource()) {
    throw new RuntimeException("body");
}
// closeでも例外が出ると、close側はsuppressedに入る
状況主例外抑制例外
try本体だけで例外try本体の例外なし
closeだけで例外closeの例外なし
try本体とcloseで例外try本体の例外closeの例外
複数closeで例外最初に主例外になったもの残りが抑制される
try-with-resourcesは「closeの順番」と「どの例外が主になるか」を分けて追います。
