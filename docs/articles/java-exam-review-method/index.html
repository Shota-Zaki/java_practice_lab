模試後の復習方法

模試は解いた後に点数より原因分析をします。

分類

  • 知識不足
  • 読み違い
  • 時間不足
  • コンパイル可否の見落とし

復習

不正解問題だけでなく、自信があったのに間違えた問題を優先します。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：学習手順・復習設計。この記事は知識そのものより、学習方法と復習方法を整えるための内容です。間違えた理由を分類し、次の演習で同じ判断ミスを減らすことを目的にします。

本文で必ず押さえること

  • インスタンスに属するものと、クラスに属するstaticなものを分ける。
  • コンストラクタは戻り値型を書かない。メソッドと見た目が似ていても別物。
  • オーバーロードはコンパイル時に引数型で決まる。実行時のオーバーライドとは分ける。

この後の演習で見るポイント

  • 呼び出し元の変数型、引数型、アクセス修飾子を順に確認する。
  • thisとsuperが何を指しているかを明確にする。
  • コンストラクタチェーンでは、最初にどのコンストラクタへ制御が移るかを見る。

ここで落ちる典型ミス

  • staticメソッドからインスタンスフィールドへ直接触れると思い込む。
  • privateメンバが継承先から自由に使えると考える。
  • 同名メソッドを見てすぐオーバーライドと判断する。引数が違えばオーバーロード。

演習前の小課題

同じ名前のメソッドが複数あるコードでは、先に「候補メソッド一覧」を作り、引数型でどれが選ばれるかを判定してください。
