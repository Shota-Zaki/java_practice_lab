Java SilverのString不変性で落ちるパターン

replace・substring・concatの戻り値を受け取らない問題を例で整理します。

Stringは変更されない

Stringのメソッドは、元の文字列を変更せず、新しい文字列を返します。

戻り値を捨てると変化しない

str.replace("a", "b"); と書いてもstr自体は変わりません。

StringBuilderとの違い

StringBuilderはappendなどで自身の中身を変更します。この違いは頻出です。

例

String s = "abc";
s.replace("a", "x");
System.out.println(s); // abc
s = s.replace("a", "x");
System.out.println(s); // xbc

試験・学習での注意

  • 実行結果を見る前に、まずコンパイルできる形か確認します。
  • 似た構文でも、型・戻り値・アクセス修飾子で結果が変わります。
  • 暗記だけでなく、なぜその結果になるかを言語化してください。
