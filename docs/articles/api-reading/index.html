Java API仕様の読み方

頻出メソッドを丸暗記だけでなく、仕様として読むための観点を整理します。

見るべき5点

メソッド名、引数、戻り値、オブジェクトを変更するか、例外が発生する条件の5つを確認します。

Stringの例

Stringは不変です。replace、substring、concatは新しい文字列を返します。戻り値を受け取らなければ元の変数の参照先は変わりません。

ArrayListの例

addは要素を追加します。setは置換して古い値を返します。removeは引数がintかObjectかで意味が変わります。

ArrayList<Integer> list = new ArrayList<>();
list.add(1);
list.add(2);
list.remove(1); // 添字1を削除。値1ではない

試験での注意

メソッド名を知っていても、戻り値や副作用を間違えると落ちます。特にString、StringBuilder、ArrayList、Arrays、Integerは優先して整理してください。
