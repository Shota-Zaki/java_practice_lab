Java Silverのswitch文とswitch式

switchは、文と式の違い、フォールスルー、yield、caseラベルの型が頻出です。

1. switch文はフォールスルーする

int n = 1;
switch (n) {
    case 1: System.out.print("A");
    case 2: System.out.print("B"); break;
    default: System.out.print("C");
}

break がないため case 2 に流れ、出力は AB です。

2. switch式は値を返す

String result = switch (n) {
    case 1 -> "A";
    case 2 -> "B";
    default -> "C";
};

switch式は式なので、結果を変数へ代入できます。末尾のセミコロンも必要です。

3. yieldが必要なケース

String result = switch (n) {
    case 1: yield "A";
    default: yield "Z";
};

コロン形式のswitch式では、値を返すために yield を使います。

4. caseラベルは定数

int x = 1;
switch (n) {
    case x: break; // コンパイルエラー
}

case にはコンパイル時定数が必要です。final int x = 1; なら使える場合があります。

5. nullの扱い

String s = null;
switch (s) {
    case "A": System.out.println("A");
}

Java Silverの範囲では、null をswitch対象にすると実行時例外になる問題として出やすいです。

確認ポイント

  • break があるか。
  • 文なのか式なのか。
  • -> なのか : なのか。
  • 全ケースで値を返しているか。
  • case が定数か。
