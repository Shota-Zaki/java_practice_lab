Java Silver 頻出

Java Silverのswitch文とswitch式

switchは、文と式の違い、フォールスルー、yield、caseラベルの型が頻出です。

1. switch文はフォールスルーする

int n = 1;
switch (n) {
    case 1: System.out.print("A");
    case 2: System.out.print("B"); break;
    default: System.out.print("C");
}

break がないため case 2 に流れ、出力は AB です。

2. switch式は値を返す

String result = switch (n) {
    case 1 -> "A";
    case 2 -> "B";
    default -> "C";
};

switch式は式なので、結果を変数へ代入できます。末尾のセミコロンも必要です。

3. yieldが必要なケース

String result = switch (n) {
    case 1: yield "A";
    default: yield "Z";
};

コロン形式のswitch式では、値を返すために yield を使います。

4. caseラベルは定数

int x = 1;
switch (n) {
    case x: break; // コンパイルエラー
}

case にはコンパイル時定数が必要です。final int x = 1; なら使える場合があります。

5. nullの扱い

String s = null;
switch (s) {
    case "A": System.out.println("A");
}

Java Silverの範囲では、null をswitch対象にすると実行時例外になる問題として出やすいです。

確認ポイント

  • break があるか。
  • 文なのか式なのか。
  • -> なのか : なのか。
  • 全ケースで値を返しているか。
  • case が定数か。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java11 / Java17 Silver共通。Silver共通範囲では、バージョン固有構文に逃げず、型、参照、制御構文、クラス、継承、例外の基本ルールだけで判定します。Java17固有の記事はJava17 Silver側に分けています。

本文で必ず押さえること

  • 演算子の優先順位だけでなく、左から評価される部分と短絡評価を分ける。
  • if、for、whileは「条件判定→本体→更新」の順番を紙に書くとミスが減る。
  • switch文とswitch式は、値を返すかどうか、break/yieldの役割が違う。

この後の演習で見るポイント

  • 条件式の左辺で結果が確定するか確認する。
  • ループは初期値、条件、更新後の値を1周ずつ表にする。
  • switchはフォールスルーの有無と、defaultの位置を確認する。

ここで落ちる典型ミス

  • &&と&、||と|を同じものとして扱う。
  • 後置インクリメントを、式の中でもすぐ増えた値として使ってしまう。
  • switch式でyieldが必要な場所と不要な場所を混同する。

演習前の小課題

a++ + ++a のような式は暗算で処理しないでください。評価された値と、変数に残る値を別々に書いて判定します。
