Streamパイプラインの追跡

複数の中間操作が並ぶコードを、左から丸暗記せず、要素単位で追跡する方法を整理します。

パイプラインは「全filter→全map」ではなく、要素が順に流れる

実装上の最適化はありますが、学習では各要素が上流から下流へ通過すると考えると追いやすくなります。

List<String> result = Stream.of("ant", "bear", "cat")
    .filter(s -> s.length() >= 4)
    .map(String::toUpperCase)
    .toList();
// [BEAR]
要素filtermap結果
"ant"false実行されない除外
"bear"true"BEAR"残る
"cat"false実行されない除外

peekは確認用。結果を作る主役にしない

peekは主にデバッグ用途です。副作用を書けますが、終端操作がなければ実行されず、短絡操作があれば一部しか実行されないことがあります。

Stream.of("A", "B", "C")
    .peek(System.out::print)
    .limit(2)
    .count(); // A B だけが対象になる可能性がある
悪い判断

peekに書いた処理が必ず全要素分実行されると考える。

良い判断

終端操作と短絡操作を確認して、実際に流れる要素だけを見る。
