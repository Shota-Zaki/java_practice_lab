並行処理入門

ExecutorService、Future、shutdownは、タスク投入、結果取得、終了処理の3つに分けて読みます。

ExecutorServiceはタスクを実行する場所

submitでタスクを渡すと、結果を表すFutureが返ります。shutdownは新規タスク受付を止めるだけで、投入済みタスクを即座に消すわけではありません。

ExecutorService ex = Executors.newSingleThreadExecutor();
Future<Integer> f = ex.submit(() -> 10 + 20);
System.out.println(f.get()); // 30
ex.shutdown();
操作意味注意
executeRunnableを投入戻り値なし。例外の扱いに注意。
submitCallable/Runnableを投入Futureが返る。
get結果を待って取得例外を投げる可能性がある。待機も発生する。
shutdown新規受付停止投入済みタスクは実行される。
shutdownNow停止を試みる必ず即停止する保証ではない。

並行処理の出力順は固定しない

複数スレッドで println するコードでは、順序が保証されているか明示的な同期や get による待機がなければ、出力順を決めつけてはいけません

ExecutorService ex = Executors.newFixedThreadPool(2);
ex.submit(() -> System.out.print("A"));
ex.submit(() -> System.out.print("B"));
ex.shutdown();
// ABとは限らない
並行処理は、正確な順序が保証されている箇所と、保証されていない箇所を分けるのが最重要です。
