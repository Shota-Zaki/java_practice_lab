Java Bronzeのif文とループを読む

Bronze向けにif・for・whileの基本と、初学者が間違えやすい条件式をまとめる。

if文は条件式を見る

if文では、条件式がtrueならブロックが実行されます。elseは直前の未対応のifに結びつきます。

if (x > 0) {
    System.out.println("plus");
} else {
    System.out.println("zero or minus");
}

ループで見る場所

  • 初期値
  • 継続条件
  • 更新式
  • break / continueの有無

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Bronze。Bronzeでは、細かい新機能よりも「文の実行順序」「変数の値」「条件式の真偽」を正確に追うことを優先します。record、sealed、Streamなどの高度な構文・APIはここでは追いません。

本文で必ず押さえること

  • 演算子の優先順位だけでなく、左から評価される部分と短絡評価を分ける。
  • if、for、whileは「条件判定→本体→更新」の順番を紙に書くとミスが減る。
  • switch文とswitch式は、値を返すかどうか、break/yieldの役割が違う。

この後の演習で見るポイント

  • 条件式の左辺で結果が確定するか確認する。
  • ループは初期値、条件、更新後の値を1周ずつ表にする。
  • switchはフォールスルーの有無と、defaultの位置を確認する。

ここで落ちる典型ミス

  • &&と&、||と|を同じものとして扱う。
  • 後置インクリメントを、式の中でもすぐ増えた値として使ってしまう。
  • switch式でyieldが必要な場所と不要な場所を混同する。

演習前の小課題

a++ + ++a のような式は暗算で処理しないでください。評価された値と、変数に残る値を別々に書いて判定します。
