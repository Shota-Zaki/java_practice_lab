模試後の復習ルート

模試は点数を出して終わりではありません。間違えた問題を、通常演習と学習記事へ戻して潰すための手順です。

復習の基本順序

間違えた問題だけを先に見る

全問を読み返すより、まず不正解・迷った問題を抽出します。正解した問題でも、理由を説明できなかったものは見直し対象です。

タグで弱点を分ける

Stream、Collectors、ジェネリクス、例外、継承、コマンドなど、論点名で分けます。単に「ミス」と処理すると再発します。

通常演習へ戻る

模試の同じ問題を暗記するのではなく、対応する通常演習で似た判定を反復します。

記事で根拠を確認する

解説だけで終わらせず、該当記事でAPI仕様・コンパイル条件・例外条件を確認します。

もう一度、間違えた問題だけ解く

最後に間違えた問題だけ再挑戦します。解けたかではなく、判定順を説明できたかを確認します。

Goldでよく戻る場所

記事Stream API 詳細中間操作、終端操作、遅延評価、reduce、collect。記事Collectors完全整理toMap、groupingBy、partitioningBy、下流Collector。記事ジェネリクス不変性、ワイルドカード、PECS、型消去。記事module-info.javarequires、exports、opens、module-path。

Silverでよく戻る場所

記事型・変数・リテラル型変換、定数式、数値昇格、var。記事演算子・制御構造評価順、短絡評価、switch、ループ。記事String・配列・ArrayList不変性、参照、remove、List.of。記事継承・interfaceoverride、field hiding、polymorphism。

復習時に見るべき判定

間違え方戻る場所確認すること
コンパイルエラーを見落とすコンパイルエラーの読み方宣言型、スコープ、checked例外、static文脈
出力結果を間違えるコード読解・演算子・String評価順、参照、可変/不変
API仕様を忘れる覚える表戻り値、変更有無、例外条件
Goldの複合問題で崩れるGold記事＋通常演習Stream/Optional/Collectors/Genericsの型の流れ
