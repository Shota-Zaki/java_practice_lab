頻出ひっかけパターン総まとめ

問題で落ちるのは、派手な論点よりも小さな違いです。似た構文を横並びで整理します。

比較正しい理解危険な誤解
== / equals()==は参照同一性。equals()は実装次第。中身が同じなら==もtrue。
++x / x++前置は増やしてから使う。後置は使ってから増やす。どちらも常に同じ。
&& / &&&は短絡評価。&は右辺も評価。booleanでは完全に同じ。
String / StringBuilderStringは不変。StringBuilderは可変。replaceやappendはどちらも元を変える。
length / length() / size()配列はlength、Stringはlength()、Listはsize()。全部length()。
ArrayList.remove(1) / remove(Integer.valueOf(1))前者はインデックス削除、後者は要素削除。どちらも値1を削除。
List.of() / Arrays.asList()List.ofはnull不可・変更不可。Arrays.asListは固定サイズ。どちらも普通のArrayList。
オーバーロード / オーバーライドオーバーロードは引数違い。オーバーライドは親メソッドの再定義。戻り値だけ違えばオーバーロード。
フィールド / メソッドフィールドは変数の型、オーバーライドメソッドは実体の型。どちらも実体の型。
this() / super()どちらもコンストラクタ先頭行のみ。途中で自由に呼べる。
switch文 / switch式式は値を返す。->はフォールスルーなし。switchならすべてbreakが必要。
checked / uncheckedcheckedは処理義務あり。RuntimeException系は処理義務なし。例外は全部catch必須。

見直し時の問い

  1. これは値比較か、参照比較か。
  2. このメソッドは元のオブジェクトを変えるか、新しい値を返すだけか。
  3. この変数はフィールドか、ローカル変数か。
  4. この呼び出しはオーバーロードか、オーバーライドか。
  5. この例外はコンパイル時に処理を要求されるか。

コンパイルエラーと実行時例外を混ぜない

分類例判断
コンパイルエラー型不一致、未初期化ローカル変数、アクセス不可実行前に止まる。出力はない。
実行時例外null参照、配列範囲外、キャスト失敗そこまでの出力は残る可能性がある。
正常出力型・アクセス・例外がすべて通る最後に評価順を追う。

この単元をコードに適用する手順

  1. まず宣言を読む。型、修飾子、メソッド名、引数を固定する。
  2. 次に変更される値を探す。代入、メソッド呼び出し、副作用を表にする。
  3. 最後に出力を組み立てる。途中で止まるなら、そこまでの出力だけを残す。
理解した気になっているだけなら、似たコードに変えた瞬間に間違えます。本文の例を1行ずつ書き換えて、どこで結果が変わるか確認してください。
