Java Silverの数値昇格で間違える理由

byte・short・charの計算結果、複合代入、キャストを例で整理します。

byteやshortは計算でintになる

byte同士、short同士の計算でも結果は基本的にintです。そのためbyteへそのまま代入できません。

複合代入は暗黙キャストを含む

b += 1 は b = (byte)(b + 1) に近い処理を行います。b = b + 1 とはコンパイル可否が変わります。

charも数値として計算される

charは文字ですが、計算では数値として扱われます。出力時に文字として出るか数値として出るかは式の型で変わります。

例

byte b = 1;
// b = b + 1; // コンパイルエラー
b += 1;       // OK

試験・学習での注意

  • 実行結果を見る前に、まずコンパイルできる形か確認します。
  • 似た構文でも、型・戻り値・アクセス修飾子で結果が変わります。
  • 暗記だけでなく、なぜその結果になるかを言語化してください。
