コンストラクタで落ちるパターン完全整理

コンストラクタは「new時に呼ばれる初期化処理」です。戻り値型、this()/super()、暗黙super、初期化順、継承されない性質をまとめて判定します。

コンストラクタとは

コンストラクタはインスタンス生成時に実行される初期化処理です。メソッドに似ていますが、メソッドではありません。

コンストラクタクラス名と同じ名前で、戻り値型を書かない。new 時に実行される。
初期化インスタンスを使える状態にすること。フィールドへ値を入れる処理が代表。
デフォルトコンストラクタコンストラクタを1つも書かない場合だけコンパイラが追加する引数なしコンストラクタ。
暗黙superコンストラクタ先頭に何も書かない場合に追加される super()。
class Sample {
    Sample() { }      // コンストラクタ
    void Sample() { } // メソッド。コンストラクタではない
}
戻り値型を書いたらコンストラクタではありません。void も戻り値型です。

判定順

  1. クラス名と同じ名前か。
  2. 戻り値型を書いていないか。
  3. this(...) または super(...) が先頭行か。
  4. 親クラスに呼べるコンストラクタがあるか。
  5. フィールド初期化、初期化ブロック、コンストラクタ本体の順に実行されるか。

デフォルトコンストラクタは常にあるわけではない

class A {
    // A() が暗黙に追加される
}
new A(); // OK

class B {
    B(String s) {}
}
// new B(); // NG: B() はない

自分でコンストラクタを1つでも書いた場合、引数なしコンストラクタは自動では追加されません。

this() と super() は先頭行だけ

class A {
    A(int n) {}
}
class B extends A {
    B() {
        this(10);
    }
    B(int n) {
        super(n);
    }
}

this(...) は同じクラスの別コンストラクタ、super(...) は親コンストラクタを呼びます。どちらもコンストラクタの先頭行にしか書けません。

class C extends A {
    C() {
        // int x = 10;
        // super(x); // NG: 先頭行ではない
        super(10);
    }
}

暗黙superで落ちるケース

class Parent {
    Parent(String s) {}
}

class Child extends Parent {
    Child() {
        // 暗黙に super(); が入る
        // Parent() が存在しないためコンパイルエラー
    }
}

サブクラスのコンストラクタだけ見ても不十分です。親に引数なしコンストラクタがあるかを確認します。

初期化順

  1. フィールドはまずデフォルト値になる。
  2. 親クラス側の初期化が先。
  3. 自クラスのフィールド初期化とインスタンス初期化ブロックが、書いた順に実行される。
  4. 自クラスのコンストラクタ本体が実行される。
class Sample {
    int a = print("field", 1);
    { a = print("block", 2); }
    Sample() { a = print("constructor", 3); }

    static int print(String label, int value) {
        System.out.println(label);
        return value;
    }
}
new Sample();

出力は field → block → constructor です。

コンストラクタは継承されない

サブクラスはスーパークラスのコンストラクタを継承しません。親コンストラクタを使いたい場合は、サブクラスのコンストラクタから super(...) で呼びます。

class Parent {
    Parent(String s) {}
}
class Child extends Parent {
    Child(String s) {
        super(s);
    }
}
// new Child();      // NG
new Child("A");     // OK

不可能・誤りパターン

コード判定理由
void A(){}コンストラクタではない戻り値型がある
this(); super();不可両方は書けない
System.out.println(); super();不可super() が先頭行ではない
親に引数ありだけ、子で super()不可呼べる親コンストラクタがない
コンストラクタの再帰呼び出し不可this() が循環する
