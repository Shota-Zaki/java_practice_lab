記事一覧参照比較型・配列文字列演算・制御クラス設計継承例外メソッド表単語集解き方
同一性
同じオブジェクトを参照しているかという概念。参照型の==は同一性を見る。
同値性
値や内容が等しいかという概念。equals()で表現することが多いが、実装次第。
参照のコピー
参照型変数を代入したとき、オブジェクトではなく参照がコピーされること。
文字列プール
同じ文字列リテラルを共有する領域。Stringの==問題で重要。
不変オブジェクト
生成後に内部状態が変わらないオブジェクト。Stringが代表。
可変オブジェクト
生成後に内部状態を変更できるオブジェクト。StringBuilderが代表。
数値昇格
byteやshortなどが演算時にintへ昇格すること。
暗黙の型変換
キャストなしで安全に変換できる型変換。intからlongなど。
明示的キャスト
データ欠損の可能性がある変換をプログラマが明示すること。
ローカル変数型推論
varにより右辺からローカル変数の型を推論する仕組み。
オーバーロード
同じ名前で引数リストが異なるメソッドを定義すること。
オーバーライド
継承したメソッドをサブクラスで再定義すること。
ポリモーフィズム
サブクラスのインスタンスをスーパークラス型で扱える性質。
フィールド隠蔽
サブクラスで同名フィールドを定義したとき、フィールドは変数の型で決まること。
defaultメソッド
インタフェースに実装を持たせるメソッド。競合時は実装クラスで解決が必要。
sealed class
継承できるクラスを限定するクラス。直接サブクラス側にも修飾が必要。
record
データ保持用クラスを簡潔に宣言する構文。アクセサやequalsが自動生成される。
checked例外
コンパイラが処理を要求する例外。catchまたはthrowsが必要。
unchecked例外
RuntimeException系。コンパイラによる処理義務はない。
短絡評価
&&や||で右辺評価が省略されること。副作用がある式では結果が変わる。
到達不能コード
絶対に実行されないとコンパイラが判断するコード。コンパイルエラーになる。

用語を覚えるだけで終わらせない

例えば「ポリモーフィズム」を覚えるだけでは不十分です。問題では「変数の型で呼べるメソッドが決まり、実行されるメソッドは実体の型で決まる」という判定に使えなければ意味がありません。
