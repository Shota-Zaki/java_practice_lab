同一性
同じオブジェクトを参照しているかという概念。参照型の==は同一性を見る。
同値性
値や内容が等しいかという概念。equals()で表現することが多いが、実装次第。
参照のコピー
参照型変数を代入したとき、オブジェクトではなく参照がコピーされること。
文字列プール
同じ文字列リテラルを共有する領域。Stringの==問題で重要。
不変オブジェクト
生成後に内部状態が変わらないオブジェクト。Stringが代表。
可変オブジェクト
生成後に内部状態を変更できるオブジェクト。StringBuilderが代表。
数値昇格
byteやshortなどが演算時にintへ昇格すること。
暗黙の型変換
キャストなしで安全に変換できる型変換。intからlongなど。
明示的キャスト
データ欠損の可能性がある変換をプログラマが明示すること。
ローカル変数型推論
varにより右辺からローカル変数の型を推論する仕組み。
オーバーロード
同じ名前で引数リストが異なるメソッドを定義すること。
オーバーライド
継承したメソッドをサブクラスで再定義すること。
ポリモーフィズム
サブクラスのインスタンスをスーパークラス型で扱える性質。
フィールド隠蔽
サブクラスで同名フィールドを定義したとき、フィールドは変数の型で決まること。
defaultメソッド
インタフェースに実装を持たせるメソッド。競合時は実装クラスで解決が必要。
sealed class
継承できるクラスを限定するクラス。直接サブクラス側にも修飾が必要。
record
データ保持用クラスを簡潔に宣言する構文。アクセサやequalsが自動生成される。
checked例外
コンパイラが処理を要求する例外。catchまたはthrowsが必要。
unchecked例外
RuntimeException系。コンパイラによる処理義務はない。
短絡評価
&&や||で右辺評価が省略されること。副作用がある式では結果が変わる。
到達不能コード
絶対に実行されないとコンパイラが判断するコード。コンパイルエラーになる。

基礎概念の追加用語

用語短い意味混同しやすい点
宣言型変数を宣言したときの型。呼べるメンバを決める。実体とは別。
実体new で作られた実際のオブジェクト。overrideメソッドの実行内容を決める。
参照オブジェクトへのリンク情報。参照型変数はオブジェクトそのものを持つわけではない。
実引数呼び出し時に渡す値。仮引数とは別。
仮引数メソッド側で受け取る変数。Javaは参照型でも参照値のコピーを渡す。
throw例外オブジェクトを投げる文。throws と混同しない。
throws例外を投げる可能性の宣言。例外を発生させる処理ではない。
伝播処理されなかった例外が呼び出し元へ戻ること。catchされるまで上位へ戻る。

用語を覚えるだけで終わらせない

例えば「ポリモーフィズム」を覚えるだけでは不十分です。問題では「変数の型で呼べるメソッドが決まり、実行されるメソッドは実体の型で決まる」という判定に使えなければ意味がありません。

コンパイルエラーと実行時例外を混ぜない

分類例判断
コンパイルエラー型不一致、未初期化ローカル変数、アクセス不可実行前に止まる。出力はない。
実行時例外null参照、配列範囲外、キャスト失敗そこまでの出力は残る可能性がある。
正常出力型・アクセス・例外がすべて通る最後に評価順を追う。

この単元をコードに適用する手順

  1. まず宣言を読む。型、修飾子、メソッド名、引数を固定する。
  2. 次に変更される値を探す。代入、メソッド呼び出し、副作用を表にする。
  3. 最後に出力を組み立てる。途中で止まるなら、そこまでの出力だけを残す。
理解した気になっているだけなら、似たコードに変えた瞬間に間違えます。本文の例を1行ずつ書き換えて、どこで結果が変わるか確認してください。
