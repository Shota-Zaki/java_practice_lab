1. 全問題共通の判定順

  1. 構文エラーがないか。 括弧、セミコロン、クラス名、ファイル名、import。
  2. 型が合うか。 代入、戻り値、引数、配列の次元、ジェネリクス。
  3. アクセスできるか。 private、protected、パッケージ、staticから非static。
  4. checked例外を処理しているか。 catchまたはthrows。
  5. 到達不能コードがないか。 return、throw、continue、break直後。
  6. 実行時例外がないか。 null、配列範囲外、キャスト、parseInt。
  7. 最後に出力を追う。 評価順、参照、ループ、switch、finally。

2. 参照比較問題

確認見る場所
new の回数別インスタンスが作られているか
代入参照コピーか、再代入か
==同じ参照先か
equals()クラスがオーバーライドしているか

3. 出力追跡問題

出力問題は、変数表を作るのが一番安全です。i、j、num、result の値を1行ずつ更新します。暗算で飛ばすと前置/後置やcontinueでずれます。

4. 例外問題

例外典型原因
NullPointerExceptionnull参照でメソッド・フィールドにアクセス
ArrayIndexOutOfBoundsException配列の範囲外アクセス
ClassCastException実体と互換性のないキャスト
NumberFormatException不正文字列を数値変換

この記事を演習で使える知識に変える

対象レベル：学習手順・復習設計。この記事は知識そのものより、学習方法と復習方法を整えるための内容です。間違えた理由を分類し、次の演習で同じ判断ミスを減らすことを目的にします。

本文で必ず押さえること

  • この記事は知識そのものより、学習手順の改善が目的です。
  • 間違いを「知らなかった」で終わらせず、判断手順のどこが雑だったかに分解します。
  • 演習の正答率だけでなく、根拠を言語化できるかを確認します。

この後の演習で見るポイント

  • 解いた直後に、正解理由と誤答理由を1文ずつ書く。
  • 間違えた問題をタグで分類する。型、参照、例外、継承、APIなどに分ける。
  • 翌日に同じ問題を見て、選択肢を見ずに判断手順を再現する。

ここで落ちる典型ミス

  • 正解した問題を復習対象から外す。偶然正解を放置すると弱点が残る。
  • 解説を読んで納得しただけで、再現できると思い込む。
  • 広く浅く進めすぎて、同じ型のミスを繰り返す。

演習前の小課題
