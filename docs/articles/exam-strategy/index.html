実践模試の解き方

模試A〜Dを「解いて終わり」にしないための使い方です。

時間配分

60問90分なので、1問あたり平均1分30秒です。ただし全問を均等に読む必要はありません。短い知識問題は30〜45秒で処理し、長いコード読解に時間を残します。

捨てる判断

長い継承・例外・配列問題で2分以上止まったら見直しマークを付けて進みます。全体を一周できない方が損失は大きいです。

復習方法

復習では正解だけ見ても意味が薄いです。自分が選んだ誤答がどの誤解から来たかをメモに残してください。特に「コンパイルエラーを実行結果として追っていた」「参照比較と同値比較を混同した」「Stringの不変性を忘れた」は頻出です。

コンパイルエラーと実行時例外を混ぜない

分類例判断
コンパイルエラー型不一致、未初期化ローカル変数、アクセス不可実行前に止まる。出力はない。
実行時例外null参照、配列範囲外、キャスト失敗そこまでの出力は残る可能性がある。
正常出力型・アクセス・例外がすべて通る最後に評価順を追う。

この単元をコードに適用する手順

  1. まず宣言を読む。型、修飾子、メソッド名、引数を固定する。
  2. 次に変更される値を探す。代入、メソッド呼び出し、副作用を表にする。
  3. 最後に出力を組み立てる。途中で止まるなら、そこまでの出力だけを残す。
理解した気になっているだけなら、似たコードに変えた瞬間に間違えます。本文の例を1行ずつ書き換えて、どこで結果が変わるか確認してください。

よくある誤解と修正

誤解正しい見方確認方法
見た目が似ていれば同じ動きになる型、戻り値、参照、修飾子が1つ違うだけで結果は変わる。行ごとに「型」「値」「参照先」をメモする。
実行結果だけ追えばよいコンパイルできなければ実行されない。例外で止まれば後続行は動かない。コンパイル可否 → 実行時例外 → 出力の順に判定する。
暗記した用語で判断できる用語は入口にすぎない。コード上のどの行に効いているかまで見る。本文の例を1行変更し、結果が変わる理由を説明する。

ミニドリル

次の形式で、自分の答えを文章にしてください。正解だけを出しても、弱点は潰れません。

  1. このコードはコンパイルできるか。
  2. 実行時に例外が起きるなら、どの行で起きるか。
  3. 正常に動くなら、出力は何か。
  4. 自分が間違えた場合、原因は「型」「参照」「戻り値」「アクセス」「例外」「評価順」のどれか。
学習時は、選択肢を見る前に自分の判定を作ってください。選択肢を先に見ると、もっともらしい誤答に引っ張られます。

最後に自分で確認すること

この記事を読み終えたら、次の3点を口頭で説明できるか確認します。説明できない場合は、まだ「読んだだけ」で、コードに適用できる状態ではありません。

  • この単元で最初に見るべき場所はどこか。
  • コンパイルエラー、実行時例外、正常出力をどの順番で分けるか。
  • 自分が間違えやすい引っかけは、型・参照・戻り値・アクセス・例外のどれか。
