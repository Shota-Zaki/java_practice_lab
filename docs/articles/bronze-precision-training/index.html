Bronze演習の読み方：基本問題を雑に解かない

Bronzeで落とす原因は、難問ではなく「基本用語」「文として成立するか」「値の変化」を飛ばすことです。

先に結論

Bronzeでは、難しい知識よりも、コードを雑に読まないことが点数に直結します。特に、変数、型、mainメソッド、if、for、配列、クラスとインスタンスを区別できているかを見ます。

読む順番

  1. 文として成立するか。型、変数名、セミコロン、波かっこ、mainメソッドの形を見る。
  2. 変数の値がどこで変わるか。代入、インクリメント、if、forの実行回数を追う。
  3. オブジェクトを作っているか。クラス名だけを見て、インスタンスが存在すると勘違いしない。

悪い読み方と修正

悪い読み方

「見たことがある形だからA」と選ぶ。これは偶然当たりを増やすだけで、復習不能です。

良い読み方

「3行目でxが10、5行目でxが11、条件はtrueなので出力は11」のように、行番号と状態で説明します。

ミニ例

int x = 3;
x = x + 2;
System.out.println(x);

この場合、答えは5です。重要なのは「x + 2を計算してからxに戻す」という代入の流れを言語化できることです。

復習の基準

正解したかより、「なぜ他の選択肢が違うか」を1つ説明できるかを基準にしてください。説明できない正解は、次の問題では再現できません。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Bronze。Bronzeでは、細かい新機能よりも「文の実行順序」「変数の値」「条件式の真偽」を正確に追うことを優先します。record、sealed、Streamなどの高度な構文・APIはここでは追いません。

本文で必ず押さえること

  • 用語を覚えるだけでなく、コード上でどの判断に使う知識かを確認する。
  • 先にコンパイル可否、次に実行時の流れ、最後に出力を考える。
  • 自信がない箇所は、変数の型と値を紙に分けて書く。

この後の演習で見るポイント

  • 本文の例を少し変えても同じ判断ができるか確認する。
  • 選択肢を消去するときは、なぜ誤りかを言葉で説明する。
  • 関連する演習では、正解番号ではなく判断根拠を残す。

ここで落ちる典型ミス

  • 用語の意味だけ覚えて、コード読解に接続しない。
  • 「なんとなく見たことがある」で選択肢を選ぶ。
  • 解説を読んだあとに、自力で再判定しない。

演習前の小課題

本文のコード例を1行だけ変更し、コンパイル可否・実行時の流れ・出力をもう一度判定してください。
