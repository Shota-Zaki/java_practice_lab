Java Silverの継承・オーバーライド・フィールド隠蔽

継承問題は「変数の型」と「実体の型」を分けて考えます。フィールドは変数の型、オーバーライドメソッドは実体の型で決まる、という区別が重要です。

1. ポリモーフィズムの基本

A a = new B();
a.method(); // BでオーバーライドされていればBのmethodが動く

変数の型が A でも、実体が B なら、オーバーライドされたメソッドは B 側が実行されます。

2. フィールドはオーバーライドされない

class A { int data = 10; int getData() { return data; } }
class B extends A { int data = 20; int getData() { return data; } }
A a = new B();
System.out.println(a.data);     // 10: フィールドは変数の型Aで決まる
System.out.println(a.getData());// 20: メソッドは実体Bで決まる
ここは頻出です。フィールドは隠蔽、メソッドはオーバーライド。混ぜて考えると失点します。

3. オーバーライドの条件

条件説明
メソッド名・引数が同じシグネチャが一致する
戻り値型が同じか共変戻り値サブタイプを返せる場合がある
アクセス修飾子を狭くしないpublicをprotectedにするのはNG
checked例外を広げない親より広いchecked例外は宣言できない

4. オーバーロードとの違い

オーバーロードは同じメソッド名で引数が違うものです。戻り値だけ違ってもオーバーロードにはなりません。

void test(int x) {}
void test(String x) {} // OK: 引数が違う
int test(int x) { return x; } // NG: 戻り値だけ違う

5. interfaceの抽象メソッド

インタフェースの抽象メソッドは暗黙的に public abstract です。実装クラスでは public で実装する必要があります。

interface A { void run(); }
class B implements A {
    public void run() {} // publicが必要
}

6. defaultメソッドの競合

複数のインタフェースから同じシグネチャのdefaultメソッドを継承すると、実装クラス側で明示的にオーバーライドする必要があります。

interface A { default void hello() { System.out.println("A"); } }
interface B { default void hello() { System.out.println("B"); } }
class C implements A, B {
    public void hello() {
        A.super.hello();
    }
}

7. sealed / non-sealed / final

修飾子意味
sealed継承・実装できるクラスを制限する
permits許可するサブクラスを列挙する
finalそれ以上継承させない
non-sealed制限を解除して通常の継承可能クラスに戻す

sealedクラスの直接サブクラスは、final、sealed、non-sealed のいずれかで宣言する必要があります。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java基礎〜Silver共通。この範囲は、基礎理解とSilverの読解をつなぐ部分です。まず「コンパイルできるか」「実行時にどこまで進むか」「出力は何か」の順で考えます。

本文で必ず押さえること

  • 変数の型と、実際のインスタンスの型を分ける。
  • フィールド参照は変数の型、オーバーライドメソッド呼び出しは実体の型が効く。
  • interfaceのメソッド・フィールドは暗黙の修飾子を意識する。

この後の演習で見るポイント

  • 代入が上向き変換か、下向きキャストかを確認する。
  • キャストはコンパイル可否と実行時可否を分けて判定する。
  • sealedではpermitsと、継承先のfinal/sealed/non-sealedを必ず確認する。

ここで落ちる典型ミス

  • 親型変数だから親のメソッドだけが実行される、と誤解する。オーバーライドは実体側。
  • キャストを書けば必ず安全に変換できると考える。実体が対応していなければ実行時例外になる。
  • interfaceの抽象メソッドがpackage-privateになると思い込む。基本はpublic。

演習前の小課題

Parent p = new Child(); の1行を見たら、変数型Parentと実体Childを分け、フィールド・メソッド・キャストの3つで判断してください。
