Java基礎を薄くしないための学び方

基礎はページ数を増やせば良いわけではありません。1つの概念を、定義、コード例、失敗例、次に何へつながるかまで理解する必要があります。

薄い解説の問題点

たとえば「Stringは不変です」とだけ書いても、学習者は問題で使えません。replaceの戻り値を受け取らない例、==とequalsの違い、StringBuilderとの違いまで並べて初めて使える知識になります。

濃い解説の型

  1. まず一言で結論を書く。
  2. 小さいコード例を出す。
  3. なぜそうなるかを説明する。
  4. 初心者が間違える例を出す。
  5. Silver問題ではどう問われるかを示す。

例：static

staticは「クラス側に属する」という説明だけでは不十分です。mainがstaticだからインスタンスフィールドを直接読めない、staticフィールドは共有される、staticメソッドはオーバーライドではなく隠蔽になる、というところまでつなげます。
