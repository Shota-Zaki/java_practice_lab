module-info.javaの確認

モジュールは、公開するパッケージと必要なモジュールを宣言する仕組みです。

module-info.javaの基本

モジュール名、依存するモジュール、外部へ公開するパッケージを宣言します。

module com.example.app {
    requires java.sql;
    exports com.example.app.api;
}
ディレクティブ意味注意
requires他モジュールへの依存使う側が依存を宣言する。
exportsパッケージを公開公開しないパッケージは外部からアクセス不可。
opensリフレクション向けに開く通常のコンパイル時アクセスとは別。
usesサービス利用を宣言ServiceLoaderと関係する。
provides ... with ...サービス実装を提供実装クラスを指定する。

exportsはクラス単位ではなくパッケージ単位

モジュールで公開するのはパッケージです。publicクラスであっても、そのパッケージがexportsされていなければ、他モジュールから通常アクセスできません。

module lib {
    exports lib.api;
    // lib.internal は公開していない
}
悪い判断

publicクラスならモジュール外から必ず使えると考える。

良い判断

publicかどうかに加えて、そのパッケージがexportsされているかを見る。
