配列の共変とArrayStoreException

コンパイルできるのに実行時例外になる配列代入を整理します。

配列は共変

String[]はObject[]に代入できます。

実体の型は残る

Object[]として見えていても、実体がString[]ならString以外は入れられません。

実行時に検査される

不正な要素を入れようとするとArrayStoreExceptionが発生します。

例

Object[] values = new String[2];
values[0] = "A";
// values[1] = 10; // ArrayStoreException

試験・学習での注意

  • 実行結果を見る前に、まずコンパイルできる形か確認します。
  • 似た構文でも、型・戻り値・アクセス修飾子で結果が変わります。
  • 暗記だけでなく、なぜその結果になるかを言語化してください。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java11 / Java17 Silver共通。Silver共通範囲では、バージョン固有構文に逃げず、型、参照、制御構文、クラス、継承、例外の基本ルールだけで判定します。Java17固有の記事はJava17 Silver側に分けています。

本文で必ず押さえること

  • リテラルの型を先に決める。整数は基本int、浮動小数点は基本doubleとして見る。
  • byte・short・charは、計算に入った瞬間にintへ昇格する場面がある。
  • varは「型を省略する記法」ではなく、初期化式から型を確定する仕組みとして読む。

この後の演習で見るポイント

  • 代入の右辺を先に評価し、結果の型が左辺へ入るか確認する。
  • 複合代入だけを例外扱いにせず、内部でキャスト相当の処理が入ると理解する。
  • コンパイル可否を出力結果より先に判定する。

ここで落ちる典型ミス

  • 数値が小さいからbyteへ入る、という雑な判断をしない。式になった時点で型が変わる。
  • varをフィールド、引数、戻り値型で使えると考えるのは誤り。
  • キャストを書けば常に安全、ではない。値の欠落や符号変化まで確認する。

演習前の小課題

次の3行を自分で判定します。byte b = 1; b = b + 1; b += 1; それぞれの違いを「式の型」で説明してください。
