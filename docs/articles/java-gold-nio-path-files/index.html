PathとFilesの基本

NIO.2は、Pathがパス表現、Filesが操作、という役割分担で読むと整理できます。

Pathはファイルそのものではない

Pathはファイルやディレクトリの場所を表すオブジェクトです。実体が存在するかどうかは別問題です。

Path p = Path.of("data", "sample.txt");
System.out.println(p.getFileName()); // sample.txt
API役割注意
Path.of(...)Pathを作る存在確認はしない。
Files.exists(path)存在確認戻り値はboolean。
Files.readString(path)内容を読むIOExceptionの可能性。
Files.lines(path)行Streamを返すStreamを閉じる意識が必要。
Files.copyコピー既存ファイルの扱いに注意。

normalizeとtoRealPathを混同しない

normalizeはパス文字列上の.や..を整理します。ファイルの存在確認はしません。toRealPathは実ファイルシステムに問い合わせるため、存在しない場合に例外が起きます。

Path p = Path.of("a", "..", "b");
System.out.println(p.normalize()); // b

// p.toRealPath(); // 実体がなければIOException
悪い判断

Pathを作れたのでファイルが存在すると考える。

良い判断

Path作成、存在確認、読み書き操作を別々に判断する。
