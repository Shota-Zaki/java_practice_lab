API判定ツリー

API問題は、暗記よりも判定順序です。コンパイル可否、型、実行タイミング、例外の順で切ります。

先に結論

この単元は、用語を覚えるだけでは安定しません。コードを見た瞬間に、型、戻り値、実行タイミング、副作用の有無を順番に切り分ける必要があります。

最初に見る4点

  1. importが必要な型か、パッケージが合っているか。
  2. メソッドがその型に存在するか。
  3. ラムダの引数型・戻り値型が合うか。
  4. 実行時に空、null、キー重複、閉じ忘れなどが起きるか。
段階見ること典型ミス
コンパイル前提型・メソッド・引数OptionalにStream用メソッドを書いている。
戻り値次のメソッドへつながる型collect後をStreamのままだと思う。
実行タイミング遅延評価・短絡評価peekが必ず全要素で動くと思う。
実行時例外null、空、重複、getOptional.get、toMap重複、Future.get。
Map<Integer, String> map = Stream.of("aa", "bb")
    .collect(Collectors.toMap(String::length, s -> s));
// コンパイルは通るが、キー重複で実行時例外

API問題は「コンパイルエラー」と「実行時例外」を分ける

メソッドの型が合わないならコンパイルエラーです。型は合っているが、データ条件で失敗するなら実行時例外です。この切り分けが最優先です。

悪い判断

見たことがあるAPIだから正常動作と判断する。

良い判断

型が合うか、実行条件で落ちないか、戻り値が何かを順番に確認する。

演習前チェック

演習に入る前に、次の4点を自分の言葉で確認してください。

  1. このコードはコンパイルできるか。
  2. 戻り値の型は何か。
  3. 処理はいつ実行されるか。
  4. 元のオブジェクトが変わるのか、新しい結果が返るだけなのか。
