Java Gold API判定ツリー：型・実行タイミング・戻り値で読む

GoldではAPI名の暗記ではなく、型の流れ、処理の実行タイミング、戻り値型を追う力が必要です。

先に結論

Goldは、単体の構文暗記では解けません。ラムダ式、Stream、Optional、Collectors、ジェネリクス、NIO、並行処理は、すべて「型」「実行タイミング」「副作用」の3軸で読みます。

判定ツリー

  1. 対象型は何か。ラムダ式やメソッド参照は、代入先・引数・戻り値の関数型インタフェースで形が決まります。
  2. 中間操作か終端操作か。Streamの中間操作は終端操作まで実行されません。
  3. 戻り値型は何か。map、flatMap、collect、groupingBy、Optional.mapの結果型を追います。
  4. 再利用できるか。Streamは一度終端操作を行うと再利用できません。
  5. 副作用があるか。Listの変更、Files操作、Future.get、closeのタイミングを確認します。

ジェネリクスの最低ライン

List<Integer>はList<Number>のサブタイプではありません。? extendsは読む側、? superは書く側に寄せて考えると崩れにくくなります。

Streamの最低ライン

悪い読み方

filterやmapが書かれた時点で処理されたと思い込む。

良い読み方

終端操作が現れた時点で、上から中間操作が適用されると読む。

並行処理の最低ライン

executeは戻り値なし、submitはFutureを返します。例外の見え方や結果取得のタイミングはFuture.getに関係します。スレッドが絡む問題では、出力順を断定できる条件があるかを必ず確認してください。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Gold。Goldでは、構文暗記よりもAPIの戻り値型、遅延実行、型推論、例外、リソース管理まで含めて判定します。Silverレベルの文法確認で止まると、ここでは足りません。

本文で必ず押さえること

  • 用語を覚えるだけでなく、コード上でどの判断に使う知識かを確認する。
  • 先にコンパイル可否、次に実行時の流れ、最後に出力を考える。
  • 自信がない箇所は、変数の型と値を紙に分けて書く。

この後の演習で見るポイント

  • 本文の例を少し変えても同じ判断ができるか確認する。
  • 選択肢を消去するときは、なぜ誤りかを言葉で説明する。
  • 関連する演習では、正解番号ではなく判断根拠を残す。

ここで落ちる典型ミス

  • 用語の意味だけ覚えて、コード読解に接続しない。
  • 「なんとなく見たことがある」で選択肢を選ぶ。
  • 解説を読んだあとに、自力で再判定しない。

演習前の小課題

本文のコード例を1行だけ変更し、コンパイル可否・実行時の流れ・出力をもう一度判定してください。
