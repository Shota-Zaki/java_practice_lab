Stream APIを丁寧に読む

遅延評価、短絡操作、reduce、collectを、型と実行タイミングで分解します。

遅延評価は「実行されない」ではなく「必要になるまで保留」

中間操作はパイプラインを組み立てます。終端操作が呼ばれた時点で、必要な分だけ上流から要素が流れます。

Stream<String> s = Stream.of("A", "B")
    .peek(System.out::println);
// この時点では何も表示されない
s.count(); // 終端操作で処理が流れる

短絡操作は全要素を処理しないことがある

anyMatch、allMatch、findFirst、limit は、結果が確定した時点で処理を打ち切ることがあります。出力問題では、何個の要素が実際に評価されたかを追います。

boolean result = Stream.of(1, 2, 3, 4)
    .peek(System.out::print)
    .anyMatch(n -> n > 2);
// 表示は 123 で止まる
操作戻り値注意点
mapStream<R>要素型が変わる。以後のラムダ引数の型も変わる。
flatMapStream<R>Streamの入れ子を平らにする。戻り値はStreamでなければならない。
reduce値またはOptional初期値の有無で戻り値が変わる。
collectコレクションやMapなどCollectorの指定によって戻り値が大きく変わる。

reduceは「畳み込み」

reduceは、要素を1つの値へまとめます。初期値ありなら空Streamでも初期値が返ります。初期値なしなら結果が存在しない可能性があるため、戻り値はOptionalになります。

int sum = Stream.of(1, 2, 3)
    .reduce(0, (a, b) -> a + b); // 6

Optional<Integer> max = Stream.<Integer>empty()
    .reduce((a, b) -> a > b ? a : b); // Optional.empty
Streamの深い問題は、見た目よりも「戻り値の型」と「評価される要素数」で決まります。処理の意味を雰囲気で読むと、短絡操作で落ちます。
