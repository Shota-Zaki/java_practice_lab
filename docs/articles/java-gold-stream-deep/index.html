Java GoldのStream APIを丁寧に読む

Stream APIは、データの流れを「中間操作」と「終端操作」に分けて読むと理解しやすくなります。

1. Streamは遅延評価

Stream.of("A", "B")
    .peek(System.out::println); // 何も表示されない

終端操作がないため、処理は実行されません。

Stream.of("A", "B").peek(System.out::println).count();

2. 中間操作と終端操作

  • 中間操作：filter、map、sorted、peek
  • 終端操作：forEach、collect、count、reduce

3. mapとfilter

List<String> result = List.of("a", "bb", "ccc").stream()
    .filter(s -> s.length() >= 2)
    .map(String::toUpperCase)
    .toList(); // [BB, CCC]

filter は残すかどうか、map は値を変換します。

4. reduce

int sum = Stream.of(1, 2, 3)
    .reduce(0, (a, b) -> a + b); // 6

初期値から順に畳み込みます。

5. groupingBy

Map<Integer, List<String>> m = Stream.of("a", "bb", "c")
    .collect(Collectors.groupingBy(String::length));

分類キーごとに要素を集めます。Goldでは戻り値の型を読めることが重要です。
