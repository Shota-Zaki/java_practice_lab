Java17 Silverの範囲境界：Java11・Java17・Goldを混ぜない

Java17 SilverではJava17寄りの構文を扱いますが、GoldのStream/Collectors/高度ジェネリクスまでは持ち込みません。

先に結論

Java17 Silverは、Java11 Silverの基礎に加えて、record、sealed、switch式、instanceofパターン変数などを正確に扱うコースです。ただし、Gold範囲のStream、Collectors、高度ジェネリクス、並行処理はここで深追いしません。

Java17 Silverで見るべきもの

  • record：暗黙のフィールド、アクセサ、コンパクトコンストラクタ。
  • sealed：permits、直接サブクラスのfinal/sealed/non-sealed。
  • instanceofパターン変数：使えるスコープ。
  • switch式：値を返すこと、yield、網羅性。

混ぜると危険なもの

Streamのcollect、OptionalのflatMap、高度なワイルドカード、Executor、Files.walkなどはGold側で扱います。Silverの段階でこれらを混ぜると、基礎文法の精度が落ちます。

判定順

  1. 構文として成立するか。record/sealed/switch式は形が正しいか。
  2. 型が成立するか。パターン変数やswitch式の戻り値型を見る。
  3. 例外が起きるか。null、キャスト、配列、parseを確認。
  4. 最後に出力を追う。参照、フィールド、メソッド、Stringの不変性を分ける。

試すべき復習

間違えた問題は、「Java11範囲のミス」「Java17新構文のミス」「Gold知識を持ち込んだミス」に分類してください。分類できると復習効率が上がります。

本文で必ず押さえること

  • 用語を覚えるだけでなく、コード上でどの判断に使う知識かを確認する。
  • 先にコンパイル可否、次に実行時の流れ、最後に出力を考える。
  • 自信がない箇所は、変数の型と値を紙に分けて書く。

  • 「なんとなく見たことがある」で選択肢を選ぶ。
  • 解説を読んだあとに、自力で再判定しない。
