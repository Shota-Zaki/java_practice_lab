ラムダ式と関数型インタフェース

ラムダ式は、抽象メソッドが1つだけの型へ代入される処理です。形ではなく、代入先の型から読みます。

ラムダ式だけでは型が決まらない

x -> x + 1という形だけでは、何型の引数を受け取り、何型を返すのか決まりません。代入先やメソッド引数の型によって決まります。

Function<Integer, Integer> f = x -> x + 1;
Predicate<String> p = s -> s.length() > 0;
Consumer<String> c = s -> System.out.println(s);
型抽象メソッド用途
Predicate<T>boolean test(T t)条件判定。filterなどで使う。
Function<T,R>R apply(T t)値の変換。mapなどで使う。
Consumer<T>void accept(T t)値を受け取って処理する。戻り値なし。
Supplier<T>T get()値を供給する。引数なし。

波かっこありのreturn

式形式ならreturnを書きません。ブロック形式で値を返すならreturnが必要です。

Function<String, Integer> a = s -> s.length();
Function<String, Integer> b = s -> { return s.length(); };
// Function<String, Integer> c = s -> { s.length(); }; // コンパイルエラー
悪い判断

ラムダの見た目だけで戻り値あり・なしを判断する。

良い判断

代入先の関数型インタフェースの抽象メソッドを確認する。
