javac・javaコマンドで詰まる原因

Javaのコンパイル・実行コマンドで起こりやすいミスをまとめます。

クラス名とファイル名

public class Sampleなら、基本的にファイル名はSample.javaです。大文字小文字も一致させます。

// Sample.java
public class Sample {
    public static void main(String[] args) {}
}

実行時は拡張子を付けない

javacはファイル名を指定しますが、javaはクラス名を指定します。

javac Sample.java
java Sample
// java Sample.class は誤り

パッケージがある場合

package ex; のクラスは、ルートディレクトリから ex.Sample のように完全修飾名で実行します。

javac -d build src/ex/Sample.java
java -cp build ex.Sample

試験での注意

問題文に「classファイルはない」「カレントディレクトリ」「ファイル名」が書かれている場合は、コマンド問題のヒントです。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java基礎〜Silver共通。この範囲は、基礎理解とSilverの読解をつなぐ部分です。まず「コンパイルできるか」「実行時にどこまで進むか」「出力は何か」の順で考えます。

本文で必ず押さえること

  • 用語を覚えるだけでなく、コード上でどの判断に使う知識かを確認する。
  • 先にコンパイル可否、次に実行時の流れ、最後に出力を考える。
  • 自信がない箇所は、変数の型と値を紙に分けて書く。

この後の演習で見るポイント

  • 本文の例を少し変えても同じ判断ができるか確認する。
  • 選択肢を消去するときは、なぜ誤りかを言葉で説明する。
  • 関連する演習では、正解番号ではなく判断根拠を残す。

ここで落ちる典型ミス

  • 用語の意味だけ覚えて、コード読解に接続しない。
  • 「なんとなく見たことがある」で選択肢を選ぶ。
  • 解説を読んだあとに、自力で再判定しない。

演習前の小課題

本文のコード例を1行だけ変更し、コンパイル可否・実行時の流れ・出力をもう一度判定してください。
