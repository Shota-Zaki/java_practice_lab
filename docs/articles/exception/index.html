Java Silverのtry-catch-finallyと例外処理完全整理

例外処理は、コンパイル時にチェックされるか、実行時にどこで止まるか、catchの順序、finallyの実行タイミングを整理すれば解けます。

1. checked例外とunchecked例外

分類代表例処理義務
checked例外Exception, IOExceptioncatchするかthrows宣言が必要
unchecked例外RuntimeException系処理義務なし
ErrorStackOverflowErrorなど通常処理対象ではない

2. catchの順序

catchは上から評価されます。親クラスを先に書くと、子クラスのcatchに到達できなくなります。

try {
    // ...
} catch (Exception e) {
    // ここで大半を捕まえる
} catch (IOException e) { // NG: 到達不能
}

正しくは特定の例外から一般的な例外へ並べます。

3. finallyの実行

finally は、try/catchを抜ける前に実行されます。return があっても、原則としてfinallyが先に実行されます。

static int test() {
    try {
        return 1;
    } finally {
        System.out.println("finally");
    }
}

4. try-with-resources

try-with-resourcesでは、tryブロックを抜けるときにリソースの close() が自動で呼ばれます。複数リソースは後に宣言したものから閉じられます。

try (A a = new A(); B b = new B()) {
    System.out.println("try");
}
// close順は b -> a
順番処理
1tryブロック実行
2リソースのclose
3catch
4finally

5. throw と throws

構文意味
throw例外オブジェクトを実際に投げる
throwsメソッドが例外を投げる可能性を宣言する
void test() throws IOException {
    throw new IOException();
}

6. NullPointerExceptionの典型

String s = null;
s.equals("A"); // NullPointerException
"A".equals(s); // false。例外は出ない

比較では、定数文字列を左に置くとNPEを避けられます。ただし試験では、どちらが左辺かを必ず確認します。
