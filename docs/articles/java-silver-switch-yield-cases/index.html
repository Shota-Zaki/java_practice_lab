switch文・switch式・yieldの見分け方

試験対策と実装理解の両方で使えるよう、例を中心に整理します。

switch文

switch(x){ case 1: result="A"; break; }

文なので値を返しません。

switch式

String s = switch(x){ case 1 -> "A"; default -> "B"; };

式なので値を返します。

yield

String s = switch(x){ case 1: yield "A"; default: yield "B"; };

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java17 Silver。Java17 Silverでは、Java17で扱う文法差分を確認します。ただしGold寄りのStream、Optional、Collectors、NIO、並行処理、モジュールの深掘りはここでは扱いません。

本文で必ず押さえること

  • 演算子の優先順位だけでなく、左から評価される部分と短絡評価を分ける。
  • if、for、whileは「条件判定→本体→更新」の順番を紙に書くとミスが減る。
  • switch文とswitch式は、値を返すかどうか、break/yieldの役割が違う。

この後の演習で見るポイント

  • 条件式の左辺で結果が確定するか確認する。
  • ループは初期値、条件、更新後の値を1周ずつ表にする。
  • switchはフォールスルーの有無と、defaultの位置を確認する。

ここで落ちる典型ミス

  • &&と&、||と|を同じものとして扱う。
  • 後置インクリメントを、式の中でもすぐ増えた値として使ってしまう。
  • switch式でyieldが必要な場所と不要な場所を混同する。

演習前の小課題

a++ + ++a のような式は暗算で処理しないでください。評価された値と、変数に残る値を別々に書いて判定します。
