頻出APIチェック

String、StringBuilder、ArrayList、Arraysを試験目線で整理します。

この単元で固定する判断順

  1. 対象が何かを見る。 String、StringBuilder、配列、ArrayList、List、ラッパークラスでは同じメソッド名でも意味が変わります。
  2. 戻り値を確認する。 返された値を使っているか、戻り値を捨てているかで結果が変わります。
  3. 元のオブジェクトが変わるかを見る。 String は不変、StringBuilder と ArrayList は基本的に可変です。
  4. インデックス範囲を見る。 0 始まり、終端は「含む」のか「含まない」のかを必ず分けます。
  5. null・空・範囲外の例外を見る。 コンパイルできても、実行時に NullPointerException や範囲外例外で止まることがあります。
雑に暗記すると危険です。
問題では「このメソッドを知っているか」より、「戻り値を受け取らないと変わらない」「元が変わっているから後続行に影響する」という形で落としてきます。

String：不変オブジェクトとして見る

String のメソッドは、多くの場合「新しい結果」を返します。元の文字列は変わりません。ここを間違えると、出力問題でほぼ確実に落ちます。

String s = "abc";
s.concat("de");
System.out.println(s);     // abc

s = s.concat("de");
System.out.println(s);     // abcde
メソッド戻り値元のString問題での注意
length()int変化なし配列は length フィールド。() を付けない。
charAt(int index)char変化なし0 以上 length() - 1 以下。範囲外は例外。
substring(int begin)String変化なしbegin から末尾まで。戻り値を使わないと意味がない。
substring(int begin, int end)String変化なしbegin 以上 end 未満。end の文字は含まない。
indexOf(String s)int変化なし見つからない場合は -1。例外ではない。
replace(char old, char new)String変化なしすべて置換した新しい文字列を返す。
startsWith(String)boolean変化なし大小文字は区別される。
endsWith(String)boolean変化なし末尾一致。部分一致とは違う。
equals(Object)boolean変化なし左辺が null なら呼び出し時に例外。
equalsIgnoreCase(String)boolean変化なし左辺 null は例外。右辺 null は false。
toLowerCase() / toUpperCase()String変化なし戻り値を代入しないと元は変わらない。
trim()String変化なし前後の空白を取り除いた結果を返す。中間の空白は消えない。
concat(String)String変化なし+ と似るが、null 引数は例外になり得る。

Stringの境界値：substringとcharAt

境界値は、問題作成者が最も使いやすいひっかけです。substring の第2引数は「取り出す最後の位置」ではなく、「取り出しを終える位置」です。

String s = "abcd";
System.out.println(s.substring(1, 3)); // bc
System.out.println(s.substring(4));    // 空文字
System.out.println(s.charAt(3));       // d
System.out.println(s.charAt(4));       // 実行時例外
式結果理由
s.substring(0, 4)"abcd"end は length() まで許される。
s.substring(4)空文字末尾から末尾までを取り出す扱い。
s.substring(2, 2)空文字開始位置と終了位置が同じ。
s.substring(3, 2)実行時例外開始位置が終了位置より後。
s.charAt(4)実行時例外charAt は length() と同じ位置を参照できない。

StringBuilder：可変オブジェクトとして見る

StringBuilder は中身を書き換えます。戻り値も多くの場合、自分自身への参照です。つまり、同じインスタンスを複数変数で参照していると、片方の操作がもう片方の見え方にも影響します。

StringBuilder a = new StringBuilder("A");
StringBuilder b = a;
a.append("B");
System.out.println(b); // AB
メソッド戻り値元のStringBuilder問題での注意
append(x)StringBuilder変わる末尾追加。戻り値は基本的に同じインスタンス。
insert(index, x)StringBuilder変わる指定位置の前に挿入。範囲外に注意。
delete(begin, end)StringBuilder変わるbegin 以上 end 未満を削除。
deleteCharAt(index)StringBuilder変わる1文字削除。範囲外は例外。
replace(begin, end, str)StringBuilder変わる範囲を別文字列で置換。
reverse()StringBuilder変わる戻り値を受けなくても反転済み。
length()int変化なし現在の文字数。
charAt(index)char変化なし範囲外は例外。
toString()String変化なしString と比較したいなら文字列化してから。
equals(Object)boolean変化なし内容比較ではない。基本は参照比較。
StringBuilder.equals() は頻出の罠。
new StringBuilder("a").equals(new StringBuilder("a")) は、内容が同じでも通常 false です。内容比較したいなら toString().equals(...) を使う必要があります。

配列：lengthはフィールド、固定長、初期値あり

配列はオブジェクトですが、length はメソッドではなくフィールドです。length() と書いたらコンパイルエラーです。

対象要素数取得注意
配列array.lengthフィールド。() なし。
Stringstr.length()メソッド。() あり。
ArrayListlist.size()メソッド。length ではない。
 
int[] nums = new int[3];
System.out.println(nums.length); // 3
System.out.println(nums[0]);     // 0
nums[3] = 10;                    // 実行時例外

ArrayList / List：戻り値と副作用を分ける

ArrayList は可変です。add、set、remove の戻り値がそれぞれ違うため、選択肢で狙われます。

メソッド戻り値副作用注意
add(E e)boolean末尾に追加通常 true。戻り値は追加要素ではない。
add(int index, E e)void指定位置に挿入指定位置以降が後ろへずれる。
set(int index, E e)置換前の要素指定位置を置換追加ではない。サイズは変わらない。
get(int index)指定位置の要素なし範囲外は例外。
remove(int index)削除した要素指定位置を削除後続要素のインデックスが詰まる。
remove(Object o)boolean一致要素を削除最初に一致した要素のみ削除。
contains(Object o)booleanなしequals で判定。
size()intなし配列の length と混同しない。
clear()void全要素削除空になるがリスト変数自体は null ではない。
isEmpty()booleanなしサイズ0なら true。

removeのオーバーロード：intかObjectか

remove は、引数が int ならインデックス削除、Object なら要素削除です。整数リストでは特に危険です。

var list = new java.util.ArrayList<Integer>();
list.add(10);
list.add(20);
list.add(30);

list.remove(1);                  // インデックス1を削除。20が消える
list.remove(Integer.valueOf(30)); // 要素30を削除
ここは暗記ではなくオーバーロード解決です。
remove(1) の 1 は int リテラルなので、remove(int index) が優先されます。要素の 1 を消したい場合は Integer.valueOf(1) のように Object として渡す必要があります。

List.of と Arrays.asList

どちらも短くリストを作れますが、性質が違います。変更できるか、元配列と連動するかを見ます。

作り方サイズ変更要素変更注意
new ArrayList<>()できるできる通常の可変リスト。
List.of("A", "B")できないできない変更系操作は実行時例外になり得る。
Arrays.asList(array)できないできる固定サイズ。元配列と連動する点に注意。
 
String[] array = {"A", "B"};
var list = java.util.Arrays.asList(array);
list.set(0, "X");
System.out.println(array[0]); // X
// list.add("C");            // 実行時例外

ラッパークラス・変換メソッド

ラッパークラスでは、文字列を数値に変換するメソッド、数値を文字列にするメソッド、オートボクシングとの関係が問われます。

Integer.valueOf("10")Integerラッパーを返す。

ミニ演習

演習1：Stringは変わるか

String s = "abc";
s.replace('a', 'x');
System.out.println(s);

答え: abc 。 replace は新しい文字列を返すが、戻り値を捨てている

演習2：StringBuilderは変わるか

StringBuilder sb = new StringBuilder("abc");
sb.reverse();
System.out.println(sb);

答え: cba 。 reverse は元のインスタンスを変更する

演習3：removeの意味

var list = new java.util.ArrayList<Integer>();
list.add(1); list.add(2); list.add(3);
System.out.println(list.remove(1));
System.out.println(list);

答え: まず 2 が表示され、リストは [1, 3] 。 remove(1) はインデックス削除

問題での最終チェック

  • String の戻り値を捨てていないか。
  • StringBuilder の変更が別変数にも見えていないか。
  • substring の第2引数を「含む」と誤解していないか。
  • 配列の length、文字列の length()、Listの size() を混同していないか。
  • remove(int) と remove(Object) を区別しているか。
  • List.of や Arrays.asList を通常の ArrayList と同じに扱っていないか。
  • 左辺が null のメソッド呼び出しになっていないか。

この単元で固定する判断

配列・ArrayList・List は、見た目が似ていても性質が違います。最初に「固定長か可変長か」「要素を変更できるか」「API名が何か」を分けます。

対象長さ/サイズ要素アクセス変更可否典型ミス
配列lengtha[0]長さ固定・要素変更可length() と書く
Stringlength()charAt(0)不変配列と混同する
ArrayListsize()get(0)可変長[0] で取ろうとする
List.of(...)size()get(0)変更不可add がコンパイルエラーだと思う

配列はオブジェクトですが、ArrayList とはAPIが違います。ここを雑にまとめると、問題文の1行目で落ちます。

配列は固定長のオブジェクト

配列は作成時に長さが決まり、その後に長さを変えることはできません。要素は変更できます。インデックスは0から始まり、最後の要素は length - 1 です。

int[] nums = new int[3];
System.out.println(nums.length); // 3
System.out.println(nums[0]);     // 0
nums[1] = 10;
System.out.println(nums[1]);     // 10
// System.out.println(nums[3]);  // ArrayIndexOutOfBoundsException

new int[3] は要素を3つ持つ配列です。アクセスできる添字は 0、1、2 だけです。

配列要素の初期値

配列の要素は、作成時に型ごとのデフォルト値で初期化されます。ローカル変数そのものは自動初期化されませんが、配列オブジェクトの要素は初期化される点を分けてください。

配列型要素の初期値
int[] などの整数型0
double[] などの浮動小数点型0.0
boolean[]false
char[]''
String[] などの参照型null
 
String[] names = new String[2];
System.out.println(names[0]); // null

int[] scores = new int[2];
System.out.println(scores[0]); // 0

配列宣言の形

配列の [] は型側にも変数名側にも書けます。ただし、読みやすさと誤読防止のため、基本は型側に寄せて読むのが安全です。

int[] a;      // 推奨される読み方: int配列型のa
int b[];      // これもOK
int[] x, y;   // xもyもint[]
int m[], n;   // mはint[]、nはint

int m[], n; は、m だけが配列で、n はただの int です。問題ではこの宣言の混在がよく狙われます。

配列初期化子 {...} の使い方

配列初期化子は、宣言と同時に使うか、new 型[] {...} の形で使います。宣言後に単独で {...} を代入することはできません。

int[] a = {1, 2, 3};       // OK
int[] b;
// b = {1, 2, 3};          // コンパイルエラー
b = new int[]{1, 2, 3};    // OK

var c = new int[]{1, 2, 3}; // OK
// var d = {1, 2, 3};       // コンパイルエラー

var と配列初期化子の組み合わせも頻出です。var d = {1, 2, 3}; では型推論に必要な型情報が不足します。

長さ0の配列

長さ0の配列は作れます。ただし、要素が存在しないため、どの添字にアクセスしても実行時例外です。

int[] a = new int[0];
System.out.println(a.length); // 0
// System.out.println(a[0]);  // ArrayIndexOutOfBoundsException

「長さ0の配列は作れない」と判断すると誤りです。作成可否とアクセス可否を分けます。

多次元配列は「配列の配列」

Javaの多次元配列は、厳密には配列の中に配列への参照が入っている構造です。行ごとに長さが違っても構いません。

int[][] data = new int[2][];
data[0] = new int[]{1, 2};
data[1] = new int[]{3, 4, 5};

System.out.println(data.length);    // 2
System.out.println(data[0].length); // 2
System.out.println(data[1].length); // 3

new int[2][] の時点では、外側の配列だけが作られます。内側の配列を作る前に data[0][0] へアクセスすると、data[0] が null なので NullPointerException になります。

int[][] data = new int[2][];
// System.out.println(data[0][0]); // NullPointerException

配列の代入と共変性

参照型配列では、サブクラス配列をスーパークラス配列の変数で扱えます。ただし、実体の配列型に合わない要素を入れると、コンパイルではなく実行時に ArrayStoreException が発生します。

Object[] data = new String[2];
data[0] = "A";      // OK
// data[1] = 100;    // ArrayStoreException

変数の型は Object[] でも、実体は String[] です。実体の配列に Integer を格納できないため実行時例外になります。

ArrayList は可変長

ArrayList は要素数を増減できます。配列と違い、要素数は size()、要素取得は get(index) です。

import java.util.ArrayList;

ArrayList<String> list = new ArrayList<>();
list.add("A");
list.add("B");
System.out.println(list.size()); // 2
System.out.println(list.get(0)); // A

list[0] は使えません。ArrayList は配列ではなくクラスです。

ArrayList の頻出メソッド

メソッド意味戻り値/注意
add(e)末尾に追加boolean を返す
add(index, e)指定位置に挿入戻り値は void。後ろがずれる
set(index, e)指定位置を置換古い要素を返す。サイズは変わらない
get(index)指定位置を取得範囲外なら IndexOutOfBoundsException
remove(index)指定位置を削除削除した要素を返す
remove(Object)一致する要素を削除削除できたか boolean を返す
contains(o)要素を含むかequals で判定
 
var list = new java.util.ArrayList<String>();
list.add("A");       // [A]
list.add("B");       // [A, B]
list.add(1, "C");    // [A, C, B]
String old = list.set(0, "X");
System.out.println(old);  // A
System.out.println(list); // [X, C, B]

remove のオーバーロード

ArrayList の remove は、引数が int ならインデックス削除、Object なら要素削除です。ArrayList<Integer> で特に間違えます。

var list = new java.util.ArrayList<Integer>();
list.add(1);
list.add(2);
list.add(3);

list.remove(1); // インデックス1を削除 → 要素2が消える
System.out.println(list); // [1, 3]

list.remove(Integer.valueOf(1)); // 値1を削除
System.out.println(list); // [3]

remove(1) は「値1を消す」ではありません。int 引数として最も合うため、インデックス削除になります。

List 型で受ける意味

List はインタフェース、ArrayList は実装クラスです。変数の型が List でも、実体が ArrayList なら可変リストとして動作します。

import java.util.ArrayList;
import java.util.List;

List<String> list = new ArrayList<>();
list.add("A");
System.out.println(list); // [A]

ただし、変数の型で呼べるメソッドが決まります。List に存在しない ArrayList 固有メソッドは、List 型変数からは呼べません。

List.of は変更不可

List.of は変更不可のリストを返します。add や remove はメソッドとして存在するためコンパイルは通りますが、実行時に UnsupportedOperationException が発生します。

import java.util.List;

List<String> list = List.of("A", "B");
System.out.println(list.get(0)); // A
// list.add("C");              // UnsupportedOperationException

また、List.of は null 要素を許可しません。

// List<String> list = List.of("A", null); // NullPointerException

Arrays.asList の固定サイズリスト

Arrays.asList で作ったリストは、サイズ変更ができません。ただし、既存要素の置換はできます。List.of と同じ「完全な変更不可」とは雑に同一視しないでください。

import java.util.Arrays;
import java.util.List;

List<String> list = Arrays.asList("A", "B");
list.set(0, "X");
System.out.println(list); // [X, B]
// list.add("C");        // UnsupportedOperationException

可変の ArrayList にしたいなら、new ArrayList<>(...) で包みます。

var list = new java.util.ArrayList<>(java.util.List.of("A", "B"));
list.add("C");
System.out.println(list); // [A, B, C]

ジェネリクスとダイヤモンド演算子

ArrayList<String> のように要素型を指定すると、その型以外の要素は原則として追加できません。右辺の <> は、左辺から型を推論するダイヤモンド演算子です。

ArrayList<String> list = new ArrayList<>();
list.add("A");
// list.add(10); // コンパイルエラー

一方、raw typeを使うと型安全性が崩れます。Javaでは深追いしすぎる必要はありませんが、警告とコンパイルエラーは別物です。

拡張for: 変数は要素そのものではなくコピー

拡張forのループ変数は、各要素の値または参照のコピーです。ループ変数に別の値を代入しても、配列やリストのスロットは置き換わりません。

int[] nums = {1, 2, 3};
for (int n : nums) {
    n = 0;
}
System.out.println(nums[0]); // 1

参照型の場合、参照先オブジェクトの中身を変更すれば影響しますが、ループ変数に別オブジェクトを代入しても元の配列要素は差し替わりません。

StringBuilder[] arr = { new StringBuilder("A") };
for (StringBuilder sb : arr) {
    sb.append("B");
}
System.out.println(arr[0]); // AB

for (StringBuilder sb : arr) {
    sb = new StringBuilder("X");
}
System.out.println(arr[0]); // AB

走査中の構造変更

拡張forで ArrayList を走査している最中に、リストの構造を変更すると実行時例外につながることがあります。Javaでは、まず「拡張for中に add / remove していないか」

var list = new java.util.ArrayList<String>();
list.add("A");
list.add("B");

for (String s : list) {
    // list.remove(s); // ConcurrentModificationExceptionになる可能性が高い
}

単にループ変数へ代入するだけなら構造変更ではありません。ただし、それでリストの中身が変わるわけでもありません。

範囲外アクセスの例外

対象範囲外アクセス例外
配列a[a.length]ArrayIndexOutOfBoundsException
ArrayListlist.get(list.size())IndexOutOfBoundsException
Strings.charAt(s.length())StringIndexOutOfBoundsException

いずれも「最後の有効インデックスは長さ/サイズ - 1」です。length や size() そのものは、最後のインデックスではありません。
