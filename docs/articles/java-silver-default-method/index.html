defaultメソッド競合の整理

defaultメソッドは、interfaceに実装を持たせる仕組みです。Java Silverでは「競合したときにどうなるか」が狙われます。

1. 1つだけなら継承される

interface A { default void test() { System.out.println("A"); } }
class B implements A {}
new B().test(); // A

実装クラスがメソッドを持たなければ、defaultメソッドが使われます。

2. 複数interfaceで同じdefault

interface A { default void test() {} }
interface B { default void test() {} }
class C implements A, B {} // コンパイルエラー

どちらを使うか決められないため、クラス側で実装する必要があります。

3. 解決方法

class C implements A, B {
    public void test() {
        A.super.test();
    }
}

インタフェース名.super.メソッド名() で特定のdefault実装を呼べます。

4. クラスのメソッドが優先

class Base { public void test() { System.out.println("Base"); } }
interface A { default void test() { System.out.println("A"); } }
class Child extends Base implements A {}

クラス継承のメソッドがinterfaceのdefaultメソッドより優先されます。

5. recordとの衝突

interface A { default void name() {} }
record User(String name) implements A {} // 戻り値不一致ならエラー

recordは String name() のようなアクセサを自動生成します。同名メソッドの戻り値が衝突する場合があります。
