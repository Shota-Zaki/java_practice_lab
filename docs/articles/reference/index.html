参照の同一性と同値比較

参照比較はJava Silverで最も間違えやすい分野の一つです。== は「同じものを指しているか」、equals() は「同じ内容とみなすか」を確認するためのものです。ただし、equals() の挙動はクラスによって変わります。

1. 参照変数とオブジェクトは別物

Sample s1 = new Sample(10);
Sample s2 = s1;
s1 = new Sample(10);
System.out.println(s1 == s2); // false

s2 = s1 の時点では同じオブジェクトを見ています。しかし、その後 s1 に新しいオブジェクトを代入したため、s1 と s2 は別のオブジェクトを参照します。

2. == は中身を見ない

class Sample {
    int num;
    Sample(int num) { this.num = num; }
}
Sample a = new Sample(10);
Sample b = new Sample(10);
System.out.println(a == b); // false

num が同じでも、new が2回あればオブジェクトは2つです。

3. equalsをオーバーライドしない自作クラス

Sample a = new Sample(10);
Sample b = new Sample(10);
System.out.println(a.equals(b)); // false

自作クラスで equals() をオーバーライドしなければ、基本的には Object.equals() の参照比較です。

4. Stringはequalsで内容比較

String a = new String("Java");
String b = "Java";
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

String.equals() は文字列内容を比較します。== は参照比較なので一致しません。

5. 文字列リテラルと文字列プール

String a = "Java";
String b = "Java";
System.out.println(a == b); // true

文字列リテラルは文字列プール上の同じインスタンスを共有するため、参照が一致することがあります。

6. intern()

String a = new String("Java");
String b = a.intern();
String c = "Java";
System.out.println(b == c); // true

intern() は文字列プール上の参照を返します。a 自身がプールに移動するわけではありません。

7. StringBuilderのequalsは内容比較ではない

StringBuilder a = new StringBuilder("Java");
StringBuilder b = new StringBuilder("Java");
System.out.println(a.equals(b)); // false

StringBuilder は equals() を内容比較としてオーバーライドしていません。内容を比較したいなら toString() して比較します。

8. 配列のequals

int[] a = {1, 2};
int[] b = {1, 2};
System.out.println(a.equals(b)); // false

配列の equals() も参照比較です。中身を比較するなら java.util.Arrays.equals(a, b) を使います。

9. 配列cloneは浅いコピー

char[][] a = {{'A'}, {'B'}};
char[][] b = a.clone();
System.out.println(a == b);       // false
System.out.println(a[0] == b[0]); // true

外側の配列はコピーされますが、内側の配列は同じ参照を共有します。

10. recordのequals

record Point(int x, int y) {}
System.out.println(new Point(1, 2).equals(new Point(1, 2))); // true

recordはコンポーネントを使った equals()、hashCode()、toString() を自動生成します。

11. メソッド引数は参照のコピー

static void change(StringBuilder sb) {
    sb.append("B");
}
StringBuilder s = new StringBuilder("A");
change(s);
System.out.println(s); // AB

参照のコピーが渡されます。同じオブジェクトを操作すれば呼び出し元にも影響します。

static void replace(StringBuilder sb) {
    sb = new StringBuilder("X");
}

一方で、仮引数に新しいオブジェクトを代入しても、呼び出し元の変数が差し替わるわけではありません。

覚えるべき判断

  • new が何回出たかを見る。
  • == は参照比較。
  • equals() はクラスごとの実装を見る。
  • String、ラッパー、recordは内容比較の問題になりやすい。
  • 配列と StringBuilder は内容比較にならない点に注意。
