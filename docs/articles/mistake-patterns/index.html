誤答パターン集

点数が伸びない原因は、知識不足よりも判定の癖にあることが多いです。

1. 出力を追う前にコンパイルエラーを見落とす

ローカル変数未初期化、staticからインスタンスフィールド参照、戻り値型違い、アクセス修飾子の弱体化は、実行前に落ちます。

2. 参照比較を値比較だと思う

`==` は参照の同一性です。自作クラスで `equals()` をオーバーライドしていない場合も、通常は同一性比較になります。

3. Stringのメソッドが元の文字列を書き換えると思う

`replace()` や `substring()` は新しい文字列を返します。戻り値を代入していないなら元の参照先は変わりません。

4. オーバーロードとオーバーライドを混同する

引数が違えばオーバーロードです。戻り値だけを変えて同名同引数にすることはできません。

5. catch順序を軽く見る

先に広い例外型を書くと、後続の狭い例外型が到達不能になります。

よくある誤解と修正

誤解正しい見方確認方法
見た目が似ていれば同じ動きになる型、戻り値、参照、修飾子が1つ違うだけで結果は変わる。行ごとに「型」「値」「参照先」をメモする。
実行結果だけ追えばよいコンパイルできなければ実行されない。例外で止まれば後続行は動かない。コンパイル可否 → 実行時例外 → 出力の順に判定する。
暗記した用語で判断できる用語は入口にすぎない。コード上のどの行に効いているかまで見る。本文の例を1行変更し、結果が変わる理由を説明する。

ミニドリル

次の形式で、自分の答えを文章にしてください。正解だけを出しても、弱点は潰れません。

  1. このコードはコンパイルできるか。
  2. 実行時に例外が起きるなら、どの行で起きるか。
  3. 正常に動くなら、出力は何か。
  4. 自分が間違えた場合、原因は「型」「参照」「戻り値」「アクセス」「例外」「評価順」のどれか。
学習時は、選択肢を見る前に自分の判定を作ってください。選択肢を先に見ると、もっともらしい誤答に引っ張られます。
