Java Silverのメソッド呼び出し解決まとめ

試験対策と実装理解の両方で使えるよう、例を中心に整理します。

結論

オーバーロードはコンパイル時、オーバーライドは実行時の実体で決まります。

オーバーロード

void test(Object o) {}
void test(String s) {}
test(null); // String版が選ばれる

より具体的な型が選ばれます。

オーバーライド

A a = new B();
a.run(); // BがoverrideしていればB側

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java11 / Java17 Silver共通。Silver共通範囲では、バージョン固有構文に逃げず、型、参照、制御構文、クラス、継承、例外の基本ルールだけで判定します。Java17固有の記事はJava17 Silver側に分けています。

本文で必ず押さえること

  • インスタンスに属するものと、クラスに属するstaticなものを分ける。
  • コンストラクタは戻り値型を書かない。メソッドと見た目が似ていても別物。
  • オーバーロードはコンパイル時に引数型で決まる。実行時のオーバーライドとは分ける。

この後の演習で見るポイント

  • 呼び出し元の変数型、引数型、アクセス修飾子を順に確認する。
  • thisとsuperが何を指しているかを明確にする。
  • コンストラクタチェーンでは、最初にどのコンストラクタへ制御が移るかを見る。

ここで落ちる典型ミス

  • staticメソッドからインスタンスフィールドへ直接触れると思い込む。
  • privateメンバが継承先から自由に使えると考える。
  • 同名メソッドを見てすぐオーバーライドと判断する。引数が違えばオーバーロード。

演習前の小課題

同じ名前のメソッドが複数あるコードでは、先に「候補メソッド一覧」を作り、引数型でどれが選ばれるかを判定してください。
