Gold問題 分類別QA

Gold問題を分類ごとに見直し、同一分類内での重複、解説不足、判定順の不足を確認した結果。

今回のQA方針

  1. 分類タグを付け直し、Stream、Collectors、Optional、ジェネリクス、NIO、並行処理、モジュールなどで整理。
  2. 各問題の解説に「コンパイル可否 → API仕様 → 出力・例外」の判定順を追加。
  3. 同一演習セット内でコードが完全重複していた問題は差し替え。
  4. 通常演習と模試の両方を対象にし、模試だけがテンプレ反復にならないよう確認。

重複パターンの修正

同一演習セット内のコード完全重複は、QA前が663件、QA後が0件です。

今回、同じ分類・同じ論点でも、定数・API・判定ポイントを変えた問題へ差し替えました。単なるタイトル変更ではなく、コードと選択肢、解説を合わせて更新しています。残った同型テンプレートにはQA識別コメントを付け、同一演習内の完全一致を避けています。

分類別に見るべき弱点

分類今後も見る点
Stream / Collectors遅延評価、再利用不可、toMap重複キー、groupingBy下流Collector。
OptionalorElseとorElseGetの評価タイミング、mapとflatMap、空Optionalの扱い。
ジェネリクスextends/superの読み替え、raw type、型消去、取り出し型。
NIOPathの文字列表示、normalize、resolve、IOExceptionの処理。
並行処理出力順保証、Future.get、shutdown漏れ、例外の包まれ方。
モジュールrequires/exports/opens/uses/providesの役割分担。

QA後の使い方

Gold演習は、問題を解いた後に「なぜ他の選択肢が違うか」を必ず確認してください。Goldでは知識そのものより、API仕様の細かい差分を読めるかで差が出ます。
