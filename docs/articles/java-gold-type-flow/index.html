型の流れを追う

API連鎖の問題は、メソッド名ではなく「今の式の型」を書き出すと安定します。

先に結論

この単元は、用語を覚えるだけでは安定しません。コードを見た瞬間に、型、戻り値、実行タイミング、副作用の有無を順番に切り分ける必要があります。

型を書き出すだけで難度は下がる

Stream、Optional、ジェネリクスが混ざるコードは、1行ごとの式の型を確認します。型が変わる場所は、主にmap、flatMap、collect、orElseです。

List<String> names = List.of("a", "bb");
Optional<Integer> firstLength = names.stream()  // Stream<String>
    .map(String::length)                        // Stream<Integer>
    .filter(n -> n > 1)                         // Stream<Integer>
    .findFirst();                               // Optional<Integer>
操作前の型後の型
stream()List<T>Stream<T>
mapStream<T>Stream<R>
findFirstStream<T>Optional<T>
orElseOptional<T>T
collectStream<T>Collector次第

戻り値で次のメソッドが決まる

たとえばfindFirst()の後はStreamではなくOptionalです。そのため、続けてStream用のfilterを書くのではなく、Optional用のfilterとして解釈します。

String result = names.stream()
    .filter(s -> s.length() > 1)
    .findFirst()          // Optional<String>
    .orElse("none");     // String
API問題で迷ったら、式全体を一気に読まず、各行の右端に型を書いてください。勘より速く、間違いも減ります。

演習前チェック

演習に入る前に、次の4点を自分の言葉で確認してください。

  1. このコードはコンパイルできるか。
  2. 戻り値の型は何か。
  3. 処理はいつ実行されるか。
  4. 元のオブジェクトが変わるのか、新しい結果が返るだけなのか。
