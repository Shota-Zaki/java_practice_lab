Java Goldの型の流れを読む

GoldはAPI暗記だけでは通りません。ジェネリクス、ラムダ、Streamは「型がどう流れるか」を追う必要があります。

Streamは終端操作まで動かない

var list = java.util.List.of("A", "BB", "CCC");
var stream = list.stream()
    .filter(s -> s.length() > 1)
    .map(String::length);
System.out.println("X");
System.out.println(stream.count());

filter や map は中間操作です。count のような終端操作で初めて評価されます。

PECSを実用的に覚える

void read(java.util.List<? extends Number> list) {
    Number n = list.get(0);
    // list.add(1); // 不可
}
void write(java.util.List<? super Integer> list) {
    list.add(1);
    Object o = list.get(0);
}

extends は読む側、super は書く側と覚えると実戦で使えます。ただし、完全に「読むだけ」「書くだけ」ではなく、取り出し型・追加可能型の制限として理解します。

Optionalをnull代わりに雑に使わない

java.util.Optional<String> opt = java.util.Optional.ofNullable(null);
System.out.println(opt.orElse("empty"));

Optional.of(null) は NullPointerException です。nullの可能性があるなら ofNullable を使います。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Gold。Goldでは、構文暗記よりもAPIの戻り値型、遅延実行、型推論、例外、リソース管理まで含めて判定します。Silverレベルの文法確認で止まると、ここでは足りません。

本文で必ず押さえること

  • リテラルの型を先に決める。整数は基本int、浮動小数点は基本doubleとして見る。
  • byte・short・charは、計算に入った瞬間にintへ昇格する場面がある。
  • varは「型を省略する記法」ではなく、初期化式から型を確定する仕組みとして読む。

この後の演習で見るポイント

  • 代入の右辺を先に評価し、結果の型が左辺へ入るか確認する。
  • 複合代入だけを例外扱いにせず、内部でキャスト相当の処理が入ると理解する。
  • コンパイル可否を出力結果より先に判定する。

ここで落ちる典型ミス

  • 数値が小さいからbyteへ入る、という雑な判断をしない。式になった時点で型が変わる。
  • varをフィールド、引数、戻り値型で使えると考えるのは誤り。
  • キャストを書けば常に安全、ではない。値の欠落や符号変化まで確認する。

演習前の小課題

次の3行を自分で判定します。byte b = 1; b = b + 1; b += 1; それぞれの違いを「式の型」で説明してください。
