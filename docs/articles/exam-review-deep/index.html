模試後レビューの精度を上げる方法

模試は点数を見るだけでは意味が薄いです。復習時に「何を間違えたか」ではなく「なぜ間違えたか」を分類します。

復習分類

  • コンパイルエラーを見落とした
  • 実行時例外を見落とした
  • 出力順を追い間違えた
  • 参照比較を値比較だと思った
  • API仕様を忘れていた
  • 継承のフィールドとメソッドを混ぜた
  • 問題文の「2つ選択」を見落とした

復習優先度

最優先は「自信ありで間違えた問題」です。これは知識が曖昧ではなく、誤ったルールを正しいと思い込んでいる可能性が高いからです。

復習の終わり方

解説を読んで終わりにせず、同じタグの類題を2〜3問解いてから終わると定着しやすくなります。
