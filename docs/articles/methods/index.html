メソッド問題は、戻り値、元オブジェクトを変更するか、例外条件、添字の範囲を確認します。

String

メソッド戻り値要点落とし穴
charAt(int)char指定位置の文字0始まり。範囲外で例外。
indexOf(String)int最初に見つかった位置見つからないと-1。
substring(int,int)Stringbegin以上end未満endの文字は含まない。
replace(char,char)String置換後の新文字列元のStringは変わらない。
concat(String)String連結後の新文字列戻り値を受け取らないと捨てる。
equals(Object)boolean内容比較==とは違う。
intern()String文字列プール上の参照参照比較問題で頻出。

StringBuilder

メソッド戻り値要点落とし穴
append(...)StringBuilder末尾追加自分自身を変更する。
insert(int,...)StringBuilder指定位置に挿入範囲外で例外。
delete(int,int)StringBuilderbegin以上end未満を削除Stringのsubstringと同じくend未満。
reverse()StringBuilder反転元のBuilderが変わる。
toString()StringString化内容比較したいときに使う。

ArrayList / List

メソッド戻り値要点落とし穴
add(E)boolean末尾追加通常true。
add(int,E)void指定位置に挿入置換ではなく挿入。
set(int,E)E指定位置を置換古い値を返す。
remove(int)Eインデックス削除remove(1)は値1ではなく位置1。
remove(Object)boolean一致する要素を削除Integer.valueOf(1)で値削除。
contains(Object)boolean含むか確認equals()で比較。
size()int要素数配列のlengthと混同しない。

Arrays / Object / ラッパー

メソッド要点注意
Arrays.asList(array)配列をList風に扱う固定サイズ。add/removeは不可。
Arrays.equals(a,b)一次元配列の中身比較多次元ならdeepEquals。
Objects.equals(a,b)null安全なequalsnull比較で便利。
Integer.parseInt(String)文字列をintへ変換不正文字列でNumberFormatException。
valueOfラッパー型へ変換オートボクシングとの違いに注意。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java基礎〜Silver共通。この範囲は、基礎理解とSilverの読解をつなぐ部分です。まず「コンパイルできるか」「実行時にどこまで進むか」「出力は何か」の順で考えます。

本文で必ず押さえること

  • インスタンスに属するものと、クラスに属するstaticなものを分ける。
  • コンストラクタは戻り値型を書かない。メソッドと見た目が似ていても別物。
  • オーバーロードはコンパイル時に引数型で決まる。実行時のオーバーライドとは分ける。

この後の演習で見るポイント

  • 呼び出し元の変数型、引数型、アクセス修飾子を順に確認する。
  • thisとsuperが何を指しているかを明確にする。
  • コンストラクタチェーンでは、最初にどのコンストラクタへ制御が移るかを見る。

ここで落ちる典型ミス

  • staticメソッドからインスタンスフィールドへ直接触れると思い込む。
  • privateメンバが継承先から自由に使えると考える。
  • 同名メソッドを見てすぐオーバーライドと判断する。引数が違えばオーバーロード。

演習前の小課題

同じ名前のメソッドが複数あるコードでは、先に「候補メソッド一覧」を作り、引数型でどれが選ばれるかを判定してください。
