groupingByを正確に読む

Collectors.groupingByは、分類キーと値の集約方法を分けて読むと崩れません。

groupingByの基本形

第1引数は分類キーを作る関数です。下流Collectorを省略すると、同じキーの要素がListに集まります。

Map<Integer, List<String>> map = Stream.of("a", "bb", "cc")
    .collect(Collectors.groupingBy(String::length));
// {1=[a], 2=[bb, cc]}
指定意味戻り値の形
groupingBy(classifier)キーごとにListへ集めるMap<K, List<T>>
groupingBy(classifier, counting())キーごとに件数を数えるMap<K, Long>
groupingBy(classifier, mapping(...))キーごとに値を変換して集める下流Collector次第

下流Collectorを読む

2つ目の引数がある場合、値側はList<T>とは限りません。counting()ならLong、joining()ならString、toSet()ならSetです。

Map<Integer, Long> counts = Stream.of("a", "bb", "cc")
    .collect(Collectors.groupingBy(String::length, Collectors.counting()));
// {1=1, 2=2}
悪い判断

groupingByは必ずMap>になると決めつける。

良い判断

分類キーの型と下流Collectorの戻り値を別々に見る。
