interfaceの変数が暗黙にpublic static finalになる理由

interface内のフィールドと再代入不可の罠を整理します。

interfaceのフィールドは定数

interface内に書いた変数は暗黙にpublic static finalです。

インクリメントできない

finalなので++や代入はできません。

実装クラスから見えるが変更できない

implementsしたクラスから名前だけで参照できることはありますが、変更は不可です。

例

interface A { int VALUE = 10; }
class B implements A {
    void m() {
        System.out.println(VALUE);
        // VALUE++; // 不可
    }
}

試験・学習での注意

  • 実行結果を見る前に、まずコンパイルできる形か確認します。
  • 似た構文でも、型・戻り値・アクセス修飾子で結果が変わります。
  • 暗記だけでなく、なぜその結果になるかを言語化してください。
