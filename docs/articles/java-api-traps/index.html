Java Silver頻出APIの罠まとめ

API問題は暗記だけでは足りません。戻り値、破壊的変更の有無、例外条件、引数の意味をセットで覚える必要があります。

String：戻り値を受け取らないと変わらない

String s = "abc";
s.replace('b', 'x');
System.out.println(s); // abc
s = s.replace('b', 'x');
System.out.println(s); // axc

replace、substring、concat、toLowerCase は新しい文字列を返します。元の文字列は変わりません。

StringBuilder：変更されるがequalsは中身比較ではない

StringBuilder a = new StringBuilder("A");
StringBuilder b = new StringBuilder("A");
System.out.println(a.equals(b)); // false
System.out.println(a.toString().equals(b.toString())); // true

StringBuilder は可変ですが、equals は中身比較にオーバーライドされていません。

ArrayList.removeの罠

var list = new java.util.ArrayList<Integer>();
list.add(1); list.add(2); list.add(3);
list.remove(1);                 // index 1 を削除。値2が消える
list.remove(Integer.valueOf(3)); // 値3を削除

remove(int) と remove(Object) は別メソッドです。Integer の問題では頻出です。

List.of と Arrays.asList

List.of は要素の追加・削除・置換ができません。Arrays.asList はサイズ変更不可ですが、set は可能です。この差は実行時例外問題で狙われます。
