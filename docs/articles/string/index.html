StringとStringBuilder

String問題は「不変性」「戻り値」「文字列プール」「StringBuilderとの違い」を分けて読むと安定します。

この単元の結論

文字列問題は、次の5点だけでかなり解けます

  1. Stringは不変。 メソッドは元の文字列を変更せず、新しい文字列を返す。
  2. StringBuilderは可変。 append や reverse は同じオブジェクトを変更する。
  3. == は参照比較。 文字列の内容比較は equals。
  4. 文字列リテラルはプールを使う。 new String は別インスタンス。
  5. メソッドの戻り値と範囲を確認。 substring はend未満、indexOf は見つからなければ-1。

Stringは不変

String は一度作られると中身を変更できません。toUpperCase、replace、trim、substring などは、変更後の新しい String を返します。

String s = " Java ";
s.trim();
s.replace("J", "j");
System.out.println(s); // " Java "

戻り値を受け取ると、変数が新しい文字列を参照するようになります。

String s = " Java ";
s = s.trim();
s = s.replace("J", "j");
System.out.println(s); // "java"
判断: Stringメソッドを見たら、まず「戻り値を受け取っているか」受け取っていないなら、元の文字列は変わりません。

文字列リテラルと文字列プール

文字列リテラルは、同じ内容なら文字列プール上の同じインスタンスを参照します。

String a = "abc";
String b = "abc";
System.out.println(a == b);      // true
System.out.println(a.equals(b)); // true

一方、new String("abc") は新しいインスタンスを作ります。内容は同じでも参照先は別です。

String a = "abc";
String b = new String("abc");
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

コンパイル時定数の結合

文字列リテラル同士の結合や、コンパイル時定数だけの結合は、コンパイル時に1つの文字列として扱われます。

String a = "ab";
String b = "a" + "b";
System.out.println(a == b); // true

final で初期化されたコンパイル時定数も同様です。

final String x = "a";
final String y = "b";
String z = x + y;
System.out.println(z == "ab"); // true

ただし、通常の変数が絡むと実行時の文字列連結になり、文字列プールの同一参照とは限りません。

String x = "a";
String y = "b";
String z = x + y;
System.out.println(z == "ab"); // false
System.out.println(z.equals("ab")); // true

internの判断

intern() は、文字列プール上の同じ内容の String への参照を返します。== と絡めて出されやすいです。

String a = new String("sample");
String b = a;
String c = a.intern();
String d = "sample";

System.out.println(a == b); // true: 同じ参照をコピーしただけ
System.out.println(a == c); // false: aはnewで作ったインスタンス
System.out.println(c == d); // true: どちらも文字列プール上の参照

読む順番は、newで作ったもの、参照コピーしたもの、internでプール参照になったものを分けることです。

== と equals

== は、参照型では「同じインスタンスを参照しているか」を比較します。文字列の内容比較は equals を使います。

String a = new String("A");
String b = new String("A");

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

equals はメソッドなので、左辺が null なら NullPointerException です。

String s = null;
// s.equals("A");      // NullPointerException
System.out.println("A".equals(s)); // false

nullと文字列連結

+ による文字列連結では、null は文字列 "null" として扱われます。

String s = null;
System.out.println(s + "x"); // nullx

しかし、null 参照に対してメソッドを呼ぶと NullPointerException です。

String s = null;
// System.out.println(s.concat("x")); // NullPointerException

また、concat に null を渡すのも危険です。

String s = "A";
// System.out.println(s.concat(null)); // NullPointerException

+演算子は左から評価される

+ は、数値加算と文字列連結の両方で使われます。途中で文字列が出たら、それ以降は文字列連結として進みます。

System.out.println(1 + 2 + "A"); // 3A
System.out.println("A" + 1 + 2); // A12
System.out.println("A" + (1 + 2)); // A3
判断: + が並んでいる問題は、左から順に「今は数値加算か、文字列連結か」を追います。

substringの境界

substring(begin, end) は、begin以上、end未満です。end位置の文字は含みません。

String s = "abcdef";
System.out.println(s.substring(1, 4)); // bcd
System.out.println(s.substring(1, 1)); // 空文字
System.out.println(s.substring(4));    // ef

不正な範囲は実行時例外です。

String s = "abc";
// s.substring(-1);    // StringIndexOutOfBoundsException
// s.substring(2, 1);  // StringIndexOutOfBoundsException
// s.substring(4);     // StringIndexOutOfBoundsException

StringBuilderは可変

StringBuilder は、文字列を効率よく組み立てるための可変オブジェクトです。append、insert、delete、reverse などは自身を変更します。

StringBuilder sb = new StringBuilder("ab");
sb.append("c");
System.out.println(sb); // abc

多くの変更系メソッドは、自分自身への参照を返します。そのためメソッドチェーンできます。

StringBuilder sb = new StringBuilder("a");
StringBuilder r = sb.append("b").append("c");

System.out.println(sb);     // abc
System.out.println(r);      // abc
System.out.println(sb == r); // true

StringBuilderのsubstringは変更しない

StringBuilder は可変ですが、すべてのメソッドが自身を変更するわけではありません。substring は String を返すだけで、元の StringBuilder は変わりません。

StringBuilder sb = new StringBuilder("abcdef");
String s = sb.substring(1, 3);

System.out.println(s);  // bc
System.out.println(sb); // abcdef

StringBuilderのequalsは内容比較ではない

String の equals は内容比較ですが、StringBuilder は equals を内容比較としてオーバーライドしていません。基本的に参照比較として考えます。

StringBuilder a = new StringBuilder("abc");
StringBuilder b = new StringBuilder("abc");

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // false
System.out.println(a.toString().equals(b.toString())); // true

StringBuilder と String を equals で比較しても、同じ内容とは判定されません。

StringBuilder sb = new StringBuilder("abc");
String s = "abc";

System.out.println(sb.equals(s));        // false
System.out.println(sb.toString().equals(s)); // true

参照コピーと可変オブジェクト

StringBuilder は可変なので、同じインスタンスを複数の変数で参照していると、一方から変更した結果が他方からも見えます。

StringBuilder a = new StringBuilder("A");
StringBuilder b = a;

b.append("B");
System.out.println(a); // AB
System.out.println(b); // AB
System.out.println(a == b); // true

これは String と対照的です。String は不変なので、変更系に見える処理は新しい文字列を返します。

問題での判断順

  1. 型を見る。 String か StringBuilder かで挙動が違う。
  2. 変更系メソッドの戻り値を受け取っているか。 Stringなら特に重要。
  3. == か equals かを見る。 == は参照比較。
  4. リテラル・new・intern・コンパイル時定数を分ける。 文字列プールの問題で必要。
  5. メソッドの境界を確認する。 substring、charAt、delete の範囲に注意。
  6. nullがメソッド呼び出し側にないかを見る。 左辺nullならNPE。

ミニ演習

Q1. String s="abc"; s.toUpperCase(); System.out.println(s);

abc です。 toUpperCase() の戻り値を受け取っていないため、元の String は変わりません

Q2. String a="a"; String b="b"; System.out.println(a + b == "ab");

false です。通常の変数を使った連結は実行時の文字列連結であり、リテラル "ab" と同じ参照とは限りません。内容比較なら equals を使います

Q3. StringBuilder sb = new StringBuilder("abc"); String x = sb.substring(1); System.out.println(sb);

abc です。 StringBuilder.substring は String を返すだけで、元のBuilderを変更しません

Q4. new StringBuilder("x").equals(new StringBuilder("x")) は？

false です。 StringBuilder の equals は内容比較として考えてはいけません。内容比較するなら toString() 後に比較します
