Java SilverのStringとStringBuilder

String問題は「不変性」「戻り値」「文字列プール」「StringBuilderとの違い」を分けて読むと安定します。

1. Stringは不変

String s = "abc";
s.replace('b', 'x');
System.out.println(s); // abc

replace は新しい文字列を返します。元の s は変わりません。

s = s.replace('b', 'x'); // axc

2. substringのendは含まない

String s = "abcdef";
System.out.println(s.substring(2, 5)); // cde

開始位置は含み、終了位置は含みません。添字2は c、5の手前までです。

3. concatも戻り値を受け取る

String s = "A";
s.concat("B");
System.out.println(s); // A

concat も元の文字列を変更しません。

4. +演算子の左から評価

System.out.println(1 + 2 + "A" + 3 + 4); // 3A34

文字列が出るまでは数値加算、文字列が出た後は文字列連結です。

5. internと文字列プール

String a = new String("Java");
String b = "Java";
System.out.println(a == b);          // false
System.out.println(a.intern() == b); // true

intern() の戻り値はプール上の参照です。

6. StringBuilderは可変

StringBuilder sb = new StringBuilder("A");
sb.append("B");
System.out.println(sb); // AB

StringBuilder の多くのメソッドは同じオブジェクトを変更します。

7. StringBuilderのinsert/delete

StringBuilder sb = new StringBuilder("abcd");
sb.insert(2, "X");   // abXcd
sb.delete(1, 3);      // acd ではなく、現在の文字列から範囲削除

連続して操作する問題では、毎回現在の状態を書き換えて追います。

8. StringBuilder.equalsは内容比較ではない

var a = new StringBuilder("A");
var b = new StringBuilder("A");
System.out.println(a.equals(b)); // false

内容比較したい場合は a.toString().equals(b.toString()) とします。

9. indexOf

String s = "abcabc";
System.out.println(s.indexOf("bc"));    // 1
System.out.println(s.indexOf("bc", 2)); // 4

見つからなければ -1 です。第2引数は検索開始位置です。

10. isBlankとisEmpty

System.out.println("".isEmpty());      // true
System.out.println("   ".isEmpty());   // false
System.out.println("   ".isBlank());   // true

isEmpty() は長さ0だけ、isBlank() は空白のみも対象です。

試験での注意

  • 戻り値を受け取っていない replace / substring / concat は元の文字列を変えない。
  • StringBuilder は変わるが、equals() は内容比較ではない。
  • == が出たら文字列プールと new の有無を見る。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java基礎〜Silver共通。この範囲は、基礎理解とSilverの読解をつなぐ部分です。まず「コンパイルできるか」「実行時にどこまで進むか」「出力は何か」の順で考えます。

本文で必ず押さえること

  • 値そのものを見るのか、参照先を見るのかを分ける。
  • Stringは変更されるのではなく、新しい文字列が作られる場面が多い。
  • 配列・ArrayList・StringBuilderは、同じ変数名でも参照先と中身の変化を分けて追う。

この後の演習で見るポイント

  • ==なら参照比較、equalsなら内容比較を疑う。
  • メソッド呼び出し後に戻り値を受け取っているか確認する。
  • nullを使う前にメソッド呼び出しが発生していないか確認する。

ここで落ちる典型ミス

  • Stringのreplaceやsubstringで元の文字列が変わると思い込む。
  • 配列変数の代入を「コピー」と誤解する。コピーされるのは参照であり、配列本体ではない。
  • ArrayListのremoveで、引数がインデックスなのか要素なのかを見落とす。

演習前の小課題

String s = "abc"; s.substring(1); System.out.println(s); の出力を、Stringの不変性と戻り値の扱いから説明してください。
