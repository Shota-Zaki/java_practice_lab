Java SilverのStringとStringBuilder頻出メソッドまとめ

文字列問題では、まず「変更されるのか」「戻り値を受け取っているのか」を確認します。String は不変、StringBuilder は可変です。

1. Stringは不変

String s = "abcde";
s.replace('c', 'x');
s.substring(2, 4);
System.out.println(s); // abcde

replace や substring は元の文字列を変更しません。変更後の新しい String を返します。戻り値を受け取らなければ結果は捨てられます。

s = s.replace('c', 'x'); // abxde
s = s.substring(2, 4);    // xd

2. StringBuilderは可変

StringBuilder sb = new StringBuilder("abc");
sb.append("d");
sb.insert(0, "X");
sb.delete(1, 2);
System.out.println(sb); // Xbcd

StringBuilder の多くのメソッドは自分自身を変更して、同じインスタンスへの参照を返します。

3. 文字列プールとintern()

String a = "Java";
String b = "Java";
String c = new String("Java");
String d = c.intern();

System.out.println(a == b); // true
System.out.println(a == c); // false
System.out.println(a == d); // true

文字列リテラルは文字列プールで共有されます。new String("Java") は別インスタンスを作ります。intern() はプール上の同じ内容の文字列参照を返します。

6. StringBuilderの比較に注意

StringBuilder a = new StringBuilder("A");
StringBuilder b = new StringBuilder("A");
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // false
System.out.println(a.toString().equals(b.toString())); // true

StringBuilder は String のような内容比較の equals() を期待してはいけません。内容比較したいなら toString() してから比較します。

7. テキストブロックの基本

Java Silverではテキストブロックの改行やインデントも狙われます。""" で始まり、開始区切りの直後には改行が必要です。

String text = """
    A
    B
    """;

末尾の改行が含まれるか、インデントがどの程度取り除かれるかを問う細かい問題もあります。迷ったら実際の文字列に改行が含まれている前提で追います。
