String・StringBuilder・配列・ArrayList完全整理

SilverのAPI問題は、暗記ではなく「不変か可変か」「配列かListか」「戻り値か副作用か」「範囲外か」を分けて読む単元です。ここを曖昧にすると、出力問題でほぼ確実に落とします。

最初に固定する判定軸

  1. Stringは不変。 substring、replace、trim などは新しい String を返すだけ。
  2. StringBuilderは可変。 append、insert、delete、reverse は同じオブジェクトを書き換える。
  3. 配列は固定長。 要素数は変わらない。length はフィールド。
  4. ArrayListは可変長。 add / remove で要素数が変わる。size() はメソッド。
  5. == は参照比較。 内容比較は基本的に equals。ただし StringBuilder.equals は内容比較として扱わない。
  6. 範囲外は実行時例外。 コンパイルは通っても、添字・範囲・nullで落ちる問題が多い。

Stringは不変

String は一度作られると中身を変更できません。変更しているように見えるメソッドは、変更後の新しい String を返します。

String s = " Java ";
s.trim();
s.replace("J", "j");
System.out.println(s); // " Java "

戻り値を受け取った場合だけ、変数が新しい文字列を参照します。

String s = " Java ";
s = s.trim();
s = s.replace("J", "j");
System.out.println(s); // "java"
試験では「メソッドを呼んだか」ではなく「戻り値を受け取ったか」を見ます。ここを飛ばすと、String問題は安定しません。

文字列プール・new・intern

文字列リテラルは文字列プールを使います。同じリテラルは同じインスタンスを参照します。

String a = "abc";
String b = "abc";
System.out.println(a == b);      // true
System.out.println(a.equals(b)); // true

new String("abc") は新しいインスタンスを作ります。内容が同じでも参照先は別です。

String a = "abc";
String b = new String("abc");
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

intern() は文字列プール上の同じ内容の参照を返します。

String a = new String("sample");
String b = a;
String c = a.intern();
String d = "sample";

System.out.println(a == b); // true
System.out.println(a == c); // false
System.out.println(c == d); // true

コンパイル時定数の文字列結合

リテラル同士やコンパイル時定数だけの結合は、コンパイル時に1つの文字列として扱われます。

String a = "ab";
String b = "a" + "b";
System.out.println(a == b); // true
final String x = "a";
final String y = "b";
String z = x + y;
System.out.println(z == "ab"); // true

通常の変数が絡むと、実行時の文字列連結になります。

String x = "a";
String y = "b";
String z = x + y;
System.out.println(z == "ab");      // false
System.out.println(z.equals("ab")); // true

substring・charAtの境界

substring(begin, end) は begin以上、end未満です。end の文字は含みません。

String s = "abcdef";
System.out.println(s.substring(1, 4)); // bcd
System.out.println(s.substring(1, 1)); // 空文字
System.out.println(s.substring(4));    // ef

範囲が不正なら実行時例外です。コンパイルエラーではありません。

String s = "abc";
// s.charAt(3);       // StringIndexOutOfBoundsException
// s.substring(-1);   // StringIndexOutOfBoundsException
// s.substring(2, 1); // StringIndexOutOfBoundsException
// s.substring(4);    // StringIndexOutOfBoundsException

nullとString

+ による文字列連結では、null は文字列 "null" として扱われます。

String s = null;
System.out.println(s + "x"); // nullx

しかし、null 参照に対してメソッドを呼ぶと NullPointerException です。

String s = null;
// s.equals("x");       // NullPointerException
System.out.println("x".equals(s)); // false

concat は + と違い、引数に null を渡すと落ちます。

String s = "A";
// s.concat(null); // NullPointerException

StringBuilderは可変

StringBuilder は同じオブジェクトの中身を書き換えます。戻り値を受け取らなくても、変更は残ります。

StringBuilder sb = new StringBuilder("ab");
sb.append("c");
System.out.println(sb); // abc

多くの変更系メソッドは自分自身への参照を返すため、メソッドチェーンできます。

StringBuilder sb = new StringBuilder("a");
StringBuilder r = sb.append("b").append("c");

System.out.println(sb);      // abc
System.out.println(r);       // abc
System.out.println(sb == r); // true

StringBuilderのsubstringとequals

StringBuilder.substring は String を返すだけで、元のBuilderは変わりません。

StringBuilder sb = new StringBuilder("abcdef");
String x = sb.substring(1, 3);
System.out.println(x);  // bc
System.out.println(sb); // abcdef

StringBuilder の equals は、String のような内容比較として扱ってはいけません。内容比較したいなら toString() します。

StringBuilder a = new StringBuilder("abc");
StringBuilder b = new StringBuilder("abc");
System.out.println(a == b);      // false
System.out.println(a.equals(b)); // false
System.out.println(a.toString().equals(b.toString())); // true

StringBuilderのコンストラクタと容量

new StringBuilder(10) は文字列 "10" を入れるのではなく、容量10の空Builderを作ります。

StringBuilder sb = new StringBuilder(10);
System.out.println(sb.length()); // 0
System.out.println(sb.capacity()); // 10

new StringBuilder("10") なら中身は "10" です。

StringBuilder sb = new StringBuilder("10");
System.out.println(sb.length()); // 2

配列の基本

配列は同じ型の値または参照を並べる固定長オブジェクトです。作成後に長さは変わりません。

int[] a = new int[3];
System.out.println(a.length); // 3
System.out.println(a[0]);     // 0

length はメソッドではなくフィールドです。

String s = "abc";
int[] a = {1, 2, 3};
System.out.println(s.length()); // Stringはメソッド
System.out.println(a.length);   // 配列はフィールド
// System.out.println(a.length()); // コンパイルエラー

String[] names = new String[2];

配列の添字と実行時例外

添字は0から始まり、最後は length - 1 です。

int[] a = {10, 20, 30};
System.out.println(a[0]); // 10
System.out.println(a[2]); // 30
// System.out.println(a[3]); // ArrayIndexOutOfBoundsException

添字が範囲外かどうかは、コードが実行されるまで確定しないことがあります。よって、基本的には実行時例外です。

多次元配列とジャグ配列

Javaの多次元配列は「配列の配列」です。各行の長さが違っても構いません。

int[][] a = new int[2][];
a[0] = new int[]{1, 2};
a[1] = new int[]{3, 4, 5};

System.out.println(a.length);    // 2
System.out.println(a[0].length); // 2
System.out.println(a[1].length); // 3

内側の配列を作っていない場合は null です。

int[][] a = new int[2][];
// System.out.println(a[0][0]); // NullPointerException

配列の参照コピー

配列変数には配列オブジェクトへの参照が入ります。代入すると中身をコピーするのではなく、参照をコピーします。

int[] a = {1, 2, 3};
int[] b = a;
b[0] = 99;

System.out.println(a[0]); // 99
System.out.println(a == b); // true

拡張for文と配列

拡張for文の変数は、各要素の値または参照を受け取ります。プリミティブ配列では、ループ変数を書き換えても配列要素は変わりません。

int[] a = {1, 2, 3};
for (int n : a) {
    n = 9;
}
System.out.println(a[0]); // 1

参照型配列では、参照先のオブジェクトを変更すれば見えます。ただし、ループ変数に別オブジェクトを代入しても配列要素の参照は変わりません。

StringBuilder[] arr = {new StringBuilder("A")};
for (StringBuilder sb : arr) {
    sb.append("B");
}
System.out.println(arr[0]); // AB

Arraysユーティリティ

配列をそのまま出力すると、内容ではなく型名とハッシュのような表示になります。内容確認には Arrays.toString を使います。

import java.util.Arrays;

int[] a = {3, 1, 2};
System.out.println(a);                  // [I@... のような表示
System.out.println(Arrays.toString(a)); // [3, 1, 2]
Arrays.sort(a);
System.out.println(Arrays.toString(a)); // [1, 2, 3]

多次元配列の内容表示は Arrays.deepToString です。

配列の共変性とArrayStoreException

参照型配列は共変です。String[] は Object[] として扱えます。ただし、実体が String[] なら、Integer は入れられません。

Object[] arr = new String[2];
arr[0] = "A";
// arr[1] = Integer.valueOf(10); // ArrayStoreException

これはコンパイルエラーではなく実行時例外です。宣言型だけでなく、実体型を見る必要があります。

ArrayListの基本

ArrayList は要素数を増減できるコレクションです。使うには通常 java.util.ArrayList をimportします。

import java.util.ArrayList;

ArrayList list = new ArrayList<>();
list.add("A");
list.add("B");
System.out.println(list.size()); // 2
System.out.println(list.get(0)); // A

配列は length、ArrayListは size() です。

int[] a = {1, 2, 3};
ArrayList list = new ArrayList<>();
list.add(1);

System.out.println(a.length);
System.out.println(list.size());

add・set・removeの戻り値

Silverでは、メソッドの副作用だけでなく戻り値も問われます。

import java.util.*;

ArrayList list = new ArrayList<>();
System.out.println(list.add("A")); // true
list.add("B");
System.out.println(list.set(0, "X")); // A
System.out.println(list); // [X, B]
System.out.println(list.remove(1)); // B
System.out.println(list); // [X]

add(index, e) は void なので、出力しようとするとコンパイルエラーです。

ArrayList list = new ArrayList<>();
// System.out.println(list.add(0, "A")); // コンパイルエラー

remove(int) と remove(Object)

ArrayList の最重要ひっかけです。remove(1) は「値1」ではなく「添字1」を削除しようとします。

import java.util.*;

ArrayList list = new ArrayList<>();
list.add(1);
list.add(2);
list.add(3);

System.out.println(list.remove(1)); // 2: 添字1の要素を削除
System.out.println(list);           // [1, 3]

値としての Integer.valueOf(1) を消したいなら、引数を Object 側に寄せます。

ArrayList list = new ArrayList<>(List.of(1, 2, 3));
System.out.println(list.remove(Integer.valueOf(1))); // true
System.out.println(list); // [2, 3]
ここは暗記ではなく、オーバーロード解決です。1 は int として remove(int index) に合いやすい。

List.of と Arrays.asList

List.of は変更不可のListを返します。add や set は実行時例外です。

import java.util.*;

List list = List.of("A", "B");
// list.add("C"); // UnsupportedOperationException
// list.set(0, "X"); // UnsupportedOperationException

Arrays.asList は固定サイズのListです。set は可能ですが、add / remove は不可です。また元の配列と連動します。

import java.util.*;

String[] arr = {"A", "B"};
List list = Arrays.asList(arr);
list.set(0, "X");
System.out.println(Arrays.toString(arr)); // [X, B]
// list.add("C"); // UnsupportedOperationException

変更可能な ArrayList にしたいなら、包み直します。

ArrayList list = new ArrayList<>(List.of("A", "B"));
list.add("C");
System.out.println(list); // [A, B, C]

ArrayListとnull

ArrayList は通常 null を要素として持てます。

ArrayList list = new ArrayList<>();
list.add(null);
System.out.println(list.size()); // 1
System.out.println(list.get(0)); // null

ただし、取り出した null に対してメソッドを呼べばNPEです。

ArrayList list = new ArrayList<>();
list.add(null);
// System.out.println(list.get(0).length()); // NullPointerException

出力問題での読み順

  1. コンパイル可否を見る。 import不足、メソッド名、戻り値void、型不一致を先に潰す。
  2. オブジェクトの種類を見る。 String / StringBuilder / 配列 / ArrayList を分ける。
  3. 可変か不変かを見る。 Stringは戻り値、StringBuilder・ArrayListは副作用に注意。
  4. 参照コピーを見る。 代入で中身がコピーされるわけではない。
  5. 添字と範囲を見る。 最後の添字は length - 1 または size() - 1。
  6. nullを見る。 出力なら null と表示される場合があるが、メソッド呼び出しならNPE。

読解ミニ演習

Q1. String s="abc"; s.toUpperCase(); System.out.println(s);

abc。String は不変で、戻り値を受け取っていません。

Q2. String a="a"; String b="b"; System.out.println(a + b == "ab");

false。通常の変数を使った連結は実行時処理です。内容比較なら equals です。

Q3. StringBuilder sb=new StringBuilder("abc"); String x=sb.substring(1); System.out.println(sb);

abc。StringBuilder.substring は String を返すだけで、Builder本体は変わりません。

Q4. int[] a={1,2}; int[] b=a; b[0]=9; System.out.println(a[0]);

9。配列参照をコピーしているため、同じ配列オブジェクトを見ています。

Q5. ArrayList<Integer> list = new ArrayList<>(List.of(1,2,3)); System.out.println(list.remove(1));

2。remove(1) は remove(int index) と解釈され、添字1の要素を削除します。

Q6. List<String> list = Arrays.asList("A", "B"); list.set(0, "X"); System.out.println(list);

[X, B]。Arrays.asList は固定サイズですが、set は可能です。add や remove は不可です。
