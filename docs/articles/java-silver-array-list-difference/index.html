配列とArrayListの違いをJava Silver向けに整理

試験対策と実装理解の両方で使えるよう、例を中心に整理します。

結論

配列は固定長、ArrayListは要素数を増減できます。ただしAPIの挙動には罠があります。

配列

String[] a = new String[2];
System.out.println(a[0]); // null

ArrayList.remove

List<Integer> list = new ArrayList<>(List.of(1,2,3));
list.remove(1); // 添字1を削除

remove(int)とremove(Object)の違いに注意します。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java11 / Java17 Silver共通。Silver共通範囲では、バージョン固有構文に逃げず、型、参照、制御構文、クラス、継承、例外の基本ルールだけで判定します。Java17固有の記事はJava17 Silver側に分けています。

本文で必ず押さえること

  • 値そのものを見るのか、参照先を見るのかを分ける。
  • Stringは変更されるのではなく、新しい文字列が作られる場面が多い。
  • 配列・ArrayList・StringBuilderは、同じ変数名でも参照先と中身の変化を分けて追う。

この後の演習で見るポイント

  • ==なら参照比較、equalsなら内容比較を疑う。
  • メソッド呼び出し後に戻り値を受け取っているか確認する。
  • nullを使う前にメソッド呼び出しが発生していないか確認する。

ここで落ちる典型ミス

  • Stringのreplaceやsubstringで元の文字列が変わると思い込む。
  • 配列変数の代入を「コピー」と誤解する。コピーされるのは参照であり、配列本体ではない。
  • ArrayListのremoveで、引数がインデックスなのか要素なのかを見落とす。

演習前の小課題

String s = "abc"; s.substring(1); System.out.println(s); の出力を、Stringの不変性と戻り値の扱いから説明してください。
