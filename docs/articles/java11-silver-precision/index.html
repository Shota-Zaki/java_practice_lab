Java11 Silverの頻出差分と注意点

Java11 Silverでは、Java17ほどrecordやsealedは出ませんが、var、String、配列、継承、例外は強く問われます。

varは型推論であり、動的型付けではない

var x = 10; // int
x = 20;     // OK
x = "20";  // コンパイルエラー

var は初期化式から型を決めるだけです。一度 int と決まった変数に String は代入できません。

varが使えない場所

class Sample {
    var field = 10; // 不可
    void test(var x) {} // 不可
    void run() {
        var local = 10; // 可
    }
}

Java11 Silverでは、var の使用可能箇所を正確に問われます。ローカル変数に限定して考えると安全です。

Java17との差分を見る前に

Java11 Silver対策では、まずJava11範囲の基礎を固めるべきです。record、sealed、switch式の詳細に入る前に、従来のクラス・継承・例外を安定させます。
