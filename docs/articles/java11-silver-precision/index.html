品質強化記事

Java11 Silverの頻出差分と注意点

Java11 Silverでは、Java17ほどrecordやsealedは出ませんが、var、String、配列、継承、例外は強く問われます。

varは型推論であり、動的型付けではない

var x = 10; // int
x = 20;     // OK
x = "20";  // コンパイルエラー

var は初期化式から型を決めるだけです。一度 int と決まった変数に String は代入できません。

varが使えない場所

class Sample {
    var field = 10; // 不可
    void test(var x) {} // 不可
    void run() {
        var local = 10; // 可
    }
}

Java11 Silverでは、var の使用可能箇所を正確に問われます。ローカル変数に限定して考えると安全です。

Java17との差分を見る前に

Java11 Silver対策では、まずJava11範囲の基礎を固めるべきです。record、sealed、switch式の詳細に入る前に、従来のクラス・継承・例外を安定させます。

本文で必ず押さえること

  • 用語を覚えるだけでなく、コード上でどの判断に使う知識かを確認する。
  • 先にコンパイル可否、次に実行時の流れ、最後に出力を考える。
  • 自信がない箇所は、変数の型と値を紙に分けて書く。

この後の演習で見るポイント

  • 本文の例を少し変えても同じ判断ができるか確認する。

  • 関連する演習では、正解番号ではなく判断根拠を残す。

ここで落ちる典型ミス

  • 用語の意味だけ覚えて、コード読解に接続しない。
  • 「なんとなく見たことがある」で選択肢を選ぶ。
  • 解説を読んだあとに、自力で再判定しない。
