Java Silverのstaticで間違えるポイント

static文脈、インスタンスメンバー、フィールド隠蔽を整理します。

staticからインスタンスメンバーは直接使えない

class A {
    int n;
    static void test() {
        // n++; // コンパイルエラー
    }
}

インスタンス経由でstaticにアクセスできるが非推奨

class A { static int n = 10; }
A a = new A();
System.out.println(a.n); // コンパイル可能だが推奨されない

試験注意

staticメソッドはオーバーライドではなく隠蔽になります。
