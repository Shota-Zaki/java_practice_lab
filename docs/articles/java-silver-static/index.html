staticで間違えるポイント完全整理

staticは「クラス側に属する」という意味です。mainがstaticであること、staticメソッドからインスタンスメンバを直接使えないこと、staticフィールドが共有されることを整理します。

staticとは

static が付いたメンバは、インスタンスごとではなくクラス側に属します。

staticメンバクラス側に属するフィールドやメソッド。クラス名から呼ぶのが基本。
インスタンスメンバnewで作られた各インスタンスに属するフィールドやメソッド。
static文脈staticメソッドやstatic初期化ブロックの中。特定のインスタンスが決まっていない。
共有staticフィールドはインスタンス間で1つの値を共有する。
class Counter {
    static int total; // クラスに1つ
    int value;        // インスタンスごと

    Counter() {
        total++;
        value++;
    }
}

staticとインスタンスの違い

対象所属個数基本の呼び出し
staticフィールドクラスクラスに1つClassName.field
staticメソッドクラスクラスに1つClassName.method()
インスタンスフィールドインスタンスインスタンスごとobj.field
インスタンスメソッドインスタンスインスタンス経由obj.method()
Counter a = new Counter();
Counter b = new Counter();
System.out.println(Counter.total); // 2
System.out.println(a.value);       // 1
System.out.println(b.value);       // 1

staticメソッドからインスタンスメンバを直接使えない

class Sample {
    int value = 10;
    static int count = 20;

    void instanceMethod() {}
    static void staticMethod() {}

    static void test() {
        System.out.println(count); // OK
        staticMethod();            // OK
        // System.out.println(value); // NG
        // instanceMethod();          // NG

        Sample s = new Sample();
        System.out.println(s.value); // OK
    }
}

staticメソッド内には暗黙の this がありません。だからインスタンスメンバを使うには参照が必要です。

mainがstaticである意味

main はJVMがインスタンスを作らずに呼び出す入口です。そのため static でなければなりません。

public class Main {
    int value = 10;

    public static void main(String[] args) {
        // System.out.println(value); // NG
        Main m = new Main();
        System.out.println(m.value);  // OK
    }
}

static初期化ブロック

static初期化ブロックは、クラス初期化時に1回だけ実行されます。

class Sample {
    static {
        System.out.println("static");
    }
    {
        System.out.println("instance");
    }
    Sample() {
        System.out.println("constructor");
    }
}
new Sample();
new Sample();

出力は static が1回、instance と constructor がnewごとに2回です。

staticメソッドはoverrideされない

staticメソッドはインスタンスの実体型による動的ディスパッチの対象ではありません。サブクラスで同じ形を書いても、overrideではなく隠蔽です。

class A {
    static void test() { System.out.println("A"); }
}
class B extends A {
    static void test() { System.out.println("B"); }
}
A obj = new B();
obj.test(); // A

通常はインスタンス参照からstaticメソッドを呼ばず、A.test() のようにクラス名で呼ぶべきです。

不可能・誤りパターン

コード判定理由
staticメソッドから this不可static文脈にthisはない
staticメソッドからインスタンスフィールド直接参照不可対象インスタンスが不明
トップレベル static class A不可staticトップレベルクラスはない
staticメソッドをoverride不可同名staticメソッドは隠蔽
インスタンス変数でstaticフィールドを変更可能だが非推奨実体はクラス側の共有値
