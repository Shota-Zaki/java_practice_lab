キャストとinstanceofの読み方

参照型キャスト、ClassCastException、instanceofパターンを整理します。

キャストは実体を変えない

(B)a と書いてもオブジェクトがBに変わるわけではありません。

実体が合わないと例外

実体がBでないのにBへキャストするとClassCastExceptionです。

instanceofで確認できる

instanceofは実行時の実体を確認します。Java17ではパターン変数も使えます。

例

Object o = "Java";
if (o instanceof String s) {
    System.out.println(s.length());
}

試験・学習での注意

  • 実行結果を見る前に、まずコンパイルできる形か確認します。
  • 似た構文でも、型・戻り値・アクセス修飾子で結果が変わります。
  • 暗記だけでなく、なぜその結果になるかを言語化してください。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Java11 / Java17 Silver共通。Silver共通範囲では、バージョン固有構文に逃げず、型、参照、制御構文、クラス、継承、例外の基本ルールだけで判定します。Java17固有の記事はJava17 Silver側に分けています。

本文で必ず押さえること

  • 変数の型と、実際のインスタンスの型を分ける。
  • フィールド参照は変数の型、オーバーライドメソッド呼び出しは実体の型が効く。
  • interfaceのメソッド・フィールドは暗黙の修飾子を意識する。

この後の演習で見るポイント

  • 代入が上向き変換か、下向きキャストかを確認する。
  • キャストはコンパイル可否と実行時可否を分けて判定する。
  • sealedではpermitsと、継承先のfinal/sealed/non-sealedを必ず確認する。

ここで落ちる典型ミス

  • 親型変数だから親のメソッドだけが実行される、と誤解する。オーバーライドは実体側。
  • キャストを書けば必ず安全に変換できると考える。実体が対応していなければ実行時例外になる。
  • interfaceの抽象メソッドがpackage-privateになると思い込む。基本はpublic。

演習前の小課題

Parent p = new Child(); の1行を見たら、変数型Parentと実体Childを分け、フィールド・メソッド・キャストの3つで判断してください。
