実行時例外の原因まとめ

コンパイルは通るが、実行すると止まる代表例を整理します。

コンパイルエラーとの違い

コンパイルエラーは実行前に発見される文法・型・到達可能性などの問題です。実行時例外は、文法としては正しいが、実行中の値や状態によって発生します。

NullPointerException

nullに対してメソッドやフィールドを使うと発生します。String配列を作っただけでは各要素はnullです。

String[] arr = new String[2];
System.out.println(arr[0].length()); // NullPointerException

ArrayIndexOutOfBoundsException

配列の範囲外にアクセスすると発生します。長さが3なら有効な添字は0,1,2です。

int[] nums = {1, 2, 3};
System.out.println(nums[3]); // ArrayIndexOutOfBoundsException

ClassCastException

実体の型に合わないキャストをすると発生します。参照型の宣言だけでなく、実際のインスタンス型を確認します。

Object obj = "Java";
Integer n = (Integer) obj; // ClassCastException

NumberFormatException

数字として解釈できない文字列をparseすると発生します。

int n = Integer.parseInt("10A"); // NumberFormatException
