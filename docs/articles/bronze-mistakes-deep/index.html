Java Bronzeで落ちやすい基礎ミス

Bronzeは難しいAPIより、基礎文法の読み違いで落ちます。ここではBronzeで間違いやすい基礎を、短いコードで確認します。

代入と比較を混ぜない

int x = 10;
if (x = 20) { } // コンパイルエラー。intはbooleanではない
if (x == 20) { } // 比較

Javaの if 条件は boolean だけです。C言語のように数値を条件にできません。

配列の範囲

int[] nums = {10, 20, 30};
System.out.println(nums[0]); // 10
System.out.println(nums[3]); // 実行時例外

配列の添字は0から始まります。length は要素数であり、最後の添字ではありません。

メソッドの戻り値

int add(int a, int b) {
    return a + b;
}
void print() {
    System.out.println("A");
}

void は値を返しません。return 値; を書くとコンパイルエラーです。

演習前チェック

この記事を演習で使える知識に変える

対象レベル：Bronze。Bronzeでは、細かい新機能よりも「文の実行順序」「変数の値」「条件式の真偽」を正確に追うことを優先します。record、sealed、Streamなどの高度な構文・APIはここでは追いません。

本文で必ず押さえること

  • この記事は知識そのものより、学習手順の改善が目的です。
  • 間違いを「知らなかった」で終わらせず、判断手順のどこが雑だったかに分解します。
  • 演習の正答率だけでなく、根拠を言語化できるかを確認します。

この後の演習で見るポイント

  • 解いた直後に、正解理由と誤答理由を1文ずつ書く。
  • 間違えた問題をタグで分類する。型、参照、例外、継承、APIなどに分ける。
  • 翌日に同じ問題を見て、選択肢を見ずに判断手順を再現する。

ここで落ちる典型ミス

  • 正解した問題を復習対象から外す。偶然正解を放置すると弱点が残る。
  • 解説を読んで納得しただけで、再現できると思い込む。
  • 広く浅く進めすぎて、同じ型のミスを繰り返す。

演習前の小課題

直近で間違えた問題を1つ選び、「知識不足」「読み落とし」「手順ミス」「暗記違い」のどれかに分類してください。
