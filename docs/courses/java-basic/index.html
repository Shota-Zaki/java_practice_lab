1からわかるJava基礎解説

Lesson 76〜95を追加しました。機械語・メモリ・型・実行・オブジェクト指向からSilver/Goldへの橋渡しまで扱います。

Lesson 76 CPUが命令を処理する流れ

プログラムは、最終的にはCPUが理解できる小さな命令の列として実行されます。Javaではこの途中にバイトコードとJVMが入ります。

Lesson 77 データがメモリに置かれるとはどういうことか

変数・参照・オブジェクトを理解するには、値がどこにあるかをざっくり把握する必要があります。

Lesson 78 2進数・16進数・ビット演算の入口

Javaでは整数リテラルやビット演算が出てきます。Silverでは深いビット演算より、表記と型の読み間違いに注意します。

Lesson 79 Javaの型システムを大きく見る

型とは、値に対して「何ができるか」を決めるルールです。

Lesson 80 クラスファイルと実行単位

Javaではソースファイル、クラス、クラスファイル、実行コマンドの関係を整理しておく必要があります。

Lesson 81 mainメソッドの形を正確に読む

mainメソッドはJavaプログラムの起点です。似た形の間違いが多いので、形で覚えるだけでは危険です。

Lesson 82 コンパイル時に決まること・実行時に決まること

Java Silverで重要なのは「いつエラーになるか」を分類することです。

Lesson 83 式の途中で値が変わるケース

インクリメントや短絡評価では、式を左から追う力が必要です。

Lesson 84 配列を「箱の集まり」として読む

配列問題は、長さ・添字・初期値・参照共有の4点を見ます。

Lesson 85 Stringを読むときの基本姿勢

Stringは不変です。変更系に見えるメソッドでも、元の文字列は変わりません。

Lesson 86 メソッド呼び出しは「型」と「実体」を分けて読む

継承問題では、変数の型と実際のオブジェクトの型を分けて考えます。

Lesson 87 コンストラクタ呼び出し順の考え方

インスタンス生成では、親クラスの初期化が先に行われます。

Lesson 88 例外を「種類」と「発生タイミング」で見る

例外処理は、checked / unchecked と catch順序を整理すると理解しやすくなります。

Lesson 89 interfaceを読むための最低限

interfaceは「使い方の約束」を定義します。Java Silverではdefaultメソッドや定数も重要です。

Lesson 90 recordを初めて使う前に知ること

recordはデータ保持用クラスを簡潔に書くための仕組みです。

Lesson 91 ジェネリクスを怖がらずに読む

ジェネリクスは、コレクションに入る型をコンパイル時に制限する仕組みです。

Lesson 92 ラムダ式の入口

ラムダ式は、処理そのものを値のように渡すための書き方です。

Lesson 93 Stream APIの入口

Stream APIは、コレクションなどの要素を流れとして処理する仕組みです。

Lesson 94 テストしながら学ぶ習慣

Javaは小さいコードを動かして確認すると理解が速くなります。

Lesson 95 基礎解説の総まとめ
