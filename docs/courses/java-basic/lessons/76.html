Lesson 76 CPUが命令を処理する流れ

プログラムは、最終的にはCPUが理解できる小さな命令の列として実行されます。Javaではこの途中にバイトコードとJVMが入ります。

CPU・メモリ・命令の関係JavaではなぜJVMが必要か試験・学習での注意

CPU・メモリ・命令の関係

CPUは命令を取り出し、必要な値を読み、計算し、結果をメモリやレジスタに戻します。Javaを書くときにCPU命令を直接意識する必要はありませんが、変数に値を入れる、メソッドを呼ぶ、条件で分岐するという処理は、最終的にはこの単純な処理の組み合わせです。

Java学習で重要なのは、機械語そのものではなく「プログラムは上から順に評価され、条件やループで流れが変わる」という感覚です。

JavaではなぜJVMが必要か

Javaのソースコードは直接CPUが実行するのではなく、まずjavacでバイトコードに変換されます。そのバイトコードをJVMが読み、実行環境に合わせて処理します。

Sample.java --javac--> Sample.class --JVM--> 実行

試験・学習での注意

Java SilverではCPUや機械語の詳細は問われにくいですが、javacとjavaの役割の違い、コンパイル時エラーと実行時例外の違いは非常に重要です。
