Java基礎 Lesson 8

if文と条件式

条件式、boolean、比較、短絡評価を使って分岐を正確に読む。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：条件式・boolean・短絡評価

if文は「trueなら進む」だけです。数値の0/1を条件にできる言語とは違います。

条件式とはtrue または false になる式。
booleanとは真偽値だけを扱う型。true/false。
比較演算子とは>、==など、比較結果をbooleanで返す演算子。
短絡評価とは&&や||で右側の評価が省略されること。
ブロックとは{ }で囲んだ処理のまとまり。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
条件式 / boolean条件式はtrueまたはfalseになる式。ifの条件にはbooleanが必要。
= / ===は代入。==は等価比較。意味が完全に違う。
短絡評価 / 非短絡評価&&と||は必要なければ右側を評価しない。&と|はbooleanでも両側を評価する。

基礎用語一覧で確認する

ifの基本

if (条件式) {
    条件がtrueのときの処理
} else {
    条件がfalseのときの処理
}

条件式に書けるのはtrueまたはfalseになる式。if (1)やif (0)はJavaでは使えない。

比較演算子

演算子意味例
==等しいa == b
!=等しくないa != b
<小さいa < b
>大きいa > b
<=以下a <= b
>=以上a >= b
参照型の==

Stringなどの参照型で==を使うと、内容ではなく参照先が同じかを見る。

短絡評価

&&は左がfalseなら全体がfalseなので右を評価しない。||は左がtrueなら全体がtrueなので右を評価しない。

右側が実行されない例

int x = 0;
if (x > 0 && ++x > 0) {
    System.out.println("A");
}
System.out.println(x); // 0

&や|をbooleanに使うと短絡しない。右側も評価される。

代入と比較の混同

これはコンパイルできる

boolean flag = false;
if (flag = true) {
    System.out.println("run");
}

flag = trueは代入式で、結果としてtrueになるためifに書ける。比較のつもりならflag == trueだが、通常はif (flag)で十分。

手を動かすミニ課題

次のコードでxが最終的にいくつになるかを考える。

int x = 1;
if (x > 0 || ++x > 0) {
    x++;
}
System.out.println(x);

丁寧な補講：理解の芯

if文で最初に見るべきものは、条件式がbooleanになるかどうか。Javaでは、数値の0や1を真偽値として扱わない。if (1)のような書き方はできない。

条件式は左から何となく読むのではなく、比較演算子、論理演算子、短絡評価の順に整理する。&&は左がfalseなら右を評価しない。||は左がtrueなら右を評価しない。

短絡評価は、単なる効率化ではない。右辺にメソッド呼び出しやインクリメントがある場合、実行されるかどうかが結果を変える。

条件式

ifの丸括弧内はbooleanでなければならない。

比較

==、<、>などで真偽値を作る。

短絡評価

&&と||は右辺を評価しない場合がある。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
= / ===は代入、==は比較。条件式内で代入していないか見る。
&& / &&&は短絡評価あり。&は右辺も評価する。右辺に副作用があると差が出る。
数値 / booleanJavaでは数値を真偽値として使えない。if (x)はxがbooleanでなければ不可。

コードを読むときの手順

  1. ifの条件式全体がbooleanか確認する。
  2. 比較演算子で作られるtrue/falseを判断する。
  3. 論理演算子が&&か||かを見る。
  4. 短絡評価で右辺が実行されるか判断する。
  5. 条件がtrueならブロック内、falseならelse側へ進む。

確認用コード

int a = 10;
int b = 0;
if (a < 5 && ++b > 0) {
    a++;
}
System.out.println(a + ":" + b);
確認する点このコードでの判断結論
左辺a < 5はfalse&&なので右辺を評価しない
++b実行されないbは0のまま
出力aは10、bは010:0

よくある誤解を潰す

if (1)はJavaでは使えない

C系の感覚で数値を真偽値にできると思うと間違う。

短絡評価を無視しない

右辺が実行されない場合がある。インクリメントやメソッド呼び出しが右辺にあると結果が変わる。

確認問題：答えまで言語化する

次のコードの出力を答える。

確認問題

int x = 0;
if (true || ++x > 0) {
    System.out.println(x);
}
考え方と答えを見る

出力は0。

||は左辺がtrueなら右辺を評価しない。したがって++xは実行されず、xは0のまま。

理解の流れを整える補強

if文は「条件が正しければ実行」と覚えるだけでは足りません。Javaでは、条件式の結果は必ずbooleanでなければいけません。C言語系の感覚で、0をfalse、1をtrueのように扱うことはできません。

int n = 1;
// if (n) { }      // コンパイルエラー
if (n == 1) {      // OK。比較結果はboolean
    System.out.println("ok");
}

もう1つ重要なのが短絡評価です。&&は左がfalseなら右を評価しません。||は左がtrueなら右を評価しません。これは単なる高速化ではなく、例外を避けるためにも使われます。

String s = null;
if (s != null && s.length() > 0) {
    System.out.println(s);
}

上のコードは、左側のs != nullがfalseなので、右側のs.length()は実行されません。逆に&を使うと右側まで評価され、NullPointerExceptionになります。

演算子右辺評価主な使いどころ
&&必要なときだけ評価通常の条件分岐
||必要なときだけ評価どちらか満たせばよい条件
& / |基本的に両辺を評価booleanでは意図がある場合だけ使う
条件式で見る順番

まず型がbooleanになるか確認し、その後に左から評価順序を追います。見た目の自然言語ではなく、Javaの評価順序で読んでください。

理解チェック

  • if文と条件式を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

ifと条件式の誤解

つまずき正しい切り分け
Javaでも0や1を真偽値にできると思うJavaのif条件はbooleanだけ。数値とは互換性がない。
=と==を混同する=は代入、==は比較。boolean変数では代入式もboolean値を持つため特に危険。
elseの対応先を感覚で読むelseは直前の未対応のifに結びつく。

説明できる状態の目安

  • if条件に置ける型を答えられる
  • 代入式が値を持つことを説明できる
  • dangling elseを説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：ifとboolean

int score = 80;
if (score >= 60) {
    System.out.println("pass");
}

失敗コード

int score = 80;
if (score) {
    System.out.println("pass");
}

修正版

int score = 80;
if (score >= 60) {
    System.out.println("pass");
}
判定ポイント

Javaのif条件はbooleanだけです。数値をそのまま条件にはできません。

理解確認クイズ

用語だけで終わらせず、コード判定と説明まで確認します。

Lv1 用語確認

条件式に必要な型は何ですか。

答えと切り分けを見る

Javaのif、while、forの条件式はbooleanです。数値の0/1を真偽値として扱いません。

Lv2 コード判定

次のコードはコンパイルできますか。

int n = 1;
if (n) {
    System.out.println("A");
}
答えと切り分けを見る

コンパイルできません。ifの条件式はbooleanが必要です。n == 1 のように比較式にします。

Lv3 説明問題

if (a = true) がコンパイルできる場合がある理由を説明してください。

答えと切り分けを見る

aがboolean変数なら、代入式a = true自体の値もbooleanです。そのため条件式として成立します。ただし比較ではなく代入です。
