エラーを怖がらない読み方

エラーメッセージを学習材料に変える方法を整理します。

最初に見るのは行番号

エラーには多くの場合、ファイル名と行番号が出ます。まずその場所を確認します。

1つ目のエラーを優先する

大量のエラーが出ても、最初のエラーが原因で後続エラーが連鎖していることがあります。

英語を全部訳さなくてよい

cannot find symbol、incompatible types、missing return statementなど頻出表現を覚えるだけで読める範囲が増えます。

小さな例

// error: cannot find symbol
// error: incompatible types
// error: missing return statement

確認ポイント

  • 行番号を見る
  • 最初のエラーを優先する
  • 頻出エラー表現を覚える
