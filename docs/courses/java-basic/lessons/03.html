Java基礎 Lesson 3

ソースコードから実行まで

javaファイルを書いてから、画面に結果が出るまでの流れを説明できるようにする。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

先に結論

Javaの基本フローは、.javaを書く、javacで.classにする、javaでクラス名を指定して実行する、の3段階。出力を追う前に、この流れを崩していないかを見る。

基本コマンド

javac Main.java
java Main

1. ソースコードを書く

ソースコードは人間が読むためのテキスト。拡張子は.java。publicなトップレベルクラスがあるなら、ファイル名はそのクラス名と一致させる必要がある。

Main.java

public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}

2. javacでコンパイルする

javacは、文法、型、アクセス、未初期化ローカル変数、checked例外の扱いなどを検査する。通れば.classファイルが作られる。

コード結果理由
int x = 10;通る宣言と初期化が成立している
int x = "10";通らないintにStringは代入できない
System.out.println(x);だけを書く状況次第で通らないxが宣言・初期化されていなければエラー

3. javaで実行する

javaコマンドでは、ファイル名ではなくクラス名を指定する。Main.classではなくMainと書く。

正しい実行例

java Main

初心者がやりがちな誤り

java Main.class

失敗の読み分け

失敗する段階見方例
保存前ファイル名や保存場所の問題違うフォルダに保存している
コンパイル時文法・型・アクセスの問題cannot find symbol
実行時値や実行環境の問題ArrayIndexOutOfBoundsException
順番を飛ばさない

実行時の出力を考える前に、まずコンパイルできるかを確認する。ここを飛ばすと、存在しない出力結果を追いかけることになる。

手を動かすミニ課題

次のファイルをMain.javaとして保存し、javac Main.java、java Mainの順で実行する。

public class Main {
    public static void main(String[] args) {
        System.out.println("first run");
    }
}

丁寧な補講：理解の芯

Javaの実行までの流れは、保存、コンパイル、実行の3段階。学習中はこの順序を固定する。出力を考えるのは、コンパイルできると確認してからでよい。

javac Main.javaではファイル名を指定する。java Mainではクラス名を指定する。ここを混同してjava Main.classと書くと、実行対象の指定がずれる。

また、エラーが出たら「保存したファイル」「現在のフォルダ」「publicクラス名」「mainメソッド」の4点を順番に確認する。初心者のトラブルの大半はこの範囲にある。

保存

.javaファイルとして保存する。publicクラスがあるなら名前を合わせる。

コンパイル

javacで.classを生成する。文法・型エラーはここで出る。

実行

javaでクラス名を指定する。ここから実行時例外や出力が発生する。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
ファイル名 / クラス名コンパイル時はファイル名、実行時はクラス名を意識する。javac Main.java、java Main。
.java / .class.javaはソース、.classはコンパイル結果。手で編集するのは基本的に.java。
コンパイルエラー / 実行時例外前者は.class生成前、後者は実行中。出力を問う前にコンパイル可否を見る。

コードを読むときの手順

  1. ファイル名がpublicクラス名と一致しているか見る。
  2. javac ファイル名.javaが通るか見る。
  3. .classが生成されたか見る。
  4. java クラス名で実行する。
  5. エラーが出たら、今どの段階かを言葉にする。

確認用コード

// Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}
確認する点このコードでの判断結論
ファイル名Main.javapublic class Main と一致
コンパイルjavac Main.javaMain.classができる
実行java MainOKが表示される

よくある誤解を潰す

実行時に.classを付ける

java Main.classではなくjava Main。実行対象はファイル名ではなくクラス名。

違うフォルダでコマンドを打つ

保存した場所とターミナルの現在位置が違うと、ファイルが見つからない。

確認問題：答えまで言語化する

次のファイルをHello.javaとして保存した。コンパイルと実行のコマンドを答える。

確認問題

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
考え方と答えを見る

コンパイルはjavac Hello.java。

実行はjava Hello。java Hello.classではない。

理解の流れを整える補強

このLessonの狙いは、保存 → コンパイル → 実行を1本の流れとして説明できるようにすることです。Javaは、ファイルに書いた瞬間に動くわけではありません。途中で文法・型の検査が入り、実行可能な形に変換されます。

順番操作失敗するとどうなるか
1Main.javaを保存する保存場所やファイル名を間違えるとコマンドから見つからない
2javac Main.java文法・型・ファイル名などの問題でコンパイルエラー
3java Mainクラス名指定ミス、実行時例外、出力結果の不一致が起きる

特に重要なのは、javacに渡すのはファイル名、javaに渡すのはクラス名だという点です。ここを混同すると、コードが正しくても実行できません。

// ファイル名: Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("OK");
    }
}
javac Main.java
java Main

出力結果を考えるのは、コンパイルが成功した後です。コンパイルエラーが出るコードに対して、実行結果を考えても意味がありません。この順序を守るだけで読解精度は上がります。

ここで切るべき誤解

javac Mainやjava Main.javaのような混同は、かなり基本的ですが実務でも最初に詰まりやすい箇所です。ファイルを渡す場面とクラス名を渡す場面を明確に分けてください。

理解チェック

  • ソースコードから実行までを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。
