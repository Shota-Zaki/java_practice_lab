Java基礎 Lesson 12

クラスとオブジェクト

クラス、インスタンス、フィールド、メソッドの関係を整理する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：クラス・インスタンス・メンバ

オブジェクト指向の言葉は最初にまとめて出ます。丸暗記ではなく、コード上の位置で見分けます。

クラスとはオブジェクトを作るための設計図。フィールドやメソッドを定義する。
オブジェクトとはプログラム上で扱う実体。Javaでは多くの場合、newで作る。
インスタンスとはクラスから実際に作られたオブジェクト。
メンバとはクラスの中に直接定義される要素。主にフィールド、メソッド、コンストラクタ。
フィールドとはインスタンスやクラスが保持するデータ。
メソッドとはクラスの中に定義する処理。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
クラス / オブジェクト / インスタンスクラスは設計図。オブジェクトは実体。インスタンスはクラスからnewされた実体。
メンバ / フィールド / メソッドメンバはクラス内に定義する要素。フィールドは状態、メソッドは動作。
ローカル変数 / フィールドローカル変数はメソッド内の一時的な変数。フィールドはインスタンスやクラスが持つ変数。

基礎用語一覧で確認する

クラスの基本構造

class Item {
    String name;      // フィールド
    int price;        // フィールド

    void printInfo() { // メソッド
        System.out.println(name + ":" + price);
    }
}

nameとpriceは各インスタンスが持つ状態。printInfoはその状態を使って処理するメソッド。

インスタンスは別々に状態を持つ

Item a = new Item();
Item b = new Item();

a.name = "apple";
b.name = "banana";

System.out.println(a.name); // apple
System.out.println(b.name); // banana

aとbは別々のインスタンスを指しているため、フィールドの値も別々に持てる。

参照を代入すると同じ実体を見る

Item a = new Item();
Item b = new Item();
a.name = "apple";

b = a;
b.name = "orange";

System.out.println(a.name); // orange

b = aで、bはaと同じインスタンスを指す。中身をコピーしているわけではない。

フィールドとローカル変数

種類宣言場所初期化寿命のイメージ
フィールドクラス直下デフォルト値ありインスタンスが生きている間
ローカル変数メソッド内自動初期化なしメソッド実行中
混同しない

同じintでも、フィールドなのかローカル変数なのかで初期化ルールが変わる。

手を動かすミニ課題

次のコードで、何個のItemインスタンスが作られ、最後にaとbが何を指しているかを書く。

Item a = new Item();
Item b = new Item();
a = b;

丁寧な補講：理解の芯

クラスは設計図、インスタンスはnewで作られた実体。この説明はよく出るが、重要なのは「フィールドはインスタンスごとに持てる」という点。

同じクラスから複数のインスタンスを作ると、それぞれが別々のフィールド値を持つ。片方のインスタンスのフィールドを変えても、もう片方が自動で変わるわけではない。

ただし、変数へ代入しているのはインスタンスそのものではなく参照。a = b;とすると、aもbも同じインスタンスを見るようになる。

クラス

フィールドやメソッドを定義する型・設計。

インスタンス

newでヒープ上に作られる実体。

参照

変数からインスタンスへたどるための情報。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
クラス / インスタンスクラスは定義。インスタンスは実体。newされているかを見る。
フィールド / ローカル変数フィールドはオブジェクトの状態。ローカル変数はメソッド内の一時変数。宣言場所を見る。
同じ値 / 同じインスタンス同じ値を持っていても別インスタンスの場合がある。==は参照の同一性を見る。

コードを読むときの手順

  1. クラス定義でフィールドとメソッドを確認する。
  2. newが何回実行されるか数える。
  3. 各変数がどのインスタンスを参照しているか追う。
  4. フィールド変更がどのインスタンスに対するものか確認する。
  5. 最後にメソッド呼び出しの対象を確認する。

確認用コード

class Item {
    int price;
}

Item a = new Item();
Item b = new Item();
a.price = 100;
b.price = 200;
a = b;
System.out.println(a.price);
確認する点このコードでの判断結論
new2回実行別々のItemインスタンス
a = baがbと同じ参照になる最初のインスタンスへの参照を失う
出力aはb側を見る200

よくある誤解を潰す

クラスが動いている、という表現は雑

実行時に状態を持って動くのは主にインスタンス。クラスは定義。

変数にオブジェクト本体が入っている、は違う

参照型変数に入るのは参照。代入で参照がコピーされる。

確認問題：答えまで言語化する

次の出力を答える。

確認問題

class Box { int value; }

Box x = new Box();
Box y = x;
y.value = 7;
System.out.println(x.value);
考え方と答えを見る

出力は7。

y = xにより、xとyは同じBoxインスタンスを参照する。y.valueの変更は、その同じインスタンスのフィールド変更。

理解の流れを整える補強

クラスは設計図、インスタンスは実体、という説明は入口として有効です。ただしコードを読むときは、どの変数がどのインスタンスを参照しているかまで追う必要があります。

class Item {
    int price;
}

Item a = new Item();
Item b = new Item();
a.price = 100;
b.price = 200;
a = b;
System.out.println(a.price); // 200

a = bの時点で、aはbと同じインスタンスを参照します。最初にaが参照していたインスタンスは、ほかに参照がなければ使えない状態になります。

読む対象確認すること例
new新しいインスタンスが作られるnew Item()
フィールド代入どのインスタンスの状態を変えるかa.price = 100
参照代入変数の参照先が変わるa = b
メソッド呼び出しどのインスタンスに対して呼ぶかa.print()

thisは「今動いているインスタンス自身」を指します。フィールドとローカル変数の名前が同じとき、this.priceならフィールド、priceだけなら近いスコープの変数を指す、という区別に使います。

クラスの問題で雑に読まない

クラス名だけを見ても答えは出ません。インスタンスが何個作られ、変数がどのインスタンスを参照しているかを図にすると一気に読みやすくなります。

理解チェック

  • クラスとオブジェクトを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

クラスとオブジェクトで混ざる点

つまずき正しい切り分け
クラス自体が動いていると思う通常はクラスを元にインスタンスを作り、そのインスタンスのメンバを使う。
参照型変数にオブジェクト本体が入っていると思う参照型変数にはオブジェクトへの参照が入る。
フィールドとローカル変数を同じ扱いにするフィールドはメンバ。ローカル変数はメソッド内などの一時的な変数。

説明できる状態の目安

  • クラス・オブジェクト・インスタンスを区別できる
  • 参照コピーで何がコピーされるか説明できる
  • メンバの代表例を挙げられる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：クラス・インスタンス・参照

Item a = new Item();
Item b = new Item();
a.price = 100;
b.price = 200;

失敗コード

Item a;
a.price = 100;

修正版

Item a = new Item();
a.price = 100;
判定ポイント

参照型変数を宣言しただけではインスタンスは作られていません。newが必要です。

理解確認クイズ

メンバとは何を指しますか。

答えと切り分けを見る

クラスの中に定義されるフィールド、メソッド、コンストラクタなどです。ローカル変数はメンバではありません。
