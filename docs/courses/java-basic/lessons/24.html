Java基礎 Lesson 24

ターミナル・javac・java

手元でJavaをコンパイル・実行し、エラーを読めるようにする。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：javac・java・-d・-cp

最後は手元で動かすための用語です。コンパイル対象と実行対象を混ぜないでください。

javacとはJavaソースファイルをコンパイルするコマンド。対象は基本的に.javaファイル。
javaとはコンパイル済みクラスやソースファイルモードを実行するコマンド。通常はクラス名を指定する。
-dとはコンパイル後の.classの出力先を指定するオプション。
-cpとは実行時やコンパイル時にクラスを探す場所を指定するオプション。
クラスパスとはJVMやjavacがクラスを探す検索場所。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
ターミナル / 現在地ターミナルはコマンド入力画面。現在地は相対パスの基準になる作業ディレクトリ。
-d / -cp-dはコンパイル結果の出力先。-cpは実行時やコンパイル時にクラスを探す場所。
単純クラス名 / 完全修飾クラス名Sampleは単純名。ex.Sampleのようにpackageを含めた名前が完全修飾名。

基礎用語一覧で確認する

最小コマンドセット

コマンド意味
pwd / cd現在地確認・フォルダ移動。Windows PowerShellではpwdも使える
dir / lsファイル一覧を見る
javac Main.javaソースコードをコンパイルする
java Mainクラス名を指定して実行する
java -versionJava実行環境のバージョン確認

基本の作業手順

  1. 作業用フォルダを作る。
  2. Main.javaを保存する。
  3. ターミナルでそのフォルダへ移動する。
  4. javac Main.javaを実行する。
  5. Main.classができたか確認する。
  6. java Mainを実行する。

Windows PowerShellの例

cd C:\Users\shota\Desktop\java-work
dir
javac Main.java
java Main

よくあるエラーの読み方

エラーの雰囲気主な原因
javacが認識されないJDK未インストール、またはPATH設定の問題
file not found今いるフォルダとファイルの場所が違う
class Main is public, should be declared...publicクラス名とファイル名が不一致
Could not find or load main class実行する場所、クラス名、パッケージ指定のどれかが違う
main method not foundmainメソッドの形が入口として認識されない

ファイル名と実行名を分ける

javac Main.java  // ファイル名を指定
java Main        // クラス名を指定。拡張子は付けない
雑に覚えない

javacはファイルを対象にする。javaはクラスを対象にする。この違いが分かれば、かなりのエラーを自分で切り分けられる。

手を動かすミニ課題

わざとjava Main.classを実行し、どんなエラーになるか確認する。その後java Mainで直す。エラー文を読む練習が目的。

丁寧な補講：理解の芯

ターミナル操作はJava学習の地味な壁。エディタでコードを書けても、保存場所、現在のフォルダ、コンパイルコマンド、実行コマンドが分からなければ自力で確認できない。

最小限覚えるのは、cdでフォルダ移動、dirで中身確認、javacでコンパイル、javaで実行。この4つで学習用の検証はかなり進められる。

エラーが出たら、英語を全部読む必要はない。まずファイル名、行番号、エラー種別を見る。cannot find symbolなら名前の誤りやスコープ、; expectedなら記号や文法を疑う。

現在地

ターミナルには現在操作しているフォルダがある。

コンパイル

javac ファイル名.javaで.classを作る。

実行

java クラス名でmainを実行する。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
cd / dircdは移動、dirは一覧表示。現在地を変えるのか、確認するのか。
javac / java前者はコンパイル、後者は実行。ファイル名かクラス名か。
エラーメッセージ / 出力エラーは失敗の説明。出力はプログラムが表示した結果。誰が出した文字かを区別する。

コードを読むときの手順

  1. ソースファイルを保存したフォルダを確認する。
  2. ターミナルでそのフォルダへcdする。
  3. dirで.javaファイルがあるか確認する。
  4. javac Main.javaでコンパイルする。
  5. java Mainで実行する。
  6. 失敗したら、行番号とエラー文の先頭を読む。

確認用コード

cd C:\Users\shota\Desktop\java-study
dir
javac Main.java
java Main
確認する点このコードでの判断結論
cd作業フォルダへ移動保存場所と一致させる
javacMain.javaをコンパイルMain.class生成
javaMainクラスを実行出力または実行時例外が出る

よくある誤解を潰す

保存場所と別フォルダで実行しない

ターミナルの現在地が違えば、ファイルが見つからない。まずdirで確認する。

javac Mainやjava Main.javaとは書かない

コンパイルはファイル名、実行はクラス名。この区別を固定する。

確認問題：答えまで言語化する

次の状況で最初に確認することは何か。「javac Main.javaを実行したら、ファイルが見つからないと言われた」。

考え方と答えを見る

最初に確認するのは、ターミナルの現在地とファイルの保存場所が一致しているか。

dirでMain.javaが表示されるか確認する。表示されないなら、cdで正しいフォルダへ移動するか、ファイルの保存場所を直す。

理解の流れを整える補強

このLessonの目的は、ターミナル操作そのものを難しくすることではありません。自分の手でコンパイルし、実行し、エラーを読むための最小手順を固定することです。

やることコマンド例確認すること
現在地を見るpwd / cdJavaファイルがある場所にいるか
コンパイルjavac Main.java.classが作られるか
実行java Mainクラス名を指定しているか
mkdir java-basic-test
cd java-basic-test
notepad Main.java

うまくいかないときは、まず「今いるフォルダ」「ファイル名」「publicクラス名」「実行時に指定したクラス名」を順番に確認します。エラーメッセージを読まずにファイルを何度も書き換えるのは非効率です。

ここで切るべき誤解

ターミナルで詰まる原因の多くはJava文法ではなく、場所・名前・コマンドの指定ミスです。コードを疑う前に、実行環境の前提を確認してください。

理解チェック

  • ターミナル・javac・javaを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

コマンド実行で最後に詰まる点

つまずき正しい切り分け
-dをjavaコマンドに付ける-dはjavac側でclassファイルの出力先を指定するオプション。
package付きクラスを単純名で実行するpackageがあるなら完全修飾クラス名で実行する。例：java -cp out ex.Main。
クラスパスをソースの場所だと思う実行時のクラスパスはclassファイルを探す場所。ソースファイルの場所ではない。

説明できる状態の目安

  • -dと-cpの役割を説明できる
  • package付きクラスの実行コマンドを書ける
  • javacとjavaの失敗パターンを3つ挙げられる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：javac・javaコマンド

javac -d out src/ex/Main.java
java -cp out ex.Main

失敗コード

java -d out ex.Main
java out/ex/Main

修正版

javac -d out src/ex/Main.java
java -cp out ex.Main
判定ポイント

-dはjavac側、-cpはjava側で使う代表的な指定です。package付きクラスは完全修飾名で起動します。

理解確認クイズ

用語だけで終わらせず、コード判定と説明まで確認します。

Lv1 用語確認

-d、-cp、packageの関係を説明できますか。

答えと切り分けを見る

-dはclassファイルの出力先、-cpは実行時にclassを探す場所、packageは完全修飾クラス名とディレクトリ構造に関わります。

Lv2 コード判定

package ex; のMainをoutへコンパイルした後、正しい実行はどれですか。

java -cp out ex.Main
答えと切り分けを見る

正しいです。package付きクラスは完全修飾クラス名で実行します。java -cp out Mainではありません。

Lv3 説明問題

クラスパスがソースファイルの場所を指定しない理由を説明してください。

答えと切り分けを見る

java実行時に必要なのはコンパイル済みのclassを探す場所です。ソースファイルの場所ではなく、class出力先を指定します。
