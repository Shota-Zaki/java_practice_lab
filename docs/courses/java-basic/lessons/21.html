Java基礎 Lesson 21

ビット・バイト・2進数

型や文字コードの理解に必要な最小限のコンピュータ基礎を押さえる。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

先に結論

bitは0か1の情報、byteは通常8bit。Javaの数値リテラルや文字コード、ファイルサイズを理解するために最低限必要。深追いしすぎる必要はないが、2進数・16進数の表記は読めるようにする。

bitとbyte

単位意味例
bit0または1の1桁スイッチのON/OFFのような情報
byte通常8bitファイルサイズ、メモリサイズ
KB/MB/GBbyteを大きくした単位ストレージ容量など

2進数の読み方

2進数は右から1, 2, 4, 8, 16...の重みを持つ。1になっている桁の重みを足す。

Javaの2進数リテラル

0b1010
= 8 + 2
= 10
2進数10進数
0b11
0b102
0b113
0b10008
0b111115

16進数の読み方

16進数は0〜9に加えてA〜Fを使う。Javaでは0xを付けて書く。

16進10進
0xA10
0xF15
0x1016
0xFF255

Javaの整数リテラル

int a = 10;      // 10進数
int b = 0b1010;  // 2進数。10
int c = 012;     // 8進数。10
int d = 0xA;     // 16進数。10
8進数に注意

012は10進数の12ではなく、8進数表記。初心者はここで見た目に騙されやすい。

手を動かすミニ課題

0b1001、0x1F、010を10進数に直す。

丁寧な補講：理解の芯

ビット・バイト・2進数は、Java文法そのものではないが、数値型や文字コードを理解する土台になる。必要以上に深掘りする必要はないが、byteが8ビット、1バイトが8ビット、2進数が0と1で表すことは押さえる。

2進数リテラルは0bまたは0Bで書ける。これは表記が2進数なだけで、数値としての型判定は別に行われる。0b10000000は10進数で128。byteの範囲を超える。

ビットを学ぶ目的は、暗算の速さではない。型の範囲、文字コード、整数リテラルの表記を見たときに、何が起きているかを説明できるようにすること。

bit

0または1を表す最小単位。

byte

8ビット。Javaのbyte型も8ビット符号付き整数。

2進数表記

0b1010のように0と1で数値を書く。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
表記 / 値0b1010は表記が2進数なだけで、値は10。10進数に直して考える。
byte型 / 1バイトJavaのbyteは-128〜127の整数型。単に8ビット幅というだけではない。符号付き範囲を見る。
ビット数 / 扱える範囲ビット数が増えるほど表現できる値の種類が増える。型ごとの範囲と結びつける。

コードを読むときの手順

  1. 数値リテラルの接頭辞を見る。0bなら2進数。
  2. 必要なら10進数の値に直す。
  3. 左辺の型の範囲に収まるか確認する。
  4. 演算があるなら、演算後の型も見る。
  5. 文字やバイト列の話では、文字そのものと数値表現を分ける。

確認用コード

byte b = 0b10000000;
確認する点このコードでの判断結論
2進数の値0b10000000は12810進数に直す
byte範囲-128〜127128は範囲外
結論代入不可コンパイルエラー

よくある誤解を潰す

0bが付けばbyteに入る、は違う

表記方法と型の範囲は別。値が範囲外なら入らない。

2進数はJavaだけの特殊文法、ではない

コンピュータの数値表現の基本。Javaではリテラル表記として使える。

確認問題：答えまで言語化する

0b1010は10進数でいくつか。またint x = 0b1010;はコンパイルできるか。

考え方と答えを見る

0b1010は10進数で10。8 + 2 = 10。

int x = 0b1010;はコンパイルできる。整数リテラルとしてintに代入できる値。

理解の流れを整える補強

ビット・バイト・2進数は、深く掘りすぎる必要はありません。ただし、数値リテラル、型の範囲、文字コードを理解するための最低限は必要です。ここを避けると、byteやcharの挙動が暗記になります。

表現例読み方
2進数0b10102の重みで読む。10進数では10。
8進数012先頭0に注意。10進数では10。
16進数0x0A0〜9とA〜Fで読む。10進数では10。
int a = 0b1010; // 10
int b = 012;    // 10
int c = 0x0A;   // 10

見た目が違っても、変数に入る値としては同じ10です。表記方法と値そのものを分けてください。特に先頭に0が付く整数は8進数として扱われるため、意図せず違う値になることがあります。

ここで切るべき誤解

「2進数だから別の型になる」わけではありません。整数リテラルの表記が違うだけで、基本的にはintの整数値として扱われます。

理解チェック

  • ビット・バイト・2進数を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。
