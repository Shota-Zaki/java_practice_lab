1からわかるJava基礎解説 / Lesson 21

ビット・バイト・2進数の超入門

プログラムは最終的に0と1の情報として扱われます。Javaのintやcharも、裏側ではビット列として保存されています。

なぜ2進数を知る必要があるか

Javaの試験では2進数を直接計算する問題は多くありません。ただし、byte、short、int、long、char、文字コード、オーバーフローの理解には、ビットとバイトの感覚が必要です。

1ビットは0または1のどちらかです。8ビットをまとめた単位が1バイトです。intは通常32ビット、longは64ビットとして扱われます。

10進数と2進数

人間は10進数を使いますが、コンピュータは2進数で値を保持します。たとえば10進数の5は、2進数では101です。

重要なのは丸暗記ではありません。値には範囲があり、型によって保存できる大きさが違う、という感覚です。

int a = 5;      // 10進数
int b = 0b0101; // 2進数リテラル。値は5
int c = 0x05;   // 16進数リテラル。値は5
System.out.println(a == b); // true

Java Silverで効くポイント

0bから始まる整数リテラルは2進数、0xから始まる整数リテラルは16進数です。見た目が違っても値が同じなら数値としては同じです。

ただし、longを表すには末尾にLを付けます。小文字lは数字の1と紛らわしいので、学習では大文字Lで読む癖を付けてください。
