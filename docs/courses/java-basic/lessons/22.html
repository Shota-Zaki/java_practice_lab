文字コードとUnicode

文字も内部的には数値です。char、String、文字コードの関係を理解すると、文字列問題に強くなります。

文字は数値として扱われる

Javaのcharは1文字を表す型ですが、内部的には文字コードに対応する数値です。たとえば文字AはUnicode上で65です。

char c = 'A';
System.out.println((int)c); // 65
System.out.println((char)66); // B

Stringとcharの違い

charは1文字、Stringは文字列です。シングルクォートはchar、ダブルクォートはStringです。ここを混同するとコンパイルエラーになります。

char a = 'A';     // OK
String b = "A";   // OK
// char c = "A";  // NG: Stringをcharへ代入できない
// String d = 'A'; // NG: charをStringへ直接代入できない

試験での注意

charは数値演算に参加するとintへ昇格します。char同士の足し算はcharではなくintになる点が頻出です。

char a = 'A';
char b = 'B';
System.out.println(a + b); // 文字列ABではなく数値計算になる
