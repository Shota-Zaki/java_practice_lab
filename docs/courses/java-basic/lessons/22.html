Java基礎 Lesson 22

文字コードとUnicode

char、String、文字リテラル、文字列リテラルの違いを理解する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

charとStringの違い

表記型意味
'A'char1文字
"A"String文字列オブジェクト
'AB'不可charは1文字だけ
""String長さ0の文字列

charは数値的な性質も持つ

charは文字を扱うが、内部的には文字に対応する数値を持つ。'A'は数値として扱うと65。

char c = 'A';
int n = c;
System.out.println(n); // 65

Unicodeエスケープ

Javaでは\uXXXXの形でUnicodeエスケープを書ける。基礎段階では「文字にも数値的な対応がある」と理解できれば十分。

char c = '\u0041';
System.out.println(c); // A

文字列連結での挙動

char同士を足すと数値計算になる場合がある。Stringが混ざると文字列連結になる。

System.out.println('A' + 1);      // 66
System.out.println("A" + 1);      // A1
System.out.println('A' + "B");    // AB

手を動かすミニ課題

次の出力を予測する。

char a = 'A';
char b = 'B';
System.out.println(a + b);
System.out.println("" + a + b);

丁寧な補講：理解の芯

文字コードは「文字」と「数値」を対応させる仕組み。Javaのcharは1文字を表す型で、文字リテラルはシングルクォート、文字列リテラルはダブルクォートで書く。

char c = 'A';は1文字。String s = "A";は文字列オブジェクトへの参照。この2つは似て見えるが型が違う。charとStringを混同すると代入でエラーになる。

Unicodeは多くの文字を扱うための文字集合・符号化の土台。基礎段階では、Javaが英数字だけでなく日本語なども扱える理由として理解すればよい。

char

1文字を表すプリミティブ型。シングルクォートを使う。

String

0文字以上の文字列を表す参照型。ダブルクォートを使う。

Unicode

多様な文字を数値として扱うための体系。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
'A' / "A"前者はchar、後者はString。引用符の種類を見る。
文字 / 文字列文字は1つ。文字列は0文字以上の並び。Stringは空文字""も持てる。
文字としての数字 / 数値'1'は文字、1は数値。クォートされているかを見る。

コードを読むときの手順

  1. 引用符がシングルかダブルかを見る。
  2. 左辺の型がcharかStringか確認する。
  3. 1文字か、文字列かを判断する。
  4. 数値型との変換がある場合、文字コード上の数値として扱われていないか確認する。
  5. 出力時は文字として表示されるのか、数値として表示されるのか見る。

確認用コード

char c = 'A';
int n = c;
System.out.println(n);
確認する点このコードでの判断結論
c文字Achar
int n = c文字コード上の数値へ変換代入可能
出力Aのコード値65

よくある誤解を潰す

String s = 'A';は通る、は違う

'A'はchar。Stringにそのまま代入できない。

charは短いString、ではない

charはプリミティブ型、Stringは参照型。根本的に違う。

確認問題：答えまで言語化する

次のうち型が正しいものを説明する。

確認問題

char a = 'A';
String b = "A";
char c = "A";
String d = 'A';
考え方と答えを見る

正しいのはchar a = 'A';とString b = "A";。

char c = "A";はダブルクォートなのでString。String d = 'A';はシングルクォートなのでchar。どちらも型が合わない。

理解の流れを整える補強

このLessonでは、charとStringを同じ「文字」扱いで混ぜないことが重要です。charは1文字を表すプリミティブ型、Stringは文字列を扱う参照型です。似て見えますが、代入・比較・メソッド呼び出しのルールが違います。

型例本質
char'A'1文字を表すプリミティブ型。数値的な性質もある。
String"A"文字列オブジェクトへの参照。
Unicode'A'文字を番号で表す仕組み。
char c = 'A';
String s = "A";
System.out.println(c + 1); // 66
System.out.println(s + 1); // A1

charは数値的に扱われる場面があり、Stringは文字列連結になります。この差を「見た目が似ているから同じ」と処理すると、出力結果を間違えます。

ここで切るべき誤解

シングルクォートとダブルクォートは見た目の好みではありません。'A'はchar、"A"はStringです。

理解チェック

  • 文字コードとUnicodeを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。
