Java基礎 Lesson 16

interfaceの入口

interfaceを「実装すべき扱い方」として理解する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：interface・implements・defaultメソッド

interfaceは「中身がないクラス」ではありません。外部からの扱い方を決める型です。

interfaceとはクラスが実装すべき扱い方を定める型。
implementsとはクラスがinterfaceを実装することを示すキーワード。
抽象メソッドとは本体を持たず、実装クラス側で処理を書くメソッド。
defaultメソッドとはinterface内に書ける実装付きメソッド。
実装とは決められたメソッドの具体的な処理を書くこと。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
interface / classclassは状態や処理の実体を持つ設計図。interfaceは主に扱い方・約束を定義する型。
implements / extendsclassがinterfaceを実装するのがimplements。class同士の継承はextends。
抽象メソッド / defaultメソッド抽象メソッドは実装クラスが処理を書く。defaultメソッドはinterface側に処理を持てる。

基礎用語一覧で確認する

基本形

interface Worker {
    void work();
}

class Engineer implements Worker {
    @Override
    public void work() {
        System.out.println("coding");
    }
}

Worker型として扱うならwork()を呼べる、というルールを作っている。

interfaceのメソッドはpublic扱い

interfaceの抽象メソッドは暗黙にpublic abstract。そのため、実装クラス側ではpublicで実装する必要がある。アクセスを弱くするとコンパイルエラー。

悪い例

interface Worker {
    void work();
}

class Engineer implements Worker {
    void work() { } // コンパイルエラー：publicではない
}

修正例

class Engineer implements Worker {
    public void work() { } // OK
}

interface型で受けるメリット

Worker w = new Engineer();
w.work();

変数の型をWorkerにすると、実体がEngineerでも、呼べるのはWorkerとして見えるメソッドになる。これは継承の親型変数と同じ読み方。

defaultメソッド

defaultメソッドは、interface内に実装を持てるメソッド。実装クラスが必ずしも書かなくてよい共通処理を置ける。

interface Greet {
    default void hello() {
        System.out.println("hello");
    }
}

class User implements Greet { }
new User().hello();

フィールドは定数扱い

interfaceに書いたフィールドは暗黙にpublic static final。つまり共有される定数であり、あとから代入し直せない。

手を動かすミニ課題

次のコードで、なぜrun()をpublicにしないといけないか説明する。

interface Task {
    void run();
}
class Job implements Task {
    public void run() {
        System.out.println("run");
    }
}

丁寧な補講：理解の芯

interfaceは「この型として扱うなら、この操作ができる」という約束を定義するもの。クラスの共通部品を継承するというより、外から見た扱い方を揃えるための仕組み。

インタフェースの抽象メソッドは暗黙にpublic abstract。フィールドは暗黙にpublic static final。ここを知らないと、実装クラスでアクセス修飾子を弱くしてコンパイルエラーを起こす。

クラスは1つのクラスしかextendsできないが、複数のインタフェースをimplementsできる。これは「実装の継承」と「扱い方の約束」を分けるために重要。

約束

実装クラスが持つべき操作を定義する。

implements

クラスがインタフェースを実装する。

多態性

インタフェース型の変数で、実装クラスのインスタンスを扱える。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
extends / implementsクラス継承はextends、インタフェース実装はimplements。右側がクラスかインタフェースかを見る。
抽象メソッド / defaultメソッド抽象メソッドは実装がない。defaultメソッドは実装を持てる。本体{}があるか、defaultかを見る。
interfaceのメソッド / 実装クラスのメソッドinterface側はpublic扱い。実装時もpublicにする必要がある。アクセス修飾子を弱くしていないか見る。

コードを読むときの手順

  1. インタフェースに定義されているメソッドを確認する。
  2. 実装クラスがimplementsしているか確認する。
  3. 抽象メソッドをすべて実装しているか確認する。
  4. 実装メソッドがpublicになっているか確認する。
  5. インタフェース型変数で呼べるメソッドか確認する。

確認用コード

interface Worker {
    void work();
}
class Engineer implements Worker {
    void work() {
        System.out.println("build");
    }
}
確認する点このコードでの判断結論
interface側workは暗黙にpublic外から呼べる約束
実装側void work()はデフォルトアクセスpublicより弱い
結論アクセス権を弱くしているコンパイルエラー

よくある誤解を潰す

interfaceのメソッドはデフォルトアクセス、は違う

修飾子を省略してもpublic abstract扱い。

implementsしただけで中身を書かなくてよい、は違う

具象クラスなら抽象メソッドを実装する必要がある。抽象クラスなら未実装を残せる。

確認問題：答えまで言語化する

次のコードをコンパイルできるようにするには何を直すか。

確認問題

interface A {
    void test();
}
class B implements A {
    void test() { }
}
考え方と答えを見る

Bのtestをpublicにする。

インタフェースのtestは暗黙にpublic。実装時にアクセス修飾子を弱くできないため、public void test() { }と書く。

理解の流れを整える補強

interfaceは「中身のないクラス」ではありません。より正確には、外からどう扱えるかを定める型です。実装クラスが何であっても、そのinterface型として扱えます。

interface Printer {
    void print(String text);
}

class ConsolePrinter implements Printer {
    public void print(String text) {
        System.out.println(text);
    }
}

Printer p = new ConsolePrinter();
p.print("hello");

Printer pとして受けているので、呼べるのはPrinterに定義された操作です。実体はConsolePrinterですが、変数の型はPrinterです。この考え方がポリモーフィズムの入口です。

interface内の要素扱い注意
抽象メソッド基本的にpublic abstract実装時もpublicが必要
フィールドpublic static final扱いインスタンスごとの状態ではない
defaultメソッド実装を持てる実装クラスで上書き可能
staticメソッドinterface自身に属する実装クラスのインスタンスから呼ぶものではない
interface Sample {
    int VALUE = 10; // public static final 扱い
    void test();    // public abstract 扱い
}
interfaceで最初に見ること

「何を実装しているか」より先に、「そのinterface型として何を呼べるか」を確認してください。型としての役割を見ないと、実装クラスの詳細に引っ張られます。

理解チェック

  • interfaceの入口を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

interfaceで混同する点

つまずき正しい切り分け
interfaceをインスタンス化できると思うinterface型の変数は作れるが、interface自体はnewできない。実装クラスのインスタンスを扱う。
interfaceのメソッドはprivateだと思う抽象メソッドは暗黙にpublic abstract。実装側もpublicが必要。
extendsとimplementsを混同するクラスを継承するのがextends、interfaceを実装するのがimplements。

説明できる状態の目安

  • interfaceを「扱い方の約束」として説明できる
  • implementsの意味を説明できる
  • defaultメソッドと抽象メソッドを区別できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：interfaceとimplements

interface Printable {
    void print();
}
class Report implements Printable {
    public void print() { System.out.println("R"); }
}

失敗コード

interface Printable {
    void print();
}
class Report implements Printable {
    void print() { System.out.println("R"); }
}

修正版

class Report implements Printable {
    public void print() { System.out.println("R"); }
}
判定ポイント

interfaceの抽象メソッドはpublic扱いです。実装時にアクセス範囲を狭くできません。

理解確認クイズ

interfaceの抽象メソッドは、アクセス修飾子を省略すると何扱いですか。

答えと切り分けを見る

public扱いです。実装クラスで実装するときはpublicを狭くできません。
