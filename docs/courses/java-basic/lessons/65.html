代入と初期化の違い

変数を作ること、値を入れること、初期値を持つことを分けて理解します。

宣言と代入

int x; は変数の宣言、x = 10; は代入です。int x = 10; は宣言と初期化を同時にしています。

ローカル変数は自動初期化されない

メソッド内のローカル変数は、使う前に明示的に値を入れる必要があります。

フィールドはデフォルト値を持つ

インスタンス変数やstatic変数はデフォルト値を持ちます。intなら0、booleanならfalse、参照型ならnullです。

小さな例

class A {
    int field;
    void m() {
        int local;
        // System.out.println(local); // コンパイルエラー
        System.out.println(field);
    }
}

確認ポイント

  • ローカル変数は未初期化で使えない
  • フィールドはデフォルト値を持つ
  • 宣言と代入を分けて読む
