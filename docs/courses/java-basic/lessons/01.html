Java基礎 Lesson 1

コンピュータと機械語

Javaの前に、コンピュータが何をしているかを理解する。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

結論

コンピュータは、最終的にはCPUが理解できる機械語の命令を順番に実行している。JavaのコードはそのままCPUが読むのではなく、コンパイルを経てJVMが実行できる形に変換される。

なぜJava学習に必要か

コンパイルエラーと実行時例外の違いを理解するには、『書いたコード』『変換された命令』『実行中の状態』を分けて考える必要がある。Java Silverの問題でも、コードが実行される前に落ちるのか、実行中に落ちるのかが頻繁に問われる。

機械語のイメージ

機械語は、人間向けの英語風コードではなく、CPUが処理する命令の集まり。足し算、比較、ジャンプ、メモリから値を読む、メモリへ値を書く、といった単純な命令の組み合わせで複雑な処理が作られる。

Javaとの関係

Javaでは通常、ソースコードをjavacでバイトコードに変換し、JVMがそのバイトコードを実行する。つまりJavaはCPUに直接命令を渡すのではなく、JVMという実行環境を通して動く。

初心者の誤解

『Javaを書く＝CPUがJavaを読む』ではない。CPUが直接Java文法を理解するわけではない。ここを誤解すると、コンパイル、JVM、実行時例外の役割が曖昧になる。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
