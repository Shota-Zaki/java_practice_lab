Lesson 77 データがメモリに置かれるとはどういうことか

変数・参照・オブジェクトを理解するには、値がどこにあるかをざっくり把握する必要があります。

値型のように見えるもの参照型の基本試験で落ちるポイント

値型のように見えるもの

Javaの基本データ型、たとえばintやdoubleは、変数そのものが値を持つと考えると理解しやすいです。

int a = 10;
int b = a;
b = 20;
System.out.println(a); // 10

bを変えてもaは変わりません。

参照型の基本

クラス型や配列型の変数には、オブジェクトそのものではなく、オブジェクトを指す参照が入ります。

int[] a = {1, 2};
int[] b = a;
b[0] = 9;
System.out.println(a[0]); // 9

aとbが同じ配列を見ているため、片方から中身を変えるともう片方にも影響します。

試験で落ちるポイント

==は参照の同一性を見ます。配列や自作クラスで「中身が同じだからtrue」と考えるのは危険です。
