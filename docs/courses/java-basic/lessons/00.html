Java基礎 準備

始める準備

Javaを始めるために必要なものは、最初は多くありません。JDK、エディタ、作業フォルダ、ターミナルでの確認。この4つを先に固めます。

このページのゴール

Javaのインストール先、コードを書く道具、最初の実行確認を自分で説明できる状態にする。

初出用語：JDK・エディタ・ターミナル

Javaを書き始める前に、道具の名前を先に分けます。ここを曖昧にすると、文法ではなく環境設定で詰まります。

JDKとはJavaを開発するための道具一式。javac や java が含まれる。
エディタとはソースコードを書くための道具。VS Codeやメモ帳など。
ターミナルとは文字でコマンドを入力して、コンパイルや実行を行う画面。
作業フォルダとはJavaファイルを置いて作業する場所。コマンド実行時の現在地と一致させる。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
JDK / JRE / JVMJDKは開発道具、JREは実行環境、JVMは.classを実行する仮想マシン。学習ではまずJDKを入れる。
エディタ / IDEエディタはコードを書く道具。IDEは補完・実行・デバッグまで統合した開発環境。
ターミナル / 作業フォルダターミナルはコマンドを打つ画面。作業フォルダは今いる場所で、javacの対象ファイル探索に関係する。

インストール後の確認

JDKを入れたら、PowerShellまたはターミナルで次を実行します。

java -version
javac -version

両方でバージョンが表示されれば、最低限の準備はできています。

状態意味対応
java -versionだけ通る実行環境だけ見えていて、コンパイラが見えていない可能性JDKを入れたか、PATHが通っているか確認する
javacが認識されないJDK未インストール、またはPATH設定の問題JDKを入れ直すか、インストーラーのPATH設定を確認する
古いバージョンが出る別のJavaが先に見つかっている複数JDKが入っていないか確認する

作業フォルダを作る

最初はパスで詰まらないように、英数字だけの短いフォルダ名にします。

mkdir C:\java-work
cd C:\java-work

日本語や空白を含むフォルダ名でも動くことはありますが、最初の段階では余計な原因を減らす方が合理的です。

最初に動かすコード

C:\java-workにMain.javaを作り、次を保存します。

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

保存したら、同じフォルダで次を実行します。

javac Main.java
java Main

Hello Javaと表示されれば成功です。

ここで覚えること

javacはファイル名を指定します。javaはクラス名を指定します。java Main.classではありません。

JDK / JVM / javac / java の役割

用語役割雑に言うと
JDKJava開発に必要な道具一式開発キット
JVM.classを実行する仮想マシンJavaを動かす実行基盤
javac.javaを.classへ変換するコンパイラ
javaJVMを起動し、指定したクラスのmainを実行する実行コマンド

この違いを曖昧にしたまま進むと、エラーが出たときに原因を切り分けられません。

やらない方がいいこと

非公式サイトからJDKを入れる

開発環境は公式配布元から入れる。余計なインストーラーを踏む必要はありません。

複数のJDKを同時に入れる

最初から複数入れると、どのjavaやjavacを使っているのか分からなくなります。

エディタの実行ボタンだけに頼る

ボタン実行は便利ですが、裏で何が起きているか見えにくいです。最初はjavacとjavaを一度は手で実行してください。

確認問題

次のうち、Javaを始める準備として本当に必要なものはどれか。

  • JDK
  • エディタ
  • 作業フォルダ
  • 有料IDE
  • Webサーバー
考え方と答えを見る

最初に必要なのは、JDK、エディタ、作業フォルダです。有料IDEやWebサーバーは不要です。

Javaの基礎学習では、.javaを書き、javacでコンパイルし、javaで実行できれば十分です。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

環境構築で止まる典型

つまずき正しい切り分け
java と javac を同じものとして扱うjavaは実行、javacはコンパイル。片方だけ通る状態は普通に起きる。
エディタの実行ボタンだけで進める裏でどのフォルダ・どのJDK・どのクラス名で実行しているか見えない。最初は手打ちで確認する。
作業フォルダを意識しないターミナルの現在地とファイルの置き場所がずれると、文法が正しくても動かない。

説明できる状態の目安

  • java -version と javac -version の違いを説明できる
  • javac Main.java と java Main の指定対象を説明できる
  • 作業フォルダを変えると何が変わるか説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：環境確認

java -version
javac -version

失敗コード

java Sample.java
javac Sample.class

修正版

javac Sample.java
java Sample
判定ポイント

Javaを動かす準備では、まず実行用のjavaとコンパイル用のjavacが使えるかを分けて確認します。
