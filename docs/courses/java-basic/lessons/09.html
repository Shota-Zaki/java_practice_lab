Java基礎 Lesson 9

for・while

繰り返し処理を、初期化・条件・更新・本体の順序で追えるようにする。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：ループ・break・continue

繰り返しは、初期化、条件、本文、更新の順番で追います。感覚で回数を数えないでください。

ループとは条件を満たす間、同じ処理を繰り返す構文。
初期化式とはfor文の最初に一度だけ実行される部分。
更新式とはfor文の各周回後に実行される部分。
breakとはループやswitchを抜ける文。
continueとは現在の周回の残りを飛ばして次の周回へ進む文。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
ループ / 条件式ループは繰り返し構造。条件式は続けるか止めるかを決める。
break / continuebreakはループを抜ける。continueは今回の残りを飛ばして次の繰り返しへ進む。
while / do-whilewhileは先に条件判定。do-whileは本体を1回実行してから条件判定。

基礎用語一覧で確認する

for文の読み方

0, 1, 2が表示される

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
  1. int i = 0を最初に1回だけ実行する。
  2. i < 3を判定する。
  3. trueなら本体を実行する。
  4. i++を実行する。
  5. 条件判定へ戻る。

whileとdo-while

構文条件判定のタイミング最低実行回数
while本体の前0回
do-while本体の後1回
int x = 10;
while (x < 5) {
    System.out.println("while");
}

do {
    System.out.println("do");
} while (x < 5);

breakとcontinue

breakはループ自体を終了する。continueは現在の反復をそこで終え、次の反復へ進む。for文ではcontinue後に更新式へ進む。

0, 1, 3が表示される

for (int i = 0; i < 5; i++) {
    if (i == 2) {
        continue;
    }
    if (i == 4) {
        break;
    }
    System.out.println(i);
}

到達不能コード

breakやcontinueの直後に、必ず実行されない文を書くとコンパイルエラーになることがある。

悪い例

while (true) {
    break;
    System.out.println("never"); // 到達不能
}

手を動かすミニ課題

次のコードの出力を、表にして追う。

int sum = 0;
for (int i = 1; i <= 4; i++) {
    if (i == 3) continue;
    sum += i;
}
System.out.println(sum);

丁寧な補講：理解の芯

繰り返し文は、雰囲気で追うと必ずズレる。forなら「初期化→条件判定→本体→更新→条件判定…」の順に固定して読む。whileなら条件判定が先、do-whileなら本体が先。

特にi++のタイミングを間違えやすい。for文の更新式は、本体の最後に実行される。本体に入る前ではない。

breakとcontinueも区別する。breakはループ終了、continueは今回分を打ち切って次の判定・更新へ進む。

for

回数が見えやすい繰り返し。初期化、条件、更新が一行に集まる。

while

条件がtrueの間続く。最初に条件判定する。

do-while

最低1回は本体を実行し、その後に条件判定する。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
while / do-whilewhileは先に判定。do-whileは後で判定。条件が最初からfalseのときに差が出る。
break / continuebreakは終了。continueは次へ。ループを抜けるか、次回へ進むか。
更新式 / 本体forの更新式は本体実行後。出力してから増えるのか、増えてから出力かを追う。

コードを読むときの手順

  1. 初期化式を一度だけ実行する。
  2. 条件式を判定する。falseなら終了。
  3. 本体を上から実行する。
  4. breakやcontinueがあれば流れを変更する。
  5. 更新式を実行し、再び条件式へ戻る。

確認用コード

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
確認する点このコードでの判断結論
1回目i=0、条件true、出力0、更新で1継続
2回目i=1、条件true、出力1、更新で2継続
3回目i=2、条件true、出力2、更新で3次で終了

よくある誤解を潰す

forの更新式は本体の前には実行されない

forの更新式は本体の後。出力タイミングと混同しない。

continueではループ全体は終了しない

continueは次の繰り返しへ進むだけ。終了はbreak。

確認問題：答えまで言語化する

次の出力を答える。

確認問題

for (int i = 0; i < 5; i++) {
    if (i == 2) continue;
    if (i == 4) break;
    System.out.print(i);
}
考え方と答えを見る

出力は013。

i=2ではcontinueにより出力を飛ばす。i=4ではbreakによりループを終了する。

理解の流れを整える補強

ループは「何回回るか」を暗算する前に、初期化・条件判定・本体・更新の順番を固定してください。ここを曖昧にすると、1回多い、1回少ない、無限ループ、到達不能コードを見落とします。

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}
順番処理この例
1初期化int i = 0。最初に1回だけ。
2条件判定i < 3。trueなら本体へ。
3本体println(i)を実行。
4更新i++。その後、条件判定へ戻る。

whileは条件判定が先、do-whileは本体が先です。つまりdo-whileは条件が最初からfalseでも最低1回は実行されます。

int i = 10;
do {
    System.out.println(i);
} while (i < 3); // 10が1回表示される
ここで切るべき誤解

ループは「雰囲気で3回」と読まないでください。初期値、条件、更新の3点がそろって初めて回数が決まります。

理解チェック

  • for・whileを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

ループで読み間違える点

つまずき正しい切り分け
for文を左から1回ずつ読む初期化→条件→本体→更新→条件…の順。更新式は本体の後。
continueで更新式も飛ぶと思う通常のfor文ではcontinue後に更新式へ進む。
do-whileをwhileと同じだと思うdo-whileは条件判定前に本体が最低1回実行される。

説明できる状態の目安

  • for文の実行順を説明できる
  • breakとcontinueの違いを説明できる
  • whileとdo-whileの違いを説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：ループの更新

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}

失敗コード

int i = 0;
while (i < 3) {
    System.out.println(i);
}

修正版

int i = 0;
while (i < 3) {
    System.out.println(i);
    i++;
}
判定ポイント

whileでは更新処理を書き忘れると無限ループになります。

理解確認クイズ

用語だけで終わらせず、コード判定と説明まで確認します。

Lv1 用語確認

for文の初期化、条件、更新、本体の実行順を説明できますか。

答えと切り分けを見る

初期化 → 条件判定 → 本体 → 更新 → 条件判定、の順です。更新式は本体の後に動きます。

Lv2 コード判定

次のコードは何回表示しますか。

for (int i = 0; i < 3; i++) {
    System.out.print(i);
}
答えと切り分けを見る

3回です。表示は012です。iが0,1,2のとき本体が実行され、3で条件がfalseになります。

Lv3 説明問題

拡張for文で要素変数を書き換えても配列本体が変わらない場合がある理由を説明してください。

答えと切り分けを見る

拡張for文の変数は要素値または参照のコピーを受け取ります。プリミティブ値を書き換えても配列要素そのものは書き換わりません。
