Java基礎 Lesson 4

mainメソッド

Javaプログラムの入口であるmainを、丸暗記ではなく条件として理解する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

先に結論

mainメソッドは、Javaアプリケーションの入口としてJVMが最初に探すメソッド。基本形はpublic static void main(String[] args)。丸暗記ではなく、各部品の意味を分解して覚える。

mainメソッドの基本形

public static void main(String[] args) {
    // ここから処理が始まる
}

mainを分解する

部分意味なぜ必要か
public外部から呼び出せるJVMがクラス外から呼び出すため
staticインスタンスなしで呼べるプログラム開始前にオブジェクトを作らなくてよいため
void戻り値なし入口メソッドとして戻り値を返さない
main決められた名前JVMがこの名前を探す
String[] argsコマンドライン引数起動時に渡した文字列を受け取る

argsの読み方

argsは配列。java Main apple bananaと実行すると、args[0]が"apple"、args[1]が"banana"になる。Main自体はargsには入らない。

java Main apple で実行すると apple

public class Main {
    public static void main(String[] args) {
        System.out.println(args[0]);
    }
}
引数なしの罠

引数を渡さずにargs[0]を読むと、配列の範囲外アクセスになり実行時例外が起きる。コンパイルエラーではない。

許される形と紛らわしい形

形入口として認識されるか補足
public static void main(String[] args)される基本形
public static void main(String... args)される可変長引数も配列として扱える
static public void main(String[] x)される修飾子の順序や引数名は本質ではない
public void main(String[] args)されないstaticがない
public static int main(String[] args)されない戻り値型がvoidではない

staticがあることの影響

mainはstaticなので、同じクラスのインスタンスフィールドを直接読めない。インスタンス側の値を使いたいなら、オブジェクトを作ってから参照する。

悪い例

public class Main {
    int value = 10;

    public static void main(String[] args) {
        System.out.println(value); // コンパイルエラー
    }
}

修正例

public class Main {
    int value = 10;

    public static void main(String[] args) {
        Main m = new Main();
        System.out.println(m.value);
    }
}

丁寧な補講：理解の芯

mainメソッドはJVMがアプリケーションの入口として探す特別な形のメソッド。暗記は必要だが、暗記だけでは弱い。public、static、void、String[] argsをそれぞれ意味で分解する。

staticが重要。mainはインスタンスを作らずに呼ばれる。そのため、mainからインスタンスフィールドやインスタンスメソッドを直接使おうとするとコンパイルエラーになる。

argsは起動時に渡された文字列の配列。java Main a bなら、args[0]が"a"、args[1]が"b"。クラス名Mainは配列に入らない。

入口

JVMは決められた形のmainを探して開始する。

static

開始時点でインスタンスがなくても呼べる。

args

起動時の追加情報を文字列配列として受け取る。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
mainメソッド / 普通のメソッドmainは入口としてJVMから呼ばれる。普通のメソッドは自分のコードから呼ぶ。メソッド名と引数の形を見る。
static / インスタンスstaticはクラス側。インスタンスフィールドはオブジェクト側。mainから直接使えるかで判断する。
String[] args / 出力argsは入力側。System.out.printlnは出力側。コマンドから入るのがargs、画面に出るのが出力。

コードを読むときの手順

  1. public static void mainの形を確認する。
  2. 引数がString[]またはString...か確認する。
  3. main内でインスタンスメンバーを直接参照していないか確認する。
  4. args[n]を読むなら、起動時引数の個数を確認する。

確認用コード

public class Main {
    int value = 10;

    public static void main(String[] args) {
        System.out.println(value);
    }
}
確認する点このコードでの判断結論
mainの形入口としての形はあるここはOK
valueインスタンスフィールドstaticなmainから直接参照不可
結論インスタンス生成がないコンパイルエラー

よくある誤解を潰す

mainがあるだけで何でも直接使える、は違う

mainはstaticなので、インスタンス側の値を直接使えない。

args[0]は必ずある、は違う

起動時引数を渡さなければargs.lengthは0。読むと範囲外アクセスになる。

確認問題：答えまで言語化する

次のコードでnameを表示したい。何を直すべきか。

確認問題

public class Main {
    String name = "Java";

    public static void main(String[] args) {
        System.out.println(name);
    }
}
考え方と答えを見る

mainはstaticなので、nameを直接参照できない。

Main m = new Main();でインスタンスを作り、System.out.println(m.name);のように参照する。

理解の流れを整える補強

mainメソッドは、単なる決まり文句ではなく、JVMが最初に呼び出す入口です。丸暗記で済ませると、static、引数、戻り値、メソッド名のどれが重要なのか分からなくなります。

要素意味ずれるとどうなるか
publicJVMから呼び出せる入口として公開する入口として認識されない
staticインスタンス生成なしで呼べるJVMが最初に呼び出せない
void戻り値を返さない入口の形式と合わない
String[] argsコマンドライン引数を受け取る実行時引数を受け取れない
public static void main(String[] args) {
    System.out.println(args.length);
}

argsは「実行時に外から渡された文字列の配列」です。引数を渡さずに実行しても、args自体がnullになるわけではありません。要素数0の配列として扱われます。

java Main one two
// args[0] は "one"
// args[1] は "two"

ここでjava Mainという実行コマンドのMainはクラス名であり、argsには入りません。これも混同しやすいので、早い段階で切ってください。

ここで切るべき誤解

mainは「よく見る形」だから正しいのではありません。JVMが入口として探す条件を満たしているから実行されます。

理解チェック

  • mainメソッドを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。
