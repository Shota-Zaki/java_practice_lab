エラーメッセージの読み方

Java学習ではエラー文を読む力が重要です。エラーは敵ではなく、修正箇所を教える情報です。

まず見る場所

エラーでは、ファイル名、行番号、エラーの種類、該当箇所を確認します。全部を一度に理解しようとせず、最初に発生したエラーから直すのが基本です。

Sample.java:4: error: cannot find symbol
    System.out.println(value);
                       ^

cannot find symbol

変数名、メソッド名、クラス名が見つからないときに出ます。スペルミス、スコープ外、import不足、static/instanceの勘違いが主な原因です。

incompatible types

型が合わないときに出ます。intへdoubleを代入、Stringへcharを代入、戻り値型の不一致などです。試験ではこのエラーに直結する問題が多いです。
