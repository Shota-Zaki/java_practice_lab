CPU・メモリ・ストレージの役割

プログラムが動く場所を、CPU・メモリ・ストレージに分けて理解します。

CPUは命令を処理する場所

CPUは計算や条件判定を実行する部品です。JavaのコードそのものをCPUが直接読むわけではありませんが、最終的にはJVMやOSを通じてCPUが命令を処理します。

初心者が混乱しやすい点は、コードを書いた瞬間に処理が行われるのではなく、コンパイル・読み込み・実行という段階を通ることです。

メモリは実行中の作業場所

メモリは、実行中のプログラムが変数やオブジェクトを置く作業机です。電源を切ると基本的に内容は消えます。

Javaではローカル変数、参照、オブジェクトなどがメモリ上に存在します。参照型の理解は、この「値そのもの」と「場所を指す情報」を分けるところから始まります。

ストレージは保存場所

SSDやHDDは、ソースファイル、classファイル、画像、設定ファイルなどを保存する場所です。実行時には必要な情報がメモリに読み込まれます。

小さな例

public class Main {
    public static void main(String[] args) {
        int value = 10;
        System.out.println(value);
    }
}

確認ポイント

  • CPU・メモリ・ストレージを混同しない
  • Javaの変数は実行中にメモリ上で扱われる
  • ファイルとして保存された.javaと実行中の値は別物
