Java基礎 Lesson 17

例外処理の入口

例外が発生した後の制御の流れを、try-catch-finallyで追えるようにする。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

try-catchの流れ

try {
    int[] a = new int[0];
    a[0] = 10;
    System.out.println("finish");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("error");
}

配列の範囲外アクセスで例外が発生した時点で、System.out.println("finish")には進まない。対応するcatchへ移動し、errorが表示される。

checked例外とunchecked例外

分類代表例コンパイラの扱い
checked例外IOExceptionなどcatchするかthrows宣言が必要
unchecked例外RuntimeException系catchやthrowsを強制されない
ErrorOutOfMemoryErrorなど通常のアプリ処理で回復を前提にしない

初心者段階では、RuntimeException系は「実行中の値や状態が悪いと起きることが多い」と考えると整理しやすい。

finally

finallyは、例外発生の有無に関係なく実行したい後処理を書く。ファイルや接続を閉じる処理の説明でよく出る。

例外がなければ A, C

try {
    System.out.println("A");
} catch (Exception e) {
    System.out.println("B");
} finally {
    System.out.println("C");
}

catchの順序

複数のcatchを書く場合、狭い型から広い型へ並べる。先にExceptionで受けると、その後の具体的な例外型は到達不能になる。

悪い順序

try {
    // ...
} catch (Exception e) {
} catch (NullPointerException e) {
    // コンパイルエラーになり得る
}

手を動かすミニ課題

次のコードで表示される文字を順番に書く。

try {
    System.out.println("A");
    String s = null;
    s.length();
    System.out.println("B");
} catch (NullPointerException e) {
    System.out.println("C");
} finally {
    System.out.println("D");
}

丁寧な補講：理解の芯

例外処理は、エラーをごまかす構文ではない。実行中に起きた異常を、どこで検知し、どこで処理し、処理後にどこへ進むかを決める仕組み。

try内で例外が発生すると、その時点でtryブロックの残りは実行されず、対応するcatchへ移る。finallyがあれば、例外の有無に関係なく基本的に実行される。

例外にはチェック例外と非チェック例外がある。基礎段階では、まずRuntimeException系は非チェック例外、Exception直下の多くはチェック例外、と大まかに掴む。

try

例外が発生する可能性のある処理を書く。

catch

発生した例外を受け取り、対処する。

finally

後片付けなど、最後に実行したい処理を書く。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
例外発生前 / 発生後例外が発生した行より後のtry内処理は飛ばされる。どの行で例外が起きるか見る。
catchする / しない対応するcatchがあれば処理される。なければ呼び出し元へ伝わる。例外クラスの型を見る。
finally / catchfinallyは例外処理そのものではなく、後始末用。catchされてもされなくても実行されうる。

コードを読むときの手順

  1. tryブロックを上から読む。
  2. どの行で例外が発生するか判断する。
  3. その例外を受け取れるcatchがあるか上から確認する。
  4. catch後または例外伝播前にfinallyがあるか確認する。
  5. 最終的に正常終了か、例外が外へ出るか判断する。

確認用コード

try {
    int[] a = new int[0];
    a[0] = 10;
    System.out.println("A");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("B");
} finally {
    System.out.println("C");
}
確認する点このコードでの判断結論
例外発生a[0]で範囲外アクセスArrayIndexOutOfBoundsException
try残りSystem.out.println("A")は実行されないAは出ない
catch/finallyBを出してからC出力はB C

よくある誤解を潰す

catch後にtryの続きへ戻る、は違う

例外が起きた時点でtryの残りは実行されない。catch後はtry-catch文の後へ進む。

finallyは例外があるときだけ動く、は違う

例外がなくてもfinallyは実行される。

確認問題：答えまで言語化する

次の出力を答える。

確認問題

try {
    System.out.println("A");
} catch (RuntimeException e) {
    System.out.println("B");
} finally {
    System.out.println("C");
}
考え方と答えを見る

出力はA、C。

try内で例外は発生しないためcatchは実行されない。ただしfinallyは実行される。

理解の流れを整える補強

例外処理は「エラーを消す仕組み」ではありません。正確には、通常の流れで処理を続けられない事態が起きたとき、別の流れへ制御を移す仕組みです。

try {
    int[] nums = {1, 2, 3};
    System.out.println(nums[5]);
    System.out.println("end");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("out of range");
}

nums[5]で例外が発生すると、その時点でtryブロックの残りは実行されません。"end"は表示されず、対応するcatchへ移動します。

確認すること見る場所判断
例外が発生する可能性tryブロック内どの行で止まるか
catchできるかcatchの型発生した例外の型と互換性があるか
catchの順序上から順に確認親クラスを先に書くと子クラス側に届かない
finally最後に実行例外の有無にかかわらず基本的に実行

checked例外とunchecked例外の違いは、まずコンパイラが処理を要求するかで見ます。Exceptionのサブクラスでも、RuntimeException系はuncheckedです。

雑なcatchは理解を悪くする

catch (Exception e)だけで済ませると、何が起きたのかを読めなくなります。学習段階では、どの例外がどのcatchに入るかを必ず言語化してください。

理解チェック

  • 例外処理の入口を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。
