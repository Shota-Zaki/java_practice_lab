Java基礎 Lesson 17

例外処理の入口

checked、unchecked、finallyを理解する。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

例外

通常の処理を続けられない状態を表す仕組み。throwで発生させ、try-catchで処理できる。

checked例外

ExceptionのうちRuntimeException系ではないものは、基本的にcatchするかthrowsで宣言する必要がある。

unchecked例外

RuntimeException系はコンパイラによる処理強制がない。NullPointerException、ArrayIndexOutOfBoundsException、ClassCastExceptionなど。

finally

tryやcatchの後に実行される後処理。例外が発生しても基本的に実行される。

試験での罠

catchは狭い型から広い型へ並べる。Exceptionを先にcatchすると、その後のRuntimeExceptionなどが到達不能になる場合がある。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
