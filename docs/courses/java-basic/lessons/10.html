Java基礎 Lesson 10

配列

固定長の入れ物としての配列、添字、初期値、参照のコピーを理解する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：配列・要素・添字

配列は固定長のオブジェクトです。変数には配列そのものではなく参照が入ります。

配列とは同じ型の値を番号付きで並べる固定長の入れ物。
要素とは配列の中の1つ1つの値。
添字とは要素の位置番号。Javaでは0から始まる。
lengthとは配列の要素数を持つフィールド。メソッドではない。
参照コピーとは配列本体ではなく、配列を指す参照がコピーされること。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
配列 / 要素 / 添字配列は複数の値を並べる入れ物。要素は中身1個。添字は何番目かを表す番号。
length / インデックスlengthは要素数。使える添字は0からlength - 1まで。
参照コピー / 要素コピー配列変数の代入は参照のコピー。中身を別配列に複製することとは違う。

基礎用語一覧で確認する

配列の作り方

int[] a = new int[3];
a[0] = 10;
a[1] = 20;
a[2] = 30;
式意味
int[] aintを入れる配列への参照変数
new int[3]長さ3のint配列を作る
a[0]0番目の要素
a.length配列の長さ。メソッドではなくフィールド

デフォルト値

配列をnewすると、各要素は型ごとのデフォルト値で初期化される。

要素型初期値
数値型0または0.0
booleanfalse
char\u0000
参照型null
String[] names = new String[2];
System.out.println(names[0]); // null

参照のコピー

配列変数を別の変数へ代入しても、配列の中身がコピーされるわけではない。同じ配列を指す参照がコピーされる。

int[] a = {1, 2, 3};
int[] b = a;
b[0] = 99;
System.out.println(a[0]); // 99

多次元配列

Javaの多次元配列は「配列の配列」。各行の長さを揃えなくてもよい。

int[][] table = new int[2][];
table[0] = new int[3];
table[1] = new int[1];

手を動かすミニ課題

次のコードで例外が出る行を判断する。

int[] nums = new int[3];
System.out.println(nums[2]);
System.out.println(nums[3]);

丁寧な補講：理解の芯

配列は同じ型の値または参照をまとめて扱う固定長の入れ物。作成した後に要素数は変えられない。添字は0から始まる。

配列変数は参照型。つまり、変数には配列オブジェクトへの参照が入る。配列を別変数へ代入しても、配列本体がコピーされるわけではない。同じ配列を2つの変数で見ている状態になる。

要素の初期値も押さえる。int配列なら0、boolean配列ならfalse、参照型配列ならnull。ローカル変数そのものの未初期化とは分ける。

固定長

配列の長さは作成時に決まる。後から伸び縮みしない。

0始まり

最初の要素は[0]。最後は[length - 1]。

参照型

配列変数には配列オブジェクトへの参照が入る。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
配列変数 / 配列要素配列変数は参照。要素は中に入っている値または参照。aとa[0]を分ける。
長さ / 最後の添字長さが3なら最後の添字は2。length - 1が最後。
配列コピー / 参照コピー代入は参照コピー。中身の複製ではない。b = a後に片方を変更して確認。

コードを読むときの手順

  1. 配列が作成されているか見る。
  2. 長さを確認する。
  3. アクセスしている添字が0以上length - 1以下か見る。
  4. 別変数に代入されている場合、同じ配列を参照していないか確認する。
  5. 要素の型と初期値を確認する。

確認用コード

int[] a = new int[3];
int[] b = a;
b[0] = 10;
System.out.println(a[0]);
確認する点このコードでの判断結論
a長さ3の配列を参照要素初期値は0
b = a同じ配列を参照配列本体はコピーされない
b[0] = 10同じ配列の0番を変更a[0]も10

よくある誤解を潰す

配列の最後はlength番、は間違い

添字は0始まりなので最後はlength - 1。

b = aで配列が複製される、は違う

参照がコピーされるだけ。同じ配列を指す。

確認問題：答えまで言語化する

次のコードの出力を答える。

確認問題

String[] names = new String[2];
System.out.println(names[0]);
考え方と答えを見る

出力はnull。

Stringは参照型なので、配列要素の初期値はnullになる。ローカル変数names自体はnew String[2]で初期化済み。

理解の流れを整える補強

配列は「複数の値を入れるもの」ですが、Javaでは配列そのものもオブジェクトです。変数には配列オブジェクトへの参照が入ります。このため、参照をコピーすると同じ配列を2つの変数から見る状態になります。

int[] a = {1, 2, 3};
int[] b = a;
b[0] = 99;
System.out.println(a[0]); // 99

b = aは配列の中身を複製しているのではありません。参照をコピーしています。したがって、aとbは同じ配列オブジェクトを見ています。

コード意味注意
new int[3]要素数3のint配列を作る初期値は0
a.length配列の長さメソッドではないので()不要
a[0]先頭要素添字は0から始まる
a[3]4番目の要素長さ3なら実行時例外

配列の長さは作った後に変えられません。要素を増やしたいなら、新しい配列を作るか、後で学ぶArrayListのようなコレクションを使います。

配列で最初に疑うべき箇所

添字が範囲内か、参照をコピーして同じ配列を見ていないか、初期値を勘違いしていないか。この3つを先に確認してください。

理解チェック

  • 配列を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

配列で落ちる典型

つまずき正しい切り分け
配列の長さを後から変えられると思う配列の長さは生成時に固定。増減したいならArrayListなどを使う。
length()と書く配列はフィールドのlength。Stringはメソッドのlength()。
多次元配列は必ず長方形だと思うJavaの多次元配列は配列の配列。行ごとに長さが違ってよい。

説明できる状態の目安

  • 配列の添字範囲を説明できる
  • 配列の初期値を型別に答えられる
  • 配列とStringのlengthの違いを説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：配列の添字

int[] nums = {10, 20, 30};
System.out.println(nums[0]);
System.out.println(nums[2]);

失敗コード

int[] nums = {10, 20, 30};
System.out.println(nums[3]);

修正版

int[] nums = {10, 20, 30};
System.out.println(nums[nums.length - 1]);
判定ポイント

配列の添字は0から始まります。最後の要素はlengthではなくlength - 1です。

理解確認クイズ

break と continue の違いは何ですか。

答えと切り分けを見る

breakはループを抜けます。continueは現在の周回を中断して次の周回へ進みます。
