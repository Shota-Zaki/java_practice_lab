Java基礎 Lesson 18

コンパイルエラーと実行時例外

失敗タイミングで切り分ける。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

コンパイルエラー

javacの段階で検出される。文法ミス、型不一致、未初期化ローカル変数、到達不能コード、checked例外未処理など。

実行時例外

コンパイルは通るが、実行中の値や状態によって発生する。null参照、配列範囲外、数値変換失敗、キャスト失敗など。

例

String s = null;
System.out.println(s.length()); // 実行時例外
コンパイル時にはsがnullかどうかまでは原則として判定されない。

見分け方

型・文法・アクセス・例外宣言の問題ならコンパイルエラー。値がnullか、添字が範囲外か、実行時の中身に依存するなら実行時例外。

試験での罠

出力結果を追う前に、そもそもコンパイルできるかを確認しないと間違える。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
