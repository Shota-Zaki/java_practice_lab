Java基礎 Lesson 18

コンパイルエラーと実行時例外

失敗が「実行前」なのか「実行中」なのかで原因を切り分ける。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：コンパイルエラー・実行時例外

失敗は全部同じではありません。実行前に止まるのか、実行中に止まるのかで原因が違います。

コンパイルエラーとはjavacの段階で文法や型の問題により止まること。
実行時例外とはコンパイル後、実行中の値や状態が原因で発生する例外。
スタックトレースとは例外がどのメソッド呼び出し経路で発生したかを示す出力。
到達不能コードとは絶対に実行されないためコンパイルエラーになるコード。
原因切り分けとは文法・型・参照・実行順序のどれが原因か分けること。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
コンパイルエラー / 実行時例外コンパイルエラーは実行前に止まる。実行時例外は起動後の処理中に発生する。
文法エラー / 型エラー文法エラーはJavaとして形が壊れている。型エラーは型の扱いが合わない。
スタックトレース / エラーメッセージスタックトレースは実行時にどこで落ちたかの履歴。コンパイルエラーとは読み方が違う。

基礎用語一覧で確認する

コンパイルエラーと実行時例外の切り分け表

「エラーが出る」だけで止めると成長が遅いです。いつ止まったかを先に言語化してください。

分類発生タイミング代表例結果
コンパイルエラーjavac時型不一致、未定義変数、checked例外処理漏れ.classが作られず、実行できない
実行時例外java実行中null参照、配列範囲外、整数0除算、キャスト失敗途中まで実行され、例外発生地点で通常処理が止まる
論理ミス実行後出力が期待と違う、条件式が逆エラーは出ないが意図通りではない

コンパイルエラーの典型

原因例なぜ実行前に分かるか
文法ミスint a = ;文の形が成立していない
型不一致int a = "x";代入できない型だと分かる
未初期化ローカル変数int x; println(x);値が入っていない経路が分かる
到達不能コードreturn; println();絶対に実行されないと分かる
checked例外未処理throwsもcatchもない処理強制のルールに反する

実行時例外の典型

例外起きる状況
NullPointerExceptionnull参照からメソッドやフィールドを使う
ArrayIndexOutOfBoundsException配列の範囲外の添字を使う
NumberFormatException数値に変換できない文字列を変換する
ClassCastException実体に合わない型へキャストする
ArithmeticException整数の0除算など

同じように見えても違う

値がないことをコンパイラが判断できる

int x;
System.out.println(x); // コンパイルエラー

型としてはStringなのでコンパイルはできる

String s = null;
System.out.println(s.length()); // 実行時例外

判定手順

  1. 宣言されていない名前を使っていないか。
  2. 型が合っているか。
  3. アクセスできるか。
  4. ローカル変数は初期化済みか。
  5. checked例外を処理しているか。
  6. ここまで通るなら、実行時の値を追う。

手を動かすミニ課題

次の各行を、コンパイルエラー・実行時例外・正常実行に分類する。

int a = "1";
String s = null; s.length();
int[] nums = new int[2]; nums[2] = 10;
int b = 10 / 2;

丁寧な補講：理解の芯

コンパイルエラーと実行時例外は、失敗するタイミングが違う。コンパイルエラーは実行前にjavacが止める。実行時例外は、コンパイルは通った後、実際に処理を進めている途中で起きる。

この区別は学習効率に直結する。コンパイルエラーのコードについて出力結果を考えるのは無駄。まず実行できるコードかを確認する。

逆に、配列の範囲外アクセスやnull参照のように、コンパイル時には値が確定しないものは実行時例外になりやすい。

コンパイルエラー

文法・型・アクセス・未初期化など、実行前に検出される失敗。

実行時例外

実行中の値や状態によって起きる失敗。

論理バグ

エラーは出ないが、期待した結果と違う失敗。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
型が合わない / 値が不正型が合わないならコンパイルエラー。値が実行時に不正なら例外になりやすい。int x = "1"とargs[0]を区別。
未初期化 / null未初期化ローカル変数はコンパイルエラー。null参照は実行時例外。代入されていないのか、nullが代入されているのかを見る。
エラーあり / 出力ありコンパイルエラーなら出力はない。「何が表示されるか」より先にコンパイル可否。

コードを読むときの手順

  1. コード全体が文法的に成立するか見る。
  2. 型の代入・メソッド呼び出し・アクセス修飾子を確認する。
  3. ローカル変数が初期化済みか確認する。
  4. コンパイル可能なら、実行時の値を追う。
  5. 例外が起きる行があれば、それ以降の通常処理は実行されない。

確認用コード

String s = null;
System.out.println(s.length());
確認する点このコードでの判断結論
型sはString型コンパイルは可能
値sはnull参照先がない
実行length()呼び出しNullPointerException

よくある誤解を潰す

例外が起きるコードはコンパイルできない、は違う

実行時の値によって起きる問題はコンパイルを通ることがある。

コンパイルエラーでも途中まで実行される、は違う

コンパイルに失敗したら実行されない。出力は発生しない。

確認問題：答えまで言語化する

次のコードはコンパイルエラーか、実行時例外か、正常終了か。

確認問題

public class Main {
    public static void main(String[] args) {
        int[] a = new int[1];
        System.out.println(a[1]);
    }
}
考え方と答えを見る

コンパイルは通るが、実行時例外になる。

aは長さ1なので有効な添字は0だけ。a[1]は範囲外アクセスなのでArrayIndexOutOfBoundsExceptionが起きる。

理解の流れを整える補強

Javaの失敗は、まずコンパイル前に見つかる失敗と実行中に起きる失敗に分けます。この区別をせずに「エラー」とまとめると、原因を探す場所を間違えます。

分類いつ起きるか例主な原因
コンパイルエラーjavac時cannot find symbol文法、型、名前、アクセス制御の問題
実行時例外java実行中NullPointerException実行して初めて決まる値や状態の問題
正常終了最後まで実行出力が出る文法も実行時状態も問題なし
// コンパイルエラーの例
String s = 10; // 型が合わない

// 実行時例外の例
String t = null;
System.out.println(t.length()); // NullPointerException

上の2つはどちらも「失敗」ですが、調べる段階が違います。String s = 10;は実行前にコンパイラが止めます。t.length()は文法としては正しいためコンパイルできますが、実行中にtがnullなので失敗します。

  1. まず文法・型・名前・アクセス修飾子でコンパイルできるか見る。
  2. コンパイルできるなら、実行時の値と参照を追う。
  3. 例外が起きるなら、どの行で、どの例外型かを見る。
  4. catchがあるなら、どのcatchに入るかを見る。
ここで切るべき誤解

「nullだからコンパイルエラー」は間違いです。null参照からメソッドを呼ぶコードは文法上は成立します。問題は実行時に参照先がないことです。

理解チェック

  • コンパイルエラーと実行時例外を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

エラー段階の切り分け不足

つまずき正しい切り分け
すべてを「エラー」とだけ呼ぶコンパイルエラー、実行時例外、出力違いを分けないと原因に辿り着けない。
コンパイルできたら正解だと思う実行時例外や期待出力との違いはコンパイル後に発覚する。
エラーメッセージの最初だけ見る行番号、記号の位置、symbol、locationを合わせて読む。

説明できる状態の目安

  • コンパイルエラーと実行時例外の違いを説明できる
  • javacのエラー行と^の意味を説明できる
  • 出力問題の判定順を説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：コンパイルエラーと実行時例外

String s = null;
System.out.println(s.length()); // 実行時例外

失敗コード

String s = 10; // コンパイルエラー

修正版

String s = "10";
System.out.println(s.length());
判定ポイント

コンパイルエラーは実行前に止まります。実行時例外はコンパイル後、実行中に起きます。

理解確認クイズ

オーバーロードとオーバーライドの違いは何ですか。

答えと切り分けを見る

オーバーロードは同名で引数が違うメソッドを同じクラスなどに定義すること。オーバーライドは継承先で同じシグネチャのメソッドを再定義することです。
