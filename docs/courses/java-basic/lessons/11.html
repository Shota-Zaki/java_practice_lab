Java基礎 Lesson 11

メソッドと引数

処理の分割、引数、戻り値、値渡しを混同しないようにする。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：メソッド・引数・戻り値

ここから「処理を部品化する」話に入ります。呼び出す側と呼び出される側の言葉を分けます。

static int add(int a, int b) {
    return a + b;
}

int result = add(3, 4);
メソッドとは名前を付けた処理のまとまり。呼び出されると本体の処理が実行される。
引数とはメソッドへ渡す情報。呼び出し側の値を、定義側の仮引数で受け取る。
戻り値とはメソッドが呼び出し元へ返す結果。voidなら戻り値はない。
シグネチャとはメソッド名と引数リストの組み合わせ。オーバーロード判定で重要。
値渡しとは変数の中身のコピーを渡すこと。参照型でも参照のコピーが渡る。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
メソッド / 引数 / 戻り値メソッドは処理のまとまり。引数は外から渡す情報。戻り値は処理結果として返す値。
仮引数 / 実引数メソッド定義側が仮引数。呼び出し側で渡す値が実引数。
値渡し / 参照のコピーJavaは値渡し。参照型でも、渡しているのは参照値のコピー。

基礎用語一覧で確認する

メソッドの部品

戻り値ありのメソッド

public static int add(int a, int b) {
    return a + b;
}
部分意味
publicアクセス範囲
staticクラスから呼べる
int戻り値の型
addメソッド名
int a, int b引数
return値を返してメソッドを終える

値渡しを正しく理解する

プリミティブ型を渡すと、値のコピーが渡る。メソッド内で引数を書き換えても、呼び出し元の変数は変わらない。

static void change(int x) {
    x = 99;
}

public static void main(String[] args) {
    int n = 10;
    change(n);
    System.out.println(n); // 10
}

参照型でも値渡し

参照型を渡すと、参照のコピーが渡る。同じオブジェクトを指しているので中身の変更は見える。ただし、引数変数そのものに別オブジェクトを代入しても呼び出し元は変わらない。

中身を変更する例

static void change(int[] x) {
    x[0] = 99;
}

public static void main(String[] args) {
    int[] a = {10};
    change(a);
    System.out.println(a[0]); // 99
}

参照変数を付け替えるだけの例

static void replace(int[] x) {
    x = new int[] {99};
}

public static void main(String[] args) {
    int[] a = {10};
    replace(a);
    System.out.println(a[0]); // 10
}

戻り値の注意

戻り値型がvoidでないメソッドは、値を返す必要がある。条件分岐がある場合は、すべての経路で戻り値があるかを見る。

static int sign(int n) {
    if (n > 0) {
        return 1;
    }
    // n <= 0 のとき return がないのでコンパイルエラー
}

手を動かすミニ課題

次のコードの出力を、参照のコピーという言葉を使って説明する。

static void edit(StringBuilder sb) {
    sb.append("B");
}

public static void main(String[] args) {
    StringBuilder sb = new StringBuilder("A");
    edit(sb);
    System.out.println(sb);
}

丁寧な補講：理解の芯

メソッドは処理に名前を付けて分ける仕組み。呼び出す側は引数を渡し、呼び出された側は仮引数として受け取る。必要なら戻り値を返す。

Javaの引数渡しは値渡し。プリミティブ型なら値そのもののコピー、参照型なら参照のコピーが渡る。ここを「参照渡し」と雑に覚えると、オブジェクトの中身変更と変数の付け替えを混同する。

メソッドを読むときは、名前よりもシグネチャを見る。メソッド名、引数の数・型・順番が呼び出し解決に効く。戻り値だけではオーバーロードを区別できない。

引数

呼び出し側からメソッドへ渡す情報。

戻り値

メソッドの処理結果として返す値。voidなら返さない。

値渡し

変数の中身をコピーして渡す。参照型なら参照のコピー。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
実引数 / 仮引数呼び出し側に書く値が実引数。受け取る側の変数が仮引数。呼び出し行とメソッド定義を対応させる。
オブジェクト変更 / 参照の付け替え参照先の中身変更は見える。仮引数への再代入は呼び出し元変数を変えない。p.name=...とp=new ...を区別。
戻り値 / 出力戻り値は呼び出し元へ返す値。出力は画面に表示する処理。returnとprintlnは別物。

コードを読むときの手順

  1. メソッド定義の戻り値型、名前、引数を確認する。
  2. 呼び出し側の実引数と、定義側の仮引数を対応させる。
  3. プリミティブ型か参照型かを見る。
  4. メソッド内で仮引数に再代入しているか、参照先の中身を変えているかを見る。
  5. 戻り値があるなら、呼び出し元で使っているか確認する。

確認用コード

static void change(int x) {
    x = 20;
}

public static void main(String[] args) {
    int a = 10;
    change(a);
    System.out.println(a);
}
確認する点このコードでの判断結論
渡されるものaの値10のコピーxは別変数
メソッド内x = 20aは変わらない
出力aは1010

よくある誤解を潰す

仮引数を変えても呼び出し元の変数は変わらない

Javaは値渡し。仮引数の再代入は呼び出し元変数へ戻らない。

戻り値を受け取らないと変数に反映されない処理がある

戻り値で新しい値を返す設計なら、受け取らなければ結果を捨てる。

確認問題：答えまで言語化する

次の出力を答える。

確認問題

static int add(int x) {
    x = x + 5;
    return x;
}

public static void main(String[] args) {
    int n = 10;
    add(n);
    System.out.println(n);
}
考え方と答えを見る

出力は10。

add(n)は15を返すが、戻り値を変数に代入していない。n自体は変わらない。

理解の流れを整える補強

メソッドは処理を分ける仕組みです。読むときは、呼び出し側の実引数と受け取り側の仮引数を分けてください。名前が違っても、渡される値の流れを追えば読めます。

static void change(int x) {
    x = 20;
}

public static void main(String[] args) {
    int n = 10;
    change(n);
    System.out.println(n); // 10
}

Javaは値渡しです。change(n)では、nそのものが移動するのではなく、nに入っている値10がxへコピーされます。メソッド内でxを変更しても、呼び出し側のnは変わりません。

参照型でも「参照の値」がコピーされます。オブジェクトの中身を変更すれば呼び出し側からも見えますが、仮引数そのものを別オブジェクトへ代入しても、呼び出し側の変数は差し替わりません。

static void edit(int[] data) {
    data[0] = 99;          // 配列の中身を変更
    data = new int[]{1};   // 仮引数dataの参照を差し替えるだけ
}

public static void main(String[] args) {
    int[] nums = {10, 20};
    edit(nums);
    System.out.println(nums[0]); // 99
}
ここで切るべき誤解

「参照型は参照渡し」は不正確です。Javaは常に値渡しです。参照型の場合は、オブジェクトへの参照という値がコピーされます。

理解チェック

  • メソッドと引数を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

メソッドで混同する点

つまずき正しい切り分け
引数と戻り値を同じものとして扱う引数は呼び出し側から渡す値。戻り値はメソッドから返す値。
仮引数を書き換えると呼び出し元の変数も必ず変わると思うJavaは値渡し。参照型でも「参照値」がコピーされる。
voidでも値を返せると思うvoidは戻り値なし。return;は可能だが値は返せない。

説明できる状態の目安

  • 仮引数と実引数を区別できる
  • 戻り値型とreturn文の関係を説明できる
  • Javaの値渡しを一文で説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：メソッドの引数と戻り値

static int add(int a, int b) {
    return a + b;
}

失敗コード

static int add(int a, int b) {
    System.out.println(a + b);
}

修正版

static int add(int a, int b) {
    return a + b;
}
判定ポイント

戻り値型がvoidでないメソッドは、対応する型の値をreturnする必要があります。

理解確認クイズ

戻り値型が void のメソッドは値を返せますか。

答えと切り分けを見る

返せません。return; で終了することはできますが、return 1; のように値を返すとコンパイルエラーです。
