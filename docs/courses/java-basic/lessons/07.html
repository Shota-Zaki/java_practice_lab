Java基礎 Lesson 7

Stringの基本

Stringの不変性と比較を理解する。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

Stringは不変

Stringのreplace、substring、concatなどは元の文字列を変更しない。変更後の文字列を新しく返す。

例

String s = "abc";
s.replace("a", "x");
System.out.println(s); // abc
戻り値を受け取っていないためsは変わらない。

比較

== は参照比較。equals() は内容比較。Stringリテラルは文字列プールに置かれるため、==がtrueになるケースもあるが、常に内容比較として使うのは危険。

intern

intern() は文字列プール上の参照を返す。Java Silverでは new String とリテラルの比較でよく問われる。

試験での罠

StringBuilderのequals()は内容比較ではなく、基本的にはObject由来の参照比較。Stringと同じ感覚で読むと間違える。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
