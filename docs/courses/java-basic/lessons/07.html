Java基礎 Lesson 7

Stringの基本

文字列がどのように扱われ、なぜ比較や変更でつまずくのかを理解する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

Stringは不変

replace、substring、toUpperCaseなどは、元のStringを書き換えない。戻り値を受け取らないと、結果は捨てられる。

悪い読み方をしやすい例

String s = "java";
s.toUpperCase();
System.out.println(s); // java

戻り値を受け取る

String s = "java";
s = s.toUpperCase();
System.out.println(s); // JAVA

==とequals

==は同じオブジェクトを参照しているかを見る。equalsは文字列の内容を比較する。文字列リテラルでは==がtrueになることもあるが、それを内容比較として使うのは危険。

String a = new String("java");
String b = "java";

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true

文字列連結

+にStringが混ざると文字列連結になる。左から評価されるため、数値計算と文字列連結が混ざると結果が変わる。

System.out.println(1 + 2 + "A"); // 3A
System.out.println("A" + 1 + 2); // A12

nullとの組み合わせ

String変数がnullのとき、その変数からメソッドを呼ぶとNullPointerExceptionになる。内容が空の文字列""とは別物。

値意味メソッド呼び出し
null何も参照していないs.length()で実行時例外
""長さ0の文字列オブジェクトs.length()は0
" "空白1文字の文字列s.length()は1

手を動かすミニ課題

次の出力を予測し、==なのかequalsなのかを説明する。

String a = "x";
String b = "x";
String c = new String("x");
System.out.println(a == b);
System.out.println(a == c);
System.out.println(a.equals(c));

丁寧な補講：理解の芯

Stringは文字列を表す参照型。プリミティブ型ではない。変数には文字列そのものではなく、Stringオブジェクトへの参照が入る。

さらにStringは不変。concatやreplaceを呼んでも、元の文字列オブジェクトが書き換わるわけではない。結果として新しい文字列が返る。返り値を受け取らないと変化しないように見える。

比較では==とequalsを分ける。==は参照の比較、equalsは文字列内容の比較として使う。

参照型

String sにはStringオブジェクトへの参照が入る。

不変

文字列操作は元を変えず、新しい文字列を返す。

比較

内容比較は基本equals。==は参照比較。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
== / equals==は同じ参照か、equalsは内容が同じか。文字列内容を見たいならequals。
文字列リテラル / new Stringリテラルは文字列プールが絡む。newは新規オブジェクト。参照比較で差が出る。
変更 / 新規作成Stringは変更されず、新しい結果が作られる。返り値を代入しているか見る。

コードを読むときの手順

  1. String変数が何を参照しているか見る。
  2. newで作ったか、リテラルかを見る。
  3. 文字列操作メソッドの返り値を代入しているか見る。
  4. 比較が==かequalsかを見る。

確認用コード

String s = "Java";
s.concat("Java");
System.out.println(s);
確認する点このコードでの判断結論
初期値sは"Java"を参照OK
concat新しい文字列を返すが代入していないsは変わらない
出力sの参照先は元のままJava

よくある誤解を潰す

Stringは文字配列のように直接書き換わる、は違う

Stringは不変。変更したいなら返り値を受け取る。

文字列は常に==で比べればよい、は危険

同じ内容でも別オブジェクトなら==はfalseになりうる。

確認問題：答えまで言語化する

次の出力を答え、理由を説明する。

確認問題

String a = "ab";
String b = new String("ab");
System.out.println(a == b);
System.out.println(a.equals(b));
考え方と答えを見る

1行目はfalse。aとbは同じ内容だが、参照先が同じとは限らない。new Stringで別オブジェクトを作っている。

2行目はtrue。equalsは文字列内容を比較するため。

理解の流れを整える補強

Stringはよく使うので簡単に見えますが、実際には参照型・不変・文字列プールが絡みます。基礎段階では、まず「文字列操作メソッドは元を書き換えない」「内容比較はequals」の2点を確実にしてください。

String s = "abc";
s.substring(1);
System.out.println(s); // abc

このコードでbcが表示されると思ったなら、まだStringを「書き換わる箱」として見ています。substringは新しい文字列を返すだけです。戻り値を受け取らない限り、変数sの参照先は変わりません。

String s = "abc";
s = s.substring(1);
System.out.println(s); // bc
比較何を見ているか使う場面
a == b同じオブジェクトを参照しているか参照そのものを確認したいとき
a.equals(b)文字列の内容が同じか通常の文字列比較
"x".equals(a)定数側から比較aがnullの可能性があるとき
ここで切るべき誤解

==がtrueになる文字列の例を見て「文字列は==で比較できる」と結論づけるのは間違いです。たまたま同じ参照を見ているだけの場合があります。

理解チェック

  • Stringの基本を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。
