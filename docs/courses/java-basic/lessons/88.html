Lesson 88 例外を「種類」と「発生タイミング」で見る

例外処理は、checked / unchecked と catch順序を整理すると理解しやすくなります。

checked例外unchecked例外catch順序

checked例外

ExceptionやIOExceptionなどは、投げる可能性があるなら処理または宣言が必要です。

unchecked例外

RuntimeException系は、コンパイラが処理を強制しません。NullPointerExceptionやArrayIndexOutOfBoundsExceptionが代表です。

catch順序

catch (Exception e) { }
catch (IOException e) { } // 到達不能でコンパイルエラー

広い型を先に捕まえると、狭い型のcatchに到達できません。
