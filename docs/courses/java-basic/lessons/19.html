Java基礎 Lesson 19

Javaファイルとクラス名

ファイル名、publicクラス、トップレベルクラス、実行クラスを混同しない。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：トップレベルクラス・ファイル名・パッケージ

Javaファイルとクラスは1対1とは限りません。ただしpublicクラス名とファイル名には制約があります。

トップレベルクラスとは他のクラスの中ではなく、ファイル直下に定義されたクラス。
public classとは外部から利用できる公開クラス。1つのファイルに最大1つ。
ファイル名一致とはpublicトップレベルクラスがある場合、ファイル名をそのクラス名に合わせるルール。
パッケージとはクラスを分類し、名前の衝突を避ける仕組み。
完全修飾名とはパッケージ名を含めたクラス名。例: ex.Main。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
トップレベルクラス / ネストしたクラストップレベルクラスは他のクラスの外側に直接書くクラス。ネストしたクラスはクラス内に書くクラス。
publicクラス / ファイル名publicトップレベルクラスがある場合、ファイル名はそのクラス名と一致必須。
package / ディレクトリpackage宣言はクラスの所属名。通常はディレクトリ構造と対応させる。

基礎用語一覧で確認する

トップレベルクラス

他のクラスの内側ではなく、ファイル直下に書かれたクラスをトップレベルクラスと呼ぶ。

どちらもトップレベルクラス

class A { }
class B { }

publicクラスとファイル名

ファイル内容ファイル名結果
public class Main { }Main.javaOK
public class Main { }Sample.javaコンパイルエラー
class Main { }Sample.javaOK
public class A { } public class B { }どちらでも不可publicトップレベルクラスが複数あるためエラー

実行するクラス名

javaコマンドで指定するのは、ファイル名ではなくクラス名。mainメソッドを持つクラスが複数あれば、それぞれ実行できる。

A.javaでなくても、publicがなければ保存名の自由度がある

class A {
    public static void main(String[] args) {
        System.out.println("A");
    }
}
class B {
    public static void main(String[] args) {
        System.out.println("B");
    }
}
javac Sample.java
java A
java B

パッケージが絡む場合

package宣言がある場合、ディレクトリ構造とクラス名の対応も意識する必要がある。基礎段階では、まず「publicクラス名とファイル名」を確実に押さえる。

手を動かすミニ課題

次のコードをTest.javaとして保存した場合、コンパイルできるか判定する。

public class Main {
    public static void main(String[] args) { }
}
class Test { }

丁寧な補講：理解の芯

Javaファイルとクラス名のルールは、最初に固めておくべき実務的な基礎。publicなトップレベルクラスがある場合、ソースファイル名はそのクラス名と一致しなければならない。

1つのファイルに複数のトップレベルクラスを書くことはできる。ただし、publicなトップレベルクラスは1つまで。publicでないクラスはファイル名と一致していなくてもよい。

実行時はファイル名ではなく、mainメソッドを持つクラス名を指定する。publicでないクラスでも、mainがあれば実行できる場合がある。

トップレベルクラス

他のクラスの中ではなく、ファイル直下に書かれたクラス。

publicクラス

外部から見えるクラス。ファイル名と一致が必要。

実行クラス

mainを持ち、java クラス名で起動するクラス。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
ファイル名 / publicクラス名publicトップレベルクラスがあるなら一致が必要。public class XならX.java。
public / 非publicpublicトップレベルクラスは1ファイル1つまで。非publicは複数可。修飾子を見る。
コンパイル対象 / 実行対象コンパイルはファイル、実行はクラス。javac A.java、java A。

コードを読むときの手順

  1. ファイル内のトップレベルクラスを数える。
  2. publicなトップレベルクラスがあるか確認する。
  3. publicクラス名とファイル名が一致するか見る。
  4. mainメソッドを持つクラスを探す。
  5. 実行時はmainを持つクラス名を指定する。

確認用コード

// A.java
public class A { }
class B {
    public static void main(String[] args) {
        System.out.println("B");
    }
}
確認する点このコードでの判断結論
ファイル名A.javapublic class A と一致
クラスB非publicトップレベルクラス同じファイルに書ける
実行Bにmainがあるjava Bで実行可能

よくある誤解を潰す

1ファイル1クラスしか書けない、は言い過ぎ

複数のトップレベルクラスを書ける。ただしpublicは1つまで。

ファイル名を指定して実行する、は違う

javaに渡すのはクラス名。.javaや.classではない。

確認問題：答えまで言語化する

次のコードがSample.javaに保存されている。コンパイルできるか。

確認問題

public class Main {
}
class Sample {
}
考え方と答えを見る

コンパイルできない。

publicなトップレベルクラスはMainなので、ファイル名はMain.javaである必要がある。Sample.javaとは一致しない。

理解の流れを整える補強

このLessonでは、ファイル名・クラス名・実行名を分けて考えることが中心です。Javaでは、1つのソースファイルに複数のトップレベルクラスを書けます。ただし、publicなトップレベルクラスがある場合、ファイル名はそのクラス名と一致する必要があります。

観点見る場所判断
保存名Sample.javapublic class Sampleがあるなら一致が必要
トップレベルクラスファイル直下のclass複数書けるが、publicは原則1つ
実行名java クラス名mainを持つクラス名を指定する
// ファイル名: A.java
public class A {
    public static void main(String[] args) {
        System.out.println("A");
    }
}

class B {
    public static void main(String[] args) {
        System.out.println("B");
    }
}

この場合、javac A.javaでコンパイルできます。実行時はjava Aでもjava Bでも、対応するmainがあれば実行できます。ファイル名と実行クラス名は同じ場面で使うものではありません。

ここで切るべき誤解

「ファイル名と同じクラスだけが実行できる」は誤りです。実行できるかは、対象クラスが適切なmainメソッドを持つかで判断します。

理解チェック

  • Javaファイルとクラス名を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

ファイル名とクラス名で詰まる点

つまずき正しい切り分け
public classが複数書けると思うトップレベルのpublic classは1ファイルに最大1つ。ファイル名と一致が必要。
publicでないクラスは実行できないと思うpublicでなくても、正しいmainがありクラス名を指定すれば実行できる場合がある。
java A.classと実行するjavaコマンドに指定するのはクラス名。ファイル名ではない。

説明できる状態の目安

  • public classとファイル名の関係を説明できる
  • 1ファイル複数classのclass生成を説明できる
  • javacとjavaで指定するものの違いを説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：ファイル名とpublic class

失敗コード

修正版

public class Main { }

判定ポイント
