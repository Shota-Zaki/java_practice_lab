Java基礎 Lesson 2

OS・JDK・JVM

Javaを動かす部品を分解し、どこで何が起きるのかを整理する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：OS・JDK・JVM・PATH

Javaが動くまでには、OS、JDK、JVM、PATHが関係します。役割を混ぜないでください。

OSとはWindowsやmacOSなど、アプリとハードウェアの間を管理する基本ソフト。
JDKとはJavaの開発キット。コンパイル用のjavacと実行用のjavaを含む。
JREとはJava実行環境。現在の学習ではJDKに含まれるものとして見る。
PATHとはコマンド名だけでプログラムを呼び出すための検索先設定。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
OS / JVMOSはPC全体を管理する基本ソフト。JVMはJavaプログラムを動かす実行環境。
javac / javajavacは.javaを.classにする。javaは.classを起動する。役割は逆にできない。
PATH / classpathPATHはコマンド自体を探す設定。classpathは実行時にクラスを探す場所。

基礎用語一覧で確認する

JDKは「開発者用の工具箱」

Javaを書くなら基本的にJDKを入れる。JDKには、ソースコードをコンパイルするjavac、プログラムを起動するjava、標準ライブラリなどが含まれる。

インストール確認の例

javac -version
java -version

この2つのコマンドが動くなら、少なくともJavaをコンパイル・実行する入口は用意できている。

JVMは「Java用の実行環境」

JVMは.classファイルを読み込み、バイトコードを実行する。クラスを読み込む、メモリを確保する、例外を発生させる、不要になったオブジェクトを回収する、といった処理にも関わる。

重要

JVMがあるから、同じJavaコードを複数のOS上で動かしやすい。ただしOS差が完全に消えるわけではない。ファイルパスや文字コードなど、環境差が出る部分もある。

エラー切り分けの考え方

状況疑う場所例
javacが見つからないJDKのインストールまたはPATHjavac : 用語が認識されません
javacで失敗するソースコードの文法・型・アクセス; expected、cannot find symbol
java実行後に失敗する実行中の値・main・クラスパスNullPointerException、Could not find or load main class

悪い例

JDKとJVMを同じものとして扱うと、次のような判断ミスが起きる。

  • javacがないのに、コードの文法ミスだと思い込む。
  • NullPointerExceptionなのに、コンパイル設定の問題だと思い込む。
  • java Main.classのように、実行コマンドの指定を間違える。

手を動かすミニ課題

PowerShellで次を実行し、どちらがJDK/JVMのどの部分に関係するかを説明する。

javac -version
java -version

javacはコンパイル、javaは実行。両方が使える状態にするのが、最初の環境構築のゴール。

丁寧な補講：理解の芯

OS・JDK・JVMは、役割が違う。OSはWindowsなどの土台、JDKはJava開発用の道具箱、JVMはJavaプログラムを動かす実行環境。この3つを混ぜると、環境構築でもエラー調査でも判断が鈍る。

javacが動かないならJDKやPATHの問題を疑う。javacでエラーが出るならコードの問題を疑う。java実行後に例外が出るなら、実行時の値や処理の流れを疑う。

要するに、エラーを見たら「どの層で失敗したか」を先に決める。ここを飛ばして感覚で直そうとすると、原因と無関係な場所をいじることになる。

OS

ファイル保存、プロセス、メモリ、標準入出力などを管理する土台。

JDK

javac、java、標準ライブラリなどを含む開発道具。

JVM

.classを読み込み、Javaプログラムを実行する仮想マシン。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
JDK / JREJDKは開発用。JREは実行用の環境という位置づけ。現代はJDKを入れておけば学習には十分。javacが必要ならJDK。
javac / javajavacはコンパイル、javaは実行。ファイル名を渡すか、クラス名を渡すかも違う。
環境エラー / コードエラーコマンドが見つからないのは環境。文法・型の指摘はコード。エラーメッセージの先頭を見る。

コードを読むときの手順

  1. javac -versionでコンパイラが使えるか確認する。
  2. java -versionで実行コマンドが使えるか確認する。
  3. ソースコードを保存したフォルダでコマンドを実行しているか確認する。
  4. エラーが出たら、JDK・コンパイル・実行のどの段階かに分類する。

確認用コード

javac -version
java -version
確認する点このコードでの判断結論
javacコンパイラの確認動かなければJDK/PATHを疑う
java実行環境の確認動かなければPATHまたはインストールを疑う
両方OKコードをコンパイル・実行する準備がある次はファイル名とクラス名を見る

よくある誤解を潰す

JVMだけでは開発できない

学習ではコンパイルが必要なのでjavacを含むJDKが必要。

環境問題をコードで直そうとしない

javacが認識されない状態でコードを直しても意味がない。

確認問題：答えまで言語化する

PowerShellでjavacが認識されず、javaは認識される。このとき最初に疑うべきものは何か。

考え方と答えを見る

最初に疑うのはJDKのインストール状態、またはPATH設定。

javaだけ使える状態でも、javacが使えないならコンパイルできない。コードの文法を直す前に環境を確認する。

理解の流れを整える補強

このLessonでは、OS・JDK・JVMの役割を混ぜないことが最重要です。全部を「Javaを動かすもの」とまとめてしまうと、インストール、コンパイル、実行、エラー調査で何を確認すべきか分からなくなります。

部品主な役割詰まったときに見る場面
OSファイル、プロセス、画面、入出力などを管理するパス、権限、コマンド実行、ファイル位置
JDKJava開発に必要な道具一式。javacも含むjavacが見つからない、コンパイルできない
JVMコンパイル済みのバイトコードを実行するjava実行時のクラス読み込みや例外

たとえばjavacが見つからないなら、コードの文法以前にJDKやPATHの問題です。一方、java Mainで例外が出るなら、JVMが実行を始めた後の問題です。この区別は、初心者が無駄に時間を溶かしやすい箇所です。

javac Main.java  // JDKのコンパイラを使う
java Main        // JVMでMainクラスを実行する

この2つのコマンドは似ていますが、担当が違います。javacは「作る」、javaは「動かす」です。動かす前に作れていないなら、実行結果を考えても意味がありません。

ここで切るべき誤解

JDKとJVMを同じものとして扱うと、原因調査が雑になります。開発道具の問題なのか、実行環境の問題なのか、コードの問題なのかを切り分けてください。

理解チェック

  • OS・JDK・JVMを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

JDK・JVM周りの詰まり方

つまずき正しい切り分け
JDKとJVMを同じ意味で使うJDKは開発道具一式。JVMはclassファイルを実行する仮想マシン。
JREを入れれば開発できると思うJREは実行環境。コンパイルするには通常JDKが必要。
PATHを「Javaの機能」と思うPATHはOSの設定。コマンド名だけで実行ファイルを探すための仕組み。

説明できる状態の目安

  • JDKに含まれる代表的なコマンドを言える
  • JVMが実行する対象が.javaではなく.classだと説明できる
  • PATHが通っていないと何が起きるか説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：JDK・JVMの切り分け

// 開発時
javac Main.java

// 実行時
java Main

失敗コード

// JVMだけあればコンパイルもできる、という理解

修正版

JDK: javacなど開発道具を含む
JVM: classファイルを実行する仕組み
判定ポイント

JDKとJVMを混同すると、環境エラーの原因を見誤ります。

理解確認クイズ

変数、値、型の説明として正しいものはどれですか。

答えと切り分けを見る

変数は値を入れる名前付きの箱、値は実際のデータ、型はその値の扱い方を決めるルールです。
