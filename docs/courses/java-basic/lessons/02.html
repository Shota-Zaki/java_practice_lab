Java基礎 Lesson 2

OS・JDK・JVM

Javaを動かす部品の役割を整理する。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

結論

OSはアプリケーションを動かす土台、JDKは開発道具一式、JVMはJavaプログラムを実行する仮想マシン。学習ではJDKとJVMの違いを必ず分ける。

JDK

JDKにはjavac、javaコマンド、標準ライブラリ、開発に必要なツールが含まれる。Javaを書く人が入れるのは基本的にJDK。

JVM

JVMは.classファイルのバイトコードを実行する。メモリ管理、例外処理、クラスロードなどもJVMが関与する。

JRE

JREは実行環境を指す言葉だが、現在の学習ではJDKを入れれば十分なことが多い。試験ではJDK/JRE/JVMの言葉の違いが問われることがある。

実務的な見方

エラーが出たとき、javacで失敗したならコンパイル時の問題、javaで実行してから失敗したなら実行時の問題。この区別が最重要。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
