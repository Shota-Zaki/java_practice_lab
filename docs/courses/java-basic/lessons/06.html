Java基礎 Lesson 6

数値型と型変換

数値の範囲、リテラル、型変換、計算時の型の変化を整理する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：リテラル・型変換・キャスト

数値は見た目だけで判断しません。リテラルの型と、代入先の型を分けます。

リテラルとはコードに直接書いた値。例: 10、10L、"abc"。
型変換とはある型の値を別の型として扱うこと。
キャストとは明示的に型変換を指定すること。例: (int) 3.5。
オーバーフローとは型の範囲を超えて値が回り込むこと。
数値昇格とは演算時に小さい数値型がintなどへ広がること。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
リテラル / 変数リテラルはコードに直接書いた値。変数は実行中に値を保持する名前。
拡大変換 / 縮小変換安全な方向が拡大変換。情報が落ちうる方向が縮小変換で、基本はキャストが必要。
キャスト / 自動変換キャストは明示的な型変換。自動変換はコンパイラが許す範囲で行われる変換。

基礎用語一覧で確認する

代表的な数値型

型大まかな役割注意点
byte小さい整数計算時はintになりやすい
short小さい整数リテラル代入と計算結果代入を分ける
int整数の基本整数リテラルの基本型
long大きい整数Lを付けることがある
float単精度小数Fを付けないと小数リテラルは基本double
double小数の基本小数リテラルの基本型

リテラル代入と計算結果代入は一致しない

byte b = 10;は通る。10がbyteの範囲内で、定数として明らかだから。しかしbyte c = b + 1;は通らない。b + 1の計算結果はintとして扱われるから。

悪い例

byte b = 10;
byte c = b + 1; // コンパイルエラー

修正例

byte b = 10;
byte c = (byte) (b + 1); // 明示的にキャスト

複合代入は暗黙のキャストを含む

+=などの複合代入は、単純な+とは挙動が違う。x += 1は、計算後に左辺の型へ戻す処理を含む。

通常の加算

byte x = 1;
x = x + 1; // コンパイルエラー

複合代入

byte x = 1;
x += 1; // OK

小数リテラルの罠

小数リテラルは基本的にdouble。float f = 1.0;は、doubleをfloatへ入れようとしているためコンパイルエラーになる。

float f1 = 1.0;   // コンパイルエラー
float f2 = 1.0F;  // OK
double d = 1.0;   // OK

手を動かすミニ課題

次のうちコンパイルできるものを、理由付きで判定する。

short a = 100;
short b = a + 1;
long c = 10;
float d = 10.0F;
float e = 10.0;

丁寧な補講：理解の芯

数値型は「どれも数字」ではない。扱える範囲、整数か小数か、計算時にどの型へ昇格するかが違う。Javaではこの型の違いを厳しく見る。

整数リテラルは基本的にint、小数リテラルは基本的にdoubleとして扱われる。だからfloat f = 1.0;はそのままだと通らない。1.0Fのように明示する必要がある。

byteやshortは特に注意。変数同士の計算ではintに昇格しやすい。代入できるかどうかは「右辺の型」と「左辺の型」の関係で見る。

範囲

小さい型に大きい値は入らない。必要ならキャストが必要。

リテラル型

整数は基本int、小数は基本double。

数値昇格

byteやshortの計算結果は基本intになりやすい。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
代入可能 / キャスト必要狭い型へ広い型を入れるなら基本キャストが必要。左辺と右辺の型を見る。
整数 / 小数10は整数、10.0は小数。小数はdouble扱いが基本。
リテラル / 変数定数リテラルは範囲内なら特別に代入できる場合がある。変数計算は別。byte b = 10;とbyte b = a + 1;を区別する。

コードを読むときの手順

  1. 数値が整数か小数かを見る。
  2. リテラルのデフォルト型を判断する。
  3. 演算があるなら、演算後の型を判断する。
  4. 左辺の型に代入できるか確認する。
  5. 必要なら明示的キャストまたは接尾辞を使う。

確認用コード

byte a = 10;
byte b = 20;
byte c = a + b;
確認する点このコードでの判断結論
a + bbyte同士の計算intへ昇格
左辺byte cintをそのまま入れられない
結論暗黙変換不可コンパイルエラー

よくある誤解を潰す

byte + byteはbyteにならない

小さい整数型の計算はintとして扱われるのが基本。

float f = 1.0;はそのままでは通らない

1.0はdouble。1.0Fまたはキャストが必要。

確認問題：答えまで言語化する

次のうち、なぜコンパイルエラーになるかを説明する。

確認問題

short s = 10;
s = s + 1;
考え方と答えを見る

s + 1の結果はshortではなくintとして扱われる。

intをshort変数へそのまま代入できないため、コンパイルエラーになる。s = (short)(s + 1);のような明示的キャストが必要。

理解の流れを整える補強

数値型で重要なのは、代入できるかと計算した結果の型が何かを分けることです。初心者は「値が範囲内なら大丈夫」と考えがちですが、変数を使った計算では別のルールが働きます。

byte a = 10;       // OK。10はbyteの範囲内
byte b = 20;       // OK
byte c = 10 + 20;  // OK。定数式として30と判断できる
// byte d = a + b; // コンパイルエラー。byte同士の計算結果はint

a + bは、値としては30になりそうでも、式の型はintです。byteやshortは、計算時にintへ昇格する、と覚えてください。これは暗記ではなく、Javaが小さい整数型の計算を安全に扱うための基本ルールです。

コード見るポイント結果
short s = 100;整数リテラルが範囲内OK
short s = 100000;shortの範囲外コンパイルエラー
int i = 10L;longをintに入れようとしているコンパイルエラー
float f = 1.0;小数リテラルは基本doubleコンパイルエラー
float f = 1.0F;FでfloatリテラルにしているOK
問題でも実務でも危ない読み方

「右辺の答えが小さいから入る」と読むのは危険です。リテラルなのか、変数を含む式なのか、式の型は何かを順番に見てください。

理解チェック

  • 数値型と型変換を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

型変換で落ちる典型

つまずき正しい切り分け
小さい型同士の演算結果も小さい型だと思うbyteやshortは演算時に基本的にintへ昇格する。
複合代入を単純代入と同じだと思うa += bには暗黙キャストが入る場合がある。a = a + bと完全同一ではない。
charを文字専用だと思うcharは数値的にも扱われる。ただしshortと自由に互換ではない。

説明できる状態の目安

  • 拡大変換と縮小変換を区別できる
  • byte + byte の結果型を説明できる
  • +=でコンパイルが通る理由を説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：数値型と型変換

int a = 10;
long b = a;

失敗コード

long a = 10L;
int b = a;

修正版

long a = 10L;
int b = (int) a;
判定ポイント

広い型から狭い型へ入れるときは、値が収まるかではなく型としてキャストが必要です。

理解確認クイズ

String の replace を呼んだあと、元の文字列変数は変わりますか。

答えと切り分けを見る

変わりません。Stringは不変です。戻り値を受け取らないと変更後の文字列は使えません。
