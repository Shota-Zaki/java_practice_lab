Java基礎 Lesson 6

数値型と型変換

数値の範囲、リテラル、型変換、計算時の型の変化を整理する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

代表的な数値型

型大まかな役割注意点
byte小さい整数計算時はintになりやすい
short小さい整数リテラル代入と計算結果代入を分ける
int整数の基本整数リテラルの基本型
long大きい整数Lを付けることがある
float単精度小数Fを付けないと小数リテラルは基本double
double小数の基本小数リテラルの基本型

リテラル代入と計算結果代入は違う

byte b = 10;は通る。10がbyteの範囲内で、定数として明らかだから。しかしbyte c = b + 1;は通らない。b + 1の計算結果はintとして扱われるから。

悪い例

byte b = 10;
byte c = b + 1; // コンパイルエラー

修正例

byte b = 10;
byte c = (byte) (b + 1); // 明示的にキャスト

複合代入は暗黙のキャストを含む

+=などの複合代入は、単純な+とは挙動が違う。x += 1は、計算後に左辺の型へ戻す処理を含む。

通常の加算

byte x = 1;
x = x + 1; // コンパイルエラー

複合代入

byte x = 1;
x += 1; // OK

小数リテラルの罠

小数リテラルは基本的にdouble。float f = 1.0;は、doubleをfloatへ入れようとしているためコンパイルエラーになる。

float f1 = 1.0;   // コンパイルエラー
float f2 = 1.0F;  // OK
double d = 1.0;   // OK

手を動かすミニ課題

次のうちコンパイルできるものを、理由付きで判定する。

short a = 100;
short b = a + 1;
long c = 10;
float d = 10.0F;
float e = 10.0;

丁寧な補講：理解の芯

数値型は「どれも数字」ではない。扱える範囲、整数か小数か、計算時にどの型へ昇格するかが違う。Javaではこの型の違いを厳しく見る。

整数リテラルは基本的にint、小数リテラルは基本的にdoubleとして扱われる。だからfloat f = 1.0;はそのままだと通らない。1.0Fのように明示する必要がある。

byteやshortは特に注意。変数同士の計算ではintに昇格しやすい。代入できるかどうかは「右辺の型」と「左辺の型」の関係で見る。

範囲

小さい型に大きい値は入らない。必要ならキャストが必要。

リテラル型

整数は基本int、小数は基本double。

数値昇格

byteやshortの計算結果は基本intになりやすい。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
代入可能 / キャスト必要狭い型へ広い型を入れるなら基本キャストが必要。左辺と右辺の型を見る。
整数 / 小数10は整数、10.0は小数。小数はdouble扱いが基本。
リテラル / 変数定数リテラルは範囲内なら特別に代入できる場合がある。変数計算は別。byte b = 10;とbyte b = a + 1;を区別する。

コードを読むときの手順

  1. 数値が整数か小数かを見る。
  2. リテラルのデフォルト型を判断する。
  3. 演算があるなら、演算後の型を判断する。
  4. 左辺の型に代入できるか確認する。
  5. 必要なら明示的キャストまたは接尾辞を使う。

確認用コード

byte a = 10;
byte b = 20;
byte c = a + b;
確認する点このコードでの判断結論
a + bbyte同士の計算intへ昇格
左辺byte cintをそのまま入れられない
結論暗黙変換不可コンパイルエラー

よくある誤解を潰す

byte + byteはbyte、ではない

小さい整数型の計算はintとして扱われるのが基本。

float f = 1.0;は通る、ではない

1.0はdouble。1.0Fまたはキャストが必要。

確認問題：答えまで言語化する

次のうち、なぜコンパイルエラーになるかを説明する。

確認問題

short s = 10;
s = s + 1;
考え方と答えを見る

s + 1の結果はshortではなくintとして扱われる。

intをshort変数へそのまま代入できないため、コンパイルエラーになる。s = (short)(s + 1);のような明示的キャストが必要。

理解の流れを整える補強

数値型で重要なのは、代入できるかと計算した結果の型が何かを分けることです。初心者は「値が範囲内なら大丈夫」と考えがちですが、変数を使った計算では別のルールが働きます。

byte a = 10;       // OK。10はbyteの範囲内
byte b = 20;       // OK
byte c = 10 + 20;  // OK。定数式として30と判断できる
// byte d = a + b; // コンパイルエラー。byte同士の計算結果はint

a + bは、値としては30になりそうでも、式の型はintです。byteやshortは、計算時にintへ昇格する、と覚えてください。これは暗記ではなく、Javaが小さい整数型の計算を安全に扱うための基本ルールです。

コード見るポイント結果
short s = 100;整数リテラルが範囲内OK
short s = 100000;shortの範囲外コンパイルエラー
int i = 10L;longをintに入れようとしているコンパイルエラー
float f = 1.0;小数リテラルは基本doubleコンパイルエラー
float f = 1.0F;FでfloatリテラルにしているOK
問題でも実務でも危ない読み方

「右辺の答えが小さいから入る」と読むのは危険です。リテラルなのか、変数を含む式なのか、式の型は何かを順番に見てください。

理解チェック

  • 数値型と型変換を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。
