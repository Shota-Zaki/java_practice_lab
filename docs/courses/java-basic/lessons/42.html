オーバーロード解決の入口

同じ名前のメソッドが複数ある場合の選ばれ方を理解します.

このページの目的：Javaを丸暗記ではなく、動く仕組みから理解するための補助解説です。

オーバーロードとは

メソッド名が同じでも、引数の型・数・順序が違えば別メソッドとして定義できます。

void test(int x) {}
void test(long x) {}
void test(String x) {}

基本的な選ばれ方

完全一致が最優先です。次に拡大変換、ボクシング、可変長引数などが候補になります。

void test(long x) { System.out.println("long"); }
void test(Integer x) { System.out.println("Integer"); }
test(10); // long

試験での注意

nullを渡す問題では、より具体的な参照型が選ばれます。候補が曖昧ならコンパイルエラーです。
