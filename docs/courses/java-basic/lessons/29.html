1からわかるJava基礎解説 / Lesson 29

オブジェクト指向の設計感覚

クラス、フィールド、メソッドを単なる文法ではなく、役割分担として理解します。

クラスは設計図という説明の限界

クラスは設計図、インスタンスは実体という説明は入口として有効です。ただし実際には、状態と振る舞いをまとめた単位と考える方が実用的です。

class User {
    private String name;
    public User(String name) {
        this.name = name;
    }
    public String getName() {
        return name;
    }
}

状態と振る舞い

nameのようなデータが状態、getNameのような処理が振る舞いです。フィールドを直接公開しすぎると、どこからでも状態を書き換えられてしまいます。

試験での注意

privateフィールド、publicメソッド、コンストラクタ、thisの使い分けは頻出です。特に引数名とフィールド名が同じときはthisを使わないと意図通りになりません。
