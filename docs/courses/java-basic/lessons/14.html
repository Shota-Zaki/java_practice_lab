Java基礎 Lesson 14

コンストラクタ

newしたときの初期化処理と、コンストラクタ呼び出しの順序を理解する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：コンストラクタ・this・super

コンストラクタはメソッドではありません。newの直後に初期化するための特別な構文です。

コンストラクタとはインスタンス生成時に実行される初期化処理。クラス名と同じ名前で戻り値型を書かない。
デフォルトコンストラクタとはコンストラクタを1つも書かない場合に、コンパイラが自動追加する引数なしコンストラクタ。
thisとは現在のインスタンス自身を指す参照。
this()とは同じクラス内の別コンストラクタを呼ぶ構文。
super()とはスーパークラスのコンストラクタを呼ぶ構文。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
コンストラクタ / メソッドコンストラクタはnew時の初期化処理。戻り値型を書かず、クラス名と同じ名前。
デフォルトコンストラクタ / 自作コンストラクタコンストラクタを1つも書かないと自動追加。1つでも書くと自動追加されない。
this() / super()this()は同じクラスの別コンストラクタ。super()は親クラスのコンストラクタ。どちらも先頭行限定。

基礎用語一覧で確認する

基本形

class Item {
    String name;

    Item(String name) {
        this.name = name;
    }
}

Item item = new Item("apple");

new Item("apple")のタイミングでコンストラクタが呼ばれ、フィールドnameへ値が入る。

メソッドではない

項目コンストラクタメソッド
名前クラス名と同じ自由に付ける
戻り値型書かないvoidやintなどを書く
呼び出しnew時に呼ばれる明示的に呼ぶ
目的初期化処理の実行

戻り値型を書くとメソッドになる

class Sample {
    void Sample() { } // これはコンストラクタではなくメソッド
}

デフォルトコンストラクタ

コンストラクタを1つも書かない場合だけ、引数なしコンストラクタが自動で用意される。

自動生成される

class A { }
new A(); // OK

1つ書いたので自動生成されない

class B {
    B(String name) { }
}
new B(); // コンパイルエラー

this()とsuper()

this()は同じクラスの別コンストラクタを呼ぶ。super()は親クラスのコンストラクタを呼ぶ。どちらもコンストラクタの先頭にしか書けない。

class Item {
    String name;
    int price;

    Item() {
        this("unknown", 0);
    }

    Item(String name, int price) {
        this.name = name;
        this.price = price;
    }
}

手を動かすミニ課題

次のコードがなぜコンパイルエラーになるか説明する。

class Parent {
    Parent(String name) { }
}

class Child extends Parent {
    Child() { }
}

丁寧な補講：理解の芯

コンストラクタは、newされた直後の初期化処理。戻り値型を書かず、クラス名と同じ名前にする。メソッドではないので、voidを書いたら普通のメソッドになってしまう。

コンストラクタの先頭では、明示しなくても親クラスのコンストラクタ呼び出しが行われる。自分でthis(...)またはsuper(...)を書くなら、必ずコンストラクタの最初の文にする必要がある。

初期化の順序を読むときは、フィールド初期化、コンストラクタ本体、親子関係の順序を分ける。基礎段階では、まず「newしたらコンストラクタが動く」「名前と戻り値型に注意」の2点を固める。

名前

コンストラクタ名はクラス名と同じ。

戻り値型なし

voidも書かない。書くとメソッドになる。

初期化

インスタンス生成時にフィールドを適切な状態にする。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
コンストラクタ / メソッドコンストラクタは戻り値型なし。メソッドは戻り値型がある。void ClassName()はコンストラクタにならない。
this() / super()this()は同じクラスの別コンストラクタ、super()は親コンストラクタ。最初の文にしか書けない。
デフォルトコンストラクタ / 自作コンストラクタコンストラクタを書かない場合だけ、引数なしコンストラクタが自動追加される。1つでも書いたら自動追加されない。

コードを読むときの手順

  1. クラス名と同じ名前の宣言を探す。
  2. 戻り値型が書かれていないか確認する。
  3. newの引数と一致するコンストラクタがあるか確認する。
  4. this()やsuper()が最初の文か確認する。
  5. フィールドがどの値で初期化されるか追う。

確認用コード

class Item {
    String name;

    void Item(String name) {
        this.name = name;
    }
}

Item item = new Item("pen");
確認する点このコードでの判断結論
void Item戻り値型voidがあるコンストラクタではなくメソッド
new Item("pen")String引数コンストラクタを探す存在しない
結論一致するコンストラクタなしコンパイルエラー

よくある誤解を潰す

コンストラクタにvoidを書かない

それはコンストラクタではなく、クラス名と同じ名前のメソッドになる。

自作コンストラクタを書くと引数なしは自動生成されない

1つでもコンストラクタを書くと、引数なしのデフォルトコンストラクタは自動生成されない。

確認問題：答えまで言語化する

次のコードはなぜコンパイルエラーになるか。

確認問題

class User {
    User(String name) { }
}

public class Main {
    public static void main(String[] args) {
        User u = new User();
    }
}
考え方と答えを見る

User(String name)を自分で定義しているため、引数なしコンストラクタは自動生成されない。

new User()に対応する引数なしコンストラクタが存在しないため、コンパイルエラーになる。

理解の流れを整える補強

コンストラクタは、newされた直後のインスタンスを使える状態に整える処理です。メソッドに似ていますが、戻り値型を書かない、クラス名と同じ名前にする、生成時に呼ばれる、という点が違います。

class User {
    String name;

    User(String name) {
        this.name = name;
    }
}

ここでのthis.nameはフィールド、右辺のnameは引数です。コンストラクタは、外から受け取った値をインスタンスの状態へセットする役割を持ちます。

状況何が起きるか注意
コンストラクタを1つも書かない引数なしのデフォルトコンストラクタが用意される自動で作られる
引数ありコンストラクタを書くデフォルトコンストラクタは自動生成されないnew User()は不可
this(...)を書く同じクラスの別コンストラクタを呼ぶ先頭行にしか書けない
super(...)を書く親クラスのコンストラクタを呼ぶ先頭行にしか書けない
class Item {
    String name;

    Item() {
        this("unknown");
    }

    Item(String name) {
        this.name = name;
    }
}
よくあるミス

コンストラクタにvoidを書くと、コンストラクタではなく通常メソッドになります。見た目が似ていても、生成時には呼ばれません。

理解チェック

  • コンストラクタを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

コンストラクタで落ちる典型

つまずき正しい切り分け
コンストラクタをメソッドだと思うコンストラクタはインスタンス生成時の初期化処理。戻り値型を書かない。
デフォルトコンストラクタが常にあると思う引数ありでも自作コンストラクタを書くと、引数なしのデフォルトコンストラクタは自動追加されない。
this()やsuper()をどこにでも書けると思うコンストラクタ呼び出しはコンストラクタの先頭行でなければならない。

説明できる状態の目安

  • コンストラクタとメソッドの違いを説明できる
  • デフォルトコンストラクタが追加される条件を説明できる
  • this()とsuper()の先頭行ルールを説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：コンストラクタ

class User {
    User(String name) {
        System.out.println(name);
    }
}

失敗コード

class User {
    void User(String name) {
        System.out.println(name);
    }
}

修正版

class User {
    User(String name) {
        System.out.println(name);
    }
}
判定ポイント

コンストラクタには戻り値型を書きません。voidを書くと同名メソッドになります。

理解確認クイズ

用語だけで終わらせず、コード判定と説明まで確認します。

Lv1 用語確認

コンストラクタとメソッドの違いを説明できますか。

答えと切り分けを見る

コンストラクタはインスタンス生成時の初期化処理で、クラス名と同じ名前で戻り値型を書きません。メソッドは通常の処理です。

Lv2 コード判定

次のSampleはコンストラクタを定義していますか。

class Sample {
    void Sample() {
        System.out.println("A");
    }
}
答えと切り分けを見る

定義していません。voidがあるため、これはSampleという名前のメソッドです。コンストラクタではありません。

Lv3 説明問題

this() または super() をコンストラクタの先頭行に置く必要がある理由を説明してください。

答えと切り分けを見る

インスタンス初期化では、どのコンストラクタを先に使うかを最初に確定する必要があります。そのためthis()やsuper()はコンストラクタ本体の先頭にしか置けません。
