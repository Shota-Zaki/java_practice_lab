Java基礎 Lesson 14

コンストラクタ

生成時の初期化と呼び出し順。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

コンストラクタ

newしたときに呼ばれる初期化処理。クラス名と同じ名前で、戻り値型を書かない。

デフォルトコンストラクタ

コンストラクタを1つも書かない場合だけ、引数なしコンストラクタが自動で用意される。1つでも書くと自動では作られない。

this()とsuper()

this()は同じクラスの別コンストラクタ、super()は親クラスのコンストラクタを呼ぶ。どちらもコンストラクタの先頭にしか書けない。

親子の順序

サブクラスのオブジェクト生成時は、親クラスのコンストラクタが先に実行され、その後サブクラスのコンストラクタが実行される。

試験での罠

親に引数なしコンストラクタがないのに、子コンストラクタでsuper(引数)を書かないとコンパイルエラー。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
