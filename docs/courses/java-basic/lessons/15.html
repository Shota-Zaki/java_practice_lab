Java基礎 Lesson 15

継承の入口

extends、親子関係、オーバーライド、型の見方を整理する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：継承・オーバーライド・ポリモーフィズム

継承は「コードを使い回すため」だけで覚えると危険です。型として扱える範囲が広がる点が重要です。

継承とは既存クラスを元にして、新しいクラスを定義する仕組み。
スーパークラスとは継承元の親クラス。
サブクラスとは継承して作られる子クラス。
オーバーライドとはサブクラスで、親クラスのメソッドを同じシグネチャで再定義すること。
ポリモーフィズムとは同じ型の変数で、実体に応じて異なる動作をさせる仕組み。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
継承 / is-a関係継承は既存クラスを元に新しいクラスを作る仕組み。子は親の一種として扱える。
オーバーライド / オーバーロードオーバーライドは親メソッドの再定義。オーバーロードは同名で引数違いの別メソッド。
宣言型 / 実体型呼べるメソッドは宣言型で決まり、オーバーライド先は実体型で決まる。

基礎用語一覧で確認する

継承の基本

class Animal {
    void speak() {
        System.out.println("...");
    }
}

class Dog extends Animal {
    @Override
    void speak() {
        System.out.println("bow");
    }
}

親型の変数で子を参照する

Animal a = new Dog();
a.speak(); // bow

aの宣言型はAnimalだが、実体はDog。speakはオーバーライドされているため、実行時にはDog側が呼ばれる。

呼べるかどうかは宣言型で決まる

実体がDogでも呼べない

class Dog extends Animal {
    void run() { }
}

Animal a = new Dog();
a.run(); // コンパイルエラー

Animal型の変数からは、Animalとして見えるメソッドしか呼べない。実体が何かは、コンパイル時の呼び出し可否には使われない。

フィールドはオーバーライドされない

class A {
    String name = "A";
}
class B extends A {
    String name = "B";
}

A x = new B();
System.out.println(x.name); // A

メソッドとフィールドは同じ仕組みで動かない。フィールドアクセスは変数の宣言型で決まる。

継承で引き継がないもの

  • コンストラクタは引き継がない。
  • privateメンバーは子クラスから直接アクセスできない。
  • 親に引数なしコンストラクタがない場合、子は明示的にsuper(引数)を書く必要がある。

手を動かすミニ課題

次のコードでコンパイルできない行と、実行されるメソッドを分けて説明する。

class A { void test() { System.out.println("A"); } }
class B extends A {
    void test() { System.out.println("B"); }
    void onlyB() { }
}
A a = new B();
a.test();
a.onlyB();

丁寧な補講：理解の芯

継承は「親クラスの機能を子クラスが引き継ぐ」仕組み。ただし、何でも引き継ぐわけではない。コンストラクタは引き継がれないし、privateメンバーには子クラスから直接アクセスできない。

継承で重要なのは、差分を足すことよりも、同じ型として扱えること。親型の変数で子インスタンスを参照できる。これがポリモーフィズムの入口。

メソッド呼び出しでは、実行時のインスタンス側のオーバーライドが効く。一方、フィールドアクセスは変数の型で決まる。この違いは後で頻出するので、基礎段階で言葉だけでも分けておく。

extends

子クラスが親クラスを拡張する。

is-a関係

子は親の一種として扱える。

オーバーライド

親のメソッドを子で再定義する。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
継承 / インスタンス生成継承しただけでは実体は作られない。newが必要。クラス定義と実行を分ける。
コンストラクタ / メソッドメソッドは継承されうるが、コンストラクタは継承されない。子の生成時は親コンストラクタを呼ぶが、子のコンストラクタとして使えるわけではない。
メソッド / フィールドオーバーライドはメソッド。フィールドは同名でも隠蔽であり動きが違う。呼び出しているのが()付きかを見る。

コードを読むときの手順

  1. クラスのextends関係を確認する。
  2. 変数の型と、実際にnewしている型を分ける。
  3. 呼び出しているのがメソッドかフィールドか見る。
  4. メソッドならオーバーライドされているか確認する。
  5. アクセス修飾子やコンストラクタの継承不可に注意する。

確認用コード

class A {
    String name = "A";
    String getName() { return "A"; }
}
class B extends A {
    String name = "B";
    String getName() { return "B"; }
}
A obj = new B();
System.out.println(obj.name);
System.out.println(obj.getName());
確認する点このコードでの判断結論
変数型A objフィールドはA側を見る
実体new B()メソッドはB側のオーバーライドが効く
出力obj.nameはA、obj.getName()はBA / B

よくある誤解を潰す

親のコンストラクタを子はそのまま使えない

コンストラクタは継承されない。必要なら子で定義し、super(...)を呼ぶ。

親型変数でもオーバーライドされた子のメソッドが動く

オーバーライドされたメソッドは実行時の実体側で決まる。

確認問題：答えまで言語化する

次の出力を答える。

確認問題

class Parent {
    void show() { System.out.println("P"); }
}
class Child extends Parent {
    void show() { System.out.println("C"); }
}
Parent p = new Child();
p.show();
考え方と答えを見る

出力はC。

変数の型はParentだが、実際のインスタンスはChild。showはオーバーライドされたメソッドなので、実行時の実体であるChild側が呼ばれる。

理解の流れを整える補強

継承は「親の機能を子が使える」だけで終わらせると危険です。Javaで重要なのは、変数の型で呼べるメソッドが決まり、実行時の実体でオーバーライド先が決まるという2段階です。

class Animal {
    void speak() { System.out.println("?"); }
}
class Dog extends Animal {
    void speak() { System.out.println("wan"); }
    void run() { System.out.println("run"); }
}

Animal a = new Dog();
a.speak(); // wan
// a.run(); // コンパイルエラー

aの宣言型はAnimalなので、コンパイル時にAnimalとして扱われます。そのためrunは呼べません。一方、speakはAnimalに存在し、Dogでオーバーライドされているため、実行時にはDog側が呼ばれます。

確認すること決まるタイミング例
そのメソッドを呼べるかコンパイル時変数の宣言型を見る
どの実装が動くか実行時実体のクラスを見る
フィールド参照基本的に宣言型で決まるフィールドはオーバーライドされない
コンストラクタ継承されない子の生成時に親のコンストラクタは呼ばれる
ここで切るべき誤解

中身がDogでも、Animal型変数からはDog専用メソッドを呼べません。呼べるメソッドは、まず変数の宣言型で決まります。

理解チェック

  • 継承の入口を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

継承で誤解しやすい点

つまずき正しい切り分け
継承はコードをコピーすることだと思う概念的には機能を引き継ぐが、単純なソースコードコピーではない。
コンストラクタも継承されると思うコンストラクタは継承されない。ただし生成時にスーパークラスのコンストラクタは呼ばれる。
フィールドも動的に切り替わると思うオーバーライドされるのはメソッド。フィールドアクセスは参照変数の型で決まる。

説明できる状態の目安

  • extendsの意味を説明できる
  • オーバーライドとオーバーロードを区別できる
  • ポリモーフィズムで動的に選ばれる対象を説明できる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：継承とオーバーライド

class Parent {
    void hello() { System.out.println("P"); }
}
class Child extends Parent {
    @Override
    void hello() { System.out.println("C"); }
}

失敗コード

class Child extends Parent {
    @Override
    void hello(String s) { System.out.println(s); }
}

修正版

class Child extends Parent {
    @Override
    void hello() { System.out.println("C"); }
}
判定ポイント

オーバーライドはメソッド名だけでなく、引数リストまで一致する必要があります。

理解確認クイズ

用語だけで終わらせず、コード判定と説明まで確認します。

Lv1 用語確認

継承、スーパークラス、サブクラス、overrideを説明できますか。

答えと切り分けを見る

継承は既存クラスをもとに新しいクラスを作る仕組みです。継承元がスーパークラス、継承先がサブクラス、overrideは親のメソッドを子で再定義することです。

Lv2 コード判定

次のコードはoverrideになりますか。

class A { void test(int n) {} }
class B extends A { void test(String s) {} }
答えと切り分けを見る

overrideになりません。引数型が違うためオーバーロードです。

Lv3 説明問題

Parent p = new Child(); で呼び出すメソッドがChild側になる場合がある理由を説明してください。

答えと切り分けを見る

インスタンスの実体がChildで、メソッドは実行時の実体に基づいて動的に選択されるためです。フィールド直接参照とは分けます。
