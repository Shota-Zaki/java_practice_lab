Java基礎 Lesson 15

継承の入口

extends、型、オーバーライドの入口。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

継承

extendsで親クラスの機能を引き継ぐ。Javaのクラス継承は単一継承。

ポリモーフィズム

親型の変数で子オブジェクトを参照できる。ただし変数の型で呼べるメソッドが決まる。実行されるオーバーライドメソッドは実体側で決まる。

フィールド

フィールドはオーバーライドされない。親型変数から読むと親側のフィールドを見る。ここはメソッドと違う。

privateメソッド

privateメソッドは継承先からオーバーライドできない。同名メソッドを書いても別物。

試験での罠

A a = new B(); a.onlyB(); はコンパイルエラー。実体がBでも、変数型AにonlyBがなければ呼べない。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
