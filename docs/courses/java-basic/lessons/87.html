Lesson 87 コンストラクタ呼び出し順の考え方

インスタンス生成では、親クラスの初期化が先に行われます。

暗黙のsuper親に無引数コンストラクタがない場合thisとsuperの位置

暗黙のsuper

コンストラクタの先頭にthis(...)やsuper(...)がなければ、暗黙にsuper()が入ります。

親に無引数コンストラクタがない場合

class A { A(int x) {} }
class B extends A { B() {} } // コンパイルエラー

B()から呼べるA()がないためです。

thisとsuperの位置

this(...)またはsuper(...)はコンストラクタの先頭に1回だけ書けます。
