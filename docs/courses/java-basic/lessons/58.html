コンパイラとインタプリタの違い

Javaのコンパイルと実行の位置づけを、初心者向けに整理します。

コンパイルとは変換すること

コンパイルは、人間が読みやすいソースコードを、実行しやすい形式へ変換する処理です。Javaでは.javaから.classへ変換されます。

Javaはバイトコードを実行する

Javaのclassファイルにはバイトコードが入っています。CPUの機械語そのものではなく、JVMが解釈・最適化して実行する形式です。

試験で重要な点

Java Silverでは、実行結果を考える前にコンパイルできるかを見る必要があります。文法違反、型違反、未処理例外などは実行前に止まります。

小さな例

javac Sample.java
java Sample

確認ポイント

  • .javaをjavacで.classへ変換する
  • javaコマンドはclassを実行する
  • コンパイルエラーなら実行結果は存在しない
