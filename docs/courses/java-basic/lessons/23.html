メモリ・スタック・ヒープの考え方

参照型、new、==、メソッド呼び出しを理解するには、スタックとヒープのイメージが必要です。

ざっくりした考え方

ローカル変数はメソッド実行中の作業場所に置かれます。newで作られるオブジェクトは別の領域に置かれ、変数はその場所を指す参照を持ちます。

厳密なJVM実装の話ではなく、Java Silverでは「変数」と「オブジェクト」を分けて考えることが重要です。

Sample s1 = new Sample();
Sample s2 = s1;
s1 = new Sample();
System.out.println(s1 == s2); // false

なぜfalseになるか

s2には最初に作ったオブジェクトへの参照が残ります。一方、s1は新しいオブジェクトを指し直しています。中身が同じかどうかではなく、同じ実体を指しているかが==の判定です。

試験での注意

メソッドに参照型を渡しても、渡されるのは参照のコピーです。メソッド内でフィールドを書き換えると元のオブジェクトに影響しますが、引数変数そのものを別オブジェクトに代入しても呼び出し元の変数は変わりません。
