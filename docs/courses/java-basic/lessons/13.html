Java基礎 Lesson 13

static

クラス側とインスタンス側を分ける。

このレッスンのゴール

用語を暗記するだけではなく、Javaのコードを読んだときに「なぜそうなるか」を説明できる状態にする。

staticとは

staticメンバーはインスタンスではなくクラスに属する。インスタンスを作らなくてもクラス名からアクセスできる。

mainとの関係

mainはstaticなので、同じクラスのインスタンスフィールドやインスタンスメソッドを直接呼べない。

例

class A {
  int x = 10;
  static void test() {
    // System.out.println(x); // コンパイルエラー
  }
}

staticフィールド

全インスタンスで共有される。インスタンスごとに別々の値を持つフィールドとは違う。

試験での罠

staticメソッドはオーバーライドではなく隠蔽になる。インスタンスメソッドと同じ感覚で読むと間違える。

確認ポイント

  • 言葉の定義を自分の言葉で説明できるか。
  • コード例のどこで値や参照が変わるか追えるか。
  • コンパイル前に決まることと、実行中に決まることを分けられるか。
