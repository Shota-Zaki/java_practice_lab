Java基礎 Lesson 13

static

クラス側に属するものと、インスタンス側に属するものを切り分ける。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：static・クラスメンバ・static文脈

staticは「共有」だけでは不十分です。インスタンスなしで使えるクラス側の要素として見ます。

staticとはインスタンスではなくクラスに属することを示す修飾子。
クラスメンバとはstaticなフィールドやメソッド。クラス名から使える。
インスタンスメンバとはnewされた各インスタンスに属するフィールドやメソッド。
static文脈とはstaticメソッドやstatic初期化ブロックの中。インスタンスなしで実行される。
共有とはstaticフィールドが全インスタンスで同じ1つの値を持つこと。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
static / インスタンスstaticはクラス側に属する。インスタンスメンバはnewされた個体に属する。
クラスメンバ / インスタンスメンバクラスメンバは全体で共有。インスタンスメンバは個体ごとに別。
static文脈 / thisstatic文脈では特定のインスタンスがないため、thisは使えない。

基礎用語一覧で確認する

staticフィールド

class Counter {
    static int total;
    int value;
}

Counter a = new Counter();
Counter b = new Counter();
a.value = 1;
b.value = 2;
Counter.total = 100;

valueはインスタンスごとに別。totalはクラス側に1つ。

staticメソッド

staticメソッドはインスタンスなしで呼べる。その代わり、インスタンスフィールドやインスタンスメソッドを直接使えない。

悪い例

class Sample {
    int value = 10;

    static void print() {
        System.out.println(value); // コンパイルエラー
    }
}

修正例

class Sample {
    int value = 10;

    static void print() {
        Sample s = new Sample();
        System.out.println(s.value);
    }
}

呼び出し方

staticメンバーはクラス名で呼ぶのが基本。インスタンス変数からも呼べる場合があるが、読み手を混乱させるため避けるべき。

Math.max(3, 5);
Integer.parseInt("123");
System.out.println("OK");

static初期化の考え方

staticフィールドはクラスが使われるタイミングで準備される。インスタンス生成とは別軸で考える。初心者段階では、まず「staticはクラス側」「非staticはインスタンス側」と切るだけでよい。

手を動かすミニ課題

次のコードのcountが共有される理由を説明する。

class User {
    static int count;

    User() {
        count++;
    }
}

new User();
new User();
System.out.println(User.count);

丁寧な補講：理解の芯

staticは「インスタンスに属さず、クラス側に属する」という意味で捉える。初心者がやりがちな誤解は、staticを「便利だからどこからでも使えるもの」と考えること。これは設計を壊す。

staticメソッドからインスタンスフィールドを直接使えない理由は単純。どのインスタンスの値を使うのか決まっていないから。インスタンスを指定すれば使える。

逆に、インスタンスメソッドからstaticメンバーを使うことはできる。ただし、状態の置き場所がクラス側なのかインスタンス側なのかを意識しないと、値が共有されて予想外の動きになる。

staticフィールド

クラスに1つ紐づく値。全インスタンスで共有される。

staticメソッド

インスタンスなしで呼べるメソッド。

インスタンスメンバー

各オブジェクトに紐づく値や動作。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
クラス側 / インスタンス側staticはクラス側。非staticはインスタンス側。static修飾子の有無を見る。
共有 / 個別staticフィールドは共有。インスタンスフィールドは個別。newを複数回したときに値が分かれるか。
直接参照 / インスタンス経由staticから非staticを直接参照できない。対象オブジェクトが指定されているか見る。

コードを読むときの手順

  1. メンバーにstaticが付いているか見る。
  2. 今いるメソッドがstaticかどうかを見る。
  3. staticメソッド内で非staticを直接使っていないか確認する。
  4. インスタンスを生成して対象を指定しているか確認する。
  5. 値が共有されるべきものか、個別に持つべきものか判断する。

確認用コード

class Counter {
    static int total;
    int count;
}

Counter a = new Counter();
Counter b = new Counter();
a.total++;
a.count++;
System.out.println(b.total + ":" + b.count);
確認する点このコードでの判断結論
totalstaticフィールドaとbで共有
countインスタンスフィールド各インスタンスで別
出力b.totalは1、b.countは01:0

よくある誤解を潰す

staticは何でも解決しない

staticにすると状態が共有される。個別状態までstaticにすると設計が破綻する。

staticメンバはインスタンス所属にならない

書ける場合があっても、意味としてはクラス側に属する。

確認問題：答えまで言語化する

次のコードがコンパイルエラーになる理由を説明する。

確認問題

public class Main {
    int value = 10;
    static void print() {
        System.out.println(value);
    }
}
考え方と答えを見る

printはstaticメソッドで、インスタンスなしで呼ばれる。

valueはインスタンスフィールドなので、どのインスタンスのvalueか決まらない。Main m = new Main();のように対象を作ってm.valueとする必要がある。

理解の流れを整える補強

staticは「便利だから付けるもの」ではありません。インスタンスに属するのか、クラスに属するのかを切り分けるための指定です。ここが曖昧だと、なぜmainからフィールドを直接使えないのかが理解できません。

class Counter {
    int instanceCount = 0;
    static int classCount = 0;

    void add() {
        instanceCount++;
        classCount++;
    }
}

instanceCountはインスタンスごとに別々です。classCountはクラス側に1つです。複数のCounterインスタンスを作っても、classCountは共有されます。

場所使えるもの理由
インスタンスメソッドインスタンスメンバ、staticメンバ対象インスタンスがあるため
staticメソッドstaticメンバ特定のインスタンスがないため
main基本はstatic側main自体がstaticだから
class Sample {
    int value = 10;

    static void print() {
        // System.out.println(value); // コンパイルエラー
        Sample s = new Sample();
        System.out.println(s.value); // OK
    }
}
staticの判断基準

「オブジェクトごとに違う状態か？」を先に考えてください。違うならインスタンス側、全体で1つならstatic側です。

理解チェック

  • staticを、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

staticで落ちる典型

つまずき正しい切り分け
staticを「速い」という意味で覚えるstaticはインスタンスに属さずクラス側に属するという意味で読む。
staticメソッドからインスタンスフィールドを直接使うインスタンスが特定できないため直接参照できない。
staticフィールドをインスタンスごとの値と思うstaticフィールドはクラス側で共有される。

説明できる状態の目安

  • staticメンバとインスタンスメンバの違いを説明できる
  • mainがstaticである理由を説明できる
  • static文脈で直接使えないものを答えられる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：static文脈

class Counter {
    static int count;
    static void up() { count++; }
}

失敗コード

class Counter {
    int count;
    static void up() { count++; }
}

修正版

class Counter {
    static int count;
    static void up() { count++; }
}
判定ポイント

staticメソッドからインスタンスフィールドへ直接アクセスすることはできません。

理解確認クイズ

用語だけで終わらせず、コード判定と説明まで確認します。

Lv1 用語確認

staticメンバとインスタンスメンバを説明できますか。

答えと切り分けを見る

staticメンバはクラスに属します。インスタンスメンバはnewされた各インスタンスに属します。

Lv2 コード判定

次のコードはコンパイルできますか。

class Sample {
    int value = 10;
    static void test() {
        System.out.println(value);
    }
}
答えと切り分けを見る

コンパイルできません。staticメソッドからインスタンスフィールドvalueを直接参照できません。

Lv3 説明問題

mainがstaticである理由を説明してください。

答えと切り分けを見る

javaコマンドはインスタンスを作る前に入口を呼び出します。そのためmainはクラスから直接呼べるstaticである必要があります。
