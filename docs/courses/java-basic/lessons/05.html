Java基礎 Lesson 5

変数・値・型

変数に何が入っているのかを、プリミティブ型と参照型に分けて理解する。

このレッスンのゴール

暗記ではなく、コードを読んだときに「なぜそうなるか」を説明できる状態にする。

初出用語：変数・値・型

変数は「箱」とだけ覚えると限界があります。値そのものか、参照かを分けます。

変数とは値や参照を保持する名前付きの領域。
値とは計算や比較の対象になるデータそのもの。例: 10、true。
型とは値の種類と扱い方を決めるルール。例: int、String。
プリミティブ型とは数値や真偽値など、値そのものを扱う基本型。
参照型とはオブジェクトの場所を指す参照を扱う型。

このLessonで先に区別する言葉

似た言葉を同じ意味で読まないこと。ここを分けると、後のコード読解で迷いにくくなります。

用語切り分け
変数 / 値 / 型変数は値を入れる名前付きの場所。型はその値の扱い方を決めるルール。
宣言 / 代入 / 初期化宣言は変数を作ること。代入は値を入れること。初期化は最初の代入。
プリミティブ型 / 参照型プリミティブ型は値そのもの。参照型はオブジェクトへの参照を扱う。

基礎用語一覧で確認する

プリミティブ型と参照型

分類代表例変数に入るものnull
プリミティブ型int, double, char, boolean値そのもの入らない
参照型String, 配列, 自作クラスオブジェクトを指す参照入る
int n = 10;        // nには数値10が入る
String s = "A";    // sにはStringオブジェクトへの参照が入る
String t = null;   // tは何も参照していない

ローカル変数とフィールドの初期値

メソッド内で宣言したローカル変数は、自動初期化されない。使う前に必ず代入が必要。クラスのフィールドは、型ごとのデフォルト値で初期化される。

差が出る例

public class Main {
    static int field;

    public static void main(String[] args) {
        int local;
        System.out.println(field); // 0
        System.out.println(local); // コンパイルエラー
    }
}

型は「扱い方」を決める

型は単なるラベルではない。その変数に何を代入できるか、どんな演算ができるか、どのメソッドを呼べるかを決める。

コード結果理由
int x = 10;OKintに整数値を代入できる
int x = null;エラープリミティブ型にnullは入らない
String s = null;OK参照型はnullを持てる
boolean b = 1;エラーJavaのbooleanは数値ではない

悪い例と修正

悪い例：初期化前に使っている

public class Main {
    public static void main(String[] args) {
        int count;
        count++;
        System.out.println(count);
    }
}

修正例

public class Main {
    public static void main(String[] args) {
        int count = 0;
        count++;
        System.out.println(count);
    }
}

手を動かすミニ課題

次の変数について、プリミティブ型か参照型か、初期化なしで使えるかを判断する。

int a;
String b;
int[] c;
boolean d;

メソッド内のローカル変数なら、どれも初期化前には使えない。型の分類と初期化ルールは別問題。

丁寧な補講：理解の芯

変数を「箱」とだけ覚えると、参照型で必ず詰まる。Javaでは、プリミティブ型の変数には値そのもの、参照型の変数にはオブジェクトへの参照が入る。この違いが、代入、比較、メソッド呼び出し、配列で効いてくる。

ローカル変数とフィールドの違いも重要。ローカル変数は自動初期化されない。フィールドは型ごとのデフォルト値で初期化される。ここを曖昧にすると、「なぜこっちは0で、こっちはコンパイルエラーなのか」が説明できない。

型は「入れ物の名前」ではなく「扱い方のルール」。intなら整数演算、booleanなら真偽値、Stringなら文字列オブジェクトへの参照として扱う。

プリミティブ型

値そのものを持つ。nullは入らない。

参照型

オブジェクトを指す参照を持つ。nullを持てる。

型

代入できる値、使える演算、呼べるメソッドを決める。

初心者が混同しやすい境界線

混同しやすいもの違い見分け方
ローカル変数 / フィールドローカル変数は自動初期化されない。フィールドは初期値を持つ。宣言場所を見る。
値 / 参照intは値、Stringや配列は参照。型がプリミティブか参照型かを見る。
代入 / コピー代入は変数の中身をコピーする。参照型なら参照がコピーされる。オブジェクト自体が複製されるとは限らない。

コードを読むときの手順

  1. 変数の宣言場所を見る。ローカルかフィールドかを決める。
  2. 型を見る。プリミティブ型か参照型かを決める。
  3. 代入式の右辺が、その型に入れられるか確認する。
  4. 使う前に初期化済みか確認する。
  5. 参照型ならnullの可能性も考える。

確認用コード

public class Main {
    static int field;

    public static void main(String[] args) {
        int local;
        System.out.println(field);
        System.out.println(local);
    }
}
確認する点このコードでの判断結論
fieldstaticフィールドデフォルト値0で初期化済み
localローカル変数初期化されていない
結論local使用で失敗コンパイルエラー

よくある誤解を潰す

ローカル変数は自動で0にならない

自動初期化されるのはフィールド。ローカル変数は使う前に代入が必要。

参照型の代入でオブジェクト本体はコピーされない

コピーされるのは参照。2つの変数が同じオブジェクトを見ることがある。

確認問題：答えまで言語化する

次のコードはコンパイルできるか。できない場合は理由を説明する。

確認問題

public class Main {
    public static void main(String[] args) {
        int x;
        String s = null;
        System.out.println(s);
        System.out.println(x);
    }
}
考え方と答えを見る

コンパイルできない。

sは参照型なのでnullを代入できる。問題はx。ローカル変数xは初期化されないまま使われているため、コンパイルエラーになる。

理解の流れを整える補強

このLessonの中心は、変数を「値が入る箱」とだけ覚えないことです。箱という比喩は入口としては使えますが、Javaを読むときはそこで止まると危険です。プリミティブ型の変数には値そのものが入り、参照型の変数にはオブジェクトそのものではなく参照が入ります。

見る対象確認すること判断
宣言int n; / String s;変数名と型が決まる。まだ使える値があるとは限らない。
初期化n = 10; / s = "Java";その変数を読んでよい状態になる。
再代入n = 20; / s = "Java";変数が保持する値、または参照が差し替わる。
使用System.out.println(n);ここまでに必ず初期化されている必要がある。

特にローカル変数は、宣言しただけでは使えません。フィールドには型ごとの初期値がありますが、ローカル変数には自動初期化がありません。この差を曖昧にしたまま進むと、後のメソッド、コンストラクタ、クラスの問題で毎回つまずきます。

public class Sample {
    int field; // フィールド。intの初期値0が入る

    public void test() {
        int local; // ローカル変数。まだ使えない
        // System.out.println(local); // コンパイルエラー
        System.out.println(field);    // 0
    }
}
ここで切るべき誤解

「変数を宣言したら何か値が入っている」は雑すぎます。フィールドなのかローカル変数なのか、プリミティブ型なのか参照型なのかで話が変わります。

理解チェック

  • 変数・値・型を、用語だけでなく自分の言葉で説明できる。
  • コードを見たとき、コンパイル前に決まることと実行中に決まることを分けられる。
  • 悪い例がなぜ失敗するのかを、文法・型・参照・実行順序のどれかで説明できる。

このLessonのつまずきチェック

用語を覚えただけでは不十分です。実際に間違えやすい読み方を先に潰します。

変数・値・型で混ざる点

つまずき正しい切り分け
変数そのものが値だと思う変数は値を入れる入れ物または参照を保持する名前。値そのものではない。
型を「見た目」だけで判断するリテラルには基本型がある。整数は基本的にint、小数は基本的にdouble。
ローカル変数の初期値を期待するローカル変数は自動初期化されない。使う前に明示的な代入が必要。

説明できる状態の目安

  • 値・型・変数の違いを説明できる
  • ローカル変数とフィールドの初期値の違いを説明できる
  • リテラルの基本型を答えられる
試験対策としての使い方

この3点を自分の言葉で説明できない場合、問題演習で似た形を見たときに運任せになります。戻ってコードを1つ実行してください。

正しいコード / 失敗コード / 修正版

このLessonの要点を、成功例と失敗例で切り分けます。試験では「なぜ失敗するか」を説明できる状態が必要です。

正しいコード：変数・値・型

int age = 20;
String name = "Java";

失敗コード

int age = "20";
String name = 100;

修正版

int age = 20;
String name = "100";
判定ポイント

変数には型があります。代入できる値は、その型と互換性がある必要があります。

理解確認クイズ

用語だけで終わらせず、コード判定と説明まで確認します。

Lv1 用語確認

変数、値、型、リテラルを分けて説明できますか。

答えと切り分けを見る

変数は値を入れる名前、値は実データ、型は扱い方のルール、リテラルはコード上に直接書いた値です。

Lv2 コード判定

次のコードはコンパイルできますか。

int a;
System.out.println(a);
答えと切り分けを見る

コンパイルできません。ローカル変数は使う前に初期化が必要です。フィールドの自動初期値とは分けます。

Lv3 説明問題

int a = 10; のうち、型・変数名・値はどれですか。

答えと切り分けを見る

intが型、aが変数名、10が値です。10は整数リテラルでもあります。
