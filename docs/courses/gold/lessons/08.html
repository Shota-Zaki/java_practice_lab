Java Gold：総合確認

ラムダ式、Stream、ジェネリクス、NIO、並行処理、モジュールの要点をまとめます。

ポータルへ戻るGoldトップ
学習記事上部

総合確認で見るポイント

  • ラムダ式では、対象型となる関数型インタフェースを先に確認する
  • Streamでは、中間操作と終端操作、戻り値の型を追う
  • ジェネリクスでは、読み取り中心か書き込み中心かを分ける
  • NIOでは、PathとFilesの役割、IOExceptionの扱いを見る
  • 並行処理では、タスクの実行順を決めつけない
  • モジュールでは、requires、exports、opensの役割を区別する

確認例

List<String> list = List.of("a", "bb", "ccc");
Map<Integer, List<String>> m = list.stream()
    .collect(Collectors.groupingBy(String::length));

このコードでは、要素を文字列長で分類します。戻り値はMapで、キーはInteger、値はList<String>です。

復習の優先順位

まずラムダ式とStreamの型の流れを固め、次にジェネリクス、NIO、並行処理、モジュールへ進むと安定します。
